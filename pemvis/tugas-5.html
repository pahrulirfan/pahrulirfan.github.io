Soal: Post Manager

Buatlah aplikasi desktop untuk mengelola data post menggunakan API nyata (https://api.pahrul.my.id/api/posts). Aplikasi harus mampu melakukan operasi CRUD lengkap dengan UI yang responsif.

Fitur yang Harus Diimplementasikan

  1. Tampilkan Daftar Posts — ambil semua posts via GET, tampilkan dalam tabel dengan kolom: ID, Title, Author, Status
  2. Detail Post — klik baris tabel untuk menampilkan detail lengkap post (title, body, author, slug, status, comments) di panel samping
  3. Tambah Post — form input untuk POST post baru (title, body, author, slug, status), tampilkan ID yang dikembalikan server
  4. Edit Post — pilih post dari tabel, ubah data via PUT, tampilkan konfirmasi sukses
  5. Hapus Post — pilih post dari tabel, hapus via DELETE dengan konfirmasi dialog terlebih dahulu (cascade delete — semua comments ikut terhapus)
  6. Threading — semua API call harus berjalan di thread terpisah, UI tidak boleh freeze
  7. State Handling — tampilkan status loading saat request berjalan, pesan error jika gagal
Ketentuan
  • Endpoint yang digunakan: /api/posts dan /api/posts/{id}
  • Implementasikan semua 4 HTTP method: GET, POST, PUT, DELETE
  • Semua network request HARUS di thread terpisah — tidak boleh ada UI freeze
  • Implementasikan error handling: timeout dan connection error
  • Tombol Edit dan Hapus hanya aktif jika ada baris yang dipilih di tabel
  • Field slug harus unik — validasi 422 dari server harus ditangani dan ditampilkan ke user
