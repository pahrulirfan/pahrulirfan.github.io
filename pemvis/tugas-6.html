Tugas 6 — Visualisasi Data

Buat aplikasi dashboard PySide6 yang menampilkan data tabular, ringkasan, filter, dan chart Matplotlib

Indikator Penilaian

Setelah menyelesaikan tugas ini, mahasiswa mampu:
  1. Membaca dan mengolah data tabular dari CSV Kaggle atau SQLite
  2. Menampilkan data mentah ke dalam QTableWidget
  3. Mengintegrasikan Matplotlib ke PySide6 menggunakan FigureCanvasQTAgg
  4. Membuat dashboard dengan filter, ringkasan data, dan chart yang dapat berubah

Ketentuan Pengumpulan

Pengumpulan
  1. Simpan file Python dengan nama yang sesuai
  2. Test program dan pastikan berjalan tanpa error
  3. Ambil screenshot pada setiap aplikasi yang dijalankan
  4. Buat file Readme.md untuk menjelaskan secara singkat tugas yang dibuat dan menampilkan hasil screenshot
  5. Upload ke GitHub, buat repository dengan nama T6-week12 (upload file .py, file Readme.md dan screenshot aplikasi berjalan)
  6. Submit link GitHub ke platform Berajah
Tambahan
  1. Tambahkan komentar Nama, NIM, dan Kelas di awal file
  2. Pastikan kode rapi dan mudah dibaca

Soal: Dashboard Visualisasi Data

Buat aplikasi PySide6 yang menampilkan data dalam bentuk dashboard. Mahasiswa sangat dianjurkan memakai dataset nyata dari Kaggle agar data yang divisualisasikan lebih realistis dan bervariasi.

Fitur yang Harus Diimplementasikan

  1. Gunakan data minimal 50 baris, boleh dari CSV Kaggle atau SQLite
  2. Tampilkan data mentah di QTableWidget
  3. Tampilkan minimal dua jenis chart menggunakan Matplotlib
  4. Sediakan filter kategori atau pilihan tipe chart
  5. Sediakan tombol refresh dan tombol export chart ke PNG
Rekomendasi Dataset Kaggle

Dataset yang direkomendasikan adalah Supermarket Sales Dataset dari Kaggle: https://www.kaggle.com/datasets/faresashraf1001/supermarket-sales.

Dataset ini berisi transaksi penjualan supermarket dengan kolom seperti Invoice ID, Branch, City, Customer Type, Gender, Product Line, Unit Price, Quantity, Tax 5%, Total, Date, Time, Payment, cogs, gross income, dan Rating.

Dataset ini cocok untuk dashboard karena bisa dianalisis berdasarkan cabang toko, kota, jenis pelanggan, gender, kategori produk, metode pembayaran, total transaksi, dan tren penjualan berdasarkan tanggal. Mahasiswa yang memakai dataset Kaggle, menuliskan alamat dataset, dan menjelaskan makna kolom utama akan mendapat nilai lebih.

Alternatif SQLite

Jika tidak memakai Kaggle, mahasiswa boleh memakai SQLite dengan minimal 50 data. Database harus memiliki tabel yang jelas, misalnya tabel penjualan dengan kolom id, tanggal, kategori, produk, jumlah, harga, dan total. Aplikasi tetap wajib menampilkan data di tabel dan menampilkan minimal dua chart.

Mahasiswa yang menambahkan fitur CRUD untuk data SQLite akan mendapat nilai lebih. CRUD yang dimaksud meliputi tambah data, tampil data, ubah data, hapus data, dan refresh dashboard setelah data berubah.

Ketentuan Tugas
  • Dashboard harus dibuat menggunakan PySide6
  • Chart harus tampil langsung di aplikasi PySide6, bukan membuka window Matplotlib terpisah
  • Gunakan struktur project yang rapi, misalnya memisahkan file data loader, chart widget, dan window utama
  • Jika memakai Kaggle, sertakan alamat dataset dan penjelasan singkat isi dataset di Readme.md
  • Jika memakai SQLite, sertakan file database .db dan jelaskan struktur tabelnya di Readme.md
  • UI harus tetap rapi saat window di-resize
