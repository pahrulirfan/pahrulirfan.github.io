1. Instalasi dan Setup Environment

Persiapan lingkungan pengembangan untuk pemrograman visual dengan PySide6

Apa yang Dibutuhkan?

Sebelum memulai pemrograman visual dengan PySide6, Anda perlu menginstal beberapa komponen berikut:

  • Python 3.9 atau lebih baru - Bahasa pemrograman utama
  • PySide6 - Library GUI berbasis Qt 6
  • Virtual Environment - Untuk isolasi dependensi proyek
  • VS Code / PyCharm - Editor kode (opsional tapi disarankan)

Langkah-langkah Instalasi

  1. Install Python
    Download Python dari python.org. Saat instalasi di Windows, pastikan mencentang "Add Python to PATH".
  2. Verifikasi Instalasi Python
    Buka terminal/command prompt dan jalankan perintah berikut.
$ python --version
Python 3.11.5

$ pip --version
pip 23.2.1 from ...
  1. Buat Folder Proyek dan Virtual Environment
    Virtual environment mengisolasi library proyek agar tidak bentrok dengan proyek lain.
# Buat folder proyek
$ mkdir belajar-pyside6
$ cd belajar-pyside6

# Buat virtual environment
$ python -m venv venv

# Aktivasi (Windows)
$ venv\Scripts\activate

# Aktivasi (Mac/Linux)
$ source venv/bin/activate

(venv) $ # Tanda venv sudah aktif
  1. Install PySide6
    Pastikan virtual environment sudah aktif, lalu install PySide6.
(venv) $ pip install PySide6

Collecting PySide6
Downloading PySide6-6.6.1-...
Successfully installed PySide6-6.6.1
  1. Verifikasi Instalasi PySide6
    Pastikan PySide6 terinstal dengan benar.
(venv) $ python -c "import PySide6; print(PySide6.__version__)"
6.6.1
Catatan Penting

Selalu aktifkan virtual environment sebelum menjalankan program PySide6. Jika tidak, Python tidak akan menemukan library PySide6.

# struktur_minimal.py
        # Struktur paling sederhana

        import sys
        from PySide6.QtWidgets import QApplication, QWidget

        # 1. Buat objek aplikasi (WAJIB PERTAMA)
        app = QApplication(sys.argv)

        # 2. Buat jendela
        window = QWidget()
        window.setWindowTitle("Aplikasi Pertama")
        window.resize(400, 300)

        # 3. Tampilkan jendela
        window.show()

        # 4. Jalankan event loop (WAJIB TERAKHIR)
        sys.exit(app.exec())

Penjelasan Detail Setiap Baris

Baris 4: import sys

Modul sys menyediakan akses ke variabel dan fungsi yang berinteraksi dengan Python interpreter.

  • sys.argv - List berisi argumen command line
  • sys.exit() - Keluar dari program dengan kode status

Baris 5: from PySide6.QtWidgets import ...

Mengimpor class yang dibutuhkan dari modul QtWidgets:

  • QApplication - Class untuk mengelola aplikasi GUI
  • QWidget - Class dasar untuk semua widget

Baris 8: app = QApplication(sys.argv)

Membuat instance QApplication. Ini WAJIB dibuat sebelum widget apapun.

  • sys.argv berisi argumen dari command line
  • Qt akan memproses argumen khusus seperti --style, --platform
  • Hanya boleh ada SATU QApplication per program
# Contoh sys.argv
$ python app.py --style Fusion

# Di dalam program:
sys.argv = ['app.py', '--style', 'Fusion']

Baris 11-13: Membuat dan Konfigurasi Window

        window = QWidget()                        # Buat jendela kosong
        window.setWindowTitle("Aplikasi Pertama") # Judul di title bar
        window.resize(400, 300)                   # Ukuran: lebar x tinggi (pixel)

Baris 16: window.show()

Menampilkan jendela ke layar. Tanpa pemanggilan ini, jendela tidak akan terlihat meskipun sudah dibuat.

Perbedaan show() dan hide()

show() - Menampilkan widget
hide() - Menyembunyikan widget
setVisible(True/False) - Alternatif untuk show/hide

Baris 19: sys.exit(app.exec())

Baris ini melakukan dua hal:

  1. app.exec() - Menjalankan event loop. Program akan "berhenti" di sini dan menunggu interaksi user (klik, ketik, dll)
  2. sys.exit() - Setelah event loop selesai (user menutup jendela), keluar dari program dengan kode status yang benar
Apa itu Event Loop?

Event loop adalah proses yang terus berjalan menunggu dan memproses event:

  • Klik mouse
  • Ketikan keyboard
  • Resize jendela
  • Timer
  • Dan event lainnya

Tanpa event loop (app.exec()), program akan langsung berakhir setelah show() dan jendela akan hilang dalam sekejap.

# struktur_oop.py
        # Struktur dengan Class (Direkomendasikan)

        import sys
        from PySide6.QtWidgets import QApplication, QWidget

        class MainWindow(QWidget):
            """
            Class utama untuk jendela aplikasi.
            Mewarisi (inherit) dari QWidget.
            """
            
            def __init__(self):
                # Panggil constructor parent class
                super().__init__()
                
                # Panggil method untuk setup UI
                self.setup_ui()
            
            def setup_ui(self):
                """Method untuk mengatur tampilan UI"""
                self.setWindowTitle("Aplikasi OOP")
                self.resize(500, 400)
                self.setMinimumSize(300, 200)

        def main():
            """Fungsi utama untuk menjalankan aplikasi"""
            app = QApplication(sys.argv)
            
            window = MainWindow()
            window.show()
            
            sys.exit(app.exec())

        # Entry point program
        if __name__ == "__main__":
            main()

Penjelasan Struktur OOP

Class MainWindow(QWidget)

Membuat class baru yang mewarisi dari QWidget. Dengan inheritance, MainWindow mendapat semua kemampuan QWidget plus method custom yang kita tambahkan.

Method __init__(self)

Constructor yang dipanggil saat object dibuat. Di sini kita:

  1. Memanggil super().__init__() untuk inisialisasi parent class
  2. Memanggil self.setup_ui() untuk mengatur tampilan

Method setup_ui(self)

Method terpisah untuk konfigurasi UI. Memisahkan setup UI dari __init__ membuat kode lebih rapi dan mudah dikelola.

if __name__ == "__main__"

Kondisi ini memastikan main() hanya dijalankan jika file dieksekusi langsung, bukan saat di-import sebagai modul.

# Dijalankan langsung - main() AKAN dipanggil
$ python struktur_oop.py

# Di-import - main() TIDAK dipanggil
>>> import struktur_oop
>>> # Tidak terjadi apa-apa

Perbandingan: Prosedural vs OOP

Aspek Prosedural OOP (Class)
Kompleksitas Sederhana, sedikit kode Lebih banyak kode awal
Organisasi Semua kode di satu tempat Terstruktur dalam class/method
Reusability Sulit digunakan ulang Mudah digunakan ulang
Maintenance Sulit untuk kode besar Mudah di-maintain
Cocok untuk Testing, script kecil Aplikasi produksi
Standar Industri Tidak Ya

3. Widget Tree (Hirarki Widget)

Memahami hubungan parent-child antar widget dalam aplikasi PySide6

Konsep Parent-Child

Dalam PySide6, semua widget tersusun dalam struktur pohon (tree). Setiap widget dapat memiliki parent (induk) dan children (anak). Konsep ini penting untuk:

  • Memory Management - Ketika parent dihapus, semua child otomatis ikut dihapus
  • Layout - Child widget diposisikan relatif terhadap parent
  • Event Propagation - Event dapat diteruskan dari child ke parent

                    |       |

import sys
        from PySide6.QtWidgets import (
            QApplication, QWidget, QLabel, 
            QPushButton, QVBoxLayout
        )

        class ContohHirarki(QWidget):
            def __init__(self):
                super().__init__()
                self.setup_ui()
            
            def setup_ui(self):
                self.setWindowTitle("Contoh Hirarki")
                
                # Membuat layout (parent untuk widget di dalamnya)
                layout = QVBoxLayout()
                
                # Membuat widget child
                label = QLabel("Ini adalah label")
                tombol = QPushButton("Ini adalah tombol")
                
                # Menambahkan child ke layout
                # Layout otomatis menjadi parent
                layout.addWidget(label)
                layout.addWidget(tombol)
                
                # Set layout ke window (self)
                # self menjadi parent dari layout
                self.setLayout(layout)

        app = QApplication(sys.argv)
        window = ContohHirarki()
        window.show()
        sys.exit(app.exec())
Hirarki dari Kode di Atas
        ContohHirarki (QWidget) -- PARENT UTAMA
            |
            +-- QVBoxLayout
                    |
                    +-- QLabel ("Ini adalah label")
                    |
                    +-- QPushButton ("Ini adalah tombol")
Mengapa Hirarki Penting?

Ketika Anda menutup jendela utama (parent), semua widget di dalamnya (children) akan otomatis dihapus dari memory. Ini mencegah memory leak dan mempermudah pengelolaan resource.

5. Widget Umum di PySide6

Mengenal berbagai widget yang sering digunakan dalam aplikasi GUI

Apa itu Widget?

Widget adalah komponen visual yang membentuk antarmuka pengguna. Semua elemen yang terlihat di aplikasi (tombol, label, input) adalah widget. Di PySide6, semua widget merupakan turunan dari class QWidget.

Daftar Widget Umum

QLabel - Menampilkan Teks atau Gambar

Widget untuk menampilkan teks statis atau gambar. Tidak bisa diedit oleh user.

Ini adalah contoh QLabel
Contoh QLabel

Tombol yang dapat diklik user untuk memicu aksi tertentu.

Mekanisme komunikasi antar widget dalam PySide6

Apa itu Signal dan Slot?

Signal dan Slot adalah mekanisme komunikasi di Qt. Ketika sebuah event terjadi (misal: tombol diklik), widget akan mengirim signal. Signal ini dapat dihubungkan ke slot (fungsi) yang akan dieksekusi.

                self.tombol = QPushButton("Klik Saya")
                
                # MENGHUBUNGKAN SIGNAL KE SLOT
                # widget.signal.connect(slot)
                self.tombol.clicked.connect(self.tombol_diklik)
                
                # Layout
                layout = QVBoxLayout()
                layout.addWidget(self.label)
                layout.addWidget(self.tombol)
                self.setLayout(layout)
            
            # INI ADALAH SLOT (fungsi yang dipanggil)
            def tombol_diklik(self):
                self.label.setText("Tombol sudah diklik!")

        app = QApplication(sys.argv)
        window = ContohSignalSlot()
        window.show()
        sys.exit(app.exec())

import sys
        from PySide6.QtWidgets import (
            QApplication, QWidget, QLabel, 
            QPushButton, QLineEdit, QVBoxLayout
        )

        class ContohStyling(QWidget):
            def __init__(self):
                super().__init__()
                self.setup_ui()
            
            def setup_ui(self):
                self.setWindowTitle("Contoh Styling")
                self.resize(400, 300)
                
                # Styling untuk window
                self.setStyleSheet("background-color: #f5f6fa;")
                
                layout = QVBoxLayout()
                
                # Label dengan styling
                label = QLabel("Form Login")
                label.setStyleSheet("""
                    font-size: 24px;
                    font-weight: bold;
                    color: #2c3e50;
                    padding: 10px;
                """)
                
                # Input dengan styling
                input_field = QLineEdit()
                input_field.setPlaceholderText("Username")
                input_field.setStyleSheet("""
                    QLineEdit {
                        padding: 12px;
                        border: 2px solid #dcdde1;
                        border-radius: 5px;
                        font-size: 14px;
                    }
                    QLineEdit:focus {
                        border-color: #3498db;
                    }
                """)
                
                layout.addWidget(label)
                layout.addWidget(input_field)
                layout.addStretch()
                self.setLayout(layout)

        app = QApplication(sys.argv)
        window = ContohStyling()
        window.show()
        sys.exit(app.exec())
