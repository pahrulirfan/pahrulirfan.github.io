Tugas 1: Kalkulator dengan Event Handling

Buatlah aplikasi kalkulator sederhana yang memanfaatkan event handling dan signal/slot untuk validasi input real-time.

Contoh Tampilan Aplikasi

🧮 Kalkulator — □ ✕
⚠ Input harus berupa angka
Hasil: —
⚠ Input tidak valid — tombol Hitung dinonaktifkan

Fitur yang Harus Diimplementasikan

  • Input angka pertama dan kedua menggunakan QLineEdit
  • Pilihan operasi (+, -, ×, ÷) menggunakan QComboBox
  • Validasi real-time — border merah jika input bukan angka, pesan error muncul
  • Tombol "Hitung" hanya aktif (setEnabled) jika kedua input valid
  • Keyboard shortcut: Enter untuk menghitung, Escape untuk clear
  • Konfirmasi QMessageBox saat window akan ditutup
  • Hasil ditampilkan di area output yang terpisah

Tugas 2: Drawing Canvas

Buatlah aplikasi canvas sederhana yang memanfaatkan mouse event dan custom signal untuk menggambar bebas.

Contoh Tampilan Aplikasi

🎨 Drawing Canvas — □ ✕
Warna:
Ukuran: 5px
Double-click untuk clear
X: 245 | Y: 112 | Warna: Merah | Brush: 5px

Fitur yang Harus Diimplementasikan

  • Area kanvas (QWidget dengan override paintEvent) yang bisa menggambar saat mouse di-drag
  • Pilihan warna menggunakan custom signal yang dikirim ke canvas
  • Pilihan ukuran brush menggunakan QSlider
  • Koordinat mouse ditampilkan real-time di StatusBar
  • Double-click untuk clear canvas
  • Tombol "Clear" dengan konfirmasi QMessageBox

Tugas 3: Form Multi-Step

Buatlah form registrasi multi-step dengan validasi real-time dan custom signal untuk notifikasi perpindahan antar step.

Contoh Tampilan Aplikasi

📋 Form Registrasi — □ ✕
✓
2
3
Data Pribadi Kontak Akun
Step 2: Informasi Kontak
⚠ Nomor minimal 10 digit
Step 2 dari 3 — Lengkapi semua field untuk melanjutkan

Fitur yang Harus Diimplementasikan

  • Step 1 — Data Pribadi: nama, tanggal lahir (QDateEdit), jenis kelamin (QRadioButton)
  • Step 2 — Kontak: email, telepon, alamat
  • Step 3 — Akun: username, password, confirm password
  • Validasi real-time di setiap field — border hijau jika valid, merah jika tidak
  • Custom signal untuk notifikasi saat berpindah step
  • Progress indicator (QProgressBar atau widget custom) yang update via signal
  • Tombol "Lanjut" hanya aktif jika semua field di step tersebut valid
  • Halaman review data sebelum submit
