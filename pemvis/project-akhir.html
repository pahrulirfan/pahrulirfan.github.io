Final Project
Pemrograman Visual

Semester Genap 2025 / 2026

Tentang Proyek Akhir

Setiap kelompok (2–3 mahasiswa) diwajibkan mengembangkan sebuah aplikasi GUI berbasis PySide6. Topik bebas dipilih sesuai minat kelompok, namun harus memiliki manfaat yang jelas dan relevansi nyata — bukan sekadar demo atau latihan sederhana.

Proyek ini mengevaluasi kemampuan kelompok dan invidu dalam merancang aplikasi visual secara utuh, meliputi struktur antarmuka, pengelolaan data, interaksi pengguna, dan dokumentasi berdasarkan materi yang telah diplajri.

Contoh Topik yang Sesuai

  • Manajemen inventaris
  • Pencatatan keuangan
  • Sistem peminjaman buku/barang
  • Reservasi / penjadwalan
  • Manajemen tugas / proyek

Contoh Topik Terlalu Kecil

  • Kalkulator biasa
  • Konversi satuan sederhana
  • Form biodata tanpa pengelolaan data
  • CRUD tanpa alur aplikasi yang jelas

Ketentuan Kelompok

  1. Pembagian Tugas yang Jelas — setiap anggota memiliki tanggung jawab yang terdokumentasi dengan jelas di laporan dan README.
  2. Riwayat Commit GitHub — setiap anggota wajib memiliki commit di repository sebagai bukti kontribusi nyata.
  3. Interview Individu — setiap anggota akan diinterview secara individu. Nilai interview bersifat personal — setiap anggota harus memahami keseluruhan aplikasi, bukan hanya bagian yang dikerjakan sendiri.

Fitur Wajib

  1. Struktur Antarmuka Multi-Halaman — minimal 3 halaman/view fungsional dengan navigasi yang jelas (QStackedWidget, QTabWidget, sidebar, dll.). Setiap anggota kelompok bertanggung jawab atas minimal satu halaman/modul.
  2. Layout, Menu Bar & Status Bar — layout rapi dan responsif terhadap resize, menu bar berisi minimal File/Help/Exit, status bar menampilkan nama & NIM semua anggota (tidak dapat diedit)
  3. Pengelolaan Data — pilih salah satu: SQLite (min. 2 tabel dengan relasi, min. 5 kolom per tabel, CRUD lengkap di semua entitas utama) atau API
  4. Tampilan Data & Pencarian — data ditampilkan via QTableView / QTableWidget atau list custom, wajib ada search dan filter/sorting (keduanya, bukan salah satu)
  5. Validasi Input & Feedback — semua form wajib divalidasi (field kosong, format angka/tanggal, panjang karakter), konfirmasi hapus via QMessageBox, form input menggunakan QDialog, aplikasi tidak boleh crash karena input tidak valid
  6. Styling — terapkan setStyleSheet() atau file .qss secara konsisten di seluruh halaman — bukan hanya satu komponen
  7. Export Data — minimal 2 format output (contoh: CSV + PDF), atau 1 format dengan opsi filter/rentang data yang diekspor
  8. Fitur Tambahan Kelompok — wajib minimal 2 fitur tambahan di luar fitur dasar (contoh: visualisasi data/chart, import CSV, login & manajemen sesi, tema gelap/terang, notifikasi/pengingat, dashboard ringkasan, multithreading, dll.)

Fitur Opsional (Bonus)

  • Qt Designer (+5%) — UI dibuat menggunakan Qt Designer (.ui) dan terintegrasi dengan baik ke Python
  • AI / ML (+10%) — fitur AI/ML sederhana yang relevan dengan topik aplikasi (prediksi, klasifikasi, rekomendasi, dll.)
  • Fitur Kreatif (+5%) — fitur tambahan lain yang berguna, kreatif, dan meningkatkan nilai aplikasi

Total bonus maksimal +20%. Fitur bonus tidak menggantikan fitur wajib yang belum terpenuhi.

Catatan Teknis

  • Wajib menggunakan PySide6 (bukan PyQt5 / PyQt6)
  • UI boleh dibuat manual di Python atau menggunakan Qt Designer (.ui) — jika pakai Qt Designer, file .ui wajib disertakan
  • Aplikasi harus dapat dijalankan langsung setelah instalasi dependensi dari requirements.txt
  • Kode harus terorganisir rapi, contoh struktur yang disarankan:
project-root/
├── main.py
├── requirements.txt
├── README.md
├── ui/            # tampilan / file .ui
├── database/      # jika pakai SQLite
├── api/           # jika pakai API
├── models/        # model data / tabel
├── utils/         # helper (export, dll.)
└── assets/        # ikon, gambar, .qss

Instruksi Pengumpulan

1. Repository GitHub

Nama repository mengikuti format:

pv26-finalproject-namatopik
# Contoh: pv26-finalproject-inventoryapp
  • Seluruh source code (terstruktur)
  • File requirements.txt
  • File .ui jika menggunakan Qt Designer
  • README.md — judul, deskripsi singkat, nama & NIM anggota, cara menjalankan, screenshot, pembagian tugas

2. Laporan PDF

  • Deskripsi aplikasi dan latar belakang pemilihan topik
  • Pembagian tugas setiap anggota kelompok
  • Penjelasan fitur-fitur utama yang diimplementasikan
  • Penjelasan alur program secara singkat (diagram atau deskripsi)
  • Skema database atau alur data (jika relevan)
  • Screenshot tampilan antarmuka dan fungsionalitas
  • Bukti kkontribusi setiap anggota kelompok dalam mengerjakan coding
  • Kendala yang dihadapi dan cara mengatasinya

3. Submit

Kumpulkan Laporan PDF dan Link Repository GitHub melalui platform Berajah sesuai tenggat waktu yang ditentukan.
