Mini Project Pemrograman Visual

Panduan pengerjaan project akhir PySide6 dengan struktur, database, styling, dan SoC yang konsisten

Capaian Mini Project

Setelah menyelesaikan mini project ini, mahasiswa mampu:
  1. Membangun aplikasi GUI PySide6 dengan topik orisinal
  2. Mengintegrasikan SQLite untuk penyimpanan data persisten
  3. Menerapkan styling QSS melalui file eksternal
  4. Mengorganisir project dengan prinsip Separation of Concerns

Pendahuluan

Mahasiswa diwajibkan membuat aplikasi GUI menggunakan PySide6 dengan topik bebas sesuai pilihan masing-masing, sebagai implementasi materi yang telah dipelajari dari pertemuan 1 sampai 7.

Aplikasi harus menunjukkan kompetensi dalam penggunaan komponen PySide6, layout, integrasi database, styling QSS, dan arsitektur Separation of Concerns (SoC).

2. Fitur Wajib

Bagian ini cukup menjelaskan apa saja yang wajib ada dalam aplikasi. Mahasiswa tidak perlu terpaku pada contoh tampilan tertentu, tetapi semua poin berikut harus terpenuhi.

Checklist Fitur Wajib
  1. Form input minimal 5 field dengan kombinasi komponen PySide6 yang sesuai kebutuhan aplikasi.
  2. Interaksi komponen menggunakan signals dan slots.
  3. Layout rapi dan terstruktur menggunakan layout manager seperti QVBoxLayout, QHBoxLayout, QGridLayout, atau QFormLayout.
  4. Tampilan data/output menggunakan komponen seperti QTableWidget, QListWidget, QTextEdit, atau QLabel.
  5. Integrasi SQLite dengan minimal operasi Create dan Read, serta data tetap tersimpan setelah aplikasi ditutup.
  6. Menu bar yang memiliki menu Tentang Aplikasi berisi nama aplikasi, deskripsi, nama mahasiswa, dan NIM.
  7. Form tambah/edit di dialog terpisah, sedangkan jendela utama fokus menampilkan data.
  8. Minimal satu dialog konfirmasi, misalnya memakai QMessageBox.
  9. Nama dan NIM tampil di antarmuka utama dan tidak bisa diedit pengguna.
  10. Styling QSS dari file eksternal, bukan ditulis langsung di satu widget saja.
  11. Struktur project mengikuti Separation of Concerns, jadi UI, logika, database, dan style dipisah ke file/modul yang jelas.
Penekanan

Fokus penilaian bukan pada dekorasi halaman instruksi ini, tetapi pada apakah seluruh kebutuhan inti di atas benar-benar ada dan berjalan di project mahasiswa.

3. Instruksi Pengumpulan

1

Repository GitHub

Upload project ke GitHub dengan format nama repository:

format_repository: pv26-miniproject-[namatopik]-[NIM]

Contoh: pv26-miniproject-todolist-F1Dxxxxx

Repository harus berisi:

  • File source Python yang terorganisir sesuai struktur SoC
  • File .qss untuk styling
  • File database SQLite .db atau database dibuat otomatis saat aplikasi pertama kali dijalankan
  • File README.md berisi nama aplikasi, deskripsi singkat, cara menjalankan, dan teknologi yang digunakan
2

Video YouTube (Penjelasan Fitur CRUD)

Buat dan upload video ke YouTube yang menjelaskan alur pengerjaan salah satu fitur CRUD pada aplikasi Anda.

Persyaratan Video
  • Durasi 5-10 menit
  • Pilih satu fitur untuk dijelaskan: Create, Update, atau Delete
  • Sertakan perkenalan singkat, walkthrough kode, demonstrasi fitur, dan alur View → Controller → Database
  • Set visibilitas video ke Public atau Unlisted

Contoh: [PV26] Mini Project - [Topik] - [Nama Anda] - [NIM]

Contoh: [PV26] Mini Project - Todo List - Pahrul Irfan - F1Dxxxxxxx

3

Laporan PDF

Buat laporan PDF yang memuat komponen berikut:

  1. Halaman cover berisi judul, nama mahasiswa, NIM, dan kelas
  2. Deskripsi aplikasi dan tujuan project
  3. Struktur project dan penerapan SoC
  4. Desain database
  5. Fitur utama dan cara kerjanya
  6. Screenshot antarmuka
  7. Link YouTube
  8. Link GitHub
Ketentuan

Laporan PDF dikumpulkan bersama kode di dalam repository GitHub.

4

Submit via Berajah

Kumpulkan hanya link repository GitHub pada kolom pengumpulan di Berajah.

5. Catatan Akhir

Hal yang Harus Diperhatikan
  • Project wajib dikerjakan secara individu. Plagiarisme atau duplikasi kode berakibat nilai nol.
  • Aplikasi yang tidak dapat dijalankan atau tidak memenuhi fitur wajib akan mendapat pengurangan poin signifikan.
  • Semua kode harus menggunakan PySide6. Penggunaan PyQt5 atau framework lain tidak diterima.
  • Penggunaan Qt Designer bersifat optional.
Nilai Tambah
  • Kode yang bersih, mudah dibaca, dan diberi komentar seperlunya sangat dianjurkan.
  • Aplikasi yang kreatif dan unik akan mendapat apresiasi.
