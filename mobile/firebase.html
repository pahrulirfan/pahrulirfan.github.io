Praktikum: Firebase Integration

Tujuan Pembelajaran

  • Memahami ekosistem Firebase dan layanan-layanannya
  • Mampu mengonfigurasi Firebase project dan menghubungkannya ke aplikasi Android
  • Mampu mengimplementasikan Firebase Authentication (Email & Google Sign-In)
  • Mampu melakukan operasi CRUD pada Cloud Firestore
  • Mampu menggunakan Firebase Storage untuk upload dan download file
  • Memahami Firestore Security Rules dasar

Prasyarat

  • Akun Google (untuk mengakses Firebase Console)
  • Android Studio versi terbaru
  • Materi praktikum sebelumnya (Activity, Fragment, RecyclerView, Navigation, Room)
  • Koneksi internet yang stabil

Mengapa Cloud Backend?

Pada praktikum sebelumnya, semua data tersimpan secara lokal di device (SharedPreferences, Internal Storage, Room). Ini artinya data hanya ada di satu device. Dengan cloud backend seperti Firebase, data bisa:

                    LOCAL STORAGE (Praktikum sebelumnya):

                    ┌────────────┐ ┌────────────┐
                    │ Device A │ │ Device B │
                    │ ┌────────┐ │ │ ┌────────┐ │
                    │ │ Data X │ │ Data TIDAK │ │ │ │
                    │ │ Data Y │ │ bisa diakses │ │ (kosong)│ │
                    │ └────────┘ │ dari device lain │ └────────┘ │
                    └────────────┘ └────────────┘

                    CLOUD BACKEND (Firebase):

                    ┌────────────┐ ┌──────────────┐ ┌────────────┐
                    │ Device A │ │ FIREBASE │ │ Device B │
                    │ │←──→│ CLOUD │←──→│ │
                    │ Baca/Tulis│ │ ┌──────────┐ │ │ Baca/Tulis│
                    │ data │ │ │ Data X │ │ │ data │
                    │ │ │ │ Data Y │ │ │ │
                    │ │ │ └──────────┘ │ │ │
                    └────────────┘ └──────────────┘ └────────────┘
                    Data SINKRON di semua device

Firebase Overview

Apa itu Firebase?

Firebase adalah platform pengembangan aplikasi dari Google yang menyediakan berbagai layanan backend-as-a-service (BaaS). Dengan Firebase, developer tidak perlu membuat dan mengelola server sendiri. Firebase menyediakan infrastruktur siap pakai untuk autentikasi, database, storage, analytics, dan banyak lagi.

Keuntungan Menggunakan Firebase

  • Serverless — Tidak perlu membuat, deploy, atau maintain server
  • Realtime — Data otomatis sinkron ke semua client secara real-time
  • Scalable — Infrastruktur Google yang scale secara otomatis
  • Gratis untuk memulai — Free tier yang cukup untuk development dan aplikasi kecil
  • SDK resmi — Library untuk Android, iOS, Web, Flutter, dll
  • Terintegrasi — Semua layanan Firebase saling terhubung

Layanan Firebase yang Akan Dipelajari

Layanan Fungsi Analogi
Authentication Autentikasi user (login/register) Seperti satpam yang menjaga siapa yang boleh masuk
Cloud Firestore Database NoSQL cloud untuk menyimpan data terstruktur Seperti lemari arsip digital di cloud
Cloud Storage Menyimpan file (gambar, video, dokumen) Seperti Google Drive khusus untuk aplikasi

Layanan Firebase Lainnya (Tidak Dibahas)

Layanan Fungsi
Realtime Database Database realtime sederhana (pendahulu Firestore)
Cloud Messaging (FCM) Push notification
Crashlytics Crash reporting
Analytics Analisis perilaku user
Remote Config Konfigurasi app tanpa update
Cloud Functions Server-side logic
App Distribution Distribusi app untuk testing

Setup Firebase Project

Langkah 1: Buat Project di Firebase Console

1

Buka Firebase Console

Buka browser dan kunjungi console.firebase.google.com. Login dengan akun Google Anda.

2

Buat Project Baru

Klik tombol "Create a project" atau "Add project".

3

Masukkan Nama Project

Beri nama project, misalnya "StudentDirectoryApp". Firebase akan otomatis membuat project ID yang unik. Klik Continue.

4

Google Analytics (Opsional)

Anda bisa mengaktifkan atau menonaktifkan Google Analytics. Untuk praktikum ini, bisa dinonaktifkan. Klik "Create project".

5

Project Siap

Tunggu beberapa saat hingga project selesai dibuat, lalu klik "Continue" untuk masuk ke Firebase Dashboard.

Langkah 2: Hubungkan Firebase ke Android

Cara 1: Melalui Firebase Console (Manual)

1

Klik ikon Android

Di dashboard Firebase, klik ikon Android untuk menambahkan aplikasi Android.

2

Masukkan Package Name

Masukkan Android package name aplikasi Anda. Ini harus sama persis dengan applicationId di file build.gradle.kts (contoh: com.example.studentcontactapp). Anda bisa menemukan package name di file build.gradle.kts (Module: app) pada bagian namespace atau applicationId.

3

Masukkan SHA-1 (Untuk Google Sign-In)

SHA-1 fingerprint wajib untuk fitur Google Sign-In. Cara mendapatkannya:

Mendapatkan SHA-1 melalui Terminal Android Studio 
                        # Cara 1: Melalui Gradle Task di Android Studio
                        # Buka panel Gradle (kanan atas) →
                        # app → Tasks → android → signingReport
                        # SHA-1 akan muncul di panel Run

                        # Cara 2: Melalui Terminal
                        # Windows:
                        keytool -list -v -keystore "%USERPROFILE%\.android\debug.keystore" -alias androiddebugkey
                        -storepass android -keypass android

                        # macOS/Linux:
                        keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android
                        -keypass android

                        # Contoh output:
                        # SHA1: DA:39:A3:EE:5E:6B:4B:0D:32:55:BF:EF:95:60:18:90:AF:D8:07:09
4

Download google-services.json

Klik "Register app", lalu download file google-services.json.

5

Letakkan file di project

Pindahkan file google-services.json ke folder app/ di project Android Studio Anda (sejajar dengan folder src).

Lokasi google-services.json 
                        MyProject/
                        ├── app/
                        │ ├── google-services.json ← LETAKKAN DI SINI
                        │ ├── src/
                        │ │ └── main/
                        │ │ ├── java/
                        │ │ └── res/
                        │ └── build.gradle.kts
                        ├── build.gradle.kts
                        └── settings.gradle.kts

Penting: Keamanan google-services.json

  • File ini berisi konfigurasi Firebase project Anda
  • Jangan upload ke repository publik (tambahkan ke .gitignore)
  • Jika berubah konfigurasi di console, download ulang file ini

Apa itu Firebase BoM?

Firebase Bill of Materials (BoM) memastikan semua library Firebase yang Anda gunakan kompatibel satu sama lain. Dengan BoM, Anda tidak perlu menentukan versi untuk setiap library Firebase — cukup tentukan versi BoM, dan semua library akan mengikuti versi yang kompatibel.

Build dan jalankan aplikasi

Jika berhasil build tanpa error, koneksi Firebase sudah benar. Anda bisa memverifikasi di Firebase Console — di halaman project, akan ada notifikasi bahwa app berhasil terhubung.

Firebase Authentication

Pengenalan Firebase Authentication

Firebase Authentication menyediakan layanan backend, SDK, dan UI library siap pakai untuk mengautentikasi user ke aplikasi Anda. Firebase Auth mendukung berbagai metode autentikasi:

Metode Keterangan Status di Praktikum
Email/Password Register dan login dengan email & password Dibahas
Google Sign-In Login menggunakan akun Google Dibahas
Facebook Login Login menggunakan akun Facebook Tidak dibahas
Phone Number Login dengan OTP via SMS Tidak dibahas
Anonymous Login tanpa credentials (guest) Tidak dibahas

Mengaktifkan Authentication di Firebase Console

1

Di Firebase Console, pilih project Anda

2

Di panel kiri, klik Build → Authentication

3

Klik "Get started"

4

Di tab "Sign-in method", aktifkan:

  • Email/Password → Enable → Save
  • Google → Enable → pilih support email → Save

Implementasi AuthActivity untuk Email/Password

AuthActivity.kt 
                        package com.example.myapp

                        import android.content.Intent
                        import android.os.Bundle
                        import android.view.View
                        import android.widget.Button
                        import android.widget.ProgressBar
                        import android.widget.TextView
                        import android.widget.Toast
                        import androidx.appcompat.app.AppCompatActivity
                        import com.google.android.material.textfield.TextInputEditText
                        import com.google.firebase.auth.FirebaseAuth
                        import com.google.firebase.auth.FirebaseUser

                        class AuthActivity : AppCompatActivity() {

                        // Instance Firebase Auth
                        private lateinit var auth: FirebaseAuth

                        private lateinit var etEmail: TextInputEditText
                        private lateinit var etPassword: TextInputEditText
                        private lateinit var progressBar: ProgressBar

                        override fun onCreate(savedInstanceState: Bundle?) {
                        super.onCreate(savedInstanceState)
                        setContentView(R.layout.activity_auth)

                        // Inisialisasi Firebase Auth
                        auth = FirebaseAuth.getInstance()

                        // Bind views
                        etEmail = findViewById(R.id.etEmail)
                        etPassword = findViewById(R.id.etPassword)
                        progressBar = findViewById(R.id.progressBar)

                        // Tombol Login
                        findViewById<Button>(R.id.btnLogin).setOnClickListener {
                        loginWithEmail()
                        }

                        // Tombol Register
                        findViewById<Button>(R.id.btnRegister).setOnClickListener {
                        registerWithEmail()
                        }

                        // Lupa Password
                        findViewById<TextView>(R.id.tvForgotPassword).setOnClickListener {
                        resetPassword()
                        }
                        }

                        // =============================================
                        // Cek apakah user sudah login saat app dibuka
                        // =============================================
                        override fun onStart() {
                        super.onStart()
                        val currentUser = auth.currentUser
                        if (currentUser != null) {
                        // User sudah login, langsung ke MainActivity
                        navigateToMain(currentUser)
                        }
                        }

                        // =============================================
                        // REGISTER dengan Email dan Password
                        // =============================================
                        private fun registerWithEmail() {
                        val email = etEmail.text.toString().trim()
                        val password = etPassword.text.toString().trim()

                        if (!validateInput(email, password)) return

                        showLoading(true)

                        auth.createUserWithEmailAndPassword(email, password)
                        .addOnCompleteListener(this) { task ->
                        showLoading(false)

                        if (task.isSuccessful) {
                        // Register berhasil
                        val user = auth.currentUser
                        Toast.makeText(this, "Registrasi berhasil!", Toast.LENGTH_SHORT).show()
                        navigateToMain(user)
                        } else {
                        // Register gagal
                        Toast.makeText(this,
                        "Registrasi gagal: ${task.exception?.message}",
                        Toast.LENGTH_LONG
                        ).show()
                        }
                        }
                        }

                        // =============================================
                        // LOGIN dengan Email dan Password
                        // =============================================
                        private fun loginWithEmail() {
                        val email = etEmail.text.toString().trim()
                        val password = etPassword.text.toString().trim()

                        if (!validateInput(email, password)) return

                        showLoading(true)

                        auth.signInWithEmailAndPassword(email, password)
                        .addOnCompleteListener(this) { task ->
                        showLoading(false)

                        if (task.isSuccessful) {
                        // Login berhasil
                        val user = auth.currentUser
                        Toast.makeText(this, "Login berhasil!", Toast.LENGTH_SHORT).show()
                        navigateToMain(user)
                        } else {
                        // Login gagal
                        Toast.makeText(this,
                        "Login gagal: ${task.exception?.message}",
                        Toast.LENGTH_LONG
                        ).show()
                        }
                        }
                        }

                        // =============================================
                        // RESET PASSWORD
                        // =============================================
                        private fun resetPassword() {
                        val email = etEmail.text.toString().trim()
                        if (email.isEmpty()) {
                        etEmail.error = "Masukkan email untuk reset password"
                        return
                        }

                        auth.sendPasswordResetEmail(email)
                        .addOnCompleteListener { task ->
                        if (task.isSuccessful) {
                        Toast.makeText(this,
                        "Email reset password telah dikirim ke $email",
                        Toast.LENGTH_LONG
                        ).show()
                        } else {
                        Toast.makeText(this,
                        "Gagal: ${task.exception?.message}",
                        Toast.LENGTH_LONG
                        ).show()
                        }
                        }
                        }

                        // =============================================
                        // VALIDASI INPUT
                        // =============================================
                        private fun validateInput(email: String, password: String): Boolean {
                        if (email.isEmpty()) {
                        etEmail.error = "Email harus diisi"
                        etEmail.requestFocus()
                        return false
                        }

                        if (!android.util.Patterns.EMAIL_ADDRESS.matcher(email).matches()) {
                        etEmail.error = "Format email tidak valid"
                        etEmail.requestFocus()
                        return false
                        }

                        if (password.isEmpty()) {
                        etPassword.error = "Password harus diisi"
                        etPassword.requestFocus()
                        return false
                        }

                        if (password.length < 6) { etPassword.error="Password minimal 6 karakter"
                            etPassword.requestFocus() return false } return true }
                            //=============================================// NAVIGASI KE MAIN
                            //=============================================private fun navigateToMain(user:
                            FirebaseUser?) { val intent=Intent(this, MainActivity::class.java) startActivity(intent)
                            finish() } private fun showLoading(show: Boolean) { progressBar.visibility=if (show)
                            View.VISIBLE else View.GONE } }

Login dengan Google Sign-In

Google Sign-In memungkinkan user login dengan akun Google mereka. Ini adalah metode login paling populer karena hampir semua pengguna Android memiliki akun Google.

Prasyarat Google Sign-In

  • SHA-1 fingerprint sudah didaftarkan di Firebase Console
  • Google Sign-In sudah diaktifkan di Authentication → Sign-in method
  • File google-services.json sudah diperbarui setelah menambahkan SHA-1
1

Dapatkan Web Client ID

Di Firebase Console → Authentication → Sign-in method → Google → Web SDK configuration → Web client ID. Copy ID ini. Atau Anda bisa menemukannya di file google-services.json pada bagian client_type: 3.

2

Simpan Web Client ID di strings.xml

res/values/strings.xml 
                        <resources>
                        <string name="app_name">Student Directory</string>
                        <string
                        name="default_web_client_id">YOUR_WEB_CLIENT_ID.apps.googleusercontent.com</string>
                        </resources>
3

Implementasi Google Sign-In di AuthActivity

Tambahkan kode berikut di AuthActivity.kt

AuthActivity.kt - Tambahan Google Sign-In 
                        // Tambahkan import berikut di bagian atas file:
                        import androidx.activity.result.contract.ActivityResultContracts
                        import com.google.android.gms.auth.api.signin.GoogleSignIn
                        import com.google.android.gms.auth.api.signin.GoogleSignInClient
                        import com.google.android.gms.auth.api.signin.GoogleSignInOptions
                        import com.google.android.gms.common.api.ApiException
                        import com.google.firebase.auth.GoogleAuthProvider

                        class AuthActivity : AppCompatActivity() {

                        private lateinit var auth: FirebaseAuth
                        private lateinit var googleSignInClient: GoogleSignInClient

                        // ... variabel lainnya ...

                        // ActivityResultLauncher untuk Google Sign-In
                        private val googleSignInLauncher = registerForActivityResult(
                        ActivityResultContracts.StartActivityForResult()
                        ) { result ->
                        val task = GoogleSignIn.getSignedInAccountFromIntent(result.data)
                        try {
                        val account = task.getResult(ApiException::class.java)
                        // Berhasil mendapatkan akun Google, autentikasi ke Firebase
                        firebaseAuthWithGoogle(account.idToken!!)
                        } catch (e: ApiException) {
                        showLoading(false)
                        Toast.makeText(this, "Google Sign-In gagal: ${e.message}", Toast.LENGTH_SHORT).show()
                        }

Cloud Firestore

Pengenalan Cloud Firestore

Cloud Firestore adalah database NoSQL yang fleksibel, scalable, dan realtime dari Google. Berbeda dengan database relasional (SQL), Firestore menyimpan data dalam bentuk dokumen yang diorganisir dalam koleksi.

Firestore vs Realtime Database vs Room

Aspek Room (Lokal) Realtime DB Firestore
Lokasi Device lokal Cloud Cloud
Tipe SQL (Relasional) NoSQL (JSON tree) NoSQL (Dokumen)
Query SQL lengkap Terbatas Query kompleks
Offline Selalu offline Ya Ya
Realtime Tidak Ya Ya
Multi-device Tidak Ya Ya
Rekomendasi Data lokal Simple realtime Production apps

Struktur Data Firestore

Firestore menggunakan model data Collection → Document → Field.

                        Firestore Data Model:

                        ┌───────────────────────────────────────────────────────────────┐
                        │ Database (Firestore) │
                        │ │
                        │ Collection: "students" │
                        │ ├── Document: "auto_id_001" │
                        │ │ ├── name: "Ahmad Fauzi" │
                        │ │ ├── nim: "2024001" │
                        │ │ ├── prodi: "T. Informatika" │
                        │ │ ├── email: "ahmad@student.ac.id" │
                        │ │ ├── semester: 3 │
                        │ │ └── createdAt: Timestamp │
                        │ │ │
                        │ ├── Document: "auto_id_002" │
                        │ │ ├── name: "Budi Santoso" │
                        │ │ ├── nim: "2024002" │
                        │ │ ├── prodi: "S. Informasi" │
                        │ │ ├── email: "budi@student.ac.id" │
                        │ │ ├── semester: 3 │
                        │ │ └── createdAt: Timestamp │
                        │ │ │
                        │ └── Document: "auto_id_003" │
                        │ └── ... │
                        │ │
                        │ Collection: "users" │
                        │ ├── Document: "uid_firebase_001" │
                        │ │ ├── displayName: "Ahmad" │
                        │ │ ├── email: "ahmad@gmail.com" │
                        │ │ └── photoUrl: "https://..." │
                        │ └── ... │
                        │ │
                        └───────────────────────────────────────────────────────────────┘

                        Istilah Penting:
                        • Collection = Kumpulan dokumen (seperti tabel di SQL)
                        • Document = Satu unit data (seperti baris di SQL)
                        • Field = Property dalam dokumen (seperti kolom di SQL)
                        • Document ID = Identifier unik (bisa auto-generate atau custom)

Realtime Listener (Snapshot Listener)

Salah satu fitur terkuat Firestore adalah kemampuan mendengarkan perubahan data secara realtime. Artinya, jika ada user lain yang mengubah data, tampilan Anda akan otomatis ter-update tanpa perlu refresh manual.

Realtime Listener 
                        import com.google.firebase.firestore.ListenerRegistration

                        class HomeFragment : Fragment() {

                        private var snapshotListener: ListenerRegistration? = null

                        // =============================================
                        // Setup realtime listener
                        // =============================================
                        private fun setupRealtimeListener() {
                        snapshotListener = db.collection("students")
                        .orderBy("name")
                        .addSnapshotListener { snapshots, error ->
                        if (error != null) {
                        Toast.makeText(requireContext(), "Error: ${error.message}", Toast.LENGTH_SHORT).show()
                        return@addSnapshotListener
                        }

                        if (snapshots != null) {
                        val students = mutableListOf<Student>()
                        for (doc in snapshots) {
                        val student = doc.toObject(Student::class.java)
                        student.id = doc.id
                        students.add(student)
                        }

                        // Update UI otomatis saat ada perubahan
                        studentList.clear()
                        studentList.addAll(students)
                        adapter.notifyDataSetChanged()
                        }
                        }
                        }

                        override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
                        super.onViewCreated(view, savedInstanceState)
                        setupRealtimeListener() // Mulai mendengarkan
                        }

                        override fun onDestroyView() {
                        super.onDestroyView()
                        snapshotListener?.remove() // PENTING: Hentikan listener saat view dihancurkan
                        }
                        }
                        Perbandingan get() vs addSnapshotListener():

                        get() - Sekali baca:
                        App ──── request ────→ Firestore
                        App ←── response ──── Firestore
                        (selesai, tidak ada update lagi)

                        addSnapshotListener() - Realtime:
                        App ──── subscribe ──→ Firestore
                        App ←── data awal ──── Firestore
                        App ←── update 1 ──── Firestore (ada perubahan)
                        App ←── update 2 ──── Firestore (ada perubahan lagi)
                        App ←── update 3 ──── Firestore (terus menerus)
                        App ──── remove() ───→ Firestore (berhenti)

Penting: Selalu Remove Listener

Selalu hentikan snapshot listener di onDestroyView() atau onDestroy(). Jika tidak, listener akan terus berjalan dan menyebabkan memory leak serta penggunaan bandwidth yang tidak perlu.

Firestore Security Rules

Security Rules mengontrol siapa yang bisa membaca dan menulis data di Firestore. Rules wajib dikonfigurasi sebelum aplikasi di-publish.

Firestore Security Rules (di Firebase Console → Firestore → Rules) 
                        rules_version = '2';
                        service cloud.firestore {
                        match /databases/{database}/documents {

                        // ===== Rules untuk collection "students" =====
                        match /students/{studentId} {

                        // Hanya user yang login bisa membaca
                        allow read: if request.auth != null;

                        // Hanya user yang login bisa membuat data
                        // dan field userId harus sama dengan uid user
                        allow create: if request.auth != null
                        && request.resource.data.userId == request.auth.uid;

                        // Hanya pemilik data yang bisa update dan delete
                        allow update, delete: if request.auth != null
                        && resource.data.userId == request.auth.uid;
                        }

                        // ===== Rules untuk collection "users" =====
                        match /users/{userId} {
                        // User hanya bisa baca/tulis data miliknya sendiri
                        allow read, write: if request.auth != null
                        && request.auth.uid == userId;
                        }
                        }
                        }

Penjelasan Rules

  • request.auth != null → User harus sudah login
  • request.auth.uid → UID user yang sedang request
  • request.resource.data → Data yang akan ditulis
  • resource.data → Data yang sudah ada di database

                        Toast.makeText(requireContext(), "Foto dihapus", Toast.LENGTH_SHORT).show()
                        ivProfilePhoto.setImageResource(R.drawable.ic_default_avatar)

                        // Hapus URL dari Firestore juga
                        db.collection("users").document(userId)
                        .update("photoUrl", "")
                        }
                        .addOnFailureListener { e ->
                        Toast.makeText(requireContext(), "Gagal hapus: ${e.message}", Toast.LENGTH_SHORT).show()
                        }
                        }

Storage Security Rules

Storage Rules (Firebase Console → Storage → Rules) 
                        rules_version = '2';
                        service firebase.storage {
                        match /b/{bucket}/o {

                        // Profile images: hanya pemilik yang bisa upload/delete
                        match /profile_images/{userId}/{allPaths=**} {
                        allow read: if request.auth != null;
                        allow write: if request.auth != null
                        && request.auth.uid == userId
                        && request.resource.size < 5 * 1024 * 1024 // Max 5MB &&
                            request.resource.contentType.matches('image/.*'); // Hanya gambar } } }
