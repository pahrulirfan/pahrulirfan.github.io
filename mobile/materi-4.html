Pengertian Activity

Activity adalah komponen fundamental dalam aplikasi Android yang menyediakan layar (screen) untuk berinteraksi dengan pengguna. Setiap Activity merepresentasikan satu layar dengan user interface.

Karakteristik Activity

  • Setiap Activity adalah subclass dari AppCompatActivity
  • Setiap Activity harus didaftarkan di AndroidManifest.xml
  • Activity memiliki lifecycle yang dikelola oleh sistem Android
  • Aplikasi bisa memiliki multiple Activities
  • Activity bisa memiliki satu atau lebih layout

Activity Lifecycle

Activity memiliki siklus hidup (lifecycle) yang terdiri dari beberapa state. Memahami lifecycle penting agar kamu tahu di mana harus meletakkan kode — misalnya kapan mulai memutar musik, kapan menyimpan data, kapan melepas kamera.

Analogi Lifecycle

Bayangkan Activity seperti sebuah toko:

  • onCreate → toko sedang dibangun dan didekorasi
  • onStart → toko sudah buka, orang bisa melihat dari luar
  • onResume → toko aktif, pelanggan bisa masuk dan berinteraksi
  • onPause → ada gangguan kecil (telepon masuk), toko masih terlihat tapi tidak bisa dilayani penuh
  • onStop → toko tertutup, tidak terlihat dari luar
  • onRestart → toko dibuka kembali setelah sempat tutup
  • onDestroy → toko dibongkar total

Skenario Nyata Setiap Callback

Berikut penjelasan kapan tepatnya setiap method dipanggil beserta contoh kode yang relevan.

1. onCreate() — Activity pertama kali dibuat

Kapan dipanggil: Saat aplikasi pertama kali dibuka, atau setelah Activity dihancurkan lalu dibuat ulang (misal rotasi layar).

Skenario: Kamu membuka aplikasi → onCreate() langsung dipanggil.

Contoh onCreate()
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    setContentView(R.layout.activity_main)

    // Tempat yang tepat untuk:
    // - Menghubungkan layout XML
    // - Inisialisasi variabel dan view
    // - Setup RecyclerView, Adapter
    // - Mengambil data dari Intent

    val btnSubmit = findViewById<Button>(R.id.btnSubmit)
    val namaUser = intent.getStringExtra("NAMA") ?: "Tamu"

    btnSubmit.setOnClickListener {
        // aksi tombol
    }
}

2. onStart() — Activity mulai terlihat

Kapan dipanggil: Setelah onCreate(), atau saat Activity kembali terlihat setelah sebelumnya di-stop.

Skenario: Kamu menekan tombol Home lalu kembali ke aplikasi → onStart() dipanggil lagi.

Contoh onStart()
override fun onStart() {
    super.onStart()

    // Tempat yang tepat untuk:
    // - Mendaftarkan BroadcastReceiver
    // - Mulai memantau perubahan data yang perlu ditampilkan

    Log.d("Lifecycle", "Activity sekarang terlihat oleh user")
}

3. onResume() — Activity aktif dan siap digunakan

Kapan dipanggil: Setiap kali Activity berada di foreground dan siap menerima input dari user. Ini adalah state "normal" saat aplikasi digunakan.

Skenario: Notifikasi muncul lalu kamu dismiss → Activity kembali ke foreground → onResume() dipanggil.

Contoh onResume() — mulai kamera/sensor
private var mediaPlayer: MediaPlayer? = null

override fun onResume() {
    super.onResume()

    // Tempat yang tepat untuk:
    // - Mulai memutar musik/video
    // - Mengaktifkan kamera
    // - Mulai update lokasi GPS
    // - Melanjutkan animasi

    mediaPlayer?.start() // lanjutkan musik yang sempat di-pause
    Log.d("Lifecycle", "Activity aktif, user bisa berinteraksi")
}

4. onPause() — Activity kehilangan fokus (masih terlihat sebagian)

Kapan dipanggil: Saat ada Activity lain yang muncul di depan tapi tidak menutupi seluruh layar — misalnya dialog, notifikasi popup, atau saat berpindah ke Activity lain.

Skenario nyata:

  • Kamu sedang main game → ada telepon masuk → game masih terlihat di belakang → onPause() dipanggil
  • Kamu menekan tombol untuk berpindah Activity → sebelum Activity baru muncul, onPause() Activity lama dipanggil dulu
  • Muncul dialog permission (izin kamera, lokasi) → onPause() dipanggil
Contoh onPause() — pause musik dan simpan draft
override fun onPause() {
    super.onPause()

    // Tempat yang tepat untuk:
    // - Pause musik/video (karena user tidak fokus ke app ini)
    // - Simpan data sementara (draft) agar tidak hilang
    // - Pause animasi
    // - Lepas kamera jika tidak dibutuhkan di background

    mediaPlayer?.pause() // pause musik

    // Simpan teks yang sedang diketik ke SharedPreferences
    val draft = etCatatan.text.toString()
    getSharedPreferences("draft", MODE_PRIVATE)
        .edit()
        .putString("isi_catatan", draft)
        .apply()

    Log.d("Lifecycle", "onPause: Activity kehilangan fokus")

    // JANGAN lakukan operasi berat di sini (network call, database query)
    // karena onPause harus selesai cepat agar Activity baru bisa tampil
}

5. onStop() — Activity tidak terlihat sama sekali

Kapan dipanggil: Saat Activity tertutup sepenuhnya — user menekan Home, berpindah ke Activity lain yang menutupi seluruh layar, atau layar mati.

Skenario: Kamu menekan tombol Home → aplikasi masuk background → onStop() dipanggil.

Contoh onStop() — simpan data permanen dan lepas resource
override fun onStop() {
    super.onStop()

    // Tempat yang tepat untuk:
    // - Simpan data ke database secara permanen
    // - Hentikan animasi berat
    // - Unregister BroadcastReceiver
    // - Lepas resource yang tidak dibutuhkan di background

    // Simpan data ke database (Room/SQLite)
    lifecycleScope.launch {
        database.catatanDao().simpan(catatan)
    }

    Log.d("Lifecycle", "onStop: Activity tidak terlihat")
}

6. onRestart() — Activity kembali dari stopped

Kapan dipanggil: Saat user kembali ke Activity yang sebelumnya di-stop (bukan dihancurkan). Dipanggil sebelum onStart().

Skenario: Kamu menekan Home → buka aplikasi lain → kembali ke aplikasi ini → onRestart() → onStart() → onResume().

Contoh onRestart() — refresh data
override fun onRestart() {
    super.onRestart()

    // Tempat yang tepat untuk:
    // - Refresh data yang mungkin berubah saat app di background
    // - Cek apakah ada update dari server

    Log.d("Lifecycle", "onRestart: kembali dari background")
    refreshDataDariServer() // perbarui tampilan jika ada data baru
}

7. onDestroy() — Activity dihancurkan

Kapan dipanggil: Saat user menekan tombol Back (finish), atau sistem Android menghancurkan Activity karena kebutuhan memori, atau terjadi rotasi layar.

Skenario: Kamu menekan tombol Back → onDestroy() dipanggil → Activity hilang dari memori.

Contoh onDestroy() — bersihkan semua resource
override fun onDestroy() {
    super.onDestroy()

    // Tempat yang tepat untuk:
    // - Lepas semua resource yang masih dipegang
    // - Hentikan thread/coroutine yang berjalan
    // - Release MediaPlayer, kamera, koneksi database

    mediaPlayer?.release() // bebaskan resource MediaPlayer
    mediaPlayer = null

    Log.d("Lifecycle", "onDestroy: Activity dihancurkan")
}

Urutan Lifecycle Berdasarkan Skenario

Skenario Urutan Callback yang Dipanggil
Buka aplikasi pertama kali onCreate → onStart → onResume
Tekan tombol Home onPause → onStop
Kembali ke app setelah Home onRestart → onStart → onResume
Muncul dialog / notifikasi popup onPause (Activity masih terlihat sebagian)
Dismiss dialog, kembali fokus onResume
Pindah ke Activity lain Activity A: onPause → Activity B: onCreate → onStart → onResume → Activity A: onStop
Tekan tombol Back onPause → onStop → onDestroy
Rotasi layar (portrait ↔ landscape) onPause → onStop → onDestroy → onCreate → onStart → onResume

Kode Lengkap dengan Log untuk Praktikum

Buat project baru, copy kode ini ke MainActivity.kt, lalu jalankan dan perhatikan output di Logcat (View → Tool Windows → Logcat) sambil melakukan berbagai aksi di emulator.

MainActivity.kt — Lifecycle Demo Lengkap 
package com.example.lifecycledemo

import android.os.Bundle
import android.util.Log
import android.widget.TextView
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {

    companion object {
        private const val TAG = "LifecycleDemo"
    }

    // TextView untuk menampilkan log di layar
    private lateinit var tvLog: TextView
    private val logHistory = StringBuilder()

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        tvLog = findViewById(R.id.tvLog)

        addLog("onCreate() ← Buka aplikasi / rotasi layar")
        // Lakukan inisialisasi di sini
    }

    override fun onStart() {
        super.onStart()
        addLog("onStart() ← Activity mulai terlihat")
    }

    override fun onResume() {
        super.onResume()
        addLog("onResume() ← Activity aktif, siap digunakan")
        // Mulai kamera, GPS, animasi, musik di sini
    }

    override fun onPause() {
        super.onPause()
        addLog("onPause() ← Ada dialog/notifikasi, atau pindah Activity")
        // Pause musik, simpan draft di sini
        // Jangan operasi berat di sini!
    }

    override fun onStop() {
        super.onStop()
        addLog("onStop() ← Tekan Home / Activity tertutup penuh")
        // Simpan data ke database, lepas resource berat di sini
    }

    override fun onRestart() {
        super.onRestart()
        addLog("onRestart() ← Kembali dari background")
    }

    override fun onDestroy() {
        super.onDestroy()
        addLog("onDestroy() ← Tekan Back / sistem hancurkan Activity")
        // Lepas semua resource di sini
    }

    private fun addLog(message: String) {
        Log.d(TAG, message)
        logHistory.append("• $message\n")
        // Tampilkan di layar agar mudah diamati
        if (::tvLog.isInitialized) {
            tvLog.text = logHistory.toString()
        }
    }
}

Cara Mengamati Lifecycle di Logcat

  1. Jalankan aplikasi di emulator
  2. Buka Logcat di Android Studio (pojok bawah)
  3. Filter dengan kata kunci LifecycleDemo
  4. Coba lakukan aksi berikut dan perhatikan log yang muncul:
    • Tekan tombol Home
    • Kembali ke aplikasi dari Recent Apps
    • Tekan tombol Back
    • Rotasi layar (Ctrl+F11 di emulator)
    • Buka aplikasi lain lalu kembali

Kesalahan Umum

  • Menyimpan data hanya di onDestroy → berbahaya, karena onDestroy tidak selalu dipanggil jika sistem mematikan app secara paksa. Simpan data penting di onPause atau onStop.
  • Membuka koneksi di onCreate tapi tidak menutupnya → menyebabkan memory leak. Selalu pasangkan: buka di onResume, tutup di onPause.
  • Operasi berat di onPause → memperlambat transisi ke Activity berikutnya. onPause harus selesai secepat mungkin.

Pengertian Intent

Intent adalah objek messaging yang digunakan untuk meminta aksi dari komponen lain. Intent dapat digunakan untuk:

  • Memulai Activity baru
  • Memulai Service
  • Mengirim Broadcast
  • Berkomunikasi antar komponen

Jenis Intent

Jenis Keterangan Contoh Penggunaan
Explicit Intent Menentukan komponen target secara spesifik Berpindah ke Activity lain dalam aplikasi yang sama
Implicit Intent Mendeklarasikan aksi umum tanpa menentukan komponen Membuka browser, share content, kirim email

Explicit Intent

Explicit Intent digunakan ketika kita tahu persis komponen mana yang ingin dipanggil. Biasanya digunakan untuk navigasi antar Activity dalam satu aplikasi.

Analogi Explicit Intent

Bayangkan kamu mengirim surat dan sudah tahu persis alamat tujuannya. Kamu tulis nama dan alamat lengkap penerimanya — surat pasti sampai ke orang yang tepat. Itulah Explicit Intent: kamu secara langsung menyebutkan Activity mana yang ingin dibuka.

Berbeda dengan Implicit Intent yang seperti mengirim surat tanpa alamat dan membiarkan kantor pos mencari siapa yang bisa menerimanya.

Langkah 3 — Cek AndroidManifest.xml

Setiap Activity wajib terdaftar di AndroidManifest.xml. Android Studio melakukan ini otomatis, tapi penting untuk memahami strukturnya.

AndroidManifest.xml 
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:theme="@style/Theme.ExplicitIntentDemo">

        <!-- Activity pertama: MainActivity sebagai launcher (halaman awal) -->
        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>

        <!-- Activity kedua: SecondActivity (otomatis ditambahkan Android Studio) -->
        <activity
            android:name=".SecondActivity"
            android:exported="false" />

    </application>

</manifest>

Penjelasan AndroidManifest.xml

  • android:name=".MainActivity" → nama class Activity (titik di depan = shorthand untuk package name)
  • intent-filter dengan MAIN dan LAUNCHER → menandai Activity ini sebagai halaman pertama yang dibuka saat aplikasi dijalankan
  • android:exported="false" → Activity tidak bisa dibuka dari aplikasi lain (aman untuk Activity internal)
  • Jika Activity tidak terdaftar di sini, aplikasi akan crash saat mencoba membukanya

Langkah 5 — Tulis Kode MainActivity.kt

Buka MainActivity.kt dan tambahkan logika untuk berpindah ke SecondActivity saat tombol diklik.

MainActivity.kt 
package com.example.explicitintentdemo

import android.content.Intent   // ← import Intent
import android.os.Bundle
import android.widget.Button
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main) // ← hubungkan dengan layout XML

        // Ambil referensi tombol dari layout menggunakan ID-nya
        val btnGoToSecond = findViewById<Button>(R.id.btnGoToSecond)

        // Pasang listener: kode di dalam blok ini dijalankan saat tombol diklik
        btnGoToSecond.setOnClickListener {

            // Buat Explicit Intent:
            // - Parameter 1 (this)                  → context saat ini (Activity ini)
            // - Parameter 2 (SecondActivity::class.java) → Activity tujuan yang ingin dibuka
            val intent = Intent(this, SecondActivity::class.java)

            // Jalankan Intent → Android akan membuka SecondActivity
            startActivity(intent)
        }
    }
}

Langkah 7 — Tulis Kode SecondActivity.kt

Buka SecondActivity.kt dan tambahkan logika tombol kembali.

SecondActivity.kt 
package com.example.explicitintentdemo

import android.os.Bundle
import android.widget.Button
import androidx.appcompat.app.AppCompatActivity

class SecondActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_second) // ← hubungkan dengan layout XML

        val btnBack = findViewById<Button>(R.id.btnBack)

        btnBack.setOnClickListener {
            // finish() → menutup Activity ini dan kembali ke Activity sebelumnya
            // Lebih baik dari membuat Intent baru ke MainActivity
            // karena tidak membuat tumpukan Activity baru di back stack
            finish()
        }
    }
}

finish() vs Intent ke MainActivity

  • finish() → menutup Activity saat ini, kembali ke Activity sebelumnya di back stack. Direkomendasikan.
  • startActivity(Intent(this, MainActivity::class.java)) → membuat instance MainActivity baru di atas stack. Ini menyebabkan penumpukan Activity dan boros memori.

Kesalahan Umum yang Sering Terjadi

  • Activity tidak terdaftar di Manifest → aplikasi crash dengan error ActivityNotFoundException. Solusi: pastikan Activity ada di AndroidManifest.xml.
  • ID tidak cocok → findViewById mengembalikan null dan crash. Pastikan ID di XML dan Kotlin sama persis.
  • Lupa import Intent → error compile. Tambahkan import android.content.Intent di bagian atas file.
  • Menggunakan startActivity() tanpa Intent → tidak bisa, Intent wajib dibuat terlebih dahulu.

Implicit Intent

Implicit Intent tidak menentukan komponen target secara langsung, melainkan mendeklarasikan aksi yang ingin dilakukan. Sistem Android akan mencari komponen yang dapat menangani Intent tersebut.

Analogi Implicit Intent

Kalau Explicit Intent seperti mengirim surat ke alamat yang sudah kamu tahu, Implicit Intent seperti mengumumkan: "Saya butuh seseorang yang bisa membuka website!" — lalu sistem Android yang mencari aplikasi mana yang sanggup melakukannya (browser, dll).

Langkah 6 — Implementasi ImplicitIntentActivity.kt

Buka ImplicitIntentActivity.kt dan isi dengan kode berikut. Perhatikan package name sudah disesuaikan dengan project ExplicitIntentDemo.

ImplicitIntentActivity.kt 
package com.example.explicitintentdemo

import android.content.Intent
import android.net.Uri
import android.os.Bundle
import android.widget.Button
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity

class ImplicitIntentActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_implicit_intent)
        
        setupButtons()

        // Tombol kembali ke SecondActivity
        findViewById<Button>(R.id.btnBack).setOnClickListener {
            finish()
        }
    }
    
    private fun setupButtons() {
        // 1. Membuka URL di Browser
        findViewById<Button>(R.id.btnOpenBrowser).setOnClickListener {
            val url = "https://www.google.com"
            val intent = Intent(Intent.ACTION_VIEW, Uri.parse(url))
            safeStartActivity(intent, "Tidak ada browser yang tersedia")
        }
        
        // 2. Membuka Dialer untuk Panggilan Telepon
        findViewById<Button>(R.id.btnDial).setOnClickListener {
            val phoneNumber = "08123456789"
            val intent = Intent(Intent.ACTION_DIAL)
            intent.data = Uri.parse("tel:$phoneNumber")
            safeStartActivity(intent, "Tidak ada aplikasi telepon")
        }
        
        // 3. Mengirim Email
        findViewById<Button>(R.id.btnSendEmail).setOnClickListener {
            val intent = Intent(Intent.ACTION_SENDTO).apply {
                data = Uri.parse("mailto:")
                putExtra(Intent.EXTRA_EMAIL, arrayOf("contoh@email.com"))
                putExtra(Intent.EXTRA_SUBJECT, "Subject Email")
                putExtra(Intent.EXTRA_TEXT, "Isi pesan email...")
            }
            safeStartActivity(intent, "Tidak ada aplikasi email")
        }
        
        // 4. Mengirim SMS
        findViewById<Button>(R.id.btnSendSms).setOnClickListener {
            val phoneNumber = "08123456789"
            val message = "Halo, ini pesan SMS!"
            val intent = Intent(Intent.ACTION_SENDTO).apply {
                data = Uri.parse("smsto:$phoneNumber")
                putExtra("sms_body", message)
            }
            safeStartActivity(intent, "Tidak ada aplikasi SMS")
        }
        
        // 5. Share Text
        findViewById<Button>(R.id.btnShare).setOnClickListener {
            val shareText = "Ini adalah teks yang akan dibagikan!"
            val intent = Intent(Intent.ACTION_SEND).apply {
                type = "text/plain"
                putExtra(Intent.EXTRA_TEXT, shareText)
            }
            startActivity(Intent.createChooser(intent, "Bagikan melalui"))
        }
        
        // 6. Buka Lokasi di Maps
        findViewById<Button>(R.id.btnOpenMap).setOnClickListener {
            val latitude = -6.2088
            val longitude = 106.8456
            val label = "Jakarta"
            val uri = Uri.parse("geo:$latitude,$longitude?q=$latitude,$longitude($label)")
            val intent = Intent(Intent.ACTION_VIEW, uri)
            safeStartActivity(intent, "Tidak ada aplikasi maps")
        }
    }
    
    private fun safeStartActivity(intent: Intent, errorMessage: String) {
        // Cek apakah ada aplikasi yang bisa menangani intent
        if (intent.resolveActivity(packageManager) != null) {
            startActivity(intent)
        } else {
            Toast.makeText(this, errorMessage, Toast.LENGTH_SHORT).show()
        }
    }
}

Perhatian

Saat menggunakan Implicit Intent, selalu gunakan resolveActivity() atau try-catch untuk mengecek apakah ada aplikasi yang dapat menangani intent. Jika tidak ada, aplikasi akan crash.

Common Intent Actions

Action Kegunaan
ACTION_VIEW Menampilkan data (URL, lokasi, dll)
ACTION_SEND Mengirim/share data ke aplikasi lain
ACTION_SENDTO Mengirim ke alamat tertentu (email, SMS)
ACTION_DIAL Membuka dialer dengan nomor tertentu
ACTION_CALL Langsung melakukan panggilan (perlu permission)
ACTION_PICK Memilih data dari content provider
ACTION_EDIT Mengedit data

Passing Data dengan Intent

Intent dapat digunakan untuk mengirim data dari satu Activity ke Activity lain menggunakan method putExtra(). Mari mulai dari contoh paling sederhana dulu — cukup 2 input saja!

Analogi Passing Data

Bayangkan kamu mengisi formulir di loket A, lalu petugas membawa formulir itu ke loket B. Itulah yang dilakukan putExtra() — memasukkan data ke dalam "amplop" (Intent), lalu Activity tujuan membuka amplop itu dan membaca isinya.

Langkah 3 — MainActivity.kt (Pengirim)

Ambil teks dari EditText, validasi dulu, baru kirim via Intent.

MainActivity.kt
package com.example.passingdatademo

import android.content.Intent
import android.os.Bundle
import android.widget.Button
import android.widget.EditText
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        val etNama  = findViewById<EditText>(R.id.etNama)
        val etPesan = findViewById<EditText>(R.id.etPesan)
        val btnKirim = findViewById<Button>(R.id.btnKirim)

        btnKirim.setOnClickListener {
            val nama  = etNama.text.toString().trim()
            val pesan = etPesan.text.toString().trim()

            // --- VALIDASI ---
            if (nama.isEmpty()) {
                etNama.error = "Nama tidak boleh kosong!"
                etNama.requestFocus()
                return@setOnClickListener  // hentikan, jangan lanjut
            }
            if (pesan.isEmpty()) {
                etPesan.error = "Pesan tidak boleh kosong!"
                etPesan.requestFocus()
                return@setOnClickListener  // hentikan, jangan lanjut
            }

            // --- KIRIM DATA ---
            // Semua validasi lolos, baru buat Intent dan kirim data
            val intent = Intent(this, HasilActivity::class.java)
            intent.putExtra("NAMA", nama)   // kunci: "NAMA", nilai: isi EditText
            intent.putExtra("PESAN", pesan) // kunci: "PESAN", nilai: isi EditText
            startActivity(intent)
        }
    }
}

Langkah 4 — HasilActivity.kt (Penerima)

Ambil data dari Intent menggunakan kunci yang sama saat mengirim.

HasilActivity.kt
package com.example.passingdatademo

import android.os.Bundle
import android.widget.Button
import android.widget.TextView
import androidx.appcompat.app.AppCompatActivity

class HasilActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_hasil)

        // Ambil data dari Intent menggunakan kunci yang sama
        // ?: "..." adalah nilai default jika data tidak ada (null safety)
        val nama  = intent.getStringExtra("NAMA")  ?: "Tidak ada nama"
        val pesan = intent.getStringExtra("PESAN") ?: "Tidak ada pesan"

        // Tampilkan ke layar
        findViewById<TextView>(R.id.tvNama).text  = "Halo, $nama!"
        findViewById<TextView>(R.id.tvPesan).text = "Pesanmu: $pesan"

        // Tombol kembali
        findViewById<Button>(R.id.btnKembali).setOnClickListener {
            finish() // tutup Activity ini, kembali ke MainActivity
        }
    }
}

Poin Penting dari Contoh Sederhana

  • putExtra("KUNCI", nilai) → memasukkan data ke Intent sebelum dikirim
  • getStringExtra("KUNCI") → mengambil data di Activity tujuan dengan kunci yang sama
  • return@setOnClickListener → menghentikan eksekusi di dalam lambda jika validasi gagal
  • ?: "default" → nilai cadangan jika data null (Elvis operator)
  • Kunci harus sama persis saat mengirim dan menerima — typo = data tidak terbaca

FormActivity.kt - Mengirim Data 
package com.example.helloworld

import android.content.Intent
import android.os.Bundle
import android.widget.Button
import android.widget.CheckBox
import android.widget.RadioGroup
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import com.google.android.material.textfield.TextInputEditText

class FormActivity : AppCompatActivity() {
    
    private lateinit var etName: TextInputEditText
    private lateinit var etEmail: TextInputEditText
    private lateinit var etAge: TextInputEditText
    private lateinit var rgGender: RadioGroup
    private lateinit var cbAgree: CheckBox
    private lateinit var btnSubmit: Button
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_form)
        
        initViews()
        setupSubmitButton()
    }
    
    private fun initViews() {
        etName = findViewById(R.id.etName)
        etEmail = findViewById(R.id.etEmail)
        etAge = findViewById(R.id.etAge)
        rgGender = findViewById(R.id.rgGender)
        cbAgree = findViewById(R.id.cbAgree)
        btnSubmit = findViewById(R.id.btnSubmit)
    }
    
    private fun setupSubmitButton() {
        btnSubmit.setOnClickListener {
            if (validateInput()) {
                sendDataToResultActivity()
            }
        }
    }
    
    private fun validateInput(): Boolean {
        val name = etName.text.toString().trim()
        val email = etEmail.text.toString().trim()
        val age = etAge.text.toString().trim()
        
        // Validasi nama
        if (name.isEmpty()) {
            etName.error = "Nama tidak boleh kosong"
            etName.requestFocus()
            return false
        }
        
        // Validasi email
        if (email.isEmpty()) {
            etEmail.error = "Email tidak boleh kosong"
            etEmail.requestFocus()
            return false
        }
        
        if (!android.util.Patterns.EMAIL_ADDRESS.matcher(email).matches()) {
            etEmail.error = "Format email tidak valid"
            etEmail.requestFocus()
            return false
        }
        
        // Validasi usia
        if (age.isEmpty()) {
            etAge.error = "Usia tidak boleh kosong"
            etAge.requestFocus()
            return false
        }
        
        // Validasi gender
        if (rgGender.checkedRadioButtonId == -1) {
            Toast.makeText(this, "Pilih jenis kelamin", Toast.LENGTH_SHORT).show()
            return false
        }
        
        // Validasi checkbox
        if (!cbAgree.isChecked) {
            Toast.makeText(this, "Anda harus menyetujui syarat dan ketentuan", Toast.LENGTH_SHORT).show()
            return false
        }
        
        return true
    }
    
    private fun sendDataToResultActivity() {
        val name = etName.text.toString().trim()
        val email = etEmail.text.toString().trim()
        val age = etAge.text.toString().toInt()
        val gender = when (rgGender.checkedRadioButtonId) {
            R.id.rbMale -> "Laki-laki"
            R.id.rbFemale -> "Perempuan"
            else -> ""
        }
        
        // Membuat intent dengan data
        val intent = Intent(this, ResultActivity::class.java).apply {
            putExtra(ResultActivity.EXTRA_NAME, name)
            putExtra(ResultActivity.EXTRA_EMAIL, email)
            putExtra(ResultActivity.EXTRA_AGE, age)
            putExtra(ResultActivity.EXTRA_GENDER, gender)
        }
        
        startActivity(intent)
    }
}

class ResultActivity : AppCompatActivity() {
    
    companion object {
        const val EXTRA_NAME = "extra_name"
        const val EXTRA_EMAIL = "extra_email"
        const val EXTRA_AGE = "extra_age"
        const val EXTRA_GENDER = "extra_gender"
    }
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_result)

        val name = intent.getStringExtra(EXTRA_NAME) ?: "Tidak ada data"
        val email = intent.getStringExtra(EXTRA_EMAIL) ?: "Tidak ada data"
        val age = intent.getIntExtra(EXTRA_AGE, 0)
        val gender = intent.getStringExtra(EXTRA_GENDER) ?: "Tidak ada data"
        
        // Menampilkan data
        findViewById<TextView>(R.id.tvName).text = "Nama: $name"
        findViewById<TextView>(R.id.tvEmail).text = "Email: $email"
        findViewById<TextView>(R.id.tvAge).text = "Usia: $age tahun"
        findViewById<TextView>(R.id.tvGender).text = "Jenis Kelamin: $gender"
        
        // Tombol kembali
        findViewById<Button>(R.id.btnBack).setOnClickListener {
            finish()
        }
    }
}

Mengirim Object dengan Parcelable

Apa itu Parcelable?

Bayangkan kamu ingin mengirim sebuah formulir biodata dari satu ruangan ke ruangan lain. Kamu tidak bisa melempar orangnya langsung — kamu harus menuliskan semua datanya di selembar kertas, lalu kertas itu dikirim, dan di ruangan tujuan kertas itu dibaca kembali.

Itulah yang dilakukan Parcelable. Ia mengubah sebuah object Kotlin menjadi format yang bisa "dibungkus" dan dikirim melalui Intent, lalu dibuka kembali di Activity tujuan.

  • Parcelable = mekanisme bawaan Android untuk serialisasi object
  • @Parcelize = anotasi Kotlin yang otomatis menghasilkan kode Parcelable (tidak perlu tulis manual)
  • Lebih cepat dari Serializable karena dioptimasi untuk Android

Mengapa Tidak Bisa Langsung Kirim Object?

Intent hanya bisa membawa tipe data primitif (String, Int, Boolean, dll). Jika kamu punya object seperti User, kamu perlu "mengemas" object itu terlebih dahulu agar bisa dikirim.

Langkah 1 — Aktifkan Plugin Parcelize

Sebelum menggunakan @Parcelize, plugin harus diaktifkan di file konfigurasi Gradle. Buka file build.gradle.kts pada modul app:

build.gradle.kts (Module: app) 
plugins {
    id("com.android.application")
    id("org.jetbrains.kotlin.android")
    id("kotlin-parcelize") // ← Tambahkan baris ini
}
// Setelah menambahkan plugin, klik "Sync Now" di pojok kanan atas Android Studio

Langkah 2 — Buat Data Class dengan @Parcelize

Buat file baru bernama User.kt di dalam package yang sama. Cukup tambahkan anotasi @Parcelize dan implementasikan interface Parcelable — sisanya dikerjakan otomatis oleh Kotlin.

User.kt - Data Class Parcelable 
package com.example.helloworld

import android.os.Parcelable
import kotlinx.parcelize.Parcelize

// @Parcelize → anotasi yang otomatis membuat kode serialisasi
// data class  → class khusus untuk menyimpan data (otomatis punya equals, toString, copy)
// : Parcelable → menandakan class ini bisa dikirim via Intent

@Parcelize
data class User(
    val id: Int,          // ID unik pengguna
    val name: String,     // Nama lengkap
    val email: String,    // Alamat email
    val age: Int,         // Usia
    val gender: String    // Jenis kelamin
) : Parcelable

// Tanpa @Parcelize, kamu harus menulis puluhan baris kode secara manual.
// Dengan @Parcelize, cukup 1 baris anotasi!

Langkah 3 — Kirim Object dari FormActivity

Setelah data class siap, buat object User dan masukkan ke Intent menggunakan putExtra(). Gunakan sebuah konstanta sebagai "kunci" agar mudah diingat dan tidak typo.

FormActivity.kt - Mengirim Object User 
package com.example.helloworld

import android.content.Intent
import android.os.Bundle
import android.widget.Button
import androidx.appcompat.app.AppCompatActivity

class FormActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_form)

        findViewById<Button>(R.id.btnSubmit).setOnClickListener {

            // 1. Buat object User dengan data yang ingin dikirim
            val user = User(
                id = 1,
                name = "Budi Santoso",
                email = "budi@email.com",
                age = 20,
                gender = "Laki-laki"
            )

            // 2. Buat Intent menuju DetailActivity
            val intent = Intent(this, DetailActivity::class.java)

            // 3. Masukkan object ke Intent dengan putExtra(kunci, nilai)
            //    "USER_DATA" adalah kunci yang akan dipakai untuk mengambil data nanti
            intent.putExtra("USER_DATA", user)

            // 4. Jalankan Intent
            startActivity(intent)
        }
    }
}

Langkah 4 — Terima Object di DetailActivity

Di Activity tujuan, ambil object menggunakan getParcelableExtra(). Perlu pengecekan versi Android karena cara pengambilan berubah di Android 13 (API 33 / TIRAMISU).

DetailActivity.kt - Menerima Object User 
package com.example.helloworld

import android.os.Build
import android.os.Bundle
import android.widget.TextView
import androidx.appcompat.app.AppCompatActivity

class DetailActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_detail)

        // Ambil object dari Intent
        // Perlu pengecekan versi karena API berubah di Android 13 (API 33)
        val user = if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
            // Cara baru (Android 13+): lebih aman, perlu menyebutkan class-nya
            intent.getParcelableExtra("USER_DATA", User::class.java)
        } else {
            // Cara lama (Android 12 ke bawah): masih bisa dipakai tapi deprecated
            @Suppress("DEPRECATION")
            intent.getParcelableExtra("USER_DATA")
        }

        // Gunakan safe call (?.) karena user bisa null jika data tidak dikirim
        user?.let { u ->
            // Tampilkan data ke TextView
            findViewById<TextView>(R.id.tvName).text   = "Nama   : ${u.name}"
            findViewById<TextView>(R.id.tvEmail).text  = "Email  : ${u.email}"
            findViewById<TextView>(R.id.tvAge).text    = "Usia   : ${u.age} tahun"
            findViewById<TextView>(R.id.tvGender).text = "Gender : ${u.gender}"
        }
    }
}

Kenapa Ada Pengecekan Build.VERSION?

Android 13 (API 33) memperbarui cara kerja getParcelableExtra() agar lebih type-safe. Jika tidak melakukan pengecekan ini, aplikasi bisa crash di perangkat Android 13 ke atas atau menampilkan warning di versi lama. Selalu lakukan pengecekan ini saat mengambil Parcelable dari Intent.

Perbandingan: Parcelable vs Serializable

Aspek Parcelable Serializable
Kecepatan Lebih cepat Lebih lambat
Penggunaan memori Lebih efisien Lebih boros
Kemudahan penulisan Mudah dengan @Parcelize Cukup implement Serializable
Rekomendasi Android Direkomendasikan Tidak direkomendasikan
Platform Android only Java/Kotlin umum

Ringkasan Langkah-langkah

  1. Tambahkan plugin kotlin-parcelize di build.gradle.kts
  2. Buat data class dengan anotasi @Parcelize dan implementasi Parcelable
  3. Di Activity pengirim: buat object, lalu intent.putExtra("KUNCI", object)
  4. Di Activity penerima: ambil dengan getParcelableExtra("KUNCI") beserta pengecekan versi Android
  5. Gunakan ?.let { } untuk menangani kemungkinan nilai null
