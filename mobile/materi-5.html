RecyclerView

Pengenalan RecyclerView

RecyclerView adalah widget Android yang digunakan untuk menampilkan kumpulan data dalam bentuk list (daftar) atau grid secara efisien. RecyclerView merupakan penerus dari ListView dan GridView yang sudah deprecated.

Mengapa RecyclerView?

Bayangkan Anda memiliki 1000 data kontak. Jika menggunakan ScrollView dengan 1000 TextView, maka Android harus membuat 1000 View sekaligus di memori, padahal layar hanya menampilkan sekitar 10-15 item. Ini sangat boros memori dan membuat aplikasi lambat.

RecyclerView mengatasi masalah ini dengan konsep "daur ulang" (recycle). Hanya View yang terlihat di layar yang dibuat, dan saat user scroll, View yang sudah tidak terlihat akan didaur ulang untuk menampilkan data baru.

Perbandingan ListView vs RecyclerView

Fitur ListView RecyclerView
ViewHolder Pattern Opsional (manual) Wajib (built-in)
Layout Manager Hanya vertical list List, Grid, Staggered Grid
Item Animator Tidak ada Built-in animation
Item Decoration Terbatas (divider) Sangat fleksibel
Click Listener onItemClickListener Manual (lebih fleksibel)
Performa Baik Lebih baik
Status Deprecated Recommended

RecyclerView bukan satu komponen tunggal — ia adalah sistem yang terdiri dari beberapa bagian yang bekerja bersama. Pahami setiap bagiannya agar tidak bingung saat implementasi.

Menentukan bagaimana item ditampilkan. Tanpa LayoutManager, RecyclerView tidak tahu harus menampilkan item secara vertikal, horizontal, atau grid.

  • LinearLayoutManager — list vertikal / horizontal
  • GridLayoutManager — grid dengan kolom tetap
  • StaggeredGridLayoutManager — grid dengan tinggi item bervariasi

Komponen terpenting. Adapter "menerjemahkan" data (misal List<Contact>) menjadi tampilan yang bisa dirender RecyclerView. Wajib mengimplementasikan 3 method utama.

📦

✨

Opsional. Mengatur animasi default saat item ditambah (notifyItemInserted), dihapus (notifyItemRemoved), atau diperbarui (notifyItemChanged).

Langkah 3: Membuat Adapter

Adapter adalah komponen paling penting dalam RecyclerView. Adapter bertanggung jawab untuk:

  • Membuat ViewHolder (tampilan satu item)
  • Mengikat (bind) data ke ViewHolder
  • Memberitahu jumlah total item
1

Buat package adapter

Klik kanan pada package utama → New → Package → beri nama adapter

2

Buat file Kotlin baru

Klik kanan pada package adapter → New → Kotlin Class/File → beri nama ContactAdapter

adapter/ContactAdapter.kt 
package com.example.myapp.adapter

import android.view.LayoutInflater
import android.view.View
import android.view.ViewGroup
import android.widget.TextView
import androidx.recyclerview.widget.RecyclerView
import com.example.myapp.R
import com.example.myapp.model.Contact

class ContactAdapter(
    private val contactList: List<Contact>
) : RecyclerView.Adapter<ContactAdapter.ContactViewHolder>() {

    // =============================================
    // LANGKAH 1: Buat inner class ViewHolder
    // ViewHolder menyimpan referensi ke views dalam item layout
    // =============================================
    inner class ContactViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView) {
        val tvInitial: TextView = itemView.findViewById(R.id.tvInitial)
        val tvName: TextView = itemView.findViewById(R.id.tvName)
        val tvPhone: TextView = itemView.findViewById(R.id.tvPhone)
        val tvEmail: TextView = itemView.findViewById(R.id.tvEmail)
    }

    // =============================================
    // LANGKAH 2: onCreateViewHolder
    // Dipanggil saat RecyclerView membutuhkan ViewHolder baru
    // Di sini kita inflate (membuat) item layout
    // =============================================
    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): ContactViewHolder {
        val view = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_contact, parent, false)
        return ContactViewHolder(view)
    }

    // =============================================
    // LANGKAH 3: onBindViewHolder
    // Dipanggil saat RecyclerView perlu menampilkan data pada posisi tertentu
    // Di sini kita mengisi data ke views
    // =============================================
    override fun onBindViewHolder(holder: ContactViewHolder, position: Int) {
        val contact = contactList[position]

        holder.tvInitial.text = contact.name.first().uppercase()
        holder.tvName.text = contact.name
        holder.tvPhone.text = contact.phone
        holder.tvEmail.text = contact.email
    }

    // =============================================
    // LANGKAH 4: getItemCount
    // Mengembalikan jumlah total item dalam list
    // =============================================
    override fun getItemCount(): Int = contactList.size
}

LayoutManager

LayoutManager mengatur bagaimana item-item di RecyclerView ditampilkan. Ada 3 jenis LayoutManager bawaan:

1. LinearLayoutManager

Menampilkan item dalam bentuk list satu kolom (vertical) atau satu baris (horizontal).

LinearLayoutManager

rvContacts.layoutManager = LinearLayoutManager(this)

// Horizontal list
rvContacts.layoutManager = LinearLayoutManager(this, LinearLayoutManager.HORIZONTAL, false)

// Reversed vertical (item terbaru di atas)
rvContacts.layoutManager = LinearLayoutManager(this, LinearLayoutManager.VERTICAL, true)

Langkah 5: Item Click Listener

Berbeda dengan ListView yang memiliki onItemClickListener bawaan, di RecyclerView kita harus mengimplementasikan click listener secara manual. Ada beberapa cara, yang paling umum adalah menggunakan interface callback.

Modifikasi Adapter dengan Click Listener

adapter/ContactAdapter.kt (dengan Click Listener) 
package com.example.myapp.adapter

import android.view.LayoutInflater
import android.view.View
import android.view.ViewGroup
import android.widget.TextView
import androidx.recyclerview.widget.RecyclerView
import com.example.myapp.R
import com.example.myapp.model.Contact

class ContactAdapter(
    private val contactList: List<Contact>,
    private val onItemClick: (Contact) -> Unit       // Parameter callback
) : RecyclerView.Adapter<ContactAdapter.ContactViewHolder>() {

    inner class ContactViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView) {
        val tvInitial: TextView = itemView.findViewById(R.id.tvInitial)
        val tvName: TextView = itemView.findViewById(R.id.tvName)
        val tvPhone: TextView = itemView.findViewById(R.id.tvPhone)
        val tvEmail: TextView = itemView.findViewById(R.id.tvEmail)

        init {
            // Set click listener pada itemView (seluruh card)
            itemView.setOnClickListener {
                val position = adapterPosition
                if (position != RecyclerView.NO_POSITION) {
                    onItemClick(contactList[position])
                }
            }
        }
    }

    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): ContactViewHolder {
        val view = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_contact, parent, false)
        return ContactViewHolder(view)
    }

    override fun onBindViewHolder(holder: ContactViewHolder, position: Int) {
        val contact = contactList[position]
        holder.tvInitial.text = contact.name.first().uppercase()
        holder.tvName.text = contact.name
        holder.tvPhone.text = contact.phone
        holder.tvEmail.text = contact.email
    }

    override fun getItemCount(): Int = contactList.size
}

Fragment

Pengenalan Fragment

Fragment adalah komponen UI yang merepresentasikan sebagian dari tampilan dalam sebuah Activity. Fragment bisa dianggap sebagai "sub-Activity" atau "bagian modular dari Activity".

Mengapa Menggunakan Fragment?

  • Modularisasi UI - Memecah tampilan kompleks menjadi bagian-bagian yang lebih kecil dan reusable
  • Multi-pane Layout - Mendukung tampilan berbeda untuk phone dan tablet
  • Navigation - Navigasi modern di Android menggunakan Fragment, bukan Activity
  • Single Activity Architecture - Arsitektur yang direkomendasikan Google: satu Activity dengan banyak Fragment
  • Reusability - Satu Fragment bisa digunakan di berbagai Activity
    Perbandingan: Multiple Activity vs Single Activity + Fragment

    PENDEKATAN LAMA (Multiple Activity):
    ┌──────────┐  ┌──────────┐  ┌──────────┐
    │Activity A│→ │Activity B│→ │Activity C│
    │  Login   │  │   Home   │  │  Detail  │
    └──────────┘  └──────────┘  └──────────┘

    PENDEKATAN MODERN (Single Activity + Fragments):
    ┌─────────────────────────────────────────┐
    │              MainActivity                │
    │  ┌─────────────────────────────────┐    │
    │  │        NavHostFragment          │    │
    │  │  ┌────────┐ ┌────────┐ ┌─────┐ │    │
    │  │  │Fragment│→│Fragment│→│Frag.│ │    │
    │  │  │ Login  │ │  Home  │ │Detail│ │    │
    │  │  └────────┘ └────────┘ └─────┘ │    │
    │  └─────────────────────────────────┘    │
    │  ┌─────────────────────────────────┐    │
    │  │      BottomNavigationView       │    │
    │  └─────────────────────────────────┘    │
    └─────────────────────────────────────────┘

Karakteristik Fragment

  • Fragment harus di-host oleh Activity (tidak bisa berdiri sendiri)
  • Fragment memiliki lifecycle sendiri yang terikat dengan lifecycle Activity
  • Satu Activity bisa memiliki banyak Fragment
  • Fragment bisa ditambahkan, diganti, atau dihapus saat runtime

Fragment memiliki lifecycle yang mirip dengan Activity, tetapi dengan beberapa callback tambahan karena Fragment terikat dengan Activity host-nya.

Penjelasan Callback

Callback Kapan Dipanggil Kegunaan
onAttach() Fragment terhubung ke Activity Mendapatkan referensi ke Activity host
onCreate() Fragment dibuat Inisialisasi data (bukan UI)
onCreateView() Membuat tampilan Fragment Inflate layout XML
onViewCreated() View sudah dibuat Setup UI, bind views, set listeners
onStart() Fragment terlihat Mulai operasi yang terkait UI
onResume() Fragment aktif dan interaktif Resume animasi, acquire resources
onPause() Fragment kehilangan focus Pause operasi, save state
onStop() Fragment tidak terlihat Stop operasi
onDestroyView() View Fragment dihancurkan Cleanup view references
onDestroy() Fragment dihancurkan Final cleanup
onDetach() Fragment terlepas dari Activity Lepas referensi ke Activity

MainActivity.kt (dengan FragmentManager) 
package com.example.myapp

import android.os.Bundle
import android.widget.Button
import androidx.appcompat.app.AppCompatActivity
import androidx.fragment.app.Fragment

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // Tampilkan HomeFragment saat pertama kali
        if (savedInstanceState == null) {
            replaceFragment(HomeFragment())
        }

        // Tombol navigasi
        findViewById<Button>(R.id.btnHome).setOnClickListener {
            replaceFragment(HomeFragment())
        }

        findViewById<Button>(R.id.btnProfile).setOnClickListener {
            replaceFragment(ProfileFragment())
        }
    }

    // Fungsi helper untuk mengganti Fragment
    private fun replaceFragment(fragment: Fragment) {
        supportFragmentManager
            .beginTransaction()          // Mulai transaction
            .replace(                     // Ganti fragment di container
                R.id.fragmentContainer,
                fragment
            )
            .addToBackStack(null)         // Tambah ke back stack (opsional)
            .commit()                     // Eksekusi transaction
    }
}

Jenis FragmentTransaction

Method Keterangan
add(containerId, fragment) Menambahkan Fragment ke container (tidak menghapus yang lama)
replace(containerId, fragment) Mengganti Fragment yang ada dengan Fragment baru
remove(fragment) Menghapus Fragment dari container
hide(fragment) Menyembunyikan Fragment (tetap ada di memori)
show(fragment) Menampilkan Fragment yang disembunyikan
addToBackStack(name) Menambah transaction ke back stack (agar bisa back)
commit() Eksekusi transaction

Komunikasi antar Fragment

Fragment tidak boleh berkomunikasi langsung satu sama lain. Komunikasi dilakukan melalui Activity host atau menggunakan shared ViewModel.

Cara 1: Melalui Activity (Traditional)

Komunikasi Fragment → Activity → Fragment 
// ===== Interface di Fragment pengirim =====
class HomeFragment : Fragment() {

    // Interface callback
    interface OnContactSelectedListener {
        fun onContactSelected(contact: Contact)
    }

    private var listener: OnContactSelectedListener? = null

    override fun onAttach(context: Context) {
        super.onAttach(context)
        // Pastikan Activity mengimplementasikan interface
        listener = context as? OnContactSelectedListener
    }

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        val adapter = ContactAdapter(contacts) { contact ->
            // Kirim ke Activity via interface
            listener?.onContactSelected(contact)
        }
    }
}

// ===== Activity yang mengimplementasikan interface =====
class MainActivity : AppCompatActivity(), HomeFragment.OnContactSelectedListener {

    override fun onContactSelected(contact: Contact) {
        // Terima data dari HomeFragment, kirim ke DetailFragment
        val detailFragment = DetailFragment.newInstance(contact.name, contact.phone)
        replaceFragment(detailFragment)
    }
}

// ===== Fragment penerima dengan arguments =====
class DetailFragment : Fragment() {

    companion object {
        fun newInstance(name: String, phone: String): DetailFragment {
            val fragment = DetailFragment()
            val args = Bundle().apply {
                putString("ARG_NAME", name)
                putString("ARG_PHONE", phone)
            }
            fragment.arguments = args
            return fragment
        }
    }

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        val name = arguments?.getString("ARG_NAME") ?: ""
        val phone = arguments?.getString("ARG_PHONE") ?: ""

        view.findViewById<TextView>(R.id.tvName).text = name
        view.findViewById<TextView>(R.id.tvPhone).text = phone
    }
}

package com.example.myapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity
import androidx.fragment.app.Fragment
import com.google.android.material.bottomnavigation.BottomNavigationView

class BottomNavActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_bottom_nav)

        val bottomNav = findViewById<BottomNavigationView>(R.id.bottomNav)

        // Tampilkan Fragment awal
        if (savedInstanceState == null) {
            replaceFragment(HomeFragment())
        }

        // Handle bottom navigation item click
        bottomNav.setOnItemSelectedListener { item ->
            val fragment: Fragment = when (item.itemId) {
                R.id.nav_home -> HomeFragment()
                R.id.nav_search -> SearchFragment()
                R.id.nav_profile -> ProfileFragment()
                else -> HomeFragment()
            }
            replaceFragment(fragment)
            true
        }
    }

    private fun replaceFragment(fragment: Fragment) {
        supportFragmentManager.beginTransaction()
            .replace(R.id.fragmentContainer, fragment)
            .commit()
    }
}

Navigasi antar Fragment

NavController digunakan untuk melakukan navigasi. Setiap Fragment bisa mendapatkan NavController untuk berpindah ke Fragment lain.

HomeFragment.kt (dengan Navigation) 
package com.example.myapp

import android.os.Bundle
import android.view.LayoutInflater
import android.view.View
import android.view.ViewGroup
import androidx.fragment.app.Fragment
import androidx.navigation.fragment.findNavController
import androidx.recyclerview.widget.LinearLayoutManager
import androidx.recyclerview.widget.RecyclerView
import com.example.myapp.adapter.ContactAdapter
import com.example.myapp.model.Contact

class HomeFragment : Fragment() {

    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View? {
        return inflater.inflate(R.layout.fragment_home, container, false)
    }

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        val contacts = generateDummyContacts()

        val adapter = ContactAdapter(contacts) { contact ->
            // Navigasi ke DetailFragment menggunakan NavController
            val bundle = Bundle().apply {
                putString("contactName", contact.name)
                putString("contactPhone", contact.phone)
            }
            findNavController().navigate(R.id.action_home_to_detail, bundle)
        }

        val rv = view.findViewById<RecyclerView>(R.id.rvHomeContacts)
        rv.layoutManager = LinearLayoutManager(requireContext())
        rv.adapter = adapter
    }

    private fun generateDummyContacts(): List<Contact> {
        return listOf(
            Contact(1, "Ahmad Fauzi", "081234567890", "ahmad@email.com"),
            Contact(2, "Budi Santoso", "081234567891", "budi@email.com"),
            Contact(3, "Clara Wijaya", "081234567892", "clara@email.com")
        )
    }
}

Safe Args

Safe Args adalah plugin Gradle yang menghasilkan class untuk navigasi yang type-safe. Ini menghindari kesalahan key string saat passing data.

1

Tambahkan plugin Safe Args

build.gradle.kts (Project level) 
plugins {
    // ... plugin lainnya
    id("androidx.navigation.safeargs.kotlin") version "2.7.7" apply false
}
build.gradle.kts (Module: app) 
plugins {
    id("com.android.application")
    id("org.jetbrains.kotlin.android")
    id("androidx.navigation.safeargs.kotlin")  // Tambahkan ini
}
2

Gunakan Safe Args untuk navigasi

Setelah sync Gradle, Android Studio akan generate class HomeFragmentDirections dan DetailFragmentArgs berdasarkan nav_graph.xml

Navigasi dengan Safe Args 
// ===== Di HomeFragment: Navigasi dengan Safe Args =====
val adapter = ContactAdapter(contacts) { contact ->
    // Safe Args: type-safe, tidak perlu string key
    val action = HomeFragmentDirections.actionHomeToDetail(
        contactName = contact.name,
        contactPhone = contact.phone
    )
    findNavController().navigate(action)
}

// ===== Di DetailFragment: Terima args dengan Safe Args =====
class DetailFragment : Fragment() {

    // Ambil arguments menggunakan navArgs delegate
    private val args: DetailFragmentArgs by navArgs()

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        // Langsung akses property, type-safe
        view.findViewById<TextView>(R.id.tvDetailName).text = args.contactName
        view.findViewById<TextView>(R.id.tvDetailPhone).text = args.contactPhone
    }
}

MainActivity.kt (Navigation + BottomNav) 
package com.example.myapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity
import androidx.navigation.fragment.NavHostFragment
import androidx.navigation.ui.setupWithNavController
import com.google.android.material.bottomnavigation.BottomNavigationView

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // Dapatkan NavController dari NavHostFragment
        val navHostFragment = supportFragmentManager
            .findFragmentById(R.id.navHostFragment) as NavHostFragment
        val navController = navHostFragment.navController

        // Hubungkan BottomNavigationView dengan NavController
        val bottomNav = findViewById<BottomNavigationView>(R.id.bottomNav)
        bottomNav.setupWithNavController(navController)
    }
}

Keuntungan NavigationUI

Dengan setupWithNavController(), semua logika navigasi ditangani secara otomatis:

  • Klik item di BottomNav otomatis navigasi ke Fragment yang sesuai
  • Highlight item yang aktif otomatis
  • Back stack dikelola secara otomatis
  • Tidak perlu manual setOnItemSelectedListener
