RecyclerView

Pengenalan RecyclerView

RecyclerView adalah widget Android yang digunakan untuk menampilkan kumpulan data dalam bentuk list (daftar) atau grid secara efisien. RecyclerView merupakan penerus dari ListView dan GridView yang sudah deprecated.

Mengapa RecyclerView?

Bayangkan Anda memiliki 1000 data kontak. Jika menggunakan ScrollView dengan 1000 TextView, maka Android harus membuat 1000 View sekaligus di memori, padahal layar hanya menampilkan sekitar 10-15 item. Ini sangat boros memori dan membuat aplikasi lambat.

RecyclerView mengatasi masalah ini dengan konsep "daur ulang" (recycle). Hanya View yang terlihat di layar yang dibuat, dan saat user scroll, View yang sudah tidak terlihat akan didaur ulang untuk menampilkan data baru.

Perbandingan ListView vs RecyclerView

Fitur ListView RecyclerView
ViewHolder Pattern Opsional (manual) Wajib (built-in)
Layout Manager Hanya vertical list List, Grid, Staggered Grid
Item Animator Tidak ada Built-in animation
Item Decoration Terbatas (divider) Sangat fleksibel
Click Listener onItemClickListener Manual (lebih fleksibel)
Performa Baik Lebih baik
Status Deprecated Recommended

Komponen RecyclerView

RecyclerView bukan satu komponen tunggal — ia adalah sistem yang terdiri dari beberapa bagian yang bekerja bersama. Pahami setiap bagiannya agar tidak bingung saat implementasi.

Arsitektur RecyclerView
RecyclerView
LayoutManager
Mengatur posisi dan ukuran setiap item di layar
LinearGridStaggered
Adapter
Jembatan antara data dan tampilan. Membuat & mengisi ViewHolder.
ViewHolder

Menyimpan referensi view dalam satu item layout agar tidak perlu findViewById berulang

Data Source
[0][1][2][3]…
ItemDecoration
Menambahkan dekorasi visual seperti divider atau spacing antar item
ItemAnimator
Animasi saat item ditambah, dihapus, atau berubah
LayoutManager

Menentukan bagaimana item ditampilkan. Tanpa LayoutManager, RecyclerView tidak tahu harus menampilkan item secara vertikal, horizontal, atau grid.

  • LinearLayoutManager — list vertikal / horizontal
  • GridLayoutManager — grid dengan kolom tetap
  • StaggeredGridLayoutManager — grid dengan tinggi item bervariasi
Adapter

Komponen terpenting. Adapter "menerjemahkan" data (misal List<Contact>) menjadi tampilan yang bisa dirender RecyclerView. Wajib mengimplementasikan 3 method utama.

ViewHolder

Menyimpan referensi ke semua View dalam satu item layout. Dibuat sekali, lalu didaur ulang. Ini yang membuat RecyclerView jauh lebih efisien dari ListView.

ItemDecoration

Opsional. Digunakan untuk menambahkan garis pemisah (divider) atau jarak (spacing) antar item tanpa mengubah layout item itu sendiri.

ItemAnimator

Opsional. Mengatur animasi default saat item ditambah (notifyItemInserted), dihapus (notifyItemRemoved), atau diperbarui (notifyItemChanged).

Cara 1 — Langsung di build.gradle.kts (Module :app)

Cara paling sederhana. Buka file build.gradle.kts (Module :app), tambahkan di dalam blok dependencies, lalu klik Sync Now.

app/build.gradle.kts
dependencies {
    // Tambahkan dua baris ini:
    implementation("androidx.recyclerview:recyclerview:1.3.2")
    implementation("com.google.android.material:material:1.11.0")

    // Baris lainnya tetap...
    implementation("androidx.core:core-ktx:1.12.0")
    implementation("androidx.appcompat:appcompat:1.6.1")
}
Cara ini cocok untuk project kecil atau saat belajar. Versi library ditulis langsung di string.

Cara 2 — Version Catalog (libs.versions.toml)

Cara modern yang direkomendasikan Google sejak Android Studio Giraffe. Semua versi library dipusatkan di satu file libs.versions.toml, sehingga lebih mudah dikelola dan konsisten di seluruh project.

1
Buka libs.versions.toml

Ada di Gradle Scripts → libs.versions.toml. Tambahkan versi dan library di bagian yang sesuai.

2
Daftarkan versi dan library
gradle/libs.versions.toml
[versions]
# Tambahkan versi di sini
recyclerview = "1.3.2"
material     = "1.11.0"
# ... versi lainnya tetap

[libraries]
# Tambahkan library di sini
androidx-recyclerview = { group = "androidx.recyclerview", name = "recyclerview", version.ref = "recyclerview" }
material              = { group = "com.google.android.material", name = "material", version.ref = "material" }
# ... library lainnya tetap
3
Gunakan di build.gradle.kts (Module :app)

Referensikan library dengan nama alias dari toml, bukan string versi langsung.

app/build.gradle.kts (menggunakan Version Catalog)
dependencies {
    // Gunakan alias dari libs.versions.toml (autocomplete tersedia!)
    implementation(libs.androidx.recyclerview)
    implementation(libs.material)

    // Baris lainnya tetap...
}

Buat Adapter BuahAdapter.kt

Klik kanan package → New → Kotlin Class/File → Class → beri nama BuahAdapter.

BuahAdapter.kt
package com.example.daftarbuah

import android.view.LayoutInflater
import android.view.View
import android.view.ViewGroup
import android.widget.TextView
import androidx.recyclerview.widget.RecyclerView

class BuahAdapter(
    private val listBuah: List<Buah>,
    private val onItemClick: (Buah) -> Unit
) : RecyclerView.Adapter<BuahAdapter.BuahViewHolder>() {

    // ViewHolder: simpan referensi ke view dalam item_buah.xml
    inner class BuahViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView) {
        val tvInisial: TextView = itemView.findViewById(R.id.tvInisial)
        val tvNama   : TextView = itemView.findViewById(R.id.tvNama)
        val tvAsal   : TextView = itemView.findViewById(R.id.tvAsal)
        val tvHarga  : TextView = itemView.findViewById(R.id.tvHarga)
    }

    // Inflate item_buah.xml dan buat ViewHolder
    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): BuahViewHolder {
        val view = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_buah, parent, false)
        return BuahViewHolder(view)
    }

    // Isi data ke ViewHolder sesuai posisi
    override fun onBindViewHolder(holder: BuahViewHolder, position: Int) {
        val buah = listBuah[position]
        holder.tvInisial.text = buah.nama.first().uppercase()  // huruf pertama
        holder.tvNama.text    = buah.nama
        holder.tvAsal.text    = buah.asal
        holder.tvHarga.text   = buah.harga
        holder.itemView.setOnClickListener { onItemClick(buah) }
    }

    // Jumlah total item
    override fun getItemCount(): Int = listBuah.size
}
Tiga method wajib: onCreateViewHolder, onBindViewHolder, getItemCount. Huruf pertama nama buah diambil dengan buah.nama.first().uppercase().

Setup RecyclerView di MainActivity MainActivity.kt

Buka MainActivity.kt, ganti isinya dengan kode berikut.

MainActivity.kt
package com.example.daftarbuah

import android.os.Bundle
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import androidx.recyclerview.widget.LinearLayoutManager
import androidx.recyclerview.widget.RecyclerView

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // 1. Siapkan data
        val daftarBuah = listOf(
            Buah("Apel",       "Malang, Jawa Timur",    "Rp 15.000/kg"),
            Buah("Pisang",     "Lampung",                "Rp 8.000/kg"),
            Buah("Jeruk",      "Karo, Sumatera Utara",  "Rp 12.000/kg"),
            Buah("Mangga",     "Probolinggo, Jawa Timur","Rp 20.000/kg"),
            Buah("Semangka",   "Indramayu, Jawa Barat", "Rp 6.000/kg"),
            Buah("Anggur",     "Bali",                  "Rp 35.000/kg"),
            Buah("Durian",     "Medan, Sumatera Utara", "Rp 50.000/kg"),
            Buah("Rambutan",   "Binjai, Sumatera Utara","Rp 10.000/kg")
        )

        // 2. Buat adapter — klik item tampilkan Toast
        val adapter = BuahAdapter(daftarBuah) { buah ->
            Toast.makeText(this, "${buah.nama} dari ${buah.asal}", Toast.LENGTH_SHORT).show()
        }

        // 3. Pasang ke RecyclerView
        val rvBuah = findViewById<RecyclerView>(R.id.rvBuah)
        rvBuah.layoutManager = LinearLayoutManager(this)
        rvBuah.adapter = adapter
    }
}
Jalankan aplikasi sekarang — list buah sudah muncul. Klik salah satu item, Toast akan tampil. Lanjut ke langkah 5 untuk variasi tampilan.

DetailActivity.kt
package com.example.daftarbuah

import android.os.Bundle
import android.widget.Button
import android.widget.TextView
import androidx.appcompat.app.AppCompatActivity

class DetailActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_detail)

        // Terima data dari Intent
        val nama  = intent.getStringExtra("NAMA")  ?: ""
        val asal  = intent.getStringExtra("ASAL")  ?: ""
        val harga = intent.getStringExtra("HARGA") ?: ""

        // Tampilkan data
        findViewById<TextView>(R.id.tvDetailInisial).text = nama.first().uppercase()
        findViewById<TextView>(R.id.tvDetailNama).text    = nama
        findViewById<TextView>(R.id.tvDetailAsal).text    = "Asal: $asal"
        findViewById<TextView>(R.id.tvDetailHarga).text   = harga

        // Tombol kembali
        findViewById<Button>(R.id.btnKembali).setOnClickListener { finish() }
    }
}

Fragment

Pengenalan Fragment

Fragment adalah komponen UI yang merepresentasikan sebagian dari tampilan dalam sebuah Activity. Fragment bisa dianggap sebagai "sub-Activity" atau "bagian modular dari Activity".

Mengapa Menggunakan Fragment?

  • Modularisasi UI - Memecah tampilan kompleks menjadi bagian-bagian yang lebih kecil dan reusable
  • Multi-pane Layout - Mendukung tampilan berbeda untuk phone dan tablet
  • Navigation - Navigasi modern di Android menggunakan Fragment, bukan Activity
  • Single Activity Architecture - Arsitektur yang direkomendasikan Google: satu Activity dengan banyak Fragment
  • Reusability - Satu Fragment bisa digunakan di berbagai Activity

Karakteristik Fragment

  • Fragment harus di-host oleh Activity (tidak bisa berdiri sendiri)
  • Fragment memiliki lifecycle sendiri yang terikat dengan lifecycle Activity
  • Satu Activity bisa memiliki banyak Fragment
  • Fragment bisa ditambahkan, diganti, atau dihapus saat runtime

Tampilkan Fragment dari Activity MainActivity.kt

Activity bertugas memasukkan Fragment ke container menggunakan FragmentManager. Saat tombol diklik, Fragment lama diganti dengan yang baru.

MainActivity.kt
package com.example.fragmentapp

import android.os.Bundle
import android.widget.Button
import androidx.appcompat.app.AppCompatActivity
import androidx.fragment.app.Fragment

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // Tampilkan HomeFragment saat pertama kali dibuka
        loadFragment(HomeFragment())

        // Tombol Home
        findViewById<Button>(R.id.btnHome).setOnClickListener {
            loadFragment(HomeFragment())
        }

        // Tombol Profile
        findViewById<Button>(R.id.btnProfile).setOnClickListener {
            loadFragment(ProfileFragment())
        }
    }

    // Helper: ganti Fragment di dalam container
    private fun loadFragment(fragment: Fragment) {
        supportFragmentManager
            .beginTransaction()                    // mulai transaksi
            .replace(R.id.fragmentContainer, fragment) // ganti isi container
            .commit()                              // jalankan
    }
}

Kirim Data ke Fragment (Arguments) ProfileFragment.kt

Fragment tidak menerima data lewat constructor. Gunakan Bundle arguments — cara standar mengirim data ke Fragment.

ProfileFragment.kt — dengan arguments
class ProfileFragment : Fragment() {

    // companion object: factory method untuk buat Fragment dengan data
    companion object {
        fun newInstance(nama: String, nim: String): ProfileFragment {
            val fragment = ProfileFragment()
            fragment.arguments = Bundle().apply {
                putString("NAMA", nama)
                putString("NIM",  nim)
            }
            return fragment
        }
    }

    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?, savedInstanceState: Bundle?
    ): View? = inflater.inflate(R.layout.fragment_profile, container, false)

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        // Ambil data dari arguments
        val nama = arguments?.getString("NAMA") ?: "Tidak diketahui"
        val nim  = arguments?.getString("NIM")  ?: "-"

        view.findViewById<TextView>(R.id.tvNama).text = nama
        view.findViewById<TextView>(R.id.tvNim).text  = nim
    }
}

Di MainActivity.kt, ubah cara memanggil ProfileFragment:

MainActivity.kt — kirim data ke Fragment
// Sebelum:

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        loadFragment(HomeFragment())  // default

        findViewById<BottomNavigationView>(R.id.bottomNav)
            .setOnItemSelectedListener { item ->
                when (item.itemId) {
                    R.id.nav_home    -> loadFragment(HomeFragment())
                    R.id.nav_profile -> loadFragment(ProfileFragment.newInstance("Ahmad", "2024001"))
                }
                true
            }
    }

        supportFragmentManager.beginTransaction()
            .replace(R.id.fragmentContainer, fragment)
            .commit()
    }
}

Navigation Component

Pengenalan Navigation Component

Navigation Component adalah library Jetpack yang menyederhanakan implementasi navigasi dalam aplikasi Android. Library ini menyediakan framework yang konsisten untuk navigasi antar Fragment.

Komponen Utama

Komponen Fungsi
Navigation Graph (nav_graph.xml) File XML yang mendefinisikan semua destinasi (Fragment) dan aksi navigasi (action)
NavHostFragment Container khusus di layout Activity yang menampilkan Fragment dari navigation graph
NavController Object yang mengelola navigasi dalam NavHostFragment

Dari Fragment Manual ke Navigation Component

Di materi Fragment, kita sudah bisa berpindah antar Fragment menggunakan FragmentManager.beginTransaction(). Navigation Component menggantikan cara itu dengan pendekatan yang lebih terstruktur dan aman.

Buat Navigation Graph res/navigation/nav_graph.xml

Navigation Graph adalah peta navigasi aplikasi — mendefinisikan semua Fragment (destination) dan jalur perpindahan antar Fragment (action). Semua alur navigasi terpusat di satu file ini.

Buat folder: klik kanan res → New → Android Resource Directory → type: navigation. Lalu buat file nav_graph.xml di dalamnya.

<?xml version="1.0" encoding="utf-8"?>
<navigation
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/nav_graph"
    app:startDestination="@id/homeFragment">

    <!-- Fragment pertama yang ditampilkan (start destination) -->
    <fragment
        android:id="@+id/homeFragment"
        android:name="com.example.navapp.HomeFragment"
        android:label="Home">

        <!-- Action: jalur dari Home ke Detail -->
        <action
            android:id="@+id/action_home_to_detail"
            app:destination="@id/detailFragment" />

    <!-- Fragment tujuan navigasi -->
    <fragment
        android:id="@+id/detailFragment"
        android:name="com.example.navapp.DetailFragment"
        android:label="Detail">

        <!-- Data yang dikirim dari Home ke Detail -->
        <argument android:name="nama"  app:argType="string" />
        <argument android:name="nim"   app:argType="string" />
    </fragment>

    <!-- Fragment untuk BottomNav -->
    <fragment
        android:id="@+id/profileFragment"
        android:name="com.example.navapp.ProfileFragment"

Setup MainActivity — Satu Baris Saja MainActivity.kt

Ini perbedaan paling mencolok dari cara Fragment manual. MainActivity sekarang jauh lebih sederhana — tidak perlu lagi menulis logika beginTransaction() atau mengelola Fragment secara manual.

MainActivity.kt
package com.example.navapp

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity
import androidx.navigation.fragment.NavHostFragment
import androidx.navigation.ui.setupWithNavController
import com.google.android.material.bottomnavigation.BottomNavigationView

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // Ambil NavController dari NavHostFragment
        val navHost = supportFragmentManager
            .findFragmentById(R.id.navHostFragment) as NavHostFragment
        val navController = navHost.navController

        // Hubungkan BottomNav dengan NavController — satu baris, selesai!
        findViewById<BottomNavigationView>(R.id.bottomNav)
            .setupWithNavController(navController)
    }
}
Buat juga res/menu/bottom_nav_menu.xml dengan item id="@+id/homeFragment" dan id="@+id/profileFragment" — ID harus sama persis dengan ID di nav_graph.xml.

Navigasi dari HomeFragment ke DetailFragment HomeFragment.kt + DetailFragment.kt

Di Fragment, navigasi dilakukan lewat findNavController().navigate() — bukan supportFragmentManager lagi. Data dikirim via Bundle, sama seperti sebelumnya.

HomeFragment.kt — navigasi ke Detail
class HomeFragment : Fragment() {

    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?, savedInstanceState: Bundle?
    ): View? = inflater.inflate(R.layout.fragment_home, container, false)

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        val daftarMahasiswa = listOf(
            Mahasiswa("Ahmad Fauzi",  "2024001", "Teknik Informatika"),
            Mahasiswa("Budi Santoso", "2024002", "Sistem Informasi")
        )

        val adapter = MahasiswaAdapter(daftarMahasiswa) { mhs ->
            // Kirim data via Bundle, navigasi via NavController
            val bundle = Bundle().apply {
                putString("nama", mhs.nama)
                putString("nim",  mhs.nim)
            }
            findNavController().navigate(R.id.action_home_to_detail, bundle)
        }

        val rv = view.findViewById<RecyclerView>(R.id.rvMahasiswa)
        rv.layoutManager = LinearLayoutManager(requireContext())
        rv.adapter = adapter
    }
}
DetailFragment.kt — terima data
class DetailFragment : Fragment() {

    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?, savedInstanceState: Bundle?
    ): View? = inflater.inflate(R.layout.fragment_detail, container, false)

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        // Terima data dari arguments (sama seperti cara Fragment manual)
        val nama = arguments?.getString("nama") ?: ""
        val nim  = arguments?.getString("nim")  ?: ""

        view.findViewById<TextView>(R.id.tvNama).text = nama
        view.findViewById<TextView>(R.id.tvNim).text  = nim

        // Kembali ke Fragment sebelumnya
        view.findViewById<Button>(R.id.btnKembali).setOnClickListener {
            findNavController().navigateUp()
        }
    }
}

supportFragmentManager.beginTransaction()... findNavController().navigate(R.id.action_...)

activity?.onBackPressed() findNavController().navigateUp()
Jalankan aplikasi — BottomNav sudah berfungsi otomatis, dan klik item list akan membuka DetailFragment.

Safe Args — Kirim Data Lebih Aman Opsional tapi direkomendasikan

Cara Bundle di langkah 5 sudah berfungsi, tapi rawan typo pada key string. Safe Args menghasilkan class otomatis sehingga pengiriman data menjadi type-safe — IDE akan langsung error jika ada yang salah.

Tambahkan plugin di build.gradle.kts (Project level):

build.gradle.kts (Project — root)
plugins {
    // ... plugin lainnya tetap
    id("androidx.navigation.safeargs.kotlin") version "2.7.7" apply false
}

Aktifkan di build.gradle.kts (Module :app):

app/build.gradle.kts
plugins {
    id("com.android.application")
    id("org.jetbrains.kotlin.android")
    id("androidx.navigation.safeargs.kotlin")  // tambahkan ini
}

Setelah Sync, Android Studio generate class otomatis. Ganti Bundle manual dengan Safe Args:

HomeFragment.kt — dengan Safe Args
// Sebelum (Bundle manual — rawan typo):
val bundle = Bundle().apply {
    putString("nama", mhs.nama)
    putString("nim",  mhs.nim)
}
findNavController().navigate(R.id.action_home_to_detail, bundle)

// Sesudah (Safe Args — type-safe, autocomplete):
val action = HomeFragmentDirections.actionHomeToDetail(
    nama = mhs.nama,
    nim  = mhs.nim
)
findNavController().navigate(action)
DetailFragment.kt — dengan Safe Args
// Sebelum (Bundle manual):
val nama = arguments?.getString("nama") ?: ""

// Sesudah (Safe Args — langsung akses property):
private val args: DetailFragmentArgs by navArgs()

override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
    super.onViewCreated(view, savedInstanceState)
    view.findViewById<TextView>(R.id.tvNama).text = args.nama
    view.findViewById<TextView>(R.id.tvNim).text  = args.nim
}
Safe Args bersifat opsional — aplikasi tetap berjalan tanpa Safe Args. Gunakan jika project sudah cukup besar atau bekerja dalam tim.

Ubah navigasi manual (FragmentManager) menjadi Navigation Component dengan NavGraph dan NavController.

Langkah-langkah

  1. Tambahkan Dependency Navigation

    Tambahkan navigation-fragment-ktx dan navigation-ui-ktx di build.gradle.kts, lalu Sync.

  2. Buat Navigation Graph nav_graph.xml

    Buat folder res/navigation dan file nav_graph.xml. Daftarkan semua Fragment sebagai destination. Set homeFragment sebagai start destination.

  3. Konversi DetailActivity menjadi DetailFragment

    Buat DetailFragment + fragment_detail.xml. Pindahkan logic dari DetailStudentActivity ke sini.

  4. Tambahkan Action & Arguments di NavGraph

    Tambahkan action dari homeFragment ke detailFragment. Tambahkan arguments (nama, NIM, prodi, dll) di detailFragment.

  5. Update activity_main.xml

    Ganti FrameLayout dengan FragmentContainerView yang menunjuk ke NavHostFragment dan nav_graph.

  6. Setup Navigation + BottomNav di MainActivity

    Gunakan setupWithNavController(). Pastikan ID item menu di bottom_nav_menu.xml sama dengan ID Fragment di nav_graph.xml.

  7. Navigasi dari HomeFragment ke DetailFragment

    Saat item RecyclerView diklik, gunakan findNavController().navigate() dengan Bundle arguments.

  8. Terima Data di DetailFragment

    Ambil data dari arguments (bukan Intent) dan tampilkan di layout.

  9. ⭐ Bonus: Safe Args

    Tambahkan plugin Safe Args dan refactor navigasi menggunakan generated Directions & Args class — lebih type-safe dari Bundle manual.
