Sistem Penilaian

Topik: Control Flow (if, when), Range

Deskripsi Tugas

Buat program sistem penilaian yang mengkonversi nilai angka menjadi huruf dan memberikan keterangan kelulusan.

Instruksi

  1. Buat file baru dengan nama Tugas3_Penilaian.kt
  2. Minta user memasukkan nilai UTS, UAS, dan Tugas
  3. Hitung nilai akhir dengan bobot: UTS 30%, UAS 40%, Tugas 30%
  4. Konversi nilai akhir ke huruf menggunakan when dengan range
  5. Tentukan status kelulusan (Lulus jika nilai >= 60)

Konversi Nilai:

Range Nilai Huruf Keterangan
85 - 100 A Sangat Baik
70 - 84 B Baik
60 - 69 C Cukup
50 - 59 D Kurang
0 - 49 E Sangat Kurang

Contoh Output yang Diharapkan:

Output
===== SISTEM PENILAIAN ===== Masukkan Nama Mahasiswa: Budi Santoso Masukkan Nilai UTS (0-100): 75 Masukkan Nilai UAS (0-100): 80 Masukkan Nilai Tugas (0-100): 85 ===== HASIL PENILAIAN ===== Nama : Budi Santoso Nilai UTS : 75 (Bobot 30%) Nilai UAS : 80 (Bobot 40%) Nilai Tugas : 85 (Bobot 30%) ----------------------------- Nilai Akhir : 80.0 Grade : B Keterangan : Baik Status : LULUS Selamat! Anda dinyatakan LULUS.

Hint

  • Rumus nilai akhir: (uts * 0.3) + (uas * 0.4) + (tugas * 0.3)
  • Gunakan when dengan range: in 85..100 -> "A"
  • Validasi input nilai harus dalam range 0-100

Manajemen Data dengan Collections

Topik: List, Map, Collection Operations

Deskripsi Tugas

Buat program untuk mengelola data nilai mahasiswa menggunakan Collections dan operasinya.

Instruksi

  1. Buat file baru dengan nama Tugas7_Collections.kt
  2. Buat data class NilaiMahasiswa
  3. Buat list berisi minimal 10 data mahasiswa
  4. Implementasikan berbagai operasi collection

Data Class:

Data Class
Kotlin 
data class NilaiMahasiswa(
    val nim: String,
    val nama: String,
    val mataKuliah: String,
    val nilai: Int
)
Buat data class ini
Template

Operasi yang harus diimplementasikan:

  1. Tampilkan semua data mahasiswa
  2. Filter mahasiswa yang lulus (nilai >= 70)
  3. Filter mahasiswa yang tidak lulus (nilai < 70)
  4. Hitung rata-rata nilai keseluruhan
  5. Cari mahasiswa dengan nilai tertinggi
  6. Cari mahasiswa dengan nilai terendah
  7. Urutkan berdasarkan nilai (ascending dan descending)
  8. Kelompokkan mahasiswa berdasarkan grade (A/B/C/D/E)
  9. Hitung jumlah mahasiswa per grade
  10. Cari mahasiswa berdasarkan nama (contains)

Contoh Output:

Output (sebagian)
===== DATA NILAI MAHASISWA ===== No NIM Nama MataKuliah Nilai 1 2024001 Budi Santoso Pemrograman 85 2 2024002 Ani Wijaya Pemrograman 92 3 2024003 Citra Dewi Pemrograman 68 ... ===== STATISTIK ===== Total Mahasiswa : 10 Rata-rata Nilai : 76.5 Nilai Tertinggi : 92 (Ani Wijaya) Nilai Terendah : 45 (Dani Pratama) ===== MAHASISWA LULUS ===== 1. Budi Santoso - 85 (B) 2. Ani Wijaya - 92 (A) ... ===== JUMLAH PER GRADE ===== Grade A: 2 mahasiswa Grade B: 3 mahasiswa Grade C: 2 mahasiswa Grade D: 1 mahasiswa Grade E: 2 mahasiswa

Hint

  • filter { it.nilai >= 70 } untuk filter lulus
  • map { it.nilai }.average() untuk rata-rata
  • maxByOrNull { it.nilai } untuk nilai tertinggi
  • sortedByDescending { it.nilai } untuk urut descending
  • groupBy { getGrade(it.nilai) } untuk grouping

Sistem Toko Online Optional

Topik: Semua materi + Sealed Class + Lambda

Deskripsi Tugas

Buat sistem toko online sederhana yang menggabungkan semua konsep yang telah dipelajari.

Tugas Bonus

Tugas ini bersifat opsional. Nilai bonus akan ditambahkan ke nilai tugas reguler.

Fitur yang harus ada:

  1. Data Class untuk Produk, Customer, dan Order
  2. Sealed Class untuk OrderStatus (Pending, Processing, Shipped, Delivered, Cancelled)
  3. Sealed Class untuk PaymentMethod (Cash, Transfer, EWallet)
  4. Interface Discountable untuk produk yang bisa didiskon
  5. Higher-Order Function untuk menghitung total dengan custom discount calculator
  6. Collection Operations untuk filter, sort, dan grouping produk
  7. Null Safety untuk handle data yang mungkin null

Fitur Sistem:

  • Tambah produk ke keranjang
  • Hapus produk dari keranjang
  • Hitung total belanja
  • Terapkan diskon (persentase atau nominal)
  • Proses checkout dengan pilihan pembayaran
  • Tracking status pesanan
  • Tampilkan riwayat pesanan

Struktur yang Disarankan

  • Produk(id, nama, harga, kategori, stok)
  • CartItem(produk, jumlah)
  • Customer(id, nama, email, alamat?)
  • Order(id, customer, items, status, paymentMethod, totalHarga)
  • sealed class OrderStatus
  • sealed class PaymentMethod
  • class ShoppingCart
  • class TokoOnline
