Praktikum: Pengenalan Android Development

Tujuan Pembelajaran

  • Memahami lingkungan pengembangan Android
  • Mampu membuat aplikasi Hello World
  • Memahami struktur UI XML dan berbagai jenis Layout
  • Mengenal dan menggunakan Basic Widgets
  • Mampu membuat Style dan Theme
  • Memahami konsep Localization
  • Memahami konsep Activity dan Lifecycle
  • Mampu mengimplementasikan Intent untuk navigasi

Tugas Praktikum

Ketentuan Pengerjaan & Pengumpulan

  • Kerjakan salah satu saja dari soal yang tersedia (Tugas 1 atau Tugas 2)
  • Silakan membuat aplikasi sebaik mungkin — kreativitas dan kerapian tampilan akan menjadi nilai tambah
  • Kumpulkan ke repository GitHub dengan nama repo: T2-mobile
  • Link GitHub dikumpulkan melalui eLearning
  • Pastikan repository bersifat public dan project bisa di-build tanpa error
  • Sertakan file README.md berisi nama, NIM, deskripsi aplikasi, dan screenshot

Tugas 1: Aplikasi Form Biodata

Buatlah aplikasi dengan spesifikasi sebagai berikut:

  1. Halaman pertama (MainActivity) berisi form input:
    • Nama lengkap (EditText)
    • NIM (EditText dengan inputType number)
    • Program Studi (Spinner dengan pilihan)
    • Jenis Kelamin (RadioButton)
    • Hobi (CheckBox - minimal 3 pilihan)
  2. Implementasikan validasi input yang baik
  3. Gunakan Style untuk konsistensi tampilan
  4. Tombol "Tampilkan Profil" untuk berpindah ke halaman kedua
  5. Halaman kedua (ProfileActivity) menampilkan data yang diinput dengan tampilan yang menarik (gunakan CardView)
  6. Implementasikan localization (minimal 2 bahasa)

Tugas 2: Aplikasi Android dengan Activities, Intents & UI XML

Kembangkan sebuah aplikasi Android sederhana menggunakan Kotlin yang menunjukkan penggunaan Activities, Intents, dan desain UI berbasis XML. Silakan kembangkan aplikasi ini sebaik dan semenarik mungkin.

Fitur Aplikasi

Aplikasi minimal memiliki 2 halaman/screen:

  1. Registrasi Screen — berisi field input:
    • Nama lengkap (EditText)
    • NIM (EditText dengan inputType number)
    • Program Studi (Spinner/dropdown dengan pilihan)
    • Jenis Kelamin (RadioButton)
    • Hobi (CheckBox — minimal 3 pilihan)
    • Tombol "Tampilkan Profil" untuk berpindah ke halaman kedua
  2. Profil Screen — menampilkan data yang diinput dengan tampilan yang menarik (gunakan CardView jika memungkinkan)

Persyaratan Fungsional

  • Gunakan Layout Android (LinearLayout, ConstraintLayout, atau FrameLayout) untuk merancang semua komponen UI
  • Implementasikan navigasi antar activity menggunakan Intent eksplisit
  • Kirim data registrasi (nama, NIM, dll.) dari Halaman Registrasi ke Halaman Profil menggunakan Parcelable
  • Validasi field input — nama dan NIM tidak boleh kosong

Catatan Penting

  • Plagiasi akan mendapat nilai 0
  • Pahami setiap baris kode yang Anda tulis
  • Jangan ragu bertanya jika ada kesulitan
