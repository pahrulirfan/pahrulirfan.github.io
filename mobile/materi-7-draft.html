Praktikum: Konsumsi REST API di Android

Tujuan Pembelajaran

  • Memahami konsep REST API dan cara kerja komunikasi client-server
  • Mampu mengintegrasikan Retrofit untuk melakukan HTTP request di Android
  • Mampu melakukan operasi CRUD (GET, POST, PUT, DELETE) via API
  • Mampu menampilkan data dari API ke RecyclerView
  • Mampu mengimplementasikan login berbasis token (JWT/Bearer)

Prasyarat

Pastikan Anda sudah memahami materi dari praktikum sebelumnya:

  • RecyclerView dan Adapter (Materi 5)
  • Room Database dan operasi CRUD lokal (Materi 6)
  • Kotlin dasar: data class, lambda, coroutines
  • Layout XML: LinearLayout, RecyclerView, ProgressBar

Dari CRUD Lokal ke CRUD Online

Di materi sebelumnya, data disimpan di database lokal (Room) yang hanya bisa diakses dari satu device. Dengan REST API, data disimpan di server — bisa diakses dari device mana saja, oleh siapa saja yang punya izin.

Apa itu Retrofit?

Retrofit adalah library buatan Square yang menjadi standar de-facto untuk melakukan HTTP request di Android. Retrofit mengubah API endpoint menjadi interface Kotlin yang mudah dipanggil, menangani serialisasi JSON secara otomatis, dan mendukung Coroutines untuk operasi async.

Retrofit
HTTP Client
Mengubah API endpoint menjadi fungsi Kotlin yang bisa langsung dipanggil
Gson
JSON Parser
Mengkonversi JSON dari server menjadi data class Kotlin secara otomatis
Coroutines
Async
Menjalankan request di background thread tanpa memblokir UI
OkHttp
HTTP Engine
Engine HTTP di balik Retrofit, menangani koneksi, timeout, dan interceptor

API yang Digunakan di Praktikum

Praktikum ini menggunakan API Vehicle yang tersedia di api.pahrul.my.id. API ini menyediakan data kendaraan (model, tipe, produsen) dan mendukung operasi CRUD lengkap. Semua endpoint bersifat publik — tidak memerlukan autentikasi.

Endpoint Method Fungsi Response
api.pahrul.my.id/vehicles GET Ambil semua data vehicle 200 — array data
api.pahrul.my.id/vehicles/{id} GET Ambil satu vehicle berdasarkan ID 200 / 404
api.pahrul.my.id/vehicles POST Tambah vehicle baru 201 / 422
api.pahrul.my.id/vehicles/{id} PUT Update data vehicle 200 / 404
api.pahrul.my.id/vehicles/{id} DELETE Hapus vehicle 200 / 404

Struktur Data Vehicle

Setiap response dari API membungkus data dalam format standar dengan field success, message, dan data.

Contoh response GET /vehicles 
{
  "success": true,
  "message": "Data vehicle ditemukan",
  "data": [
    {
      "id": 1,
      "model": "Civic",
      "type": "Sedan",
      "manufacturer": "Honda",
      "created_at": "2026-04-30T10:00:00.000000Z",
      "updated_at": "2026-04-30T10:00:00.000000Z"
    }
  ]
}
Contoh request body POST /vehicles 
{
  "model": "Civic",
  "type": "Sedan",
  "manufacturer": "Honda"
}

Field yang Diperlukan saat POST / PUT

  • model — nama model kendaraan, wajib diisi
  • type — tipe kendaraan (Sedan, SUV, MPV, dll), wajib diisi
  • manufacturer — nama produsen (Honda, Toyota, dll), wajib diisi

Jika salah satu field kosong, API akan mengembalikan response 422 Unprocessable Entity beserta pesan validasi.

Konsep REST API

Apa itu REST API?

REST (Representational State Transfer) adalah gaya arsitektur untuk membangun layanan web. API (Application Programming Interface) adalah antarmuka yang memungkinkan dua aplikasi berkomunikasi. Jadi REST API adalah cara aplikasi berkomunikasi melalui HTTP dengan aturan-aturan tertentu.

Dalam konteks Android, aplikasi kita berperan sebagai client yang mengirim request ke server, lalu server membalas dengan data dalam format JSON.

Alur komunikasi Client → Server 
Android App (Client)
        │
        │  HTTP Request
        │  GET https://api.pahrul.my.id/vehicles
        │  Headers: Content-Type: application/json
        ▼
    Server (API)
        │
        │  HTTP Response
        │  Status: 200 OK
        │  Body: { "success": true, "data": [...] }
        ▼
Android App (Client)
        │
        └─ Parse JSON → tampilkan ke RecyclerView

HTTP Method, Status Code & JSON

HTTP Method

Setiap request ke API menggunakan salah satu HTTP method berikut. Method menentukan apa yang ingin dilakukan terhadap data di server.

Method Fungsi Contoh Ada Request Body?
GET Ambil data Tampilkan daftar vehicle Tidak
POST Kirim / buat data baru Tambah vehicle baru Ya
PUT Update seluruh data Edit data vehicle Ya
DELETE Hapus data Hapus vehicle Tidak

HTTP Status Code

Setiap response dari server menyertakan status code — angka 3 digit yang menunjukkan hasil dari request.

Kode Arti Kapan Muncul
200 OK Berhasil GET, PUT, DELETE sukses
201 Created Data berhasil dibuat POST sukses
404 Not Found Data tidak ditemukan ID tidak ada di database
422 Unprocessable Validasi gagal Field wajib tidak diisi
401 Unauthorized Tidak punya akses Token tidak valid / tidak ada
500 Server Error Error di server Bug di sisi server

Format JSON

JSON (JavaScript Object Notation) adalah format pertukaran data yang ringan dan mudah dibaca. Semua response dari API Vehicle menggunakan format JSON.

Anatomi JSON response dari API Vehicle 
{
  "success": true,          // Boolean — apakah request berhasil
  "message": "Data vehicle ditemukan",  // String — pesan dari server
  "data": {                 // Object atau Array — isi data utama
    "id": 1,                // Number
    "model": "Civic",       // String
    "type": "Sedan",        // String
    "manufacturer": "Honda",// String
    "created_at": "2026-04-30T10:00:00.000000Z"
  }
}

Wrapper Response

API Vehicle menggunakan pola wrapper response — data selalu dibungkus dalam objek dengan field success, message, dan data. Pola ini umum dipakai agar client bisa dengan mudah mengecek apakah request berhasil sebelum memproses datanya.

Uji Coba API dengan Postman

Sebelum menulis kode Android, selalu uji endpoint API terlebih dahulu menggunakan Postman. Ini memastikan API berjalan dengan benar dan kita tahu persis format request dan response-nya.

Instalasi Postman

Download Postman di postman.com/downloads (tersedia untuk Windows, Mac, Linux) atau gunakan versi web di web.postman.co.

Uji Coba GET — Ambil Semua Vehicle

LangkahYang Dilakukan
1. Buat request baruKlik New → HTTP Request, set method ke GET
2. Masukkan URLhttps://api.pahrul.my.id/vehicles → klik Send
3. Cek responseStatus 200 OK + array data vehicle dalam format JSON

Uji Coba POST — Tambah Vehicle Baru

LangkahYang Dilakukan
1. Set method POSTGanti method ke POST, URL tetap https://api.pahrul.my.id/vehicles
2. Set request bodyTab Body → raw → JSON, isi dengan data vehicle
3. Cek responseStatus 201 Created + data vehicle baru beserta ID dari server
Contoh request body POST 
{
  "model": "Avanza",
  "type": "MPV",
  "manufacturer": "Toyota"
}

Biasakan Uji API Sebelum Coding

Jika API tidak berjalan di Postman, jangan lanjut ke kode Android. Pastikan dulu response sesuai ekspektasi — cek URL, method, format body, dan status code. Ini menghemat waktu debugging yang jauh lebih lama jika langsung dicoba di Android.
