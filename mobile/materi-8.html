Praktikum: Firebase untuk Android Kotlin

Tujuan Pembelajaran

  • Memahami ekosistem Firebase dan layanan-layanannya
  • Mampu menghubungkan project Android ke Firebase Console
  • Mampu mengimplementasikan Firebase Authentication (register & login)
  • Mampu melakukan operasi CRUD data menggunakan Cloud Firestore
  • Memahami perbedaan Firestore dan Realtime Database
  • Mampu mengupload dan mengambil file menggunakan Firebase Storage

Prasyarat

Pastikan Anda sudah memahami materi dari praktikum sebelumnya:

  • Activity, Intent, dan dasar-dasar UI Android
  • RecyclerView dan Adapter
  • Kotlin dasar (data class, lambda, coroutines)
  • Memiliki akun Google untuk mengakses Firebase Console

Apa itu Firebase?

Firebase adalah platform Backend-as-a-Service (BaaS) milik Google yang menyediakan berbagai layanan backend siap pakai untuk aplikasi mobile dan web. Dengan Firebase, developer tidak perlu membangun server sendiri untuk fitur-fitur umum seperti autentikasi, database, dan penyimpanan file.

Authentication
Login / Register
Email, Google, Facebook, Phone
Cloud Firestore
NoSQL Database
Database dokumen yang scalable & realtime
Realtime Database
JSON Tree
Sinkronisasi data real-time antar device
Firebase Storage
File Storage
Upload & download gambar, video, file

Yang akan dibahas di praktikum ini

Praktikum ini fokus pada 4 layanan Firebase yang paling sering digunakan dalam pengembangan aplikasi mobile: Authentication, Cloud Firestore, Realtime Database, dan Firebase Storage. Setiap bagian dilengkapi contoh kode sederhana dan hands-on activity.

Perbedaan Firestore, Realtime Database, dan Storage

Tiga layanan ini sering membingungkan karena sama-sama menyimpan data di cloud. Perbedaan utamanya ada pada jenis data yang disimpan dan cara mengaksesnya.

Aspek Cloud Firestore Realtime Database Firebase Storage
Jenis data Data terstruktur (teks, angka, boolean, array, map) Data terstruktur (teks, angka, boolean) File biner (gambar, video, PDF, audio)
Format penyimpanan Koleksi → Dokumen → Field (seperti JSON bertingkat) Pohon JSON flat Folder & file (seperti Google Drive)
Contoh isi { name: "Budi", age: 21 } { text: "Halo!", time: 1700000 } foto_profil.jpg, laporan.pdf
Query / filter Powerful (filter, sort, compound query) Terbatas (satu field saja) Tidak ada query — akses via path/URL
Sinkronisasi realtime Ya (dengan snapshot listener) Ya (latensi sangat rendah ~50ms) Tidak — upload/download saja
Offline support Otomatis (cache lokal) Otomatis (cache lokal) Upload dilanjutkan otomatis jika koneksi putus
Kapasitas Sangat besar (skala enterprise) Besar, tapi kurang efisien untuk data kompleks Hingga 5 TB per file
Harga (free tier) 50K baca / 20K tulis per hari 10 GB storage, 1 GB/bulan download 5 GB storage, 1 GB/hari download
Butuh upgrade ke Blaze plan

Kapan Menggunakan Masing-masing?

Gunakan Firestore
Data terstruktur
  • Profil user
  • Daftar produk / catatan
  • Data yang perlu di-filter atau di-sort
  • Aplikasi dengan banyak koleksi data
Gunakan Realtime Database
Sinkronisasi cepat
  • Aplikasi chat / pesan
  • Live score / leaderboard

Kombinasi yang Umum Dipakai

Dalam satu aplikasi, ketiga layanan ini biasanya dipakai bersama-sama, bukan pilih salah satu:

  • Firestore — menyimpan data profil user, termasuk URL foto profil
  • Storage — menyimpan file foto profil yang sebenarnya
  • Realtime Database — menyimpan status online/offline user secara real-time

Contoh: saat user upload foto profil → file dikirim ke Storage → URL-nya disimpan ke Firestore → status "sedang upload" ditampilkan via Realtime Database.

Setup Firebase

Ada lima langkah yang harus dilakukan secara berurutan: membuat project Android baru, membuat project di Firebase Console, menghubungkan keduanya, menambahkan dependencies, lalu menguji koneksi.

1. Buat Project Android Baru

Langkah pertama adalah membuat project Android baru di Android Studio. Package name yang digunakan di sini harus sama persis dengan yang akan didaftarkan ke Firebase Console.

1

Buka Android Studio → New Project

Dari welcome screen klik New Project, atau dari menu File → New → New Project. Pilih template Empty Views Activity, lalu klik Next.

2

Isi Konfigurasi Project

Isi kolom berikut, lalu klik Finish:

Name PraktikumFirebase
Package name com.example.praktikumfirebase
Language Kotlin
Min SDK API 24 (Android 7.0)
Build config Groovy DSL (.gradle)
3

Tunggu Gradle Sync Selesai

Android Studio akan men-download dependencies awal. Tunggu hingga status bar di bawah menunjukkan "Gradle sync finished" sebelum melanjutkan ke langkah berikutnya.

4

Catat Package Name

Buka file build.gradle.kts (Module: app) dan catat nilai applicationId. Nilai ini harus digunakan saat mendaftarkan app ke Firebase Console.

build.gradle.kts (Module: app) — cek applicationId 
android {
    defaultConfig {
        applicationId = "com.example.praktikumfirebase"  // ← catat ini
        minSdk = 24
        targetSdk = 34
        ...
    }
}

2. Buat Project di Firebase Console

Firebase Console adalah dashboard berbasis web untuk mengelola semua layanan Firebase. Buka console.firebase.google.com dan login dengan akun Google.

1

Buka Firebase Console

Kunjungi console.firebase.google.com, lalu klik tombol "Add project" atau "Create a project".

2

Beri Nama Project

Masukkan nama project, misalnya PraktikumFirebase. Firebase akan otomatis membuat Project ID unik. Klik Continue.

3

Google Analytics (Opsional)

Untuk praktikum, pilih Disable Google Analytics agar lebih sederhana. Klik Create project.

4

Project Siap

Tunggu beberapa detik hingga project selesai dibuat, lalu klik Continue. Anda akan diarahkan ke dashboard project.

4. Tambah Dependencies Firebase

Aktifkan Layanan yang Dibutuhkan

Firebase Console kini menggunakan navigasi berbasis kategori produk di sidebar kiri. Aktifkan layanan berikut satu per satu:

3. Hubungkan Project Android ke Firebase

Ada dua cara menghubungkan Android ke Firebase: menggunakan Firebase Assistant di Android Studio (cara termudah), atau secara manual lewat Firebase Console.

Cara A — Firebase Assistant (Direkomendasikan)

  1. Di Android Studio, klik menu Tools → Firebase — panel Firebase Assistant muncul di kanan.
  2. Pilih layanan, misalnya Authentication → Authenticate using a custom authentication system.
  3. Klik Connect your app to Firebase — browser terbuka untuk login Google → pilih project Firebase yang sudah dibuat → klik Connect. Jika berhasil, statusnya berubah menjadi ✓ Connected.
  4. Klik Add the Firebase Authentication SDK to your app — dependencies otomatis ditambahkan ke build.gradle.

Cara B — Manual via Firebase Console

  1. Di Firebase Console, klik ikon Android (</>) di halaman Project Overview untuk mendaftarkan aplikasi Android baru.
  2. Masukkan Android package name yang sama persis dengan applicationId di build.gradle.kts (Module: app), misalnya com.example.praktikumfirebase → klik Register app.
  3. Download file google-services.json yang disediakan, lalu pindahkan ke folder app/ di project Android Studio.
  4. Pastikan file berada di app/google-services.json (bukan di root project) — switch view ke Project di Android Studio untuk memverifikasi lokasinya.

Package Name Harus Sama Persis

Package name di Firebase Console harus identik dengan applicationId di build.gradle.kts (Module: app). Jika berbeda, aplikasi tidak akan bisa terhubung ke Firebase dan akan crash saat dijalankan.

4. Tambah Dependencies Firebase

Jika kamu menggunakan Cara A (Firebase Assistant), plugin Google Services dan dependency firebase-auth sudah otomatis ditambahkan. Kamu hanya perlu menambahkan dependency untuk layanan lain yang belum ada, yaitu Cloud Firestore.

Sudah Otomatis oleh Firebase Assistant

Setelah mengikuti Cara A, file build.gradle.kts kamu sudah berisi:

  • Plugin google.gms.google.services di Project level dan Module level
  • Dependency firebase.auth di Module level
  • File google-services.json sudah ada di folder app/

Jangan tambahkan ulang hal-hal di atas — akan menyebabkan duplikat dan error saat build.

Tambah Dependency Firestore

Karena materi ini juga menggunakan Cloud Firestore, ada dua file yang perlu diupdate.

Langkah 1 — Tambahkan versi dan library di gradle/libs.versions.toml:

gradle/libs.versions.toml 
[versions]
# ... versi lainnya ...
firebaseAuth = "24.1.0"
firebaseFirestore = "25.1.1"  // ← tambahkan ini

[libraries]
# ... library lainnya ...
firebase-auth = { group = "com.google.firebase", name = "firebase-auth", version.ref = "firebaseAuth" }
firebase-firestore = { group = "com.google.firebase", name = "firebase-firestore", version.ref = "firebaseFirestore" }  // ← tambahkan ini

Langkah 2 — Tambahkan dependency di build.gradle.kts (Module: app):

build.gradle.kts (Module: app) — tambahkan Firestore 
dependencies {
    implementation(libs.androidx.core.ktx)
    implementation(libs.androidx.appcompat)
    implementation(libs.material)
    implementation(libs.androidx.activity)
    implementation(libs.androidx.constraintlayout)
    implementation(libs.firebase.auth)
    implementation(libs.firebase.firestore)  // ← tambahkan ini
    testImplementation(libs.junit)
    androidTestImplementation(libs.androidx.junit)
    androidTestImplementation(libs.androidx.espresso.core)
}

Verifikasi Setup

Setelah menambahkan dependency, lakukan Sync Project with Gradle Files (klik ikon gajah di toolbar atau Ctrl+Shift+O). Jika tidak ada error, setup sudah berhasil dan siap digunakan.

File Status Keterangan
google-services.json ✅ Sudah ada Otomatis oleh Firebase Assistant
Plugin google-services ✅ Sudah ada Otomatis oleh Firebase Assistant
firebase.auth ✅ Sudah ada Otomatis oleh Firebase Assistant

Langkah 2 — Tulis Kode Uji Coba di MainActivity

MainActivity.kt — uji coba koneksi Firebase 
import android.graphics.Color
import android.os.Bundle
import android.util.Log
import android.widget.Button
import android.widget.TextView
import androidx.appcompat.app.AppCompatActivity
import com.google.firebase.Firebase
import com.google.firebase.auth.auth

class MainActivity : AppCompatActivity() {

    // Tag untuk Logcat
    private val TAG = "FirebaseTest"

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        val tvStatus      = findViewById<TextView>(R.id.tvStatus)
        val btnCekKoneksi = findViewById<Button>(R.id.btnCekKoneksi)

        btnCekKoneksi.setOnClickListener {
            cekKoneksiFirebase(tvStatus)
        }

        // Cek otomatis saat app pertama dibuka
        cekKoneksiFirebase(tvStatus)
    }

    private fun cekKoneksiFirebase(tvStatus: TextView) {
        try {
            // Coba inisialisasi Firebase Auth
            val auth = Firebase.auth

            // Jika berhasil, auth tidak akan null
            val statusUser = if (auth.currentUser != null) {
                "Login sebagai: ${auth.currentUser?.email}"
            } else {
                "Tidak ada user yang login"
            }

            // Tampilkan hasil di layar
            tvStatus.text = "Firebase terhubung!\n$statusUser"
            tvStatus.setTextColor(Color.parseColor("#2e7d32"))

            // Catat di Logcat
            Log.d(TAG, "Firebase Auth berhasil diinisialisasi")
            Log.d(TAG, "Status: $statusUser")

        } catch (e: Exception) {
            // Jika gagal, tampilkan pesan error
            tvStatus.text = "Koneksi gagal!\n${e.message}"
            tvStatus.setTextColor(Color.parseColor("#c62828"))

            Log.e(TAG, "Firebase gagal diinisialisasi", e)
        }
    }
}

Firebase Authentication

Pengenalan Firebase Authentication

Firebase Authentication menyediakan layanan backend siap pakai untuk mengautentikasi pengguna. Mendukung berbagai metode login: email/password, Google, Facebook, GitHub, nomor telepon, dan lainnya.

Email & Password
Dibahas
Login klasik dengan email dan password
Google Sign-In
Tidak dibahas
Login dengan akun Google
Phone Number
Tidak dibahas
Login dengan OTP via SMS
Anonymous
Tidak dibahas
Login tanpa akun (guest mode)

Cara Kerja Firebase Auth

Saat user berhasil login, Firebase mengembalikan objek FirebaseUser yang berisi informasi user (uid, email, displayName, dll). UID ini unik per user dan digunakan sebagai identifier di database.

Aktifkan Email/Password di Firebase Console

Sebelum bisa menggunakan login email/password, aktifkan dulu di Firebase Console:

Authentication → Sign-in method → Email/Password → Enable → Save

RegisterActivity.kt

RegisterActivity.kt 
import android.content.Intent
import android.os.Bundle
import android.view.View
import android.widget.Button
import android.widget.EditText
import android.widget.ProgressBar
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import com.google.firebase.Firebase
import com.google.firebase.auth.auth

class RegisterActivity : AppCompatActivity() {

    private val auth = Firebase.auth

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_register)

        val etEmail    = findViewById<EditText>(R.id.etEmail)
        val etPassword = findViewById<EditText>(R.id.etPassword)
        val btnRegister = findViewById<Button>(R.id.btnRegister)
        val progressBar = findViewById<ProgressBar>(R.id.progressBar)

        btnRegister.setOnClickListener {
            val email    = etEmail.text.toString().trim()
            val password = etPassword.text.toString().trim()

            // Validasi input
            if (email.isEmpty() || password.isEmpty()) {
                Toast.makeText(this, "Email dan password tidak boleh kosong", Toast.LENGTH_SHORT).show()
                return@setOnClickListener
            }
            if (password.length < 6) {
                Toast.makeText(this, "Password minimal 6 karakter", Toast.LENGTH_SHORT).show()
                return@setOnClickListener
            }

            // Tampilkan loading
            progressBar.visibility = View.VISIBLE
            btnRegister.isEnabled = false

            // Daftarkan user ke Firebase
            auth.createUserWithEmailAndPassword(email, password)
                .addOnCompleteListener(this) { task ->
                    progressBar.visibility = View.GONE
                    btnRegister.isEnabled = true

                    if (task.isSuccessful) {
                        // Registrasi berhasil
                        val user = auth.currentUser
                        Toast.makeText(this, "Registrasi berhasil! UID: ${user?.uid}", Toast.LENGTH_LONG).show()
                        // Arahkan ke halaman login atau home
                        startActivity(Intent(this, LoginActivity::class.java))
                        finish()
                    } else {
                        // Registrasi gagal — tampilkan pesan error
                        Toast.makeText(this, "Gagal: ${task.exception?.message}", Toast.LENGTH_LONG).show()
                    }
                }
        }
    }
}

import com.google.firebase.auth.auth

class LoginActivity : AppCompatActivity() {

    private val auth = Firebase.auth

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_login)

        val etEmail    = findViewById<EditText>(R.id.etEmail)
        val etPassword = findViewById<EditText>(R.id.etPassword)
        val btnLogin   = findViewById<Button>(R.id.btnLogin)
        val btnGoRegister = findViewById<Button>(R.id.btnGoRegister)
        val progressBar = findViewById<ProgressBar>(R.id.progressBar)

        btnGoRegister.setOnClickListener {
            startActivity(Intent(this, RegisterActivity::class.java))
        }

        btnLogin.setOnClickListener {
            val email    = etEmail.text.toString().trim()
            val password = etPassword.text.toString().trim()

            if (email.isEmpty() || password.isEmpty()) {
                Toast.makeText(this, "Isi email dan password terlebih dahulu", Toast.LENGTH_SHORT).show()
                return@setOnClickListener
            }

            progressBar.visibility = View.VISIBLE
            btnLogin.isEnabled = false

            auth.signInWithEmailAndPassword(email, password)
                .addOnCompleteListener(this) { task ->
                    progressBar.visibility = View.GONE
                    btnLogin.isEnabled = true

                    if (task.isSuccessful) {
                        val user = auth.currentUser
                        Toast.makeText(this, "Selamat datang, ${user?.email}", Toast.LENGTH_SHORT).show()
                        startActivity(Intent(this, HomeActivity::class.java))
                        finish()
                    } else {
                        Toast.makeText(this, "Login gagal: ${task.exception?.message}", Toast.LENGTH_LONG).show()
                    }
                }
        }
    }
}

import android.content.Intent
import android.os.Bundle
import android.os.Handler
import android.os.Looper
import androidx.appcompat.app.AppCompatActivity
import com.google.firebase.Firebase
import com.google.firebase.auth.auth

class SplashActivity : AppCompatActivity() {

    private val auth = Firebase.auth
    private val splashDelay = 2000L

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_splash)
    }

    override fun onStart() {
        super.onStart()

        Handler(Looper.getMainLooper()).postDelayed({
            val currentUser = auth.currentUser

            if (currentUser != null) {
                startActivity(Intent(this, HomeActivity::class.java))
            } else {
                startActivity(Intent(this, LoginActivity::class.java))
            }

            finish()
        }, splashDelay)
    }
}

Cloud Firestore

Pengenalan Cloud Firestore

Cloud Firestore adalah database NoSQL berbasis dokumen dari Firebase. Data disimpan dalam bentuk koleksi (collection) yang berisi dokumen (document), mirip seperti folder berisi file JSON.

Fitur Cloud Firestore Realtime Database
Struktur data Koleksi → Dokumen → Field Pohon JSON
Query Lebih powerful (filter, sort, compound) Terbatas
Offline support Otomatis Otomatis
Skalabilitas Sangat baik Terbatas
Rekomendasi Untuk project baru Untuk sinkronisasi sederhana

Struktur Data Firestore

Firestore menggunakan hierarki Collection → Document → Field. Bayangkan Collection seperti tabel, Document seperti baris, dan Field seperti kolom — tapi jauh lebih fleksibel karena setiap dokumen bisa punya field yang berbeda.

Contoh struktur data Firestore — aplikasi catatan 
// Firestore Structure

// Collection: "notes"

//         ├── title    : "Belajar Firebase"
//         ├── content  : "Firestore itu mudah!"
//         ├── userId   : "uid_user_123"
//         └── createdAt: Timestamp
//
//   └── Document: "def456"
//         ├── title    : "Catatan Kedua"
//         ├── content  : "Ini catatan saya"
//         ├── userId   : "uid_user_123"
//         └── createdAt: Timestamp

// Path ke collection: db.collection("notes")
// Path ke dokumen:    db.collection("notes").document("abc123")

Semua Field Wajib Punya Default Value

Firestore menggunakan reflection untuk membuat objek saat membaca data. Jika ada field tanpa default value, aplikasi akan crash dengan error no-argument constructor. Pastikan semua field punya nilai default seperti "", 0, atau null.

Kenapa Ada Field id?

Field id tidak disimpan ke Firestore — ini hanya dipakai di sisi Android untuk menyimpan ID dokumen Firestore setelah data dibaca. Saat membaca data, kita isi manual dengan .copy(id = document.id).

NoteAdapter.kt

Adapter ini menghubungkan list data Note dengan RecyclerView. Menggunakan ListAdapter dengan DiffUtil agar update list lebih efisien — hanya item yang berubah yang di-render ulang.

NoteAdapter.kt 
import android.view.LayoutInflater
import android.view.View
import android.view.ViewGroup
import android.widget.Button
import android.widget.TextView
import androidx.recyclerview.widget.DiffUtil
import androidx.recyclerview.widget.ListAdapter
import androidx.recyclerview.widget.RecyclerView

class NoteAdapter(
    private val onEdit   : (Note) -> Unit,
    private val onDelete : (Note) -> Unit
) : ListAdapter<Note, NoteAdapter.NoteViewHolder>(DiffCallback()) {

    inner class NoteViewHolder(view: View) : RecyclerView.ViewHolder(view) {
        val tvTitle   : TextView = view.findViewById(R.id.tvTitle)
        val tvContent : TextView = view.findViewById(R.id.tvContent)
        val btnEdit   : Button   = view.findViewById(R.id.btnEdit)
        val btnDelete : Button   = view.findViewById(R.id.btnDelete)
    }

    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): NoteViewHolder {
        val view = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_note, parent, false)
        return NoteViewHolder(view)
    }

    override fun onBindViewHolder(holder: NoteViewHolder, position: Int) {
        val note = getItem(position)
        holder.tvTitle.text   = note.title
        holder.tvContent.text = note.content
        holder.btnEdit.setOnClickListener   { onEdit(note) }
        holder.btnDelete.setOnClickListener { onDelete(note) }
    }

    class DiffCallback : DiffUtil.ItemCallback<Note>() {
        override fun areItemsTheSame(old: Note, new: Note) = old.id == new.id
        override fun areContentsTheSame(old: Note, new: Note) = old == new
    }
}

submitList() Dipanggil dari Activity setiap kali data Firestore berubah — DiffUtil otomatis hitung perbedaannya

Langkah 2 — Tulis Kode NoteListActivity.kt

Activity ini melakukan tiga hal utama: setup RecyclerView dengan adapter, pasang realtime listener ke Firestore di onStart(), dan lepas listener di onStop().

NoteListActivity.kt 
import android.content.Intent
import android.os.Bundle
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import androidx.recyclerview.widget.LinearLayoutManager
import androidx.recyclerview.widget.RecyclerView
import com.google.android.material.floatingactionbutton.FloatingActionButton
import com.google.firebase.Firebase
import com.google.firebase.auth.auth
import com.google.firebase.firestore.ListenerRegistration
import com.google.firebase.firestore.Query
import com.google.firebase.firestore.firestore

class NoteListActivity : AppCompatActivity() {

    private val db   = Firebase.firestore
    private val auth = Firebase.auth
    private var listenerReg: ListenerRegistration? = null
    private lateinit var adapter: NoteAdapter

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_note_list)

        val recyclerView = findViewById<RecyclerView>(R.id.recyclerView)
        val fab          = findViewById<FloatingActionButton>(R.id.fab)

        adapter = NoteAdapter(
            onEdit = { note ->
                // Kirim data catatan ke AddEditNoteActivity (mode edit)
                val intent = Intent(this, AddEditNoteActivity::class.java)
                intent.putExtra("noteId",      note.id)
                intent.putExtra("noteTitle",   note.title)
                intent.putExtra("noteContent", note.content)
                startActivity(intent)
            },
            onDelete = { note -> deleteNote(note.id) }
        )

        recyclerView.layoutManager = LinearLayoutManager(this)
        recyclerView.adapter = adapter

        // FAB → buka AddEditNoteActivity dalam mode tambah (tanpa extra)
        fab.setOnClickListener {
            startActivity(Intent(this, AddEditNoteActivity::class.java))
        }
    }

    override fun onStart() {
        super.onStart()
        val userId = auth.currentUser?.uid ?: return

        // Pasang realtime listener — list otomatis update saat Firestore berubah
        listenerReg = db.collection("notes")
            .whereEqualTo("userId", userId)
            .orderBy("createdAt", Query.Direction.DESCENDING)
            .addSnapshotListener { snapshot, error ->
                if (error != null) {
                    Toast.makeText(this, "Error: ${error.message}", Toast.LENGTH_SHORT).show()
                    return@addSnapshotListener
                }
                val notes = snapshot?.documents?.mapNotNull { doc ->
                    doc.toObject(Note::class.java)?.copy(id = doc.id)
                } ?: emptyList()
                adapter.submitList(notes)
            }
    }

    override fun onStop() {
        super.onStop()
        listenerReg?.remove()  // Lepas listener agar tidak aktif di background
    }

    private fun deleteNote(noteId: String) {
        db.collection("notes").document(noteId)
            .delete()
            .addOnSuccessListener {
                Toast.makeText(this, "Catatan dihapus", Toast.LENGTH_SHORT).show()
            }
            .addOnFailureListener { e ->
                Toast.makeText(this, "Gagal hapus: ${e.message}", Toast.LENGTH_SHORT).show()
            }
    }
}

Jangan Lupa Remove Listener

Listener yang tidak di-remove akan terus aktif di background dan menghabiskan kuota Firestore. Selalu panggil listenerReg?.remove() di onStop().

AddEditNoteActivity.kt 
import android.os.Bundle
import android.widget.Button
import android.widget.EditText
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import com.google.firebase.Firebase
import com.google.firebase.Timestamp
import com.google.firebase.auth.auth
import com.google.firebase.firestore.firestore

class AddEditNoteActivity : AppCompatActivity() {

    private val db   = Firebase.firestore
    private val auth = Firebase.auth

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_add_edit_note)

        val etTitle   = findViewById<EditText>(R.id.etTitle)
        val etContent = findViewById<EditText>(R.id.etContent)
        val btnSave   = findViewById<Button>(R.id.btnSave)

        // Ada noteId di Intent → mode edit, tidak ada → mode tambah
        val noteId     = intent.getStringExtra("noteId")
        val isEditMode = noteId != null

        if (isEditMode) {
            etTitle.setText(intent.getStringExtra("noteTitle"))
            etContent.setText(intent.getStringExtra("noteContent"))
            btnSave.text = "Simpan Perubahan"
        }

        btnSave.setOnClickListener {
            val title   = etTitle.text.toString().trim()
            val content = etContent.text.toString().trim()

            if (title.isEmpty()) {
                etTitle.error = "Judul tidak boleh kosong"
                return@setOnClickListener
            }

            if (isEditMode) {
                // Update — hanya ubah title dan content, field lain tidak tersentuh
                db.collection("notes").document(noteId!!)
                    .update("title", title, "content", content)
                    .addOnSuccessListener {
                        Toast.makeText(this, "Catatan diperbarui", Toast.LENGTH_SHORT).show()
                        finish()
                    }
                    .addOnFailureListener { e ->
                        Toast.makeText(this, "Gagal: ${e.message}", Toast.LENGTH_SHORT).show()
                    }
            } else {
                // Tambah — buat dokumen baru dengan ID otomatis
                val note = Note(
                    title     = title,
                    content   = content,
                    userId    = auth.currentUser?.uid ?: "",
                    createdAt = Timestamp.now()
                )
                db.collection("notes").add(note)
                    .addOnSuccessListener {
                        Toast.makeText(this, "Catatan disimpan", Toast.LENGTH_SHORT).show()
                        finish()
                    }
                    .addOnFailureListener { e ->
                        Toast.makeText(this, "Gagal: ${e.message}", Toast.LENGTH_SHORT).show()
                    }
            }
        }
    }
}

AndroidManifest.xml

Ada dua hal yang perlu dilakukan di langkah ini: mendaftarkan kedua Activity baru ke manifest, dan mengubah tujuan redirect setelah login dari HomeActivity ke NoteListActivity.

Langkah 1 — Daftarkan Activity di AndroidManifest.xml

Buka AndroidManifest.xml, tambahkan dua baris untuk Activity yang baru dibuat di dalam tag <application>:

AndroidManifest.xml 
<application ... >

    <!-- Activity dari materi Authentication -->
    <activity android:name=".SplashActivity">
        <intent-filter>
            <action android:name="android.intent.action.MAIN" />
            <category android:name="android.intent.category.LAUNCHER" />
        </intent-filter>
    </activity>
    <activity android:name=".RegisterActivity" />
    <activity android:name=".LoginActivity" />
    <activity android:name=".HomeActivity" />

    <!-- Tambahkan dua baris ini untuk Firestore -->
    <activity android:name=".NoteListActivity" />
    <activity android:name=".AddEditNoteActivity" />

</application>

Langkah 2 — Update Redirect di LoginActivity.kt

Buka LoginActivity.kt, cari baris yang mengarahkan ke HomeActivity setelah login berhasil, lalu ganti dengan NoteListActivity:

LoginActivity.kt — ubah tujuan redirect setelah login 
// Sebelum:
startActivity(Intent(this, HomeActivity::class.java))

// Sesudah:
startActivity(Intent(this, NoteListActivity::class.java))

Langkah 3 — Update Redirect di SplashActivity.kt

Buka SplashActivity.kt, lakukan hal yang sama — ganti tujuan redirect jika user sudah login:

SplashActivity.kt — ubah tujuan redirect jika sudah login 
// Sebelum:
startActivity(Intent(this, HomeActivity::class.java))

// Sesudah:
startActivity(Intent(this, NoteListActivity::class.java))

Jika Muncul Error "Unresolved reference"

Pastikan NoteListActivity dan AddEditNoteActivity sudah dibuat sebelum mengubah redirect. Jika masih error setelah dibuat, coba Build → Clean Project lalu Rebuild Project.

Penutup Cloud Firestore

Di bagian ini kita sudah membangun fitur CRUD catatan lengkap menggunakan Cloud Firestore. Semua operasi — tambah, baca realtime, edit, dan hapus — sudah terintegrasi dalam dua Activity yang terhubung satu sama lain.

Checklist Setelah Menyelesaikan Cloud Firestore

  • Note.kt sudah dibuat dengan semua field punya default value
  • item_note.xml sudah dibuat dengan tvTitle, tvContent, btnEdit, btnDelete
  • NoteAdapter.kt sudah dibuat dengan DiffCallback
  • activity_note_list.xml sudah dibuat dengan RecyclerView dan FAB
  • NoteListActivity.kt sudah dibuat — list tampil realtime dari Firestore
  • activity_add_edit_note.xml sudah dibuat dengan etTitle, etContent, btnSave
  • AddEditNoteActivity.kt sudah dibuat — mode tambah dan edit berfungsi
  • NoteListActivity dan AddEditNoteActivity sudah didaftarkan di AndroidManifest.xml
  • Redirect di LoginActivity dan SplashActivity sudah diubah ke NoteListActivity

Cara Memastikan Berhasil

Jalankan aplikasi. Setelah login, aplikasi harus membuka NoteListActivity. Coba tambah catatan baru lewat FAB — catatan harus langsung muncul di list. Coba edit dan hapus catatan, pastikan perubahan langsung tercermin. Buka Firebase Console → Firestore Database → collection notes untuk memverifikasi data tersimpan dengan benar.
