1. View & ViewGroup ⏱ 15 menit

Cara Mengajar: Gunakan analogi kotak. ViewGroup = kotak besar yang bisa menampung kotak-kotak kecil. View = kotak kecil yang berisi konten. Gambar di papan tulis dulu sebelum buka kode.

// MainActivity.kt — akses semua View dari diagram
class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        val tvNama  = findViewById<TextView>(R.id.tvNama)
        val tvRole  = findViewById<TextView>(R.id.tvRole)
        val etPesan = findViewById<EditText>(R.id.etPesan)
        val btnBatal = findViewById<Button>(R.id.btnBatal)
        val btnKirim = findViewById<Button>(R.id.btnKirim)

        btnKirim.setOnClickListener {
            val pesan = etPesan.text.toString().trim()
            if (pesan.isEmpty()) {
                etPesan.error = "Pesan tidak boleh kosong"
                return@setOnClickListener
            }
            Toast.makeText(this, "${tvNama.text}: $pesan", Toast.LENGTH_SHORT).show()
            etPesan.text.clear()
        }

        btnBatal.setOnClickListener {
            etPesan.text.clear()
            etPesan.clearFocus()
        }
    }
}

3. ConstraintLayout ⏱ 20 menit

Cara Mengajar: Mulai dari visual editor drag-and-drop dulu — biarkan mahasiswa eksplorasi 5 menit. Baru tunjukkan kode XML yang dihasilkan. Ini lebih efektif daripada langsung baca XML.

💡 Analogi untuk Mahasiswa

ConstraintLayout itu seperti memasang foto di dinding dengan tali. Setiap foto (View) diikat ke dinding (parent) atau ke foto lain dengan tali (constraint). Kalau tidak ada tali, foto jatuh ke pojok kiri atas.

Aturan Wajib ConstraintLayout

⚠️ Setiap View WAJIB punya minimal:

  • 1 constraint horizontal (start/end ke parent atau view lain)
  • 1 constraint vertical (top/bottom ke parent atau view lain)

Kalau tidak, View akan "lari" ke pojok kiri atas saat runtime.

Contoh Login Screen dengan ConstraintLayout

✅ Kode ini bisa di-copy langsung

<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:padding="24dp">

    <!-- Logo di tengah atas -->
    <TextView
        android:id="@+id/tvLogo"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="🤖 MyApp"
        android:textSize="32sp"
        android:textStyle="bold"
        app:layout_constraintTop_toTopOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        android:layout_marginTop="60dp"/>

    <!-- Input Email -->
    <EditText
        android:id="@+id/etEmail"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:hint="Email"
        android:inputType="textEmailAddress"
        app:layout_constraintTop_toBottomOf="@id/tvLogo"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        android:layout_marginTop="40dp"/>

    <!-- Input Password -->
    <EditText
        android:id="@+id/etPassword"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:hint="Password"
        android:inputType="textPassword"
        app:layout_constraintTop_toBottomOf="@id/etEmail"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        android:layout_marginTop="12dp"/>

    <!-- Tombol Login -->
    <Button
        android:id="@+id/btnLogin"
        android:layout_width="0dp"
        android:layout_height="wrap_content"
        android:text="Login"
        app:layout_constraintTop_toBottomOf="@id/etPassword"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        android:layout_marginTop="24dp"/>

    <!-- Link Register di bawah -->
    <TextView
        android:id="@+id/tvRegister"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Belum punya akun? Daftar"
        android:textColor="#7F52FF"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent"

</androidx.constraintlayout.widget.ConstraintLayout>

Kotlin — Akses dan Validasi

✅ Kode ini bisa di-copy langsung

// MainActivity.kt
package com.example.myapp

import android.os.Bundle
import android.widget.*
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {

    // Deklarasi dengan lateinit — akan diinisialisasi di onCreate
    private lateinit var tvResult: TextView
    private lateinit var etNama: EditText
    private lateinit var etUsia: EditText
    private lateinit var rgGender: RadioGroup
    private lateinit var cbSetuju: CheckBox
    private lateinit var btnSubmit: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // Binding view
        tvResult  = findViewById(R.id.tvResult)
        etNama    = findViewById(R.id.etNama)
        etUsia    = findViewById(R.id.etUsia)
        rgGender  = findViewById(R.id.rgGender)
        cbSetuju  = findViewById(R.id.cbSetuju)
        btnSubmit = findViewById(R.id.btnSubmit)

        btnSubmit.setOnClickListener {
            submitForm()
        }
    }

    private fun submitForm() {
        val nama = etNama.text.toString().trim()
        val usia = etUsia.text.toString().trim()

        // Validasi
        if (nama.isEmpty()) {
            etNama.error = "Nama tidak boleh kosong"
            etNama.requestFocus()
            return
        }
        if (usia.isEmpty()) {
            etUsia.error = "Usia tidak boleh kosong"
            etUsia.requestFocus()
            return
        }
        if (rgGender.checkedRadioButtonId == -1) {
            Toast.makeText(this, "Pilih jenis kelamin!", Toast.LENGTH_SHORT).show()
            return
        }
        if (!cbSetuju.isChecked) {
            Toast.makeText(this, "Centang persetujuan dulu!", Toast.LENGTH_SHORT).show()
            return
        }

        // Ambil gender
        val gender = if (rgGender.checkedRadioButtonId == R.id.rbLaki) "Laki-laki" else "Perempuan"

        // Tampilkan hasil
        tvResult.text = "Nama: $nama\nUsia: $usia tahun\nGender: $gender"
    }
}

5. String & Color Resources ⏱ 15 menit

Cara Mengajar: Tunjukkan masalah dulu — hardcoded string di XML. Lalu tunjukkan kenapa itu buruk: kalau mau ganti teks "Login" jadi "Masuk" di 10 tempat, harus ubah 10 kali. Dengan string resource, ubah sekali saja.

🌍 Skenario Nyata

Bayangkan aplikasi kalian dipakai di 5 negara berbeda. Tanpa string resource, kalian harus buat 5 versi layout XML yang berbeda. Dengan string resource, cukup buat 5 file strings.xml.

strings.xml — Pindahkan semua teks ke sini

✅ Kode ini bisa di-copy langsung

<!-- res/values/strings.xml -->
<resources>
    <string name="app_name">Aplikasi Saya</string>

    <!-- Form -->
    <string name="hint_nama">Nama Lengkap</string>
    <string name="hint_usia">Usia</string>
    <string name="hint_email">Email</string>
    <string name="hint_password">Password</string>
    <string name="btn_submit">Submit</string>
    <string name="btn_login">Login</string>
    <string name="btn_cancel">Batal</string>

    <!-- Pesan error -->
    <string name="error_nama_kosong">Nama tidak boleh kosong</string>
    <string name="error_usia_kosong">Usia tidak boleh kosong</string>
    <string name="error_pilih_gender">Pilih jenis kelamin!</string>

    <!-- String dengan parameter -->
    <string name="sambutan">Halo, %1$s! Usia kamu %2$d tahun.</string>

    <!-- String array untuk Spinner -->
    <string-array name="daftar_kota">

        <item>Jakarta</item>
        <item>Surabaya</item>
        <item>Bandung</item>
        <item>Medan</item>

★ Mini Proyek Akhir Materi 3 ⏱ 20 menit

Cara Mengajar: Kerjakan individual atau berpasangan. Tidak harus sempurna — yang penting jalan. Di akhir, minta 2-3 mahasiswa demo hasil mereka.

🏆 Proyek: "Kartu Profil Interaktif"

Buat aplikasi dengan dua layar (satu Activity dulu):

  1. Form Input: Nama, Usia, Kota (Spinner), Hobi (CheckBox min 2), Jenis Kelamin (RadioButton)
  2. Tampilan Hasil: Setelah klik Submit, tampilkan semua data yang diinput di TextView dengan format yang rapi
  3. Validasi: Semua field wajib diisi, usia 17-60 tahun
  4. Resources: Semua string di strings.xml, semua warna di colors.xml
  5. Bonus: Tambahkan tombol "Reset" yang mengosongkan semua input

6. Activity & Lifecycle ⏱ 20 menit

Cara Mengajar: Lifecycle paling mudah dipahami dengan Log. Tambahkan Log.d() di setiap callback, jalankan di emulator, lalu tunjukkan output di Logcat saat mahasiswa menekan Home, Back, dan membuka ulang app.

🎬 Demo Lifecycle dengan Logcat

Jalankan kode di bawah, buka Logcat (filter tag "LifecycleDemo"), lalu lakukan aksi berikut sambil tunjukkan log yang muncul:

  1. Buka app → onCreate, onStart, onResume
  2. Tekan Home → onPause, onStop
  3. Buka lagi dari recent → onRestart, onStart, onResume
  4. Tekan Back → onPause, onStop, onDestroy

✅ Kode ini bisa di-copy langsung

// MainActivity.kt — Lifecycle Demo
package com.example.myapp

import android.os.Bundle
import android.util.Log
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {

    companion object {
        private const val TAG = "LifecycleDemo"
    }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        Log.d(TAG, "1. onCreate() — Activity dibuat, setup UI di sini")
    }

    override fun onStart() {
        super.onStart()
        Log.d(TAG, "2. onStart() — Activity mulai terlihat")
    }

    override fun onResume() {
        super.onResume()
        Log.d(TAG, "3. onResume() — Activity aktif, user bisa interaksi")
    }

    override fun onPause() {
        super.onPause()
        Log.d(TAG, "4. onPause() — Activity kehilangan fokus, simpan data penting di sini")
    }

    override fun onStop() {
        super.onStop()
        Log.d(TAG, "5. onStop() — Activity tidak terlihat")
    }

    override fun onRestart() {
        super.onRestart()
        Log.d(TAG, "6. onRestart() — Activity kembali dari stopped")
    }

    override fun onDestroy() {
        super.onDestroy()
        Log.d(TAG, "7. onDestroy() — Activity dihancurkan, bersihkan resource di sini")
    }
}

Jawaban: super.onCreate() melakukan inisialisasi penting dari framework Android. Tanpanya, Activity tidak bisa berfungsi.

8. Passing Data Antar Activity ⏱ 25 menit

Cara Mengajar: Ini yang paling sering dipakai di proyek nyata. Gunakan kode dari Mini Proyek M3 (form profil) — kirim data form ke Activity kedua untuk ditampilkan. Mahasiswa langsung lihat relevansinya.

FormActivity — Kirim Data

✅ Kode ini bisa di-copy langsung

// FormActivity.kt — mengirim data
class FormActivity : AppCompatActivity() {

    private lateinit var etNama: EditText
    private lateinit var etUsia: EditText
    private lateinit var btnSubmit: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_form)

        etNama    = findViewById(R.id.etNama)
        etUsia    = findViewById(R.id.etUsia)
        btnSubmit = findViewById(R.id.btnSubmit)

        btnSubmit.setOnClickListener {
            val nama = etNama.text.toString().trim()
            val usia = etUsia.text.toString().trim()

            if (nama.isEmpty()) {
                etNama.error = "Nama wajib diisi"
                return@setOnClickListener
            }

            // Kirim data via Intent extras
            val intent = Intent(this, HasilActivity::class.java)
            intent.putExtra("NAMA", nama)
            intent.putExtra("USIA", usia.toIntOrNull() ?: 0)
            intent.putExtra("WAKTU", System.currentTimeMillis())
            startActivity(intent)
        }
    }
}

        setContentView(R.layout.activity_hasil)

            Nama  : $nama
            Usia  : $usia tahun

        // Tombol kembali

        }
    }
}

🐛 Bug Sengaja #7 — Key typo

}

9. Implicit Intent ⏱ 20 menit

Cara Mengajar: Demo langsung di emulator — klik tombol "Buka Browser", tunjukkan browser terbuka. Ini selalu bikin mahasiswa antusias karena app mereka bisa "keluar" dan berinteraksi dengan app lain.

💡 Konsep Kunci

Explicit Intent = "Saya mau ke toko A yang ada di Jalan B No. C"
Implicit Intent = "Saya mau beli roti, siapapun yang jual roti silakan angkat tangan"

Demo Semua Implicit Intent

✅ Kode ini bisa di-copy langsung

// ImplicitIntentActivity.kt
class ImplicitIntentActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_implicit)

        // 1. Buka Browser
        findViewById<Button>(R.id.btnBrowser).setOnClickListener {
            val intent = Intent(Intent.ACTION_VIEW, Uri.parse("https://developer.android.com"))
            safeStart(intent, "Tidak ada browser")
        }

        // 2. Buka Dialer
        findViewById<Button>(R.id.btnTelepon).setOnClickListener {
            val intent = Intent(Intent.ACTION_DIAL, Uri.parse("tel:08123456789"))
            safeStart(intent, "Tidak ada app telepon")
        }

        // 3. Kirim Email
        findViewById<Button>(R.id.btnEmail).setOnClickListener {
            val intent = Intent(Intent.ACTION_SENDTO).apply {
                data = Uri.parse("mailto:")
                putExtra(Intent.EXTRA_EMAIL, arrayOf("dosen@kampus.ac.id"))
                putExtra(Intent.EXTRA_SUBJECT, "Pertanyaan Praktikum")
                putExtra(Intent.EXTRA_TEXT, "Halo Pak/Bu, saya ingin bertanya...")
            }
            safeStart(intent, "Tidak ada app email")
        }

        // 4. Share Teks
        findViewById<Button>(R.id.btnShare).setOnClickListener {
            val intent = Intent(Intent.ACTION_SEND).apply {
                type = "text/plain"
                putExtra(Intent.EXTRA_TEXT, "Saya sedang belajar Android di kampus!")
            }
            // createChooser menampilkan dialog pilih app
            startActivity(Intent.createChooser(intent, "Bagikan via"))
        }

        // 5. Buka Maps
        findViewById<Button>(R.id.btnMaps).setOnClickListener {
            val lat = -8.5833  // Mataram, NTB
            val lng = 116.1167
            val uri = Uri.parse("geo:$lat,$lng?q=$lat,$lng(Universitas)")
            val intent = Intent(Intent.ACTION_VIEW, uri)
            safeStart(intent, "Tidak ada app maps")
        }
    }

    // Helper: cek dulu apakah ada app yang bisa handle intent ini
    private fun safeStart(intent: Intent, errorMsg: String) {
        if (intent.resolveActivity(packageManager) != null) {
            startActivity(intent)
        } else {
            Toast.makeText(this, errorMsg, Toast.LENGTH_SHORT).show()
        }
    }
}

🐛 Bug Sengaja #8 — Tidak pakai safeStart()

Hapus pengecekan resolveActivity(), langsung panggil startActivity(intent). Di emulator yang tidak ada app tertentu, app crash dengan ActivityNotFoundException.

Tanya: "Kenapa crash? Bagaimana cara mencegahnya?" Ini mengajarkan defensive programming.
