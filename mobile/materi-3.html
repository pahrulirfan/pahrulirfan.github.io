Apa itu Android?

Android adalah sistem operasi berbasis Linux yang dirancang untuk perangkat mobile seperti smartphone dan tablet. Android dikembangkan oleh Google dan merupakan sistem operasi mobile paling populer di dunia dengan pangsa pasar lebih dari 70%.

Mengapa Kotlin?

Kotlin adalah bahasa pemrograman modern yang dikembangkan oleh JetBrains. Pada tahun 2019, Google mengumumkan Kotlin sebagai bahasa yang direkomendasikan untuk pengembangan Android. Beberapa kelebihan Kotlin:

  • Concise - Kode lebih ringkas dibanding Java, mengurangi boilerplate code
  • Safe - Null safety bawaan untuk menghindari NullPointerException
  • Interoperable - 100% kompatibel dengan Java, bisa menggunakan library Java
  • Modern - Mendukung fitur pemrograman modern seperti coroutines, extension functions

Lingkungan Pengembangan

Instalasi Android Studio

Android Studio adalah Integrated Development Environment (IDE) resmi untuk pengembangan aplikasi Android. IDE ini berbasis IntelliJ IDEA dan menyediakan fitur lengkap untuk development Android.

1

Download Android Studio

Kunjungi website resmi Android Developer di developer.android.com/studio dan download installer sesuai sistem operasi Anda (Windows, macOS, atau Linux).

2

Jalankan Installer

Jalankan file installer yang sudah didownload. Ikuti wizard instalasi dengan pengaturan default. Pada Windows, pastikan untuk mencentang opsi Android Virtual Device.

3

Setup Wizard

Saat pertama kali membuka Android Studio, akan muncul Setup Wizard. Pilih "Standard" installation untuk menginstall komponen yang diperlukan secara otomatis.

4

Download SDK Components

Android Studio akan mengunduh SDK, build tools, dan komponen lain yang diperlukan. Proses ini memerlukan koneksi internet dan bisa memakan waktu cukup lama tergantung kecepatan internet.

Setup Emulator (AVD)

AVD (Android Virtual Device) adalah emulator yang mensimulasikan perangkat Android di komputer Anda. Dengan AVD, Anda bisa menguji aplikasi tanpa memerlukan device fisik.

Membuat AVD Baru

1

Buka Device Manager melalui menu Tools → Device Manager

2

Klik tombol "Create Device"

3

Pilih kategori Phone dan pilih tipe perangkat (misalnya Pixel 6)

4

Pilih system image. Download jika belum ada. Disarankan API 33 (Android 13) atau lebih baru

5

Konfigurasi AVD (nama, orientasi, RAM) dan klik "Finish"

Tips Performa Emulator

  • Gunakan system image x86_64 untuk performa lebih baik
  • Aktifkan Hardware Acceleration (HAXM untuk Intel, Hyper-V untuk AMD)
  • Alokasikan RAM yang cukup untuk emulator (minimal 2GB)
  • Gunakan GPU host untuk rendering grafis

Penjelasan File Penting

1. AndroidManifest.xml

File manifest berisi informasi penting tentang aplikasi yang diperlukan oleh sistem Android. Setiap aplikasi harus memiliki file ini di root project.

AndroidManifest.xml 
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/Theme.HelloWorld"
        tools:targetApi="34">
        
        <!-- Deklarasi Activity -->
        <activity
            android:name=".MainActivity"
            android:exported="true">
            <!-- Intent filter untuk launcher -->
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
        
    </application>

</manifest>

2. MainActivity.kt

File Kotlin yang berisi logic dari Activity utama. Activity ini akan ditampilkan pertama kali saat aplikasi dibuka.

MainActivity.kt 
package com.example.helloworld

import android.os.Bundle
import androidx.appcompat.app.AppCompatActivity

class MainActivity : AppCompatActivity() {
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        // Menghubungkan Activity dengan layout XML
        setContentView(R.layout.activity_main)
    }
}

UI XML dan Layout

Pengenalan XML di Android

XML (eXtensible Markup Language) adalah bahasa markup yang digunakan untuk mendefinisikan tampilan UI di Android. Dengan memisahkan UI dari kode Kotlin/Java, kita mendapatkan:

  • Separation of Concerns - Memisahkan tampilan dari logic bisnis
  • Visual Editor - Bisa mendesain UI secara visual dengan drag-and-drop
  • Multiple Layouts - Mendukung berbagai ukuran dan orientasi layar
  • Resource Management - Mudah mengelola dan mengorganisir resource UI
  • Reusability - Layout bisa digunakan ulang di berbagai Activity

Resource Qualifiers (Folder dengan Suffix)

Android mendukung resource qualifiers untuk menyediakan resource alternatif berdasarkan konfigurasi device.

Qualifier Contoh Folder Keterangan
Bahasa values-id/, values-en/ Resource berdasarkan bahasa device
Orientasi layout-land/, layout-port/ Layout berbeda untuk landscape/portrait
Screen Size values-sw600dp/ Untuk layar dengan smallest width 600dp (tablet 7")
Screen Density drawable-hdpi/, drawable-xxhdpi/ Gambar untuk kepadatan layar berbeda
Night Mode values-night/ Resource untuk dark mode
API Level drawable-v24/, values-v31/ Resource untuk API level tertentu

Cara Kerja Resource Selection

Android akan otomatis memilih resource yang paling sesuai dengan konfigurasi device. Contoh:

  • Device bahasa Indonesia → values-id/strings.xml
  • Device bahasa lain → values/strings.xml (default)
  • Device landscape → layout-land/activity_main.xml
  • Device portrait → layout/activity_main.xml

Mengubah Icon Aplikasi

Icon aplikasi adalah identitas visual yang muncul di launcher, recent apps, dan berbagai tempat lainnya. Android modern menggunakan Adaptive Icons yang terdiri dari foreground dan background layer.

Metode 1: Menggunakan Image Asset Studio (Recommended)

1

Buka Image Asset Studio

Klik kanan pada folder res → New → Image Asset

2

Pilih Icon Type

Pilih "Launcher Icons (Adaptive and Legacy)" untuk membuat icon yang kompatibel dengan semua versi Android.

3

Konfigurasi Foreground Layer

  • Source Asset: Pilih Image, Clip Art, atau Text
  • Path: Browse ke file gambar Anda (PNG/SVG)
  • Trim: Yes untuk menghilangkan whitespace
  • Resize: Sesuaikan ukuran dalam safe zone
4

Konfigurasi Background Layer

  • Source Asset: Pilih Color atau Image
  • Color: Pilih warna background icon
5

Preview dan Finish

Lihat preview di berbagai bentuk (Circle, Squircle, Square, dll), kemudian klik Next → Finish

<?xml version="1.0" encoding="utf-8"?>
<adaptive-icon xmlns:android="http://schemas.android.com/apk/res/android">
    <!-- Background layer (warna atau gambar) -->
    <background android:drawable="@drawable/ic_launcher_background" />

    <foreground android:drawable="@drawable/ic_launcher_foreground" />

Tips Membuat Icon yang Baik

  • Gunakan format PNG atau WebP untuk bitmap, SVG/Vector untuk vector

String Resources (strings.xml)

File strings.xml menyimpan semua teks yang digunakan dalam aplikasi. Menggunakan string resource adalah best practice yang wajib diikuti untuk:

  • Memudahkan localization (terjemahan ke bahasa lain)
  • Menghindari hardcoded string dalam layout dan kode
  • Mempermudah maintenance dan perubahan teks
  • Konsistensi penggunaan teks di seluruh aplikasi

View dan ViewGroup

View

View adalah class dasar untuk semua komponen UI di Android. View merepresentasikan area persegi di layar yang dapat menggambar sesuatu dan merespons interaksi user. Semua widget (Button, TextView, ImageView, dll) adalah subclass dari View.

ViewGroup

ViewGroup adalah View khusus yang dapat berisi View lain (child views). ViewGroup digunakan sebagai container atau layout untuk mengatur tata letak child views. Semua layout (LinearLayout, RelativeLayout, ConstraintLayout, dll) adalah subclass dari ViewGroup.

LinearLayout

LinearLayout menyusun child views dalam satu baris (horizontal) atau satu kolom (vertical). Ini adalah layout paling sederhana dan mudah dipahami, cocok untuk layout yang simpel dan linear.

Kapan Menggunakan LinearLayout?

  • Layout sederhana dengan susunan satu arah
  • Pembagian ruang proporsional dengan weight
  • Form input sederhana
  • Nested layout kecil (hati-hati dengan performa)

Atribut Penting LinearLayout

Atribut Nilai Keterangan
android:orientation vertical | horizontal Arah susunan children
android:gravity center, start, end, top, bottom, center_horizontal, center_vertical Posisi content dalam layout
android:layout_gravity (sama dengan gravity) Posisi view dalam parent (untuk child)
android:layout_weight angka (float) Proporsi ruang yang digunakan (untuk child)
android:weightSum angka (float) Total weight untuk pembagian (opsional)
android:baselineAligned true | false Align baseline text (set false untuk performa)

RelativeLayout

RelativeLayout memposisikan child views relatif terhadap parent atau terhadap view lain. Layout ini fleksibel untuk membuat layout yang tidak bisa dicapai dengan LinearLayout biasa.

Kapan Menggunakan RelativeLayout?

  • Posisi view bergantung pada view lain
  • Menghindari nested LinearLayout
  • Layout dengan overlap views
  • Layout dengan posisi yang spesifik

Catatan Penting

  • Gunakan toStartOf/toEndOf daripada toLeftOf/toRightOf untuk RTL support
  • View yang direferensikan harus sudah dideklarasikan (atau gunakan @+id)
  • Hindari circular dependency (A depends on B, B depends on A)
  • Untuk layout kompleks, pertimbangkan ConstraintLayout

ConstraintLayout

ConstraintLayout adalah layout yang paling fleksibel dan powerful di Android. Layout ini memungkinkan pembuatan layout yang kompleks dan flat (tanpa nested layout) menggunakan sistem constraint (batasan).

Keunggulan ConstraintLayout

  • Flat hierarchy - Menghindari nested layout, meningkatkan performa
  • Visual editor - Drag-and-drop yang powerful di Android Studio
  • Responsive - Mudah membuat layout yang adaptif
  • Feature-rich - Guidelines, barriers, chains, ratio, dll
  • Recommended - Layout yang direkomendasikan Google untuk sebagian besar kasus

Konsep Dasar Constraint

Setiap view dalam ConstraintLayout harus memiliki minimal:

  • Satu constraint horizontal (start/end ke parent atau view lain)
  • Satu constraint vertical (top/bottom ke parent atau view lain)
    Constraint menghubungkan SISI view ke SISI target (parent atau view lain)

                        constraint ke TOP parent
                               ↓
                    ┌──────────●──────────┐
                    │                     │
    constraint   ●──│       VIEW          │──●  constraint
    ke START        │                     │     ke END
                    │                     │
                    └──────────●──────────┘
                               ↓
                        constraint ke BOTTOM

    ● = anchor point (titik constraint)

Atribut Constraint Dasar

Atribut Keterangan
app:layout_constraintTop_toTopOf Top view ke top target
app:layout_constraintTop_toBottomOf Top view ke bottom target
app:layout_constraintBottom_toBottomOf Bottom view ke bottom target
app:layout_constraintBottom_toTopOf Bottom view ke top target
app:layout_constraintStart_toStartOf Start view ke start target
app:layout_constraintStart_toEndOf Start view ke end target
app:layout_constraintEnd_toEndOf End view ke end target
app:layout_constraintEnd_toStartOf End view ke start target

Atribut Tambahan

Atribut Keterangan
app:layout_constraintHorizontal_bias Posisi horizontal (0.0 = start, 0.5 = center, 1.0 = end)
app:layout_constraintVertical_bias Posisi vertical (0.0 = top, 0.5 = center, 1.0 = bottom)
app:layout_constraintDimensionRatio Rasio width:height (contoh: "16:9", "1:1")
app:layout_constraintWidth_percent Lebar sebagai persentase parent (0.0 - 1.0)
app:layout_constraintHeight_percent Tinggi sebagai persentase parent (0.0 - 1.0)

Chain

Chain digunakan untuk mendistribusikan beberapa views secara horizontal atau vertical. Ada 3 jenis chain style:

    SPREAD (default) - Views didistribusikan merata
    ┌────────────────────────────────────────────────┐
    │    ┌────┐        ┌────┐        ┌────┐         │
    │    │ A  │        │ B  │        │ C  │         │
    │    └────┘        └────┘        └────┘         │
    └────────────────────────────────────────────────┘

    SPREAD_INSIDE - Views merata, pinggir menempel
    ┌────────────────────────────────────────────────┐
    │ ┌────┐           ┌────┐           ┌────┐      │
    │ │ A  │           │ B  │           │ C  │      │
    │ └────┘           └────┘           └────┘      │
    └────────────────────────────────────────────────┘

    PACKED - Views berkumpul di tengah
    ┌────────────────────────────────────────────────┐
    │           ┌────┐┌────┐┌────┐                  │
    │           │ A  ││ B  ││ C  │                  │
    │           └────┘└────┘└────┘                  │
    └────────────────────────────────────────────────┘

    WEIGHTED - Menggunakan weight seperti LinearLayout
    ┌────────────────────────────────────────────────┐
    │ ┌────┐    ┌────────────────┐    ┌────┐        │
    │ │w=1 │    │     w=2        │    │w=1 │        │
    │ └────┘    └────────────────┘    └────┘        │
    └────────────────────────────────────────────────┘

FrameLayout

FrameLayout adalah layout sederhana yang menumpuk (stack) child views di atas satu sama lain. View yang dideklarasikan terakhir akan berada di paling atas (z-order).

Kapan Menggunakan FrameLayout?

  • Container untuk satu child view
  • Container untuk Fragment
  • Overlay satu view di atas view lain
  • Badge/counter di atas icon
  • Loading overlay
    FrameLayout - Views ditumpuk (stacked)
    
    Deklarasi di XML:           Visual Result:
    ┌──────────────┐           ┌──────────────────────┐
    │ View 1       │           │ ████████████████████ │ ← View 1 (paling bawah)
    │ View 2       │    →      │ ████ View 2 ████████ │ ← View 2 (di atas View 1)
    │ View 3       │           │ ██ View 3 ██████████ │ ← View 3 (paling atas)
    └──────────────┘           └──────────────────────┘
    
    View yang dideklarasikan terakhir = paling atas (z-order tertinggi)

Perbandingan Layout

Layout Kelebihan Kekurangan Kapan Digunakan
LinearLayout Simple, mudah dipahami, weight system Perlu nested untuk layout kompleks Layout sederhana, form, distribusi proporsional
RelativeLayout Flexible positioning, mengurangi nesting Bisa kompleks, digantikan ConstraintLayout Layout dengan posisi relatif sederhana
ConstraintLayout Paling flexible, flat hierarchy, visual editor Learning curve lebih tinggi Hampir semua kasus, terutama layout kompleks
FrameLayout Simple, perfect untuk stacking Terbatas untuk satu child atau stacking Fragment container, overlay, badge

Rekomendasi

  • Default choice: ConstraintLayout untuk sebagian besar layout
  • Simple vertical/horizontal list: LinearLayout
  • Fragment container: FrameLayout
  • Overlay/stacking: FrameLayout
  • Hindari: Nested layout lebih dari 3-4 level

class MainActivity : AppCompatActivity() {
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        
        val etName = findViewById<EditText>(R.id.etName)
        val etEmail = findViewById<EditText>(R.id.etEmail)
        val btnSubmit = findViewById<Button>(R.id.btnSubmit)
        
        btnSubmit.setOnClickListener {
            // Mengambil nilai dari EditText
            val name = etName.text.toString().trim()
            val email = etEmail.text.toString().trim()
            
            // Validasi input
            if (name.isEmpty()) {
                etName.error = "Nama tidak boleh kosong"
                etName.requestFocus()
                return@setOnClickListener
            }
            
            if (email.isEmpty()) {
                etEmail.error = "Email tidak boleh kosong"
                etEmail.requestFocus()
                return@setOnClickListener
            }
            
            // Proses data
            Toast.makeText(this, "Nama: $name, Email: $email", Toast.LENGTH_SHORT).show()
        }
        
        // Text watcher untuk real-time validation
        etEmail.addTextChangedListener(object : TextWatcher {
            override fun beforeTextChanged(s: CharSequence?, start: Int, count: Int, after: Int) {}
            override fun onTextChanged(s: CharSequence?, start: Int, before: Int, count: Int) {}
            override fun afterTextChanged(s: Editable?) {
                if (!android.util.Patterns.EMAIL_ADDRESS.matcher(s.toString()).matches()) {
                    etEmail.error = "Format email tidak valid"
                }
            }
        })
    }
}

        val btnStart = findViewById<Button>(R.id.btnStart)
        
        btnStart.setOnClickListener {
            // Show loading
            progressCircular.visibility = View.VISIBLE
            
            // Simulate progress
            var progress = 0
            val handler = Handler(Looper.getMainLooper())
            
            val runnable = object : Runnable {
                override fun run() {
                    progress += 10
                    progressHorizontal.progress = progress
                    
                    if (progress < 100) {
                        handler.postDelayed(this, 500)
                    } else {
                        progressCircular.visibility = View.GONE
                        Toast.makeText(this@MainActivity, "Selesai!", Toast.LENGTH_SHORT).show()
                    }
                }
            }
            handler.post(runnable)
        }
    }
}

Activity

Pengertian Activity

Activity adalah komponen fundamental dalam aplikasi Android yang menyediakan layar (screen) untuk berinteraksi dengan pengguna. Setiap Activity merepresentasikan satu layar dengan user interface.

Karakteristik Activity

  • Setiap Activity adalah subclass dari AppCompatActivity
  • Setiap Activity harus didaftarkan di AndroidManifest.xml
  • Activity memiliki lifecycle yang dikelola oleh sistem Android
  • Aplikasi bisa memiliki multiple Activities
  • Activity bisa memiliki satu atau lebih layout

Activity Lifecycle

Activity memiliki siklus hidup (lifecycle) yang terdiri dari beberapa state. Memahami lifecycle penting untuk mengelola resource dengan benar.

onCreate()
↓
onStart()
↓
onResume()
↓
RUNNING
↓
onPause()
↓
onStop()
↓
onDestroy()

Penjelasan Callback Methods

Method Dipanggil Ketika Kegunaan
onCreate() Activity pertama kali dibuat Inisialisasi UI, binding view, setup data
onStart() Activity menjadi visible Persiapan untuk tampil ke user
onResume() Activity siap berinteraksi Start animasi, acquire camera/sensor
onPause() Activity kehilangan focus Pause animasi, save draft data
onStop() Activity tidak lagi visible Release heavy resources
onDestroy() Activity akan dihancurkan Final cleanup, release all resources
onRestart() Activity di-restart dari stopped Refresh data jika diperlukan

Intent adalah objek messaging yang digunakan untuk meminta aksi dari komponen lain. Intent dapat digunakan untuk:

  • Memulai Service
  • Mengirim Broadcast

Implementasi Implicit Intent

ImplicitIntentActivity.kt 
package com.example.helloworld

import android.content.Intent
import android.net.Uri
import android.os.Bundle
import android.widget.Button
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity

class ImplicitIntentActivity : AppCompatActivity() {
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_implicit_intent)
        
        setupButtons()
    }
    
    private fun setupButtons() {
        // 1. Membuka URL di Browser
        findViewById<Button>(R.id.btnOpenBrowser).setOnClickListener {
            val url = "https://www.google.com"
            val intent = Intent(Intent.ACTION_VIEW, Uri.parse(url))
            safeStartActivity(intent, "Tidak ada browser yang tersedia")
        }
        
        // 2. Membuka Dialer untuk Panggilan Telepon
        findViewById<Button>(R.id.btnDial).setOnClickListener {
            val phoneNumber = "08123456789"
            val intent = Intent(Intent.ACTION_DIAL)
            intent.data = Uri.parse("tel:$phoneNumber")
            safeStartActivity(intent, "Tidak ada aplikasi telepon")
        }
        
        // 3. Mengirim Email
        findViewById<Button>(R.id.btnSendEmail).setOnClickListener {
            val intent = Intent(Intent.ACTION_SENDTO).apply {
                data = Uri.parse("mailto:")
                putExtra(Intent.EXTRA_EMAIL, arrayOf("contoh@email.com"))
                putExtra(Intent.EXTRA_SUBJECT, "Subject Email")
                putExtra(Intent.EXTRA_TEXT, "Isi pesan email...")
            }
            safeStartActivity(intent, "Tidak ada aplikasi email")
        }
        
        // 4. Mengirim SMS
        findViewById<Button>(R.id.btnSendSms).setOnClickListener {
            val phoneNumber = "08123456789"
            val message = "Halo, ini pesan SMS!"
            val intent = Intent(Intent.ACTION_SENDTO).apply {
                data = Uri.parse("smsto:$phoneNumber")
                putExtra("sms_body", message)
            }
            safeStartActivity(intent, "Tidak ada aplikasi SMS")
        }
        
        // 5. Share Text
        findViewById<Button>(R.id.btnShare).setOnClickListener {
            val shareText = "Ini adalah teks yang akan dibagikan!"
            val intent = Intent(Intent.ACTION_SEND).apply {
                type = "text/plain"
                putExtra(Intent.EXTRA_TEXT, shareText)
            }
            startActivity(Intent.createChooser(intent, "Bagikan melalui"))
        }
        
        // 6. Buka Lokasi di Maps
        findViewById<Button>(R.id.btnOpenMap).setOnClickListener {
            val latitude = -6.2088
            val longitude = 106.8456
            val label = "Jakarta"
            val uri = Uri.parse("geo:$latitude,$longitude?q=$latitude,$longitude($label)")
            val intent = Intent(Intent.ACTION_VIEW, uri)
            safeStartActivity(intent, "Tidak ada aplikasi maps")
        }
    }
    
    private fun safeStartActivity(intent: Intent, errorMessage: String) {
        // Cek apakah ada aplikasi yang bisa menangani intent
        if (intent.resolveActivity(packageManager) != null) {
            startActivity(intent)
        } else {
            Toast.makeText(this, errorMessage, Toast.LENGTH_SHORT).show()
        }
    }
}

Perhatian

Saat menggunakan Implicit Intent, selalu gunakan resolveActivity() atau try-catch untuk mengecek apakah ada aplikasi yang dapat menangani intent. Jika tidak ada, aplikasi akan crash.

Common Intent Actions

Action Kegunaan
ACTION_VIEW Menampilkan data (URL, lokasi, dll)
ACTION_SEND Mengirim/share data ke aplikasi lain
ACTION_SENDTO Mengirim ke alamat tertentu (email, SMS)
ACTION_DIAL Membuka dialer dengan nomor tertentu
ACTION_CALL Langsung melakukan panggilan (perlu permission)
ACTION_PICK Memilih data dari content provider
ACTION_EDIT Mengedit data

Mengirim Data dari FormActivity

FormActivity.kt - Mengirim Data 
package com.example.helloworld

import android.content.Intent
import android.os.Bundle
import android.widget.Button
import android.widget.CheckBox
import android.widget.RadioGroup
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import com.google.android.material.textfield.TextInputEditText

class FormActivity : AppCompatActivity() {
    
    private lateinit var etName: TextInputEditText
    private lateinit var etEmail: TextInputEditText
    private lateinit var etAge: TextInputEditText
    private lateinit var rgGender: RadioGroup
    private lateinit var cbAgree: CheckBox
    private lateinit var btnSubmit: Button
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_form)
        
        initViews()
        setupSubmitButton()
    }
    
    private fun initViews() {
        etName = findViewById(R.id.etName)
        etEmail = findViewById(R.id.etEmail)
        etAge = findViewById(R.id.etAge)
        rgGender = findViewById(R.id.rgGender)
        cbAgree = findViewById(R.id.cbAgree)
        btnSubmit = findViewById(R.id.btnSubmit)
    }
    
    private fun setupSubmitButton() {
        btnSubmit.setOnClickListener {
            if (validateInput()) {
                sendDataToResultActivity()
            }
        }
    }
    
    private fun validateInput(): Boolean {
        val name = etName.text.toString().trim()
        val email = etEmail.text.toString().trim()
        val age = etAge.text.toString().trim()
        
        // Validasi nama
        if (name.isEmpty()) {
            etName.error = "Nama tidak boleh kosong"
            etName.requestFocus()
            return false
        }
        
        // Validasi email
        if (email.isEmpty()) {
            etEmail.error = "Email tidak boleh kosong"
            etEmail.requestFocus()
            return false
        }
        
        if (!android.util.Patterns.EMAIL_ADDRESS.matcher(email).matches()) {
            etEmail.error = "Format email tidak valid"
            etEmail.requestFocus()
            return false
        }
        
        // Validasi usia
        if (age.isEmpty()) {
            etAge.error = "Usia tidak boleh kosong"
            etAge.requestFocus()
            return false
        }
        
        // Validasi gender
        if (rgGender.checkedRadioButtonId == -1) {
            Toast.makeText(this, "Pilih jenis kelamin", Toast.LENGTH_SHORT).show()
            return false
        }
        
        // Validasi checkbox
        if (!cbAgree.isChecked) {
            Toast.makeText(this, "Anda harus menyetujui syarat dan ketentuan", Toast.LENGTH_SHORT).show()
            return false
        }
        
        return true
    }
    
    private fun sendDataToResultActivity() {
        val name = etName.text.toString().trim()
        val email = etEmail.text.toString().trim()
        val age = etAge.text.toString().toInt()
        val gender = when (rgGender.checkedRadioButtonId) {
            R.id.rbMale -> "Laki-laki"
            R.id.rbFemale -> "Perempuan"
            else -> ""
        }
        
        // Membuat intent dengan data
        val intent = Intent(this, ResultActivity::class.java).apply {
            putExtra(ResultActivity.EXTRA_NAME, name)
            putExtra(ResultActivity.EXTRA_EMAIL, email)
            putExtra(ResultActivity.EXTRA_AGE, age)
            putExtra(ResultActivity.EXTRA_GENDER, gender)
        }
        
        startActivity(intent)
    }
}

package com.example.helloworld

import android.os.Bundle
import android.widget.Button

import androidx.appcompat.app.AppCompatActivity

class ResultActivity : AppCompatActivity() {
    
    companion object {
        const val EXTRA_NAME = "extra_name"
        const val EXTRA_EMAIL = "extra_email"
        const val EXTRA_AGE = "extra_age"
        const val EXTRA_GENDER = "extra_gender"
    }
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_result)
        
        // Menerima data dari Intent
        val name = intent.getStringExtra(EXTRA_NAME) ?: "Tidak ada data"
        val email = intent.getStringExtra(EXTRA_EMAIL) ?: "Tidak ada data"
        val age = intent.getIntExtra(EXTRA_AGE, 0)
        val gender = intent.getStringExtra(EXTRA_GENDER) ?: "Tidak ada data"
        
        // Menampilkan data
        findViewById<TextView>(R.id.tvName).text = "Nama: $name"
        findViewById<TextView>(R.id.tvEmail).text = "Email: $email"
        findViewById<TextView>(R.id.tvAge).text = "Usia: $age tahun"
        findViewById<TextView>(R.id.tvGender).text = "Jenis Kelamin: $gender"
        
        // Tombol kembali
        findViewById<Button>(R.id.btnBack).setOnClickListener {
            finish()
        }
    }
}

Tugas Praktikum

Tugas 1: Aplikasi Form Biodata

Buatlah aplikasi dengan spesifikasi sebagai berikut:

  1. Halaman pertama (MainActivity) berisi form input:
    • Nama lengkap (EditText)
    • NIM (EditText dengan inputType number)
    • Program Studi (Spinner dengan pilihan)
    • Jenis Kelamin (RadioButton)
    • Hobi (CheckBox - minimal 3 pilihan)
  2. Implementasikan validasi input yang baik
  3. Gunakan Style untuk konsistensi tampilan
  4. Tombol "Tampilkan Profil" untuk berpindah ke halaman kedua
  5. Halaman kedua (ProfileActivity) menampilkan data yang diinput dengan tampilan yang menarik (gunakan CardView)
  6. Implementasikan localization (minimal 2 bahasa)

Kriteria Penilaian

  • Implementasi Layout (ConstraintLayout) - 20%
  • Penggunaan Widget yang tepat - 20%
  • Validasi Input - 15%
  • Style dan Theme - 15%
  • Intent dan Data Passing - 20%
  • Localization - 10%

Tugas 2: Aplikasi Kalkulator Sederhana

Buatlah aplikasi kalkulator dengan spesifikasi:

  1. Halaman pertama berisi:
    • Input angka pertama
    • Input angka kedua
    • Spinner untuk pilihan operasi (+, -, ×, ÷)
    • Tombol Hitung
  2. Halaman kedua menampilkan:
    • Angka pertama
    • Operasi yang dipilih
    • Angka kedua
    • Hasil perhitungan
  3. Implementasikan validasi:
    • Input tidak boleh kosong
    • Pembagian dengan 0 harus ditangani
  4. Gunakan Material Design components

Tugas 3: Aplikasi Intent Explorer

Buatlah aplikasi yang mengeksplorasi berbagai jenis Intent:

  1. Tombol untuk membuka website kampus
  2. Tombol untuk melakukan panggilan ke nomor kampus
  3. Tombol untuk mengirim email ke dosen
  4. Tombol untuk share text ke media sosial
  5. Tombol untuk membuka lokasi kampus di Maps
  6. Buat tampilan yang menarik dengan ikon untuk setiap tombol

Tugas 4: Mini Project - Aplikasi Catatan Sederhana

Buatlah aplikasi catatan sederhana dengan fitur:

  1. Halaman utama menampilkan daftar catatan (sementara bisa hardcode)
  2. Tombol untuk menambah catatan baru
  3. Halaman form untuk menambah catatan:
    • Judul catatan
    • Isi catatan (multiline)
    • Kategori (Spinner: Pekerjaan, Pribadi, Belajar)
    • Prioritas (RadioButton: Tinggi, Sedang, Rendah)
  4. Halaman detail catatan
  5. Implementasikan semua konsep yang telah dipelajari:
    • Layout yang baik (ConstraintLayout)
    • Berbagai widget
    • Style dan Theme
    • Localization
    • Activity dan Intent
