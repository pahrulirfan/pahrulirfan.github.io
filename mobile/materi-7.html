Apa itu Retrofit?

Retrofit adalah library buatan Square yang menjadi standar de-facto untuk melakukan HTTP request di Android. Retrofit mengubah API endpoint menjadi interface Kotlin yang mudah dipanggil, menangani serialisasi JSON secara otomatis, dan mendukung Coroutines untuk operasi async.

Konsep REST API

Apa itu REST API?

REST (Representational State Transfer) adalah gaya arsitektur untuk membangun layanan web. API (Application Programming Interface) adalah antarmuka yang memungkinkan dua aplikasi berkomunikasi. Jadi REST API adalah cara aplikasi berkomunikasi melalui HTTP dengan aturan-aturan tertentu.

Dalam konteks Android, aplikasi kita berperan sebagai client yang mengirim request ke server, lalu server membalas dengan data dalam format JSON.

Alur komunikasi Client → Server 
Android App (Client)
        │
        │  HTTP Request
        │  GET https://api.pahrul.my.id/api/vehicles
        │  Headers: Content-Type: application/json
        ▼
    Server (API)
        │
        │  HTTP Response
        │  Status: 200 OK
        │  Body: { "success": true, "data": [...] }
        ▼
Android App (Client)
        │
        └─ Parse JSON → tampilkan ke RecyclerView

HTTP Method, Status Code & JSON

HTTP Method

Setiap request ke API menggunakan salah satu HTTP method berikut. Method menentukan apa yang ingin dilakukan terhadap data di server.

Method Fungsi Contoh Ada Request Body?
GET Ambil data Tampilkan daftar vehicle Tidak
POST Kirim / buat data baru Tambah vehicle baru Ya
PUT Update seluruh data Edit data vehicle Ya
DELETE Hapus data Hapus vehicle Tidak

HTTP Status Code

Setiap response dari server menyertakan status code — angka 3 digit yang menunjukkan hasil dari request.

Kode Arti Kapan Muncul
200 OK Berhasil GET, PUT, DELETE sukses
201 Created Data berhasil dibuat POST sukses
404 Not Found Data tidak ditemukan ID tidak ada di database
422 Unprocessable Validasi gagal Field wajib tidak diisi
401 Unauthorized Tidak punya akses Token tidak valid / tidak ada
500 Server Error Error di server Bug di sisi server

Format JSON

JSON (JavaScript Object Notation) adalah format pertukaran data yang ringan dan mudah dibaca. Semua response dari API Vehicle menggunakan format JSON.

Anatomi JSON response dari API Vehicle 
{
  "success": true,          // Boolean — apakah request berhasil
  "message": "Data vehicle ditemukan",  // String — pesan dari server
  "data": {                 // Object atau Array — isi data utama
    "id": 1,                // Number
    "model": "Civic",       // String
    "type": "Sedan",        // String
    "manufacturer": "Honda",// String
    "created_at": "2026-04-30T10:00:00.000000Z"
  }
}

Wrapper Response

API Vehicle menggunakan pola wrapper response — data selalu dibungkus dalam objek dengan field success, message, dan data. Pola ini umum dipakai agar client bisa dengan mudah mengecek apakah request berhasil sebelum memproses datanya.

📌 Dua Cara Menambahkan Dependency

Android Studio versi terbaru (Ladybug 2024+) menggunakan Version Catalog (TOML) untuk mengelola dependency. Project lama mungkin masih menggunakan cara langsung di KTS.

Penjelasan Format Dependency

Dalam satu file build.gradle.kts, Anda bisa mencampur dua format:

  • implementation(libs.nama.library) → Menggunakan Version Catalog (TOML)
  • implementation("group:artifact:version") → Menulis versi langsung

Dependency bawaan Android Studio biasanya sudah menggunakan TOML, jadi biarkan saja seperti itu. Anda hanya perlu menambahkan dependency baru (Retrofit, RecyclerView, dll) dengan format versi langsung.

Sync Gradle

Setelah menambahkan dependency (baik dengan Cara A atau B), klik Sync Now yang muncul di bagian atas editor. Tunggu hingga proses sync selesai.

Troubleshooting Gradle Sync

  • Jika sync gagal: Pastikan koneksi internet stabil
  • Jika muncul error "Unresolved reference": Pastikan nama library di TOML sesuai dengan yang dipanggil di KTS (ganti titik dengan garis bawah, contoh: lifecycle.runtime.ktx → lifecycle-runtime-ktx)
  • Jika muncul error versi: Pastikan dependency ditulis di file build.gradle.kts (Module: app), bukan build.gradle.kts (Project: ...)
  • Jika masih error: Coba File → Invalidate Caches → Invalidate and Restart

Langkah 3: Tambahkan Izin Internet

Aplikasi Android harus meminta izin untuk mengakses internet. Tanpa izin ini, aplikasi tidak bisa memanggil API.

Cara menambahkan:

  1. Buka file AndroidManifest.xml di folder app/src/main
  2. Tambahkan baris <uses-permission ... /> sebelum tag <application>
AndroidManifest.xml — tambahkan izin internet 
<manifest xmlns:android="http://schemas.android.com/apk/res/android">

    <!-- Tambahkan baris ini -->
    <uses-permission android:name="android.permission.INTERNET" />

    <application
        android:allowBackup="true"
        android:theme="@style/Theme.VehicleApiApp"
        android:label="@string/app_name"
        android:supportsRtl="true">

        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>

Kenapa Perlu Izin Internet?

Android menerapkan sistem permission untuk melindungi privasi user. Tanpa deklarasi INTERNET di manifest, aplikasi tidak bisa melakukan koneksi jaringan dan akan error saat memanggil API.

Membuat Data Model

Model digunakan untuk memetakan response JSON dari API menjadi objek Kotlin. Karena API menggunakan format wrapper, kita perlu membuat model untuk data utama dan model untuk pembungkus response.

Buat file Vehicle.kt di package model. File ini berisi data class untuk merepresentasikan satu data kendaraan. Field pada data class harus mengikuti nama field pada JSON.

Cara Membuat File Kotlin

  1. Klik kanan pada package model
  2. Pilih New → Kotlin Class/File
  3. Ketik nama: Vehicle
  4. Pilih jenis: Class
  5. Tekan Enter, lalu isi dengan kode berikut
model/Vehicle.kt — Data Class 
package com.example.vehicleapiapp.model

data class Vehicle(
    val id: Int? = null,
    val model: String,
    val type: String,
    val manufacturer: String,
    val created_at: String? = null,
    val updated_at: String? = null
)

2. Buat File ApiResponse.kt — Generic Data Class

API mengembalikan response dengan bentuk success, message, dan data. Karena isi data bisa berupa satu object atau list, buat generic data class yang bisa menerima tipe data apapun.

model/ApiResponse.kt — Generic Data Class 
package com.example.vehicleapiapp.model

data class ApiResponse<T>(
    val success: Boolean,
    val message: String,
    val data: T?
)

Penjelasan <T> di ApiResponse?

<T> adalah Generic Type — placeholder untuk tipe data yang bisa diganti saat digunakan. Huruf T singkatan dari "Type", tapi bisa diganti huruf apa saja (A, B, Data, dll).

Analogi sederhana: Bayangkan <T> seperti kotak kosong yang bisa diisi berbagai jenis barang.

Saat DigunakanT Diganti DenganHasil
ApiResponse<Vehicle> Vehicle data bertipe Vehicle?
ApiResponse<List<Vehicle>> List<Vehicle> data bertipe List<Vehicle>?
ApiResponse<String> String data bertipe String?

Contoh Penggunaan di ApiService:

// GET semua vehicle → data berisi List
suspend fun getVehicles(): Response<ApiResponse<List<Vehicle>>>

// POST vehicle baru → data berisi satu Vehicle
suspend fun createVehicle(@Body vehicle: Vehicle): Response<ApiResponse<Vehicle>>

// DELETE vehicle → data berisi satu Vehicle
suspend fun deleteVehicle(@Path("id") id: Int): Response<ApiResponse<Vehicle>>

Kenapa Pakai Generic?

  • Reusable — Satu class bisa dipakai untuk berbagai tipe response
  • Type-safe — Compiler tahu tipe data yang diharapkan, mengurangi error
  • Konsisten — Semua response punya struktur yang sama (success, message, data)
  • Tanpa generic, harus buat VehicleResponse, VehicleListResponse, UserResponse, dll (repetitif!)

Penjelasan Jenis File

  • Data Class — Class khusus untuk menyimpan data, otomatis punya fungsi toString(), equals(), copy()
  • Generic <T> — Tipe data yang fleksibel, bisa diisi dengan tipe apapun (Vehicle, List<Vehicle>, dll)
  • Nullable ? — Field yang boleh kosong/null

Kenapa id Bersifat Nullable?

Saat menambah data baru, aplikasi hanya mengirim model, type, dan manufacturer. Field id dibuat oleh server, sehingga pada request POST nilainya boleh kosong.

Tahap 1: Buat ApiService dan RetrofitClient

Sebelum bisa mengambil atau mengirim data ke server, kita perlu membuat dua file di package network: ApiService.kt yang mendefinisikan semua endpoint, dan RetrofitClient.kt yang mengatur koneksi ke server. Kedua file ini menjadi fondasi untuk semua operasi CRUD di tahap berikutnya.

File 1: ApiService.kt

ApiService adalah interface yang mendaftarkan semua endpoint API yang akan dipakai aplikasi. Setiap fungsi mewakili satu endpoint dengan method HTTP-nya masing-masing.

Cara membuat file:

  1. Klik kanan pada package network
  2. Pilih New → Kotlin Class/File
  3. Ketik ApiService, pilih jenis Interface, tekan Enter
  4. Ganti seluruh isi file dengan kode berikut
network/ApiService.kt 
package com.example.vehicleapiapp.network

import com.example.vehicleapiapp.model.ApiResponse
import com.example.vehicleapiapp.model.Vehicle
import retrofit2.Response
import retrofit2.http.Body
import retrofit2.http.DELETE
import retrofit2.http.GET
import retrofit2.http.POST
import retrofit2.http.PUT
import retrofit2.http.Path

interface ApiService {

    @GET("vehicles")
    suspend fun getVehicles(): Response<ApiResponse<List<Vehicle>>>

    @POST("vehicles")
    suspend fun createVehicle(
        @Body vehicle: Vehicle
    ): Response<ApiResponse<Vehicle>>

    @PUT("vehicles/{id}")
    suspend fun updateVehicle(
        @Path("id") id: Int,
        @Body vehicle: Vehicle
    ): Response<ApiResponse<Vehicle>>

    @DELETE("vehicles/{id}")
    suspend fun deleteVehicle(
        @Path("id") id: Int
    ): Response<Unit>
}

Penjelasan Anotasi di ApiService

AnotasiPenjelasan
@GET("vehicles")HTTP GET ke BASE_URL/vehicles — untuk mengambil semua data
@POST("vehicles")HTTP POST ke BASE_URL/vehicles — untuk menambah data baru
@PUT("vehicles/{id}")HTTP PUT ke BASE_URL/vehicles/1 — untuk mengupdate data berdasarkan ID
@DELETE("vehicles/{id}")HTTP DELETE ke BASE_URL/vehicles/1 — untuk menghapus data berdasarkan ID
@Path("id")Mengganti {id} di URL dengan nilai yang dikirim. Contoh: id = 3 → URL menjadi vehicles/3
@Body vehicleObjek Vehicle diubah menjadi JSON dan dikirim sebagai body request (untuk POST dan PUT)

Kenapa DELETE Menggunakan Response<Unit>?

Perhatikan bahwa endpoint DELETE berbeda dengan endpoint lainnya:

EndpointResponse TypeAlasan
getVehicles() Response<ApiResponse<List<Vehicle>>> Mengembalikan list data vehicle
createVehicle() Response<ApiResponse<Vehicle>> Mengembalikan data vehicle yang baru dibuat
updateVehicle() Response<ApiResponse<Vehicle>> Mengembalikan data vehicle yang sudah diupdate
deleteVehicle() Response<Unit> Tidak mengembalikan data, hanya status sukses/gagal

Penjelasan: Response DELETE biasanya hanya mengembalikan status code (200 = sukses, 404 = tidak ditemukan) tanpa body data. Menggunakan Response<Unit> berarti kita hanya peduli dengan status response, tidak peduli dengan body-nya.

import okhttp3.OkHttpClient
import okhttp3.logging.HttpLoggingInterceptor
import retrofit2.Retrofit
import retrofit2.converter.gson.GsonConverterFactory

object RetrofitClient {
    private const val BASE_URL = "https://api.pahrul.my.id/api/"

    private val loggingInterceptor = HttpLoggingInterceptor().apply {
        level = HttpLoggingInterceptor.Level.BODY

    private val okHttpClient = OkHttpClient.Builder()

        .build()

            .baseUrl(BASE_URL)
            .client(okHttpClient)
            .addConverterFactory(GsonConverterFactory.create())
            .build()
            .create(ApiService::class.java)
    }
}

import android.os.Bundle

    private lateinit var progressBar: ProgressBar
    private lateinit var fabAdd: FloatingActionButton

    override fun onCreate(savedInstanceState: Bundle?) {

        setContentView(R.layout.activity_main)

import android.os.Bundle
import android.view.View
import android.widget.Button
import android.widget.ProgressBar
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.lifecycleScope
import com.example.vehicleapiapp.model.Vehicle
import com.example.vehicleapiapp.network.RetrofitClient
import com.google.android.material.textfield.TextInputEditText
import kotlinx.coroutines.launch

class AddVehicleActivity : AppCompatActivity() {

    private lateinit var etModel: TextInputEditText
    private lateinit var etType: TextInputEditText
    private lateinit var etManufacturer: TextInputEditText
    private lateinit var btnSave: Button
    private lateinit var progressBar: ProgressBar

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_add_vehicle)

        // Inisialisasi view
        etModel = findViewById(R.id.etModel)
        etType = findViewById(R.id.etType)
        etManufacturer = findViewById(R.id.etManufacturer)
        btnSave = findViewById(R.id.btnSave)
        progressBar = findViewById(R.id.progressBar)

        // Setup tombol simpan
        btnSave.setOnClickListener {
            saveVehicle()
        }
    }

    private fun saveVehicle() {
        // Ambil data dari input
        val model = etModel.text.toString().trim()
        val type = etType.text.toString().trim()
        val manufacturer = etManufacturer.text.toString().trim()

        // Validasi input
        if (model.isEmpty()) {
            etModel.error = "Model tidak boleh kosong"
            etModel.requestFocus()
            return
        }

        if (type.isEmpty()) {
            etType.error = "Tipe tidak boleh kosong"
            etType.requestFocus()
            return
        }

        if (manufacturer.isEmpty()) {
            etManufacturer.error = "Produsen tidak boleh kosong"
            etManufacturer.requestFocus()
            return
        }

        // Buat object Vehicle
        val vehicle = Vehicle(
            model = model,
            type = type,
            manufacturer = manufacturer
        )

        // Kirim ke API
        createVehicle(vehicle)
    }

    private fun createVehicle(vehicle: Vehicle) {
        lifecycleScope.launch {
            showLoading(true)

            try {
                val response = RetrofitClient.apiService.createVehicle(vehicle)

                if (response.isSuccessful) {
                    showMessage("Data berhasil ditambahkan")
                    // Kembali ke MainActivity
                    finish()
                } else {
                    showMessage("Gagal menambah data: ${response.code()}")
                }
            } catch (e: Exception) {
                showMessage("Error: ${e.message}")
            } finally {
                showLoading(false)
            }
        }
    }

    private fun showLoading(isLoading: Boolean) {
        progressBar.visibility = if (isLoading) View.VISIBLE else View.GONE
        btnSave.isEnabled = !isLoading
    }

    private fun showMessage(message: String) {
        Toast.makeText(this, message, Toast.LENGTH_SHORT).show()
    }
}

package com.example.vehicleapiapp

import android.os.Bundle
import android.view.View
import android.widget.Button
import android.widget.ProgressBar
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.lifecycleScope
import com.example.vehicleapiapp.model.Vehicle
import com.example.vehicleapiapp.network.RetrofitClient
import com.google.android.material.textfield.TextInputEditText
import kotlinx.coroutines.launch

class EditVehicleActivity : AppCompatActivity() {

    private lateinit var etModel: TextInputEditText
    private lateinit var etType: TextInputEditText
    private lateinit var etManufacturer: TextInputEditText
    private lateinit var btnUpdate: Button
    private lateinit var progressBar: ProgressBar

    private var vehicleId: Int = 0

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_edit_vehicle)

        // Inisialisasi view
        etModel = findViewById(R.id.etModel)
        etType = findViewById(R.id.etType)
        etManufacturer = findViewById(R.id.etManufacturer)
        btnUpdate = findViewById(R.id.btnUpdate)
        progressBar = findViewById(R.id.progressBar)

        // Ambil data dari Intent extras
        vehicleId = intent.getIntExtra("VEHICLE_ID", 0)
        val model = intent.getStringExtra("VEHICLE_MODEL") ?: ""
        val type = intent.getStringExtra("VEHICLE_TYPE") ?: ""
        val manufacturer = intent.getStringExtra("VEHICLE_MANUFACTURER") ?: ""

        // Validasi: pastikan ID valid
        if (vehicleId == 0) {
            showMessage("Data vehicle tidak ditemukan")
            finish()
            return
        }

        // Isi form dengan data existing
        etModel.setText(model)
        etType.setText(type)
        etManufacturer.setText(manufacturer)

        // Setup tombol update
        btnUpdate.setOnClickListener {
            updateVehicle()
        }
    }

    private fun updateVehicle() {
        // Ambil data dari input
        val model = etModel.text.toString().trim()
        val type = etType.text.toString().trim()
        val manufacturer = etManufacturer.text.toString().trim()

        // Validasi input
        if (model.isEmpty()) {
            etModel.error = "Model tidak boleh kosong"
            etModel.requestFocus()
            return
        }

        if (type.isEmpty()) {
            etType.error = "Tipe tidak boleh kosong"
            etType.requestFocus()
            return
        }

        if (manufacturer.isEmpty()) {
            etManufacturer.error = "Produsen tidak boleh kosong"
            etManufacturer.requestFocus()
            return
        }

        // Buat object Vehicle dengan data baru
        val vehicle = Vehicle(
            model = model,
            type = type,
            manufacturer = manufacturer
        )

        // Kirim update ke API
        sendUpdateToApi(vehicleId, vehicle)
    }

    private fun sendUpdateToApi(id: Int, vehicle: Vehicle) {
        lifecycleScope.launch {
            showLoading(true)

            try {
                val response = RetrofitClient.apiService.updateVehicle(id, vehicle)

                if (response.isSuccessful) {
                    showMessage("Data berhasil diupdate")
                    // Kembali ke MainActivity
                    finish()
                } else {
                    showMessage("Gagal update data: ${response.code()}")
                }
            } catch (e: Exception) {
                showMessage("Error: ${e.message}")
            } finally {
                showLoading(false)
            }
        }
    }

    private fun showLoading(isLoading: Boolean) {
        progressBar.visibility = if (isLoading) View.VISIBLE else View.GONE
        btnUpdate.isEnabled = !isLoading
    }

    private fun showMessage(message: String) {
        Toast.makeText(this, message, Toast.LENGTH_SHORT).show()
    }
}

class MainActivity : AppCompatActivity() {

    private lateinit var progressBar: ProgressBar

    override fun onCreate(savedInstanceState: Bundle?) {

        // Setup RecyclerView dengan lambda untuk Edit dan Delete

            .setPositiveButton("Hapus") { dialog, _ ->

                deleteVehicle(vehicle)
                dialog.dismiss()

            .setNegativeButton("Batal") { dialog, _ ->
                // User klik Batal
                dialog.dismiss()
            }
            .show()
    }

    private fun deleteVehicle(vehicle: Vehicle) {
        val id = vehicle.id ?: run {
            showMessage("ID vehicle tidak valid")
            return
        }

        lifecycleScope.launch {
            showLoading(true)

            try {
                val response = RetrofitClient.apiService.deleteVehicle(id)

                if (response.isSuccessful) {
                    showMessage("Data berhasil dihapus")
                    // Refresh data setelah delete
                    loadVehicles()
                } else {
                    showMessage("Gagal menghapus data: ${response.code()}")
                }
            } catch (e: Exception) {
                showMessage("Error: ${e.message}")
            } finally {
                showLoading(false)
            }
        }
    }

    private fun loadVehicles() {
        lifecycleScope.launch {
            showLoading(true)
            
            try {
                val response = RetrofitClient.apiService.getVehicles()

                    val vehicles = response.body()?.data ?: emptyList()
                    adapter.setData(vehicles)
                    
                    if (vehicles.isEmpty()) {
                        showMessage("Belum ada data vehicle")
                    }
                } else {
                    showMessage("Gagal mengambil data: ${response.code()}")
                }
            } catch (e: Exception) {
                showMessage("Error: ${e.message}")
            } finally {
                showLoading(false)
            }
        }
    }

    }

    private fun showMessage(message: String) {

    }

Kenapa Wajib?

  • Tampilkan informasi spesifik (nama item yang akan dihapus)
  • Gunakan warna merah untuk tombol delete (indikasi bahaya)

Cara Mengakses Data Setelah Response Diterima

Setelah Retrofit menerima response dan Gson mengubahnya menjadi objek Kotlin, data diakses seperti ini:

val response = RetrofitClient.apiService.login(request)

// Akses token

// Akses nama user
val name = response.body()?.data?.user?.name

// Akses pesan dari server
val message = response.body()?.message

Tanda ?. (safe call) digunakan karena setiap level bisa bernilai null — misalnya jika login gagal, data akan null sehingga akses ke data?.token tidak menyebabkan crash.

import okhttp3.OkHttpClient
import okhttp3.logging.HttpLoggingInterceptor
import retrofit2.Retrofit
import retrofit2.converter.gson.GsonConverterFactory

object RetrofitClient {
    private const val BASE_URL = "https://api.pahrul.my.id/api/"

    private val loggingInterceptor = HttpLoggingInterceptor().apply {
        level = HttpLoggingInterceptor.Level.BODY
    }

    private val okHttpClient = OkHttpClient.Builder()
        .addInterceptor(loggingInterceptor)
        .build()

    val apiService: ApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .client(okHttpClient)
            .addConverterFactory(GsonConverterFactory.create())
            .build()
            .create(ApiService::class.java)
    }
}

7. Buat MainActivity

MainActivity menangani seluruh logika login: membaca input dari form, memvalidasi, mengirim request ke API, lalu berpindah ke HomeActivity jika berhasil.

File ini sudah otomatis terbuat saat project dibuat. Buka MainActivity.kt dan ganti seluruh isinya dengan kode berikut.

Alur Kerja MainActivity

  1. User mengisi email dan password, lalu klik tombol Login
  2. Aplikasi memvalidasi input — jika kosong, tampilkan pesan error
  3. Buat objek LoginRequest dari input user
  4. Kirim request ke API menggunakan Retrofit di dalam coroutine
  5. Jika response berhasil (isSuccessful), ambil nama user dari response
  6. Buka HomeActivity dan kirim nama user via Intent
  7. Jika gagal, tampilkan pesan error dari server
MainActivity.kt 
package com.example.loginapiapp

import android.content.Intent
import android.os.Bundle
import android.view.View
import android.widget.Button
import android.widget.EditText
import android.widget.ProgressBar
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.lifecycleScope
import com.example.loginapiapp.model.LoginRequest
import com.example.loginapiapp.network.RetrofitClient
import kotlinx.coroutines.launch

class MainActivity : AppCompatActivity() {
    private lateinit var etEmail: EditText
    private lateinit var etPassword: EditText
    private lateinit var btnLogin: Button
    private lateinit var progressBar: ProgressBar

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        // Hubungkan variabel dengan view di layout
        etEmail = findViewById(R.id.etEmail)
        etPassword = findViewById(R.id.etPassword)
        btnLogin = findViewById(R.id.btnLogin)
        progressBar = findViewById(R.id.progressBar)

        // Panggil fungsi login saat tombol diklik
        btnLogin.setOnClickListener {
            login()
        }
    }

    private fun login() {
        // Ambil teks dari input dan hapus spasi di awal/akhir
        val email = etEmail.text.toString().trim()
        val password = etPassword.text.toString().trim()

        // Validasi: pastikan kedua field tidak kosong
        if (email.isEmpty() || password.isEmpty()) {
            showMessage("Email dan password wajib diisi")
            return
        }

        // Jalankan request di coroutine agar tidak memblokir UI
        lifecycleScope.launch {
            showLoading(true)

            try {
                // Buat objek request dan kirim ke API
                val request = LoginRequest(email, password)
                val response = RetrofitClient.apiService.login(request)

                if (response.isSuccessful) {
                    // Ambil nama user dari response
                    val loginData = response.body()?.data
                    val userName = loginData?.user?.name.orEmpty()

                    if (userName.isNotEmpty()) {
                        // Buka HomeActivity dan kirim nama user
                        val intent = Intent(this@MainActivity, HomeActivity::class.java)
                        intent.putExtra(HomeActivity.EXTRA_NAME, userName)
                        startActivity(intent)
                    } else {
                        showMessage("Data user tidak ditemukan")
                    }
                } else {
                    // Tampilkan pesan dari server jika login gagal
                    val errorMessage = response.body()?.message ?: "Email atau password salah"
                    showMessage(errorMessage)
                }
            } catch (e: Exception) {
                showMessage("Tidak dapat terhubung ke server: ${e.message}")
            } finally {
                // Sembunyikan loading baik berhasil maupun gagal
                showLoading(false)
            }
        }
    }

    private fun showLoading(isLoading: Boolean) {
        progressBar.visibility = if (isLoading) View.VISIBLE else View.GONE
        btnLogin.isEnabled = !isLoading
    }

    private fun showMessage(message: String) {
        Toast.makeText(this, message, Toast.LENGTH_SHORT).show()
    }
}

8. Implementasi Logout

Logout pada aplikasi berbasis token berarti menghapus token yang tersimpan di sisi client sehingga user tidak bisa lagi mengakses halaman yang memerlukan autentikasi. Pada praktikum ini, token disimpan sementara di SharedPreferences, lalu dihapus saat user logout.

Alur Logout

  1. Setelah login berhasil, simpan token ke SharedPreferences
  2. Di HomeActivity, tampilkan tombol Logout di menu toolbar
  3. Saat tombol Logout diklik, hapus token dari SharedPreferences
  4. Kembali ke MainActivity dan tutup HomeActivity agar user tidak bisa kembali dengan tombol Back

Update MainActivity — Simpan Token Setelah Login

if (response.isSuccessful) {
    val loginData = response.body()?.data
    val userName = loginData?.user?.name.orEmpty()
    val token = loginData?.token.orEmpty()      // <-- TAMBAHKAN baris ini

    if (userName.isNotEmpty()) {

    }
} else {
    val errorMessage = response.body()?.message ?: "Email atau password salah"
    showMessage(errorMessage)
}

Update HomeActivity.kt — Tambah Logika Logout

Update HomeActivity.kt untuk menangani klik tombol Logout: hapus token dari SharedPreferences, lalu kembali ke MainActivity.

HomeActivity.kt — update lengkap 
package com.example.loginapiapp

import android.content.Intent
import android.os.Bundle
import android.widget.Button
import android.widget.TextView
import androidx.appcompat.app.AlertDialog
import androidx.appcompat.app.AppCompatActivity

class HomeActivity : AppCompatActivity() {

    companion object {
        const val EXTRA_NAME = "extra_name"
    }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_home)

        // Tampilkan nama user
        val name = intent.getStringExtra(EXTRA_NAME).orEmpty()
        val tvName = findViewById<TextView>(R.id.tvName)
        tvName.text = name

        // Setup tombol logout
        val btnLogout = findViewById<Button>(R.id.btnLogout)
        btnLogout.setOnClickListener {
            showLogoutConfirmation()
        }
    }

    private fun showLogoutConfirmation() {
        AlertDialog.Builder(this)
            .setTitle("Konfirmasi Logout")
            .setMessage("Apakah Anda yakin ingin keluar?")
            .setPositiveButton("Logout") { dialog, _ ->
                logout()
                dialog.dismiss()
            }
            .setNegativeButton("Batal") { dialog, _ ->
                dialog.dismiss()
            }
            .show()
    }

    private fun logout() {
        // Hapus token dari SharedPreferences
        val prefs = getSharedPreferences("auth", MODE_PRIVATE)
        prefs.edit().remove("token").apply()

        // Kembali ke MainActivity dan tutup semua Activity di atasnya
        val intent = Intent(this, MainActivity::class.java)
        intent.flags = Intent.FLAG_ACTIVITY_NEW_TASK or Intent.FLAG_ACTIVITY_CLEAR_TASK
        startActivity(intent)
    }
}

Penjelasan Kode Logout

KodePenjelasan
prefs.edit().remove("token").apply()Menghapus token dari SharedPreferences. Setelah ini token tidak ada lagi di device
FLAG_ACTIVITY_NEW_TASKMembuat task baru untuk MainActivity
FLAG_ACTIVITY_CLEAR_TASKMenutup semua Activity yang ada di back stack. User tidak bisa kembali ke HomeActivity dengan tombol Back setelah logout
AlertDialogDialog konfirmasi sebelum logout — mencegah logout tidak sengaja

Kenapa Perlu FLAG_ACTIVITY_CLEAR_TASK?

Tanpa flag ini, setelah logout user masih bisa menekan tombol Back dan kembali ke HomeActivity tanpa login ulang. Flag CLEAR_TASK memastikan seluruh back stack dibersihkan sehingga satu-satunya halaman yang aktif adalah MainActivity.

Aplikasi Login dan Data Pasien

Buat aplikasi Android Kotlin yang memiliki fitur login menggunakan API, lalu setelah login berhasil aplikasi menampilkan daftar data pasien dari endpoint https://api.pahrul.my.id/api/pasien menggunakan RecyclerView.

Tujuan Tugas

  • Menerapkan login ke API menggunakan Retrofit
  • Menyimpan token dari response login untuk mengakses endpoint yang membutuhkan autentikasi
  • Mengirim header Authorization: Bearer {token} saat mengambil data pasien
  • Menampilkan data pasien ke RecyclerView dengan tampilan yang rapi

Fitur Method Endpoint Keterangan

Alur Aplikasi

  1. User membuka halaman login.

  2. Aplikasi mengirim request login.

  6. Tampilkan data pasien menggunakan RecyclerView.
