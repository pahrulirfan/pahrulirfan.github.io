Retrofit adalah library buatan Square yang menjadi standar de-facto untuk melakukan HTTP request di Android. Retrofit mengubah API endpoint menjadi interface Kotlin yang mudah dipanggil, menangani serialisasi JSON secara otomatis, dan mendukung Coroutines untuk operasi async.

Konsep REST API

Apa itu REST API?

REST (Representational State Transfer) adalah gaya arsitektur untuk membangun layanan web. API (Application Programming Interface) adalah antarmuka yang memungkinkan dua aplikasi berkomunikasi. Jadi REST API adalah cara aplikasi berkomunikasi melalui HTTP dengan aturan-aturan tertentu.

Dalam konteks Android, aplikasi kita berperan sebagai client yang mengirim request ke server, lalu server membalas dengan data dalam format JSON.

Alur komunikasi Client → Server 
Android App (Client)
        │
        │  HTTP Request
        │  GET https://api.pahrul.my.id/api/vehicles
        │  Headers: Content-Type: application/json
        ▼
    Server (API)
        │
        │  HTTP Response
        │  Status: 200 OK
        │  Body: { "success": true, "data": [...] }
        ▼
Android App (Client)
        │
        └─ Parse JSON → tampilkan ke RecyclerView

HTTP Method, Status Code & JSON

HTTP Method

Setiap request ke API menggunakan salah satu HTTP method berikut. Method menentukan apa yang ingin dilakukan terhadap data di server.

Method Fungsi Contoh Ada Request Body?
GET Ambil data Tampilkan daftar vehicle Tidak
POST Kirim / buat data baru Tambah vehicle baru Ya
PUT Update seluruh data Edit data vehicle Ya
DELETE Hapus data Hapus vehicle Tidak

HTTP Status Code

Setiap response dari server menyertakan status code — angka 3 digit yang menunjukkan hasil dari request.

Kode Arti Kapan Muncul
200 OK Berhasil GET, PUT, DELETE sukses
201 Created Data berhasil dibuat POST sukses
404 Not Found Data tidak ditemukan ID tidak ada di database
422 Unprocessable Validasi gagal Field wajib tidak diisi
401 Unauthorized Tidak punya akses Token tidak valid / tidak ada
500 Server Error Error di server Bug di sisi server

Format JSON

JSON (JavaScript Object Notation) adalah format pertukaran data yang ringan dan mudah dibaca. Semua response dari API Vehicle menggunakan format JSON.

Anatomi JSON response dari API Vehicle 
{
  "success": true,          // Boolean — apakah request berhasil
  "message": "Data vehicle ditemukan",  // String — pesan dari server
  "data": {                 // Object atau Array — isi data utama
    "id": 1,                // Number
    "model": "Civic",       // String
    "type": "Sedan",        // String
    "manufacturer": "Honda",// String
    "created_at": "2026-04-30T10:00:00.000000Z"
  }
}

Wrapper Response

API Vehicle menggunakan pola wrapper response — data selalu dibungkus dalam objek dengan field success, message, dan data. Pola ini umum dipakai agar client bisa dengan mudah mengecek apakah request berhasil sebelum memproses datanya.

📌 Dua Cara Menambahkan Dependency

Android Studio versi terbaru (Ladybug 2024+) menggunakan Version Catalog (TOML) untuk mengelola dependency. Project lama mungkin masih menggunakan cara langsung di KTS.

Cek dulu file build.gradle.kts (Module: app) Anda:

  • Jika ada libs.androidx.core.ktx → gunakan Cara A (TOML)
  • Jika ada "androidx.core:core-ktx:1.x.x" → gunakan Cara B (Langsung KTS)

  • Menghindari konflik versi antar module

💡 Penjelasan Format Dependency

Dalam satu file build.gradle.kts, Anda bisa mencampur dua format:

  • implementation(libs.nama.library) → Menggunakan Version Catalog (TOML)
  • implementation("group:artifact:version") → Menulis versi langsung

Dependency bawaan Android Studio biasanya sudah menggunakan TOML, jadi biarkan saja seperti itu. Anda hanya perlu menambahkan dependency baru (Retrofit, RecyclerView, dll) dengan format versi langsung.

Sync Gradle

Setelah menambahkan dependency (baik dengan Cara A atau B), klik Sync Now yang muncul di bagian atas editor. Tunggu hingga proses sync selesai.

Troubleshooting Gradle Sync

  • Jika sync gagal: Pastikan koneksi internet stabil
  • Jika muncul error "Unresolved reference": Pastikan nama library di TOML sesuai dengan yang dipanggil di KTS (ganti titik dengan garis bawah, contoh: lifecycle.runtime.ktx → lifecycle-runtime-ktx)
  • Jika muncul error versi: Pastikan dependency ditulis di file build.gradle.kts (Module: app), bukan build.gradle.kts (Project: ...)
  • Jika masih error: Coba File → Invalidate Caches → Invalidate and Restart

Langkah 3: Tambahkan Izin Internet

Aplikasi Android harus meminta izin untuk mengakses internet. Tanpa izin ini, aplikasi tidak bisa memanggil API.

Cara menambahkan:

  1. Buka file AndroidManifest.xml di folder app/src/main
  2. Tambahkan baris <uses-permission ... /> sebelum tag <application>
AndroidManifest.xml — tambahkan izin internet 
<manifest xmlns:android="http://schemas.android.com/apk/res/android">

    <!-- Tambahkan baris ini -->
    <uses-permission android:name="android.permission.INTERNET" />

    <application
        android:allowBackup="true"
        android:theme="@style/Theme.VehicleApiApp"
        android:label="@string/app_name"
        android:supportsRtl="true">

        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>

Kenapa Perlu Izin Internet?

Android menerapkan sistem permission untuk melindungi privasi user. Tanpa deklarasi INTERNET di manifest, aplikasi tidak bisa melakukan koneksi jaringan dan akan error saat memanggil API.

Membuat Data Model

Model digunakan untuk memetakan response JSON dari API menjadi objek Kotlin. Karena API menggunakan format wrapper, kita perlu membuat model untuk data utama dan model untuk pembungkus response.

1. Buat File Vehicle.kt — Data Class

Buat file Vehicle.kt di package model. File ini berisi data class untuk merepresentasikan satu data kendaraan. Field pada data class harus mengikuti nama field pada JSON.

Cara Membuat File Kotlin

  1. Klik kanan pada package model
  2. Pilih New → Kotlin Class/File
  3. Ketik nama: Vehicle
  4. Pilih jenis: Class
  5. Tekan Enter, lalu isi dengan kode berikut
model/Vehicle.kt — Data Class 
package com.example.vehicleapiapp.model

data class Vehicle(
    val id: Int? = null,
    val model: String,
    val type: String,
    val manufacturer: String,
    val created_at: String? = null,
    val updated_at: String? = null
)

2. Buat File ApiResponse.kt — Generic Data Class

API mengembalikan response dengan bentuk success, message, dan data. Karena isi data bisa berupa satu object atau list, buat generic data class yang bisa menerima tipe data apapun.

model/ApiResponse.kt — Generic Data Class 
package com.example.vehicleapiapp.model

data class ApiResponse<T>(
    val success: Boolean,
    val message: String,
    val data: T?
)

Penjelasan <T> di ApiResponse?

<T> adalah Generic Type — placeholder untuk tipe data yang bisa diganti saat digunakan. Huruf T singkatan dari "Type", tapi bisa diganti huruf apa saja (A, B, Data, dll).

Analogi sederhana: Bayangkan <T> seperti kotak kosong yang bisa diisi berbagai jenis barang.

Saat DigunakanT Diganti DenganHasil
ApiResponse<Vehicle> Vehicle data bertipe Vehicle?
ApiResponse<List<Vehicle>> List<Vehicle> data bertipe List<Vehicle>?
ApiResponse<String> String data bertipe String?

Contoh Penggunaan di ApiService:

// GET semua vehicle → data berisi List
suspend fun getVehicles(): Response<ApiResponse<List<Vehicle>>>

// POST vehicle baru → data berisi satu Vehicle
suspend fun createVehicle(@Body vehicle: Vehicle): Response<ApiResponse<Vehicle>>

// DELETE vehicle → data berisi satu Vehicle
suspend fun deleteVehicle(@Path("id") id: Int): Response<ApiResponse<Vehicle>>

Kenapa Pakai Generic?

  • Reusable — Satu class bisa dipakai untuk berbagai tipe response
  • Type-safe — Compiler tahu tipe data yang diharapkan, mengurangi error
  • Konsisten — Semua response punya struktur yang sama (success, message, data)
  • Tanpa generic, harus buat VehicleResponse, VehicleListResponse, UserResponse, dll (repetitif!)

Penjelasan Jenis File

  • Data Class — Class khusus untuk menyimpan data, otomatis punya fungsi toString(), equals(), copy()
  • Generic <T> — Tipe data yang fleksibel, bisa diisi dengan tipe apapun (Vehicle, List<Vehicle>, dll)
  • Nullable ? — Field yang boleh kosong/null

Kenapa id Bersifat Nullable?

Saat menambah data baru, aplikasi hanya mengirim model, type, dan manufacturer. Field id dibuat oleh server, sehingga pada request POST nilainya boleh kosong.

        )

    }

        etManufacturer = findViewById(R.id.etManufacturer)
        btnUpdate = findViewById(R.id.btnUpdate)
        progressBar = findViewById(R.id.progressBar)

        // Ambil data Vehicle dari Intent
        val vehicle = intent.getParcelableExtra<Vehicle>("VEHICLE_DATA")
        
        if (vehicle != null) {
            // Simpan ID untuk update nanti
            vehicleId = vehicle.id ?: 0

            // Isi form dengan data existing
            etModel.setText(vehicle.model)
            etType.setText(vehicle.type)
            etManufacturer.setText(vehicle.manufacturer)
        } else {
            showMessage("Data vehicle tidak ditemukan")
            finish()
            return
        }

        // Setup tombol update
        btnUpdate.setOnClickListener {
            updateVehicle()
        }
    }

    private fun updateVehicle() {
        // Ambil data dari input
        val model = etModel.text.toString().trim()
        val type = etType.text.toString().trim()
        val manufacturer = etManufacturer.text.toString().trim()

        // Validasi input
        if (model.isEmpty()) {
            etModel.error = "Model tidak boleh kosong"
            etModel.requestFocus()
            return
        }

        if (type.isEmpty()) {
            etType.error = "Tipe tidak boleh kosong"
            etType.requestFocus()
            return
        }

        if (manufacturer.isEmpty()) {
            etManufacturer.error = "Produsen tidak boleh kosong"
            etManufacturer.requestFocus()
            return
        }

        // Buat object Vehicle dengan data baru
        val vehicle = Vehicle(
            model = model,
            type = type,
            manufacturer = manufacturer
        )

        // Kirim update ke API
        sendUpdateToApi(vehicleId, vehicle)
    }

    private fun sendUpdateToApi(id: Int, vehicle: Vehicle) {
        lifecycleScope.launch {
            showLoading(true)

            try {
                val response = RetrofitClient.apiService.updateVehicle(id, vehicle)

                if (response.isSuccessful) {
                    showMessage("Data berhasil diupdate")
                    // Kembali ke MainActivity
                    finish()
                } else {
                    showMessage("Gagal update data: ${response.code()}")
                }
            } catch (e: Exception) {
                showMessage("Error: ${e.message}")
            } finally {
                showLoading(false)
            }
        }
    }

    private fun showLoading(isLoading: Boolean) {
        progressBar.visibility = if (isLoading) View.VISIBLE else View.GONE
        btnUpdate.isEnabled = !isLoading
    }

    private fun showMessage(message: String) {
        Toast.makeText(this, message, Toast.LENGTH_SHORT).show()
    }
}

    private lateinit var etPassword: EditText
    private lateinit var btnLogin: Button
    private lateinit var progressBar: ProgressBar

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        etEmail = findViewById(R.id.etEmail)
        etPassword = findViewById(R.id.etPassword)
        btnLogin = findViewById(R.id.btnLogin)
        progressBar = findViewById(R.id.progressBar)

        btnLogin.setOnClickListener {
            login()
        }
    }

    private fun login() {
        val email = etEmail.text.toString().trim()
        val password = etPassword.text.toString().trim()

        if (email.isEmpty() || password.isEmpty()) {
            showMessage("Email dan password wajib diisi")
            return
        }

        lifecycleScope.launch {
            showLoading(true)

            try {
                val request = LoginRequest(email, password)
                val response = RetrofitClient.apiService.login(request)

                if (response.isSuccessful) {
                    val loginData = response.body()?.data
                    val userName = loginData?.user?.name.orEmpty()

                    if (userName.isNotEmpty()) {
                        val intent = Intent(this@MainActivity, HomeActivity::class.java)
                        intent.putExtra(HomeActivity.EXTRA_NAME, userName)
                        startActivity(intent)
                    } else {
                        showMessage("Data user tidak ditemukan")
                    }
                } else {
                    showMessage("Email atau password salah")
                }
            } catch (e: Exception) {
                showMessage("Error: ${e.message}")
            } finally {
                showLoading(false)
            }
        }
    }

    private fun showLoading(isLoading: Boolean) {
        progressBar.visibility = if (isLoading) View.VISIBLE else View.GONE
        btnLogin.isEnabled = !isLoading
    }

    private fun showMessage(message: String) {
        Toast.makeText(this, message, Toast.LENGTH_SHORT).show()
    }
}

Aplikasi Login dan Data Pasien

Buat aplikasi Android Kotlin yang memiliki fitur login menggunakan API, lalu setelah login berhasil aplikasi menampilkan daftar data pasien dari endpoint https://api.pahrul.my.id/api/pasien menggunakan RecyclerView.

Tujuan Tugas

  • Menerapkan login ke API menggunakan Retrofit
  • Menyimpan token dari response login untuk mengakses endpoint yang membutuhkan autentikasi
  • Mengirim header Authorization: Bearer {token} saat mengambil data pasien
  • Menampilkan data pasien ke RecyclerView dengan tampilan yang rapi
  • Mengunggah source code dan screenshot hasil aplikasi ke GitHub

Fitur Method Endpoint Keterangan

Alur Aplikasi

  1. User membuka halaman login.

    Halaman login minimal memiliki input email, input password, tombol login, dan indikator loading.

  2. Aplikasi mengirim request login.

    Gunakan endpoint POST /api/login dengan body berisi email dan password.

  3. Jika login berhasil, simpan token.

    Token berada pada data.token. Token ini digunakan untuk mengambil data pasien.

  4. Tampilkan nama user setelah login.

    Nama user berasal dari data.user.name. Nama boleh ditampilkan di bagian atas halaman data pasien.

  5. Ambil data pasien dari API.

    Panggil endpoint GET /api/pasien dengan header Authorization: Bearer {token}.

  6. Tampilkan data pasien menggunakan RecyclerView.

    Setiap item minimal menampilkan nama pasien, tanggal lahir, jenis kelamin, alamat, dan nomor telepon.
