Praktikum: Konsumsi REST API di Android

Tujuan Pembelajaran

  • Memahami konsep REST API dan cara kerja komunikasi client-server
  • Mampu mengintegrasikan Retrofit untuk melakukan HTTP request di Android
  • Mampu melakukan operasi CRUD (GET, POST, PUT, DELETE) via API
  • Mampu menampilkan data dari API ke RecyclerView

Apa itu Retrofit?

Retrofit adalah library buatan Square yang menjadi standar de-facto untuk melakukan HTTP request di Android. Retrofit mengubah API endpoint menjadi interface Kotlin yang mudah dipanggil, menangani serialisasi JSON secara otomatis, dan mendukung Coroutines untuk operasi async.

Retrofit
HTTP Client
Mengubah API endpoint menjadi fungsi Kotlin yang bisa langsung dipanggil
Gson
JSON Parser
Mengkonversi JSON dari server menjadi data class Kotlin secara otomatis
Coroutines
Async
Menjalankan request di background thread tanpa memblokir UI
OkHttp
HTTP Engine
Engine HTTP di balik Retrofit, menangani koneksi, timeout, dan interceptor

API yang Digunakan di Praktikum

Praktikum ini menggunakan API Vehicle yang tersedia di api.pahrul.my.id. API ini menyediakan data kendaraan (model, tipe, produsen) dan mendukung operasi CRUD lengkap. Semua endpoint bersifat publik — tidak memerlukan autentikasi.

Endpoint Method Fungsi Response
api.pahrul.my.id/api/vehicles GET Ambil semua data vehicle 200 — array data
api.pahrul.my.id/api/vehicles/{id} GET Ambil satu vehicle berdasarkan ID 200 / 404
api.pahrul.my.id/api/vehicles POST Tambah vehicle baru 201 / 422
api.pahrul.my.id/api/vehicles/{id} PUT Update data vehicle 200 / 404
api.pahrul.my.id/api/vehicles/{id} DELETE Hapus vehicle 200 / 404

Struktur Data Vehicle

Setiap response dari API membungkus data dalam format standar dengan field success, message, dan data.

Contoh response GET /api/vehicles 
{
  "success": true,
  "message": "Data vehicle ditemukan",
  "data": [
    {
      "id": 1,
      "model": "Civic",
      "type": "Sedan",
      "manufacturer": "Honda",
      "created_at": "2026-04-30T10:00:00.000000Z",
      "updated_at": "2026-04-30T10:00:00.000000Z"
    }
  ]
}
Contoh request body POST /api/vehicles 
{
  "model": "Civic",
  "type": "Sedan",
  "manufacturer": "Honda"
}

Field yang Diperlukan saat POST / PUT

  • model — nama model kendaraan, wajib diisi
  • type — tipe kendaraan (Sedan, SUV, MPV, dll), wajib diisi
  • manufacturer — nama produsen (Honda, Toyota, dll), wajib diisi

Jika salah satu field kosong, API akan mengembalikan response 422 Unprocessable Entity beserta pesan validasi.

Konsep REST API

Apa itu REST API?

REST (Representational State Transfer) adalah gaya arsitektur untuk membangun layanan web. API (Application Programming Interface) adalah antarmuka yang memungkinkan dua aplikasi berkomunikasi. Jadi REST API adalah cara aplikasi berkomunikasi melalui HTTP dengan aturan-aturan tertentu.

Dalam konteks Android, aplikasi kita berperan sebagai client yang mengirim request ke server, lalu server membalas dengan data dalam format JSON.

Alur komunikasi Client → Server 
Android App (Client)
        │
        │  HTTP Request
        │  GET https://api.pahrul.my.id/api/vehicles
        │  Headers: Content-Type: application/json
        ▼
    Server (API)
        │
        │  HTTP Response
        │  Status: 200 OK
        │  Body: { "success": true, "data": [...] }
        ▼
Android App (Client)
        │
        └─ Parse JSON → tampilkan ke RecyclerView

HTTP Method, Status Code & JSON

HTTP Method

Setiap request ke API menggunakan salah satu HTTP method berikut. Method menentukan apa yang ingin dilakukan terhadap data di server.

Method Fungsi Contoh Ada Request Body?
GET Ambil data Tampilkan daftar vehicle Tidak
POST Kirim / buat data baru Tambah vehicle baru Ya
PUT Update seluruh data Edit data vehicle Ya
DELETE Hapus data Hapus vehicle Tidak

HTTP Status Code

Setiap response dari server menyertakan status code — angka 3 digit yang menunjukkan hasil dari request.

Kode Arti Kapan Muncul
200 OK Berhasil GET, PUT, DELETE sukses
201 Created Data berhasil dibuat POST sukses
404 Not Found Data tidak ditemukan ID tidak ada di database
422 Unprocessable Validasi gagal Field wajib tidak diisi
401 Unauthorized Tidak punya akses Token tidak valid / tidak ada
500 Server Error Error di server Bug di sisi server

Format JSON

JSON (JavaScript Object Notation) adalah format pertukaran data yang ringan dan mudah dibaca. Semua response dari API Vehicle menggunakan format JSON.

Anatomi JSON response dari API Vehicle 
{
  "success": true,          // Boolean — apakah request berhasil
  "message": "Data vehicle ditemukan",  // String — pesan dari server
  "data": {                 // Object atau Array — isi data utama
    "id": 1,                // Number
    "model": "Civic",       // String
    "type": "Sedan",        // String
    "manufacturer": "Honda",// String
    "created_at": "2026-04-30T10:00:00.000000Z"
  }
}

Wrapper Response

API Vehicle menggunakan pola wrapper response — data selalu dibungkus dalam objek dengan field success, message, dan data. Pola ini umum dipakai agar client bisa dengan mudah mengecek apakah request berhasil sebelum memproses datanya.

Retrofit & CRUD API

Apa itu Retrofit?

Retrofit adalah library HTTP client untuk Android yang memudahkan aplikasi mengakses REST API. Dengan Retrofit, endpoint API ditulis sebagai interface Kotlin, lalu Retrofit akan mengurus proses request, response, dan konversi JSON ke data class.

Pada praktikum ini, aplikasi Android akan mengambil dan mengelola data kendaraan dari API Vehicle. Operasi yang dibuat meliputi ambil data, tambah data, ubah data, dan hapus data.

📌 Dua Cara Menambahkan Dependency

Android Studio versi terbaru (Ladybug 2024+) menggunakan Version Catalog (TOML) untuk mengelola dependency. Project lama mungkin masih menggunakan cara langsung di KTS.

Cek dulu file build.gradle.kts (Module: app) Anda:

  • Jika ada libs.androidx.core.ktx → gunakan Cara A (TOML)
  • Jika ada "androidx.core:core-ktx:1.x.x" → gunakan Cara B (Langsung KTS)

Cara A: Menggunakan Version Catalog (TOML) — Untuk Project Baru

Jika project Anda menggunakan TOML, dependency dikelola di dua file terpisah.

Langkah A.1: Tambahkan versi library di gradle/libs.versions.toml

gradle/libs.versions.toml — tambahkan di bagian [versions] 
[versions]
agp = "8.7.3"
kotlin = "2.0.21"
coreKtx = "1.13.1"
# ... versi lain yang sudah ada ...

# Tambahkan versi library baru di bawah ini
recyclerview = "1.3.2"
lifecycleRuntimeKtx = "2.8.4"
retrofit = "2.11.0"
okhttp = "4.12.0"

Langkah A.2: Tambahkan library di gradle/libs.versions.toml

gradle/libs.versions.toml — tambahkan di bagian [libraries] 
[libraries]
androidx-core-ktx = { group = "androidx.core", name = "core-ktx", version.ref = "coreKtx" }
# ... library lain yang sudah ada ...

# Tambahkan library baru di bawah ini

Langkah A.3: Gunakan library di build.gradle.kts (Module: app)

build.gradle.kts (Module: app) — tambahkan di blok dependencies 
dependencies {

    implementation(libs.androidx.core.ktx)
    implementation(libs.androidx.appcompat)

Keuntungan Menggunakan TOML

  • Versi library dikelola di satu tempat (libs.versions.toml)
  • Mudah update versi tanpa ubah banyak file
  • Autocomplete lebih baik di Android Studio
  • Menghindari konflik versi antar module

Cara B: Langsung di KTS — Untuk Project Lama

Jika project Anda tidak menggunakan TOML, tambahkan dependency langsung di file build.gradle.kts (Module: app).

💡 Penjelasan Format Dependency

Dalam satu file build.gradle.kts, Anda bisa mencampur dua format:

  • implementation(libs.nama.library) → Menggunakan Version Catalog (TOML)
  • implementation("group:artifact:version") → Menulis versi langsung

Dependency bawaan Android Studio biasanya sudah menggunakan TOML, jadi biarkan saja seperti itu. Anda hanya perlu menambahkan dependency baru (Retrofit, RecyclerView, dll) dengan format versi langsung.

Sync Gradle

Setelah menambahkan dependency (baik dengan Cara A atau B), klik Sync Now yang muncul di bagian atas editor. Tunggu hingga proses sync selesai.

Troubleshooting Gradle Sync

  • Jika sync gagal: Pastikan koneksi internet stabil
  • Jika muncul error "Unresolved reference": Pastikan nama library di TOML sesuai dengan yang dipanggil di KTS (ganti titik dengan garis bawah, contoh: lifecycle.runtime.ktx → lifecycle-runtime-ktx)
  • Jika muncul error versi: Pastikan dependency ditulis di file build.gradle.kts (Module: app), bukan build.gradle.kts (Project: ...)
  • Jika masih error: Coba File → Invalidate Caches → Invalidate and Restart

Langkah 3: Tambahkan Izin Internet

Aplikasi Android harus meminta izin untuk mengakses internet. Tanpa izin ini, aplikasi tidak bisa memanggil API.

Cara menambahkan:

  1. Buka file AndroidManifest.xml di folder app/src/main
  2. Tambahkan baris <uses-permission ... /> sebelum tag <application>
AndroidManifest.xml — tambahkan izin internet 
<manifest xmlns:android="http://schemas.android.com/apk/res/android">

    <!-- Tambahkan baris ini -->
    <uses-permission android:name="android.permission.INTERNET" />

    <application
        android:allowBackup="true"
        android:theme="@style/Theme.VehicleApiApp"
        android:label="@string/app_name"
        android:supportsRtl="true">

        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>

Kenapa Perlu Izin Internet?

Android menerapkan sistem permission untuk melindungi privasi user. Tanpa deklarasi INTERNET di manifest, aplikasi tidak bisa melakukan koneksi jaringan dan akan error saat memanggil API.

Membuat Data Model

Model digunakan untuk memetakan response JSON dari API menjadi objek Kotlin. Karena API menggunakan format wrapper, kita perlu membuat model untuk data utama dan model untuk pembungkus response.

1. Buat File Vehicle.kt — Data Class

Buat file Vehicle.kt di package model. File ini berisi data class untuk merepresentasikan satu data kendaraan. Field pada data class harus mengikuti nama field pada JSON.

Cara Membuat File Kotlin

  1. Klik kanan pada package model
  2. Pilih New → Kotlin Class/File
  3. Ketik nama: Vehicle
  4. Pilih jenis: Class
  5. Tekan Enter, lalu isi dengan kode berikut
model/Vehicle.kt — Data Class 
package com.example.vehicleapiapp.model

data class Vehicle(
    val id: Int? = null,
    val model: String,
    val type: String,
    val manufacturer: String,
    val created_at: String? = null,
    val updated_at: String? = null
)

2. Buat File ApiResponse.kt — Generic Data Class

API mengembalikan response dengan bentuk success, message, dan data. Karena isi data bisa berupa satu object atau list, buat generic data class yang bisa menerima tipe data apapun.

model/ApiResponse.kt — Generic Data Class 
package com.example.vehicleapiapp.model

data class ApiResponse<T>(
    val success: Boolean,
    val message: String,
    val data: T?
)

❓ Apa Maksud <T> di ApiResponse?

<T> adalah Generic Type — placeholder untuk tipe data yang bisa diganti saat digunakan. Huruf T singkatan dari "Type", tapi bisa diganti huruf apa saja (A, B, Data, dll).

Analogi sederhana: Bayangkan <T> seperti kotak kosong yang bisa diisi berbagai jenis barang.

Saat DigunakanT Diganti DenganHasil
ApiResponse<Vehicle> Vehicle data bertipe Vehicle?
ApiResponse<List<Vehicle>> List<Vehicle> data bertipe List<Vehicle>?
ApiResponse<String> String data bertipe String?

Contoh Penggunaan di ApiService:

// GET semua vehicle → data berisi List
suspend fun getVehicles(): Response<ApiResponse<List<Vehicle>>>

// POST vehicle baru → data berisi satu Vehicle
suspend fun createVehicle(@Body vehicle: Vehicle): Response<ApiResponse<Vehicle>>

// DELETE vehicle → data berisi satu Vehicle
suspend fun deleteVehicle(@Path("id") id: Int): Response<ApiResponse<Vehicle>>

Kenapa Pakai Generic?

  • ✅ Reusable — Satu class bisa dipakai untuk berbagai tipe response
  • ✅ Type-safe — Compiler tahu tipe data yang diharapkan, mengurangi error
  • ✅ Konsisten — Semua response punya struktur yang sama (success, message, data)
  • ❌ Tanpa generic, harus buat VehicleResponse, VehicleListResponse, UserResponse, dll (repetitif!)

Penjelasan Jenis File

  • Data Class — Class khusus untuk menyimpan data, otomatis punya fungsi toString(), equals(), copy()
  • Generic <T> — Tipe data yang fleksibel, bisa diisi dengan tipe apapun (Vehicle, List<Vehicle>, dll)
  • Nullable ? — Field yang boleh kosong/null

Kenapa id Bersifat Nullable?

Saat menambah data baru, aplikasi hanya mengirim model, type, dan manufacturer. Field id dibuat oleh server, sehingga pada request POST nilainya boleh kosong.

GET Data dan Tampilkan ke RecyclerView

Pada tahap ini, aplikasi akan mengambil semua data vehicle dari API, lalu menampilkannya dalam RecyclerView.

1. Buat ApiService.kt — Interface

ApiService adalah interface yang berisi daftar endpoint API. Setiap fungsi diberi anotasi sesuai HTTP method. Retrofit akan mengimplementasikan interface ini secara otomatis.

network/ApiService.kt — Interface 
package com.example.vehicleapiapp.network

import com.example.vehicleapiapp.model.ApiResponse
import com.example.vehicleapiapp.model.Vehicle
import retrofit2.Response
import retrofit2.http.Body
import retrofit2.http.DELETE
import retrofit2.http.GET
import retrofit2.http.POST
import retrofit2.http.PUT
import retrofit2.http.Path

interface ApiService {
    @GET("vehicles")
    suspend fun getVehicles(): Response<ApiResponse<List<Vehicle>>>

    @POST("vehicles")
    suspend fun createVehicle(
        @Body vehicle: Vehicle
    ): Response<ApiResponse<Vehicle>>

    @PUT("vehicles/{id}")
    suspend fun updateVehicle(
        @Path("id") id: Int,
        @Body vehicle: Vehicle
    ): Response<ApiResponse<Vehicle>>

    @DELETE("vehicles/{id}")
    suspend fun deleteVehicle(
        @Path("id") id: Int
    ): Response<ApiResponse<Vehicle>>
}

Penjelasan Anotasi Retrofit

  • @GET, @POST, @PUT, @DELETE — HTTP method yang digunakan
  • @Body — Data yang dikirim dalam request body (untuk POST/PUT)
  • @Path — Parameter yang dimasukkan ke URL (contoh: /vehicles/{id})
  • suspend — Fungsi yang bisa dipanggil di coroutine (async)

2. Buat RetrofitClient.kt — Singleton Object

RetrofitClient adalah singleton object yang menyimpan konfigurasi Retrofit. Object ini hanya dibuat sekali dan bisa diakses dari mana saja.

network/RetrofitClient.kt — Singleton Object 
package com.example.vehicleapiapp.network

import okhttp3.OkHttpClient
import okhttp3.logging.HttpLoggingInterceptor
import retrofit2.Retrofit
import retrofit2.converter.gson.GsonConverterFactory

object RetrofitClient {
    private const val BASE_URL = "https://api.pahrul.my.id/api/"

    private val loggingInterceptor = HttpLoggingInterceptor().apply {
        level = HttpLoggingInterceptor.Level.BODY
    }

    private val okHttpClient = OkHttpClient.Builder()
        .addInterceptor(loggingInterceptor)
        .build()

    val apiService: ApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .client(okHttpClient)
            .addConverterFactory(GsonConverterFactory.create())
            .build()
            .create(ApiService::class.java)
    }
}

Penjelasan Komponen RetrofitClient

  • object — Singleton pattern, hanya ada satu instance di seluruh aplikasi
  • HttpLoggingInterceptor — Menampilkan log request/response di Logcat untuk debugging
  • GsonConverterFactory — Mengkonversi JSON ↔ Kotlin object secara otomatis
  • by lazy — Inisialisasi hanya dilakukan saat pertama kali diakses

Base URL Harus Diakhiri Slash

Retrofit mewajibkan baseUrl diakhiri dengan tanda /. Jika ditulis https://api.pahrul.my.id/api tanpa slash akhir, aplikasi akan error saat dijalankan.

import android.view.LayoutInflater

import android.widget.TextView

import com.example.vehicleapiapp.R

class VehicleAdapter(
    private val onEditClick: (Vehicle) -> Unit,
    private val onDeleteClick: (Vehicle) -> Unit
) : RecyclerView.Adapter<VehicleAdapter.VehicleViewHolder>() {

    private val vehicles = mutableListOf<Vehicle>()

    fun setData(newVehicles: List<Vehicle>) {
        vehicles.clear()
        vehicles.addAll(newVehicles)
        notifyDataSetChanged()
    }

    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): VehicleViewHolder {
        val view = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_vehicle, parent, false)
        return VehicleViewHolder(view)
    }

    override fun onBindViewHolder(holder: VehicleViewHolder, position: Int) {
        holder.bind(vehicles[position])
    }

    override fun getItemCount(): Int = vehicles.size

    inner class VehicleViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView) {
        private val tvModel: TextView = itemView.findViewById(R.id.tvModel)
        private val tvDetail: TextView = itemView.findViewById(R.id.tvDetail)
        private val btnEdit: Button = itemView.findViewById(R.id.btnEdit)
        private val btnDelete: Button = itemView.findViewById(R.id.btnDelete)

        fun bind(vehicle: Vehicle) {
            tvModel.text = vehicle.model
            tvDetail.text = "${vehicle.type} - ${vehicle.manufacturer}"

            btnEdit.setOnClickListener {
                onEditClick(vehicle)
            }

            btnDelete.setOnClickListener {
                onDeleteClick(vehicle)
            }
        }
    }
}

Token Authentication

Pada aplikasi yang memakai API, login tidak hanya mencocokkan email dan password. Jika login berhasil, server akan mengembalikan token. Token ini menjadi bukti bahwa user sudah terautentikasi.

Server yang digunakan pada praktikum ini memakai Laravel Sanctum. Endpoint login tersedia di https://api.pahrul.my.id/api/login. Berdasarkan dokumentasi, response sukses berisi token dan data user.

import com.example.loginapiapp.model.LoginRequest

network/RetrofitClient.kt — Singleton Object 
package com.example.loginapiapp.network

import okhttp3.OkHttpClient
import okhttp3.logging.HttpLoggingInterceptor
import retrofit2.Retrofit
import retrofit2.converter.gson.GsonConverterFactory

object RetrofitClient {
    private const val BASE_URL = "https://api.pahrul.my.id/api/"

    private val loggingInterceptor = HttpLoggingInterceptor().apply {
        level = HttpLoggingInterceptor.Level.BODY
    }

    private val okHttpClient = OkHttpClient.Builder()
        .addInterceptor(loggingInterceptor)
        .build()

    val apiService: ApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .client(okHttpClient)
            .addConverterFactory(GsonConverterFactory.create())
            .build()
            .create(ApiService::class.java)
    }
}

7. Buat MainActivity untuk Proses Login — Activity Class

MainActivity adalah Activity class yang membaca input, mengirim request login, lalu membuka HomeActivity jika login berhasil.

MainActivity.kt — Activity Class 
package com.example.loginapiapp

import android.content.Intent
import android.os.Bundle
import android.view.View
import android.widget.Button
import android.widget.EditText
import android.widget.ProgressBar
import android.widget.Toast
import androidx.appcompat.app.AppCompatActivity
import androidx.lifecycle.lifecycleScope
import com.example.loginapiapp.model.LoginRequest
import com.example.loginapiapp.network.RetrofitClient
import kotlinx.coroutines.launch

class MainActivity : AppCompatActivity() {
    private lateinit var etEmail: EditText
    private lateinit var etPassword: EditText
    private lateinit var btnLogin: Button
    private lateinit var progressBar: ProgressBar

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        etEmail = findViewById(R.id.etEmail)
        etPassword = findViewById(R.id.etPassword)
        btnLogin = findViewById(R.id.btnLogin)
        progressBar = findViewById(R.id.progressBar)

        btnLogin.setOnClickListener {
            login()
        }
    }

    private fun login() {
        val email = etEmail.text.toString().trim()
        val password = etPassword.text.toString().trim()

        if (email.isEmpty() || password.isEmpty()) {
            showMessage("Email dan password wajib diisi")
            return
        }

        lifecycleScope.launch {
            showLoading(true)

            try {
                val request = LoginRequest(email, password)
                val response = RetrofitClient.apiService.login(request)

                if (response.isSuccessful) {
                    val loginData = response.body()?.data
                    val userName = loginData?.user?.name.orEmpty()

                    if (userName.isNotEmpty()) {
                        val intent = Intent(this@MainActivity, HomeActivity::class.java)
                        intent.putExtra(HomeActivity.EXTRA_NAME, userName)
                        startActivity(intent)
                    } else {
                        showMessage("Data user tidak ditemukan")
                    }
                } else {
                    showMessage("Email atau password salah")
                }
            } catch (e: Exception) {
                showMessage("Error: ${e.message}")
            } finally {
                showLoading(false)
            }
        }
    }

    private fun showLoading(isLoading: Boolean) {
        progressBar.visibility = if (isLoading) View.VISIBLE else View.GONE
        btnLogin.isEnabled = !isLoading
    }

    private fun showMessage(message: String) {
        Toast.makeText(this, message, Toast.LENGTH_SHORT).show()
    }
}

Catatan Keamanan

Pada contoh ini token belum disimpan karena target praktikum hanya menampilkan nama user setelah login. Jika aplikasi perlu mengakses endpoint yang terkunci seperti /api/users, simpan token dengan mekanisme yang lebih aman, lalu kirim sebagai header Authorization: Bearer token.

Aplikasi Login dan Data Pasien

Buat aplikasi Android Kotlin yang memiliki fitur login menggunakan API, lalu setelah login berhasil aplikasi menampilkan daftar data pasien dari endpoint https://api.pahrul.my.id/api/pasien menggunakan RecyclerView.

Tujuan Tugas

  • Menerapkan login ke API menggunakan Retrofit
  • Menyimpan token dari response login untuk mengakses endpoint yang membutuhkan autentikasi
  • Mengirim header Authorization: Bearer {token} saat mengambil data pasien
  • Menampilkan data pasien ke RecyclerView dengan tampilan yang rapi
  • Mengunggah source code dan screenshot hasil aplikasi ke GitHub

Endpoint yang Digunakan

Fitur Method Endpoint Keterangan
Login POST https://api.pahrul.my.id/api/login Mengirim email dan password, lalu menerima token dan data user
Data Pasien GET https://api.pahrul.my.id/api/pasien Mengambil daftar pasien, wajib memakai Bearer token

Alur Aplikasi

  1. User membuka halaman login.

    Halaman login minimal memiliki input email, input password, tombol login, dan indikator loading.

  2. Aplikasi mengirim request login.

    Gunakan endpoint POST /api/login dengan body berisi email dan password.

  3. Jika login berhasil, simpan token.

    Token berada pada data.token. Token ini digunakan untuk mengambil data pasien.

  4. Tampilkan nama user setelah login.

    Nama user berasal dari data.user.name. Nama boleh ditampilkan di bagian atas halaman data pasien.

  5. Ambil data pasien dari API.

    Panggil endpoint GET /api/pasien dengan header Authorization: Bearer {token}.

  6. Tampilkan data pasien menggunakan RecyclerView.

    Setiap item minimal menampilkan nama pasien, tanggal lahir, jenis kelamin, alamat, dan nomor telepon.
