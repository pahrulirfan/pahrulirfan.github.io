Tugas Kelas: Layout Dasar & Intent

Tujuan Pembelajaran

  • Mampu menyusun tampilan menggunakan Layout XML dasar
  • Mampu menggunakan Basic Widgets (EditText, Spinner, RadioButton)
  • Mampu mengirim data antar Activity menggunakan Explicit Intent
  • Mampu membuka halaman web menggunakan Implicit Intent

Ketentuan Pengerjaan

  • Tugas dikerjakan secara individu selama sesi kelas berlangsung
  • Kumpulkan file ZIP melalui Berajah sebelum kelas berakhir
  • ZIP berisi: folder project Android + screenshot hasil aplikasi
  • Cara export project: di Android Studio → File → Export → Export Project to Zip
  • Screenshot minimal: tampilan form input dan tampilan halaman hasil
  • Soal ditentukan berdasarkan digit terakhir NIM:
    • NIM digit terakhir ganjil (1,3,5,7,9) → Tugas A
    • NIM digit terakhir genap (0,2,4,6,8) → Tugas B

Soal Tugas Kelas

Tugas A — NIM digit terakhir Ganjil

Studi Kasus: Registrasi Kegiatan Mahasiswa

Buat aplikasi Android sederhana untuk mendaftarkan diri ke sebuah kegiatan kampus.

Input yang diperlukan (4 input):

  1. Nama Lengkap — EditText
  2. NIM — EditText (inputType: number)
  3. Nama Kegiatan — Spinner (isi minimal 3 pilihan kegiatan, contoh: Seminar, Workshop, Lomba)
  4. Ketersediaan Hadir — RadioButton (Hadir / Tidak Hadir)

Ketentuan:

  1. Buat RegistrasiActivity berisi form 4 input di atas
  2. Buat KonfirmasiActivity yang menampilkan semua data yang diinputkan
  3. Gunakan Explicit Intent dengan putExtra() untuk mengirim data ke KonfirmasiActivity
  4. Tampilkan nama Anda di salah satu halaman aplikasi (boleh sebagai TextView statis)
  5. Di KonfirmasiActivity, tambahkan tombol "Info Kegiatan" yang membuka URL halaman web menggunakan Implicit Intent (ACTION_VIEW)

Fokus Penilaian

  • Data dari 4 input berhasil dikirim dan tampil di KonfirmasiActivity
  • Nama mahasiswa tampil di aplikasi
  • Implicit Intent membuka browser tanpa crash

Tugas B — NIM digit terakhir Genap

Studi Kasus: Pemesanan Tiket Wisata

Buat aplikasi Android sederhana untuk memesan tiket kunjungan wisata.

Input yang diperlukan (4 input):

  1. Nama Pemesan — EditText
  2. Jumlah Tiket — EditText (inputType: number)
  3. Destinasi — Spinner (isi minimal 3 pilihan destinasi)
  4. Kategori Pengunjung — RadioButton (Dewasa / Anak-anak)

Ketentuan:

  1. Buat PemesananActivity berisi form 4 input di atas
  2. Buat DetailPesananActivity yang menampilkan semua data yang diinputkan
  3. Gunakan Explicit Intent dengan putExtra() untuk mengirim data ke DetailPesananActivity
  4. Tampilkan nama Anda di salah satu halaman aplikasi (boleh sebagai TextView statis)
  5. Di DetailPesananActivity, tambahkan tombol "Lihat Lokasi" yang membuka URL halaman web menggunakan Implicit Intent (ACTION_VIEW)

Fokus Penilaian

  • Data dari 4 input berhasil dikirim dan tampil di DetailPesananActivity
  • Nama mahasiswa tampil di aplikasi
  • Implicit Intent membuka browser tanpa crash

Catatan Penting

  • Plagiasi akan mendapat nilai 0
  • Pahami setiap baris kode yang Anda tulis
  • Jangan ragu bertanya jika ada kesulitan

Praktikum Pemrograman Bergerak

Silakan ketik ulang kode untuk pemahaman yang lebih baik
