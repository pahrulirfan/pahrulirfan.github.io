Praktikum: Firebase untuk Android Kotlin

Tujuan Pembelajaran

  • Memahami ekosistem Firebase dan layanan-layanannya
  • Mampu menghubungkan project Android ke Firebase Console
  • Mampu menyiapkan project Android agar terhubung ke Firebase
  • Mampu melakukan operasi tulis, baca, update, dan hapus data menggunakan Realtime Database
  • Memahami perbedaan Firestore dan Realtime Database
  • Mampu mengupload dan mengambil file menggunakan Firebase Storage

Prasyarat

Pastikan Anda sudah memahami materi dari praktikum sebelumnya:

  • Activity, Intent, dan dasar-dasar UI Android
  • RecyclerView dan Adapter
  • Kotlin dasar (data class, lambda, coroutines)
  • Memiliki akun Google untuk mengakses Firebase Console

Apa itu Firebase?

Firebase adalah platform Backend-as-a-Service (BaaS) milik Google yang menyediakan berbagai layanan backend siap pakai untuk aplikasi mobile dan web. Dengan Firebase, developer tidak perlu membangun server sendiri untuk fitur-fitur umum seperti autentikasi, database, dan penyimpanan file.

Authentication
Login / Register
Email, Google, Facebook, Phone
Cloud Firestore
NoSQL Database
Database dokumen yang scalable & realtime
Realtime Database
JSON Tree
Sinkronisasi data real-time antar device
Firebase Storage
File Storage
Upload & download gambar, video, file

Yang akan dibahas di praktikum ini

Praktikum ini fokus pada dua layanan Firebase untuk penyimpanan data dan file: Realtime Database dan Firebase Storage. Firestore tetap disebut sebagai pembanding karena sering digunakan bersama Storage untuk menyimpan URL file.

Perbedaan Firestore, Realtime Database, dan Storage

Tiga layanan ini sering membingungkan karena sama-sama menyimpan data di cloud. Perbedaan utamanya ada pada jenis data yang disimpan dan cara mengaksesnya.

Contoh isi { name: "Budi", age: 21 } { text: "Halo!", time: 1700000 } foto_profil.jpg, laporan.pdf

Offline support Otomatis (cache lokal) Otomatis (cache lokal) Upload dilanjutkan otomatis jika koneksi putus

Kombinasi yang Umum Dipakai

Dalam satu aplikasi, ketiga layanan ini biasanya dipakai bersama-sama, bukan pilih salah satu:

  • Firestore — menyimpan data profil user, termasuk URL foto profil
  • Storage — menyimpan file foto profil yang sebenarnya
  • Realtime Database — menyimpan status online/offline user secara real-time

Contoh: saat user upload foto profil → file dikirim ke Storage → URL-nya disimpan ke Firestore → status "sedang upload" ditampilkan via Realtime Database.

Setup Firebase

Setup dilakukan sekali di awal: membuat project Android, membuat project Firebase, mendaftarkan app Android, memasang plugin Google Services, menambahkan SDK Firebase, lalu menguji koneksi ke Realtime Database dan Storage.

1. Buat Project Android Baru

Buat project Android di Android Studio dengan Kotlin. Package name yang digunakan di Android Studio harus sama persis dengan package name yang didaftarkan ke Firebase Console.

1

Buka Android Studio → New Project

Pilih template Empty Views Activity, lalu klik Next.

2

Isi Konfigurasi Project

Name PraktikumFirebaseStorage
Package name com.example.praktikumfirebasestorage
Language Kotlin
Min SDK API 24 (Android 7.0)
Build config Kotlin DSL (.gradle.kts)
3

Catat applicationId

Buka build.gradle.kts (Module: app) dan catat nilai applicationId. Nilai ini dipakai saat mendaftarkan aplikasi Android di Firebase.

build.gradle.kts (Module: app) 
android {
    namespace = "com.example.praktikumfirebasestorage"

    defaultConfig {
        applicationId = "com.example.praktikumfirebasestorage"
        minSdk = 24
        targetSdk = 35
    }
}

2. Buat Project di Firebase Console

Buka console.firebase.google.com, login dengan akun Google, lalu buat project baru.

1

Create Project

Klik Add project, beri nama project misalnya PraktikumFirebaseStorage, lalu lanjutkan hingga project selesai dibuat. Google Analytics boleh dimatikan untuk praktikum.

2

Aktifkan Realtime Database

Buka Databases & Storage → Realtime Database → Create database. Pilih lokasi server, lalu pilih Start in test mode untuk praktikum.

3

Aktifkan Storage

Buka Databases & Storage → Storage → Get started. Project baru biasanya harus di-upgrade ke Blaze plan sebelum Cloud Storage for Firebase bisa dibuat.

Catatan Keamanan Test Mode

Test mode membuat database atau bucket mudah dicoba saat praktikum, tetapi aksesnya longgar dan punya masa berlaku terbatas. Untuk aplikasi production, gunakan rules yang membatasi akses berdasarkan user yang login.

3. Hubungkan Project Android ke Firebase

Daftarkan aplikasi Android di Firebase Console, lalu masukkan file konfigurasi ke project Android Studio.

  1. Di halaman Project Overview, klik ikon Android untuk menambahkan app Android.
  2. Masukkan Android package name yang sama dengan applicationId, misalnya com.example.praktikumfirebasestorage.
  3. Klik Register app, lalu download file google-services.json.
  4. Pindahkan file tersebut ke folder app/, sehingga lokasinya menjadi app/google-services.json.

Nama File Harus Tepat

Pastikan nama file tetap google-services.json, bukan google-services (1).json. Jika Firebase Storage baru diaktifkan setelah file ini diunduh, download ulang file konfigurasi agar data bucket ikut masuk.

4. Tambah Dependencies Firebase

Dependency Firebase yang dipakai di materi ini berbeda dari beberapa tutorial lama. Gunakan Firebase Android BoM dan modul utama Firebase. Jangan memakai firebase-database-ktx atau firebase-storage-ktx karena modul KTX sudah tidak dipakai lagi di BoM terbaru.

Project-level build.gradle.kts

Tambahkan plugin Google Services di file build.gradle.kts level project.

build.gradle.kts (Project) 
plugins {
    id("com.android.application") version "8.7.3" apply false
    id("org.jetbrains.kotlin.android") version "2.0.21" apply false
    id("com.google.gms.google-services") version "4.4.4" apply false
}

App-level build.gradle.kts

Aktifkan plugin Google Services dan tambahkan SDK Realtime Database serta Storage.

build.gradle.kts (Module: app) 
plugins {
    id("com.android.application")
    id("org.jetbrains.kotlin.android")
    id("com.google.gms.google-services")
}

dependencies {
    implementation(platform("com.google.firebase:firebase-bom:34.13.0"))
    implementation("com.google.firebase:firebase-database")
    implementation("com.google.firebase:firebase-storage")

    // Opsional: dipakai di contoh upload foto profil berbasis user login
    implementation("com.google.firebase:firebase-auth")

    // Opsional: menampilkan gambar dari URL
    implementation("com.github.bumptech.glide:glide:4.16.0")
}

Yang Sering Bikin Error

  • Menambahkan firebase-database-ktx atau firebase-storage-ktx bersama BoM terbaru.
  • Memberi versi manual pada firebase-database dan firebase-storage saat sudah memakai BoM.
  • Meletakkan google-services.json di root project, bukan di folder app/.
  • Lupa menambahkan plugin com.google.gms.google-services di module app.

5. Uji Coba Koneksi

Setelah Gradle sync berhasil, uji koneksi dengan menulis data sederhana ke Realtime Database. Storage cukup diverifikasi dengan memastikan bucket sudah bisa diakses dari kode.

MainActivity.kt — tes Realtime Database 
import android.os.Bundle
import android.util.Log
import androidx.appcompat.app.AppCompatActivity
import com.google.firebase.Firebase
import com.google.firebase.database.database
import com.google.firebase.storage.storage

class MainActivity : AppCompatActivity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val database = Firebase.database
        val storage = Firebase.storage

        database.getReference("setup_test")
            .setValue("Firebase berhasil terhubung")
            .addOnSuccessListener {
                Log.d("FirebaseTest", "Realtime Database berhasil ditulis")
                Log.d("FirebaseTest", "Storage bucket: ${storage.reference.bucket}")
            }
            .addOnFailureListener { error ->
                Log.e("FirebaseTest", "Setup Firebase gagal", error)
            }
    }
}

Checklist Setup

  • google-services.json berada di folder app/
  • Plugin com.google.gms.google-services aktif di project dan module app
  • Dependency memakai BoM + firebase-database + firebase-storage
  • Gradle sync berhasil tanpa error
  • Node setup_test muncul di Realtime Database setelah aplikasi dijalankan

Firebase Realtime Database

Pengenalan Realtime Database

Firebase Realtime Database adalah database NoSQL berbasis cloud yang menyimpan data dalam format pohon JSON. Keunggulan utamanya adalah sinkronisasi data secara real-time ke semua client yang terhubung dalam hitungan milidetik.

Aspek Realtime Database Cloud Firestore
Struktur data Pohon JSON flat Collection → Document
Cocok untuk Chat, live score, status online Data terstruktur kompleks
Query Terbatas (satu field) Lebih powerful
Harga Berdasarkan bandwidth & storage Berdasarkan jumlah operasi
Latensi Sangat rendah (~50ms) Rendah (~100ms)

Struktur Data JSON

Semua data di Realtime Database adalah pohon JSON. Hindari nesting yang terlalu dalam karena saat Anda mengambil data dari sebuah node, semua child-nya ikut terambil.

Contoh struktur JSON di Realtime Database 
{
  "messages": {
    "-NxAbc123": {
      "text"     : "Halo semua!",
      "sender"   : "Budi",
      "timestamp": 1700000001000
    },
    "-NxDef456": {
      "text"     : "Selamat datang di Firebase!",
      "sender"   : "Ani",
      "timestamp": 1700000002000
    }
  },
  "users": {
    "uid_budi": {
      "name"  : "Budi Santoso",
      "online": true
    },
    "uid_ani": {
      "name"  : "Ani Rahayu",
      "online": false
    }
  }
}

Inisialisasi Realtime Database

Mendapatkan instance DatabaseReference 
import com.google.firebase.Firebase
import com.google.firebase.database.database

// Mendapatkan referensi ke root database
val database = Firebase.database

// Referensi ke node tertentu
val messagesRef = database.getReference("messages")
val usersRef    = database.getReference("users")

// Referensi ke node bersarang
val userRef = database.getReference("users/uid_budi")

Aktifkan Realtime Database di Firebase Console

Build → Realtime Database → Create database → Pilih lokasi server → Start in test mode → Done. Setelah aktif, salin URL database (format: https://nama-project-default-rtdb.firebaseio.com) — URL ini dibutuhkan jika ada lebih dari satu database.

Menulis Data ke Realtime Database

Ada tiga cara menulis data: setValue() untuk menulis/menimpa node, push() untuk menambah data dengan key unik otomatis, dan updateChildren() untuk update beberapa field sekaligus.

setValue() — Tulis atau Timpa Node

setValue() — menulis data ke node 
val database = Firebase.database
val userId   = Firebase.auth.currentUser?.uid ?: return

// Tulis data profil user — jika node sudah ada, akan ditimpa
val userRef = database.getReference("users/$userId")
userRef.setValue(mapOf(
    "name"   to "Budi Santoso",
    "email"  to "budi@example.com",
    "online" to true
)).addOnSuccessListener {
    Log.d("RTDB", "Data berhasil ditulis")
}.addOnFailureListener { e ->
    Log.e("RTDB", "Gagal menulis data: ${e.message}")
}

push() — Tambah Data dengan Key Unik

push() membuat key unik otomatis (mirip add() di Firestore) dan cocok untuk data list seperti pesan chat.

push() — tambah pesan baru ke list 
val database    = Firebase.database
val messagesRef = database.getReference("messages")

// Data class untuk pesan
data class Message(
    val text      : String = "",
    val sender    : String = "",
    val timestamp : Long   = 0L
)

val newMessage = Message(
    text      = "Halo dari Android!",
    sender    = Firebase.auth.currentUser?.displayName ?: "Anonymous",
    timestamp = System.currentTimeMillis()
)

// push() membuat key unik seperti "-NxAbc123"
val newRef = messagesRef.push()
newRef.setValue(newMessage)
    .addOnSuccessListener {
        Log.d("RTDB", "Pesan terkirim, key: ${newRef.key}")
    }
    .addOnFailureListener { e ->
        Log.e("RTDB", "Gagal kirim pesan: ${e.message}")
    }

updateChildren() — Update Beberapa Field

updateChildren() — update tanpa menimpa field lain 
val database = Firebase.database
val userId   = Firebase.auth.currentUser?.uid ?: return

// Hanya update field "online" dan "lastSeen"
// Field lain seperti "name" dan "email" tidak tersentuh
database.getReference("users/$userId")
    .updateChildren(mapOf(
        "online"   to false,
        "lastSeen" to System.currentTimeMillis()
    ))
    .addOnSuccessListener {
        Log.d("RTDB", "Status diperbarui")
    }

removeValue() — Hapus Node

removeValue() — hapus node beserta semua child-nya 
val messageKey = "-NxAbc123"
database.getReference("messages/$messageKey")
    .removeValue()
    .addOnSuccessListener {
        Toast.makeText(this, "Pesan dihapus", Toast.LENGTH_SHORT).show()
    }

Membaca Data & Listener

Sama seperti Firestore, Realtime Database mendukung dua cara baca: sekali baca (get()) dan listener real-time (addValueEventListener()).

get() — Baca Sekali

get() — baca data satu kali 
val database = Firebase.database
val userId   = Firebase.auth.currentUser?.uid ?: return

database.getReference("users/$userId")
    .get()
    .addOnSuccessListener { snapshot ->
        if (snapshot.exists()) {
            val name   = snapshot.child("name").getValue(String::class.java)
            val online = snapshot.child("online").getValue(Boolean::class.java)
            Log.d("RTDB", "Nama: $name, Online: $online")
        } else {
            Log.d("RTDB", "Data tidak ditemukan")
        }
    }
    .addOnFailureListener { e ->
        Log.e("RTDB", "Gagal membaca: ${e.message}")
    }

addValueEventListener() — Listener Real-time

Listener ini dipanggil pertama kali saat data dibaca, lalu dipanggil lagi setiap kali data berubah. Cocok untuk fitur chat atau status online.

addValueEventListener() — update otomatis saat data berubah 
import com.google.firebase.database.DataSnapshot
import com.google.firebase.database.DatabaseError
import com.google.firebase.database.ValueEventListener

val database    = Firebase.database
val messagesRef = database.getReference("messages")

// Simpan referensi listener untuk di-remove nanti
private lateinit var messageListener: ValueEventListener

override fun onStart() {
    super.onStart()

    messageListener = object : ValueEventListener {
        override fun onDataChange(snapshot: DataSnapshot) {
            // Dipanggil setiap kali data berubah
            val messageList = mutableListOf<Message>()

            for (child in snapshot.children) {
                val message = child.getValue(Message::class.java)
                message?.let { messageList.add(it) }
            }

            // Update RecyclerView
            adapter.submitList(messageList)
            Log.d("RTDB", "Total pesan: ${messageList.size}")
        }

        override fun onCancelled(error: DatabaseError) {
            Log.e("RTDB", "Listener dibatalkan: ${error.message}")
        }
    }

    messagesRef.addValueEventListener(messageListener)
}

override fun onStop() {
    super.onStop()
    // Selalu remove listener saat Activity tidak aktif
    messagesRef.removeEventListener(messageListener)
}

addChildEventListener() — Dengarkan Perubahan Per Item

Lebih efisien dari ValueEventListener untuk list besar karena hanya dipanggil saat ada item yang ditambah, diubah, atau dihapus — bukan seluruh list.

addChildEventListener() — efisien untuk list 
import com.google.firebase.database.ChildEventListener

messagesRef.addChildEventListener(object : ChildEventListener {
    override fun onChildAdded(snapshot: DataSnapshot, previousChildName: String?) {
        // Dipanggil saat ada pesan baru ditambahkan
        val message = snapshot.getValue(Message::class.java)
        message?.let { adapter.addItem(it) }
    }

    override fun onChildChanged(snapshot: DataSnapshot, previousChildName: String?) {
        // Dipanggil saat pesan diubah
        val message = snapshot.getValue(Message::class.java)
        message?.let { adapter.updateItem(it) }
    }

    override fun onChildRemoved(snapshot: DataSnapshot) {
        // Dipanggil saat pesan dihapus
        val key = snapshot.key
        key?.let { adapter.removeItem(it) }
    }

    override fun onChildMoved(snapshot: DataSnapshot, previousChildName: String?) { }
    override fun onCancelled(error: DatabaseError) {
        Log.e("RTDB", "Error: ${error.message}")
    }
})

Firebase Storage

Pengenalan Firebase Storage

Firebase Storage adalah layanan penyimpanan file berbasis cloud yang dioptimalkan untuk menyimpan konten yang dibuat pengguna seperti foto profil, gambar postingan, video, dan dokumen. Storage terintegrasi dengan Firebase Authentication sehingga akses file bisa dikontrol per user.

Fitur Keterangan
Tipe file Gambar, video, audio, PDF, dan file lainnya
Ukuran maksimal 5 TB per file (free tier: 5 GB total)
Keamanan Dikontrol via Firebase Storage Rules
Integrasi Bekerja bersama Firebase Auth untuk akses per user
Offline Upload/download dilanjutkan otomatis jika koneksi terputus

Struktur Path di Storage

File di Storage diorganisasi menggunakan path seperti folder, mirip sistem file biasa.

Contoh struktur path di Firebase Storage 
// Struktur path yang direkomendasikan
// images/
//   └── profile_photos/
//         └── {userId}/
//               └── profile.jpg
//   └── posts/
//         └── {userId}/
//               └── {postId}.jpg
//
// Contoh path lengkap:
// "images/profile_photos/uid_budi/profile.jpg"

Inisialisasi Firebase Storage

Mendapatkan instance StorageReference 
import com.google.firebase.Firebase
import com.google.firebase.storage.storage

// Mendapatkan instance Storage
val storage = Firebase.storage

// Referensi ke root storage
val storageRef = storage.reference

// Referensi ke path tertentu
val profileRef = storageRef.child("images/profile_photos/${Firebase.auth.currentUser?.uid}/profile.jpg")

Firebase Storage Memerlukan Upgrade Plan

Per 2024, Firebase Storage tidak lagi tersedia di Spark (free) plan. Untuk mengaktifkannya, project harus di-upgrade ke plan Blaze (Pay as you go).

Apa itu Blaze Plan?

Blaze adalah plan berbayar Firebase, tapi dengan model bayar sesuai pemakaian. Yang perlu dipahami:

  • Wajib mendaftarkan kartu kredit atau debit ke akun Google — ini hanya sebagai metode pembayaran, tidak langsung dicharge
  • Blaze tetap menyertakan free tier yang sama seperti Spark plan — selama pemakaian di bawah batas, tagihan = Rp 0
  • Biaya baru muncul jika penggunaan melebihi batas free tier
  • Untuk skala praktikum (beberapa mahasiswa, file kecil), hampir pasti tidak akan kena biaya
ResourceBatas Gratis (Blaze)Harga jika Melebihi
Storage5 GB$0.026 / GB
Download1 GB / hari$0.12 / GB
Upload operasi20.000 / hari$0.05 / 10K operasi

Alternatif jika tidak ingin mendaftarkan kartu: Gunakan Cloudinary (free tier 25 GB, tanpa kartu kredit) atau ImgBB untuk upload gambar, lalu simpan URL-nya ke Realtime Database.

Tambahkan Permission di AndroidManifest.xml

Untuk memilih gambar dari galeri, tambahkan permission berikut:

AndroidManifest.xml 
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES" />
<!-- Untuk Android 12 ke bawah -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"
    android:maxSdkVersion="32" />

Upload File ke Firebase Storage

Ada dua cara upload: dari Uri (path file di device) menggunakan putFile(), atau dari ByteArray menggunakan putBytes(). Cara paling umum adalah upload dari galeri menggunakan Uri.

Langkah 1 — Pilih Gambar dari Galeri

Membuka galeri dan mendapatkan Uri gambar 
import android.net.Uri
import androidx.activity.result.contract.ActivityResultContracts

class ProfileActivity : AppCompatActivity() {

    private var selectedImageUri: Uri? = null

    // Launcher untuk membuka galeri
    private val pickImageLauncher = registerForActivityResult(
        ActivityResultContracts.GetContent()
    ) { uri: Uri? ->
        if (uri != null) {
            selectedImageUri = uri
            // Tampilkan preview gambar yang dipilih
            findViewById<ImageView>(R.id.ivProfile).setImageURI(uri)
        }
    }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_profile)

        // Tombol untuk membuka galeri
        findViewById<Button>(R.id.btnPickImage).setOnClickListener {
            pickImageLauncher.launch("image/*")
        }

        // Tombol upload
        findViewById<Button>(R.id.btnUpload).setOnClickListener {
            selectedImageUri?.let { uri -> uploadImage(uri) }
                ?: Toast.makeText(this, "Pilih gambar terlebih dahulu", Toast.LENGTH_SHORT).show()
        }
    }
}

Langkah 2 — Upload ke Firebase Storage

uploadImage() — upload dengan progress tracking 
import com.google.firebase.Firebase
import com.google.firebase.auth.auth
import com.google.firebase.database.database
import com.google.firebase.storage.storage

private fun uploadImage(imageUri: Uri) {
    val userId     = Firebase.auth.currentUser?.uid ?: return
    val storage    = Firebase.storage
    val storageRef = storage.reference

    // Buat path unik untuk file — gunakan timestamp agar tidak bentrok
    val fileName  = "profile_${System.currentTimeMillis()}.jpg"
    val imageRef  = storageRef.child("images/profile_photos/$userId/$fileName")

    val progressBar = findViewById<ProgressBar>(R.id.progressBar)
    val btnUpload   = findViewById<Button>(R.id.btnUpload)

    progressBar.visibility = View.VISIBLE
    btnUpload.isEnabled    = false

    // Mulai upload
    val uploadTask = imageRef.putFile(imageUri)

    // Pantau progress upload
    uploadTask.addOnProgressListener { taskSnapshot ->
        val progress = (100.0 * taskSnapshot.bytesTransferred / taskSnapshot.totalByteCount).toInt()
        progressBar.progress = progress
        Log.d("Storage", "Upload progress: $progress%")
    }

    // Selesai upload
    uploadTask.addOnSuccessListener {
        progressBar.visibility = View.GONE
        btnUpload.isEnabled    = true
        Toast.makeText(this, "Gambar berhasil diupload!", Toast.LENGTH_SHORT).show()

        // Ambil download URL setelah upload selesai
        imageRef.downloadUrl.addOnSuccessListener { downloadUri ->
            Log.d("Storage", "Download URL: $downloadUri")
            // Simpan URL ke Realtime Database untuk ditampilkan nanti
            saveImageUrlToDatabase(downloadUri.toString())
        }
    }

    uploadTask.addOnFailureListener { e ->
        progressBar.visibility = View.GONE
        btnUpload.isEnabled    = true
        Toast.makeText(this, "Upload gagal: ${e.message}", Toast.LENGTH_LONG).show()
        Log.e("Storage", "Upload error", e)
    }
}

private fun saveImageUrlToDatabase(downloadUrl: String) {
    val userId = Firebase.auth.currentUser?.uid ?: return

    Firebase.database.getReference("users/$userId/photoUrl")
        .setValue(downloadUrl)
        .addOnSuccessListener {
            Log.d("RTDB", "URL foto berhasil disimpan")
        }
        .addOnFailureListener { e ->
            Log.e("RTDB", "Gagal menyimpan URL foto", e)
        }
}

Langkah 3 — Buat CloudinaryHelper.kt

Buat helper class untuk menangani upload ke Cloudinary. Simpan kredensial di sini — untuk production sebaiknya disimpan di server, tapi untuk praktikum ini cukup di client.

CloudinaryHelper.kt 
import android.content.Context
import android.net.Uri
import okhttp3.*
import okhttp3.MediaType.Companion.toMediaTypeOrNull
import okhttp3.RequestBody.Companion.toRequestBody
import org.json.JSONObject
import java.io.IOException

object CloudinaryHelper {

    // Ganti dengan kredensial Cloudinary Anda
    private const val CLOUD_NAME   = "your_cloud_name"
    private const val UPLOAD_PRESET = "your_upload_preset"  // unsigned preset

    // URL endpoint upload Cloudinary
    private val UPLOAD_URL = "https://api.cloudinary.com/v1_1/$CLOUD_NAME/image/upload"

    /**
     * Upload gambar ke Cloudinary
     * @param context  Context Android
     * @param imageUri Uri gambar yang dipilih dari galeri
     * @param onSuccess Callback dengan URL gambar jika berhasil
     * @param onError   Callback dengan pesan error jika gagal
     */
    fun uploadImage(
        context: Context,
        imageUri: Uri,
        onSuccess: (imageUrl: String) -> Unit,
        onError: (message: String) -> Unit
    ) {
        // Baca bytes dari Uri
        val inputStream = context.contentResolver.openInputStream(imageUri)
            ?: return onError("Tidak bisa membaca file gambar")
        val imageBytes = inputStream.readBytes()
        inputStream.close()

        // Buat multipart request body
        val requestBody = MultipartBody.Builder()
            .setType(MultipartBody.FORM)
            .addFormDataPart(
                "file",
                "upload_${System.currentTimeMillis()}.jpg",
                imageBytes.toRequestBody("image/jpeg".toMediaTypeOrNull())
            )
            .addFormDataPart("upload_preset", UPLOAD_PRESET)
            .build()

        val request = Request.Builder()
            .url(UPLOAD_URL)
            .post(requestBody)
            .build()

        // Jalankan di background thread
        OkHttpClient().newCall(request).enqueue(object : Callback {
            override fun onFailure(call: Call, e: IOException) {
                onError("Upload gagal: ${e.message}")
            }

            override fun onResponse(call: Call, response: Response) {
                val body = response.body?.string()
                if (response.isSuccessful && body != null) {
                    // Parse URL dari response JSON
                    val json = JSONObject(body)
                    val secureUrl = json.getString("secure_url")
                    onSuccess(secureUrl)
                } else {
                    onError("Server error: ${response.code}")
                }
            }
        })
    }
}

Langkah 4 — Implementasi di Activity

ProfileActivity.kt — upload foto profil via Cloudinary 
import android.net.Uri
import android.os.Bundle
import android.view.View
import android.widget.Button
import android.widget.ImageView
import android.widget.ProgressBar
import android.widget.Toast
import androidx.activity.result.contract.ActivityResultContracts
import androidx.appcompat.app.AppCompatActivity
import com.bumptech.glide.Glide
import com.google.firebase.Firebase
import com.google.firebase.auth.auth
import com.google.firebase.database.database

class ProfileActivity : AppCompatActivity() {

    private val auth = Firebase.auth
    private val db   = Firebase.database.reference
    private var selectedImageUri: Uri? = null

    private val pickImageLauncher = registerForActivityResult(
        ActivityResultContracts.GetContent()
    ) { uri: Uri? ->
        if (uri != null) {
            selectedImageUri = uri
            // Tampilkan preview sebelum upload
            findViewById<ImageView>(R.id.ivProfile).setImageURI(uri)
        }
    }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_profile)

        val ivProfile   = findViewById<ImageView>(R.id.ivProfile)
        val btnPick     = findViewById<Button>(R.id.btnPickImage)
        val btnUpload   = findViewById<Button>(R.id.btnUpload)
        val progressBar = findViewById<ProgressBar>(R.id.progressBar)

        // Tampilkan foto profil yang sudah ada (jika ada)
        val userId = auth.currentUser?.uid ?: return
        db.child("users/$userId/photoUrl").get()
            .addOnSuccessListener { snapshot ->
                val photoUrl = snapshot.getValue(String::class.java)
                if (!photoUrl.isNullOrEmpty()) {
                    Glide.with(this).load(photoUrl).circleCrop().into(ivProfile)
                }
            }

        btnPick.setOnClickListener {
            pickImageLauncher.launch("image/*")
        }

        btnUpload.setOnClickListener {
            val uri = selectedImageUri
            if (uri == null) {
                Toast.makeText(this, "Pilih gambar terlebih dahulu", Toast.LENGTH_SHORT).show()
                return@setOnClickListener
            }

            // Tampilkan loading
            progressBar.visibility = View.VISIBLE
            btnUpload.isEnabled    = false

            // Upload ke Cloudinary
            CloudinaryHelper.uploadImage(
                context   = this,
                imageUri  = uri,
                onSuccess = { imageUrl ->
                    // Kembali ke main thread untuk update UI
                    runOnUiThread {
                        progressBar.visibility = View.GONE
                        btnUpload.isEnabled    = true

                        // Simpan URL ke Realtime Database
                        db.child("users/$userId/photoUrl")
                            .setValue(imageUrl)
                            .addOnSuccessListener {
                                Toast.makeText(this, "Foto profil berhasil diperbarui!", Toast.LENGTH_SHORT).show()
                                // Tampilkan gambar baru
                                Glide.with(this).load(imageUrl).circleCrop().into(ivProfile)
                            }
                    }
                },
                onError = { message ->
                    runOnUiThread {
                        progressBar.visibility = View.GONE
                        btnUpload.isEnabled    = true
                        Toast.makeText(this, "Gagal upload: $message", Toast.LENGTH_LONG).show()
                    }
                }
            )
        }
    }
}
