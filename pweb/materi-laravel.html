1. Pengenalan Laravel Dasar

Apa itu Laravel?

Laravel adalah framework PHP open-source yang mengikuti pola arsitektur MVC (Model-View-Controller). Dibuat oleh Taylor Otwell pada tahun 2011, Laravel menjadi framework PHP paling populer karena sintaksnya yang elegan dan fitur-fiturnya yang lengkap.

Mengapa Framework?
  • Tanpa framework = menulis kode dari nol (routing, security, database, dll)
  • Dengan framework = fitur sudah tersedia, tinggal pakai
  • Ibarat membangun rumah: framework = fondasi + struktur rangka sudah jadi

Keunggulan Laravel

Fitur Keterangan
Eloquent ORM Interaksi database tanpa menulis SQL mentah
Blade Template Template engine yang powerful dan mudah
Artisan CLI Command line tool untuk generate kode, migrasi, dll
Migration Version control untuk skema database
Routing Sistem routing yang ekspresif dan fleksibel
Middleware Filter HTTP request (auth, CSRF, dll)
Authentication Sistem auth built-in (login, register, dll)
Tinker REPL interaktif untuk eksperimen kode
Komunitas Dokumentasi lengkap, ekosistem besar

Perbandingan PHP Native vs Laravel

Aspek PHP Native Laravel
Routing Manual (file per halaman) Centralized di routes/web.php
Database Query SQL manual Eloquent ORM / Query Builder
Template PHP campur HTML Blade (layout, component, inheritance)
Security Implementasi sendiri CSRF, XSS, SQL Injection built-in
Struktur Bebas (bisa berantakan) Terstruktur dan terstandar (MVC)

2. Arsitektur MVC Konsep

MVC (Model-View-Controller) adalah pola desain arsitektur yang memisahkan aplikasi menjadi 3 komponen utama. Pemisahan ini membuat kode lebih terorganisir, mudah dirawat, dan mudah dikembangkan secara tim.

Model

Data & Logika Bisnis

app/Models/*.php
  • Berinteraksi dengan database
  • Mendefinisikan struktur data
  • Validasi aturan bisnis
  • Relasi antar tabel (ORM)

View

Tampilan / UI

resources/views/*.blade.php
  • HTML + Blade template
  • Menampilkan data ke user
  • Tidak ada logika bisnis
  • Layout & komponen reusable

Controller

Pengendali / Perantara

app/Http/Controllers/*.php
  • Menerima request dari user
  • Memproses data via Model
  • Mengirim data ke View
  • Mengembalikan response

Tanggung Jawab Tiap Komponen di Laravel

Model — Apa datanya?
File: app/Models/Product.php
Terhubung ke tabel products di DB
Definisi kolom yang boleh diisi ($fillable)
Relasi: hasMany, belongsTo
Query: Product::all(), find()
View — Bagaimana tampilannya?
File: resources/views/products/index.blade.php

Analogi Sederhana: Bayangkan sebuah restoran — Model adalah dapur (menyimpan & mengolah data), View adalah meja makan (tampilan yang dilihat pelanggan), Controller adalah pelayan (menerima pesanan, meneruskan ke dapur, lalu menyajikan ke meja).

3. Jembatan: OOP PHP Penting!

Sebelum masuk Laravel, kamu perlu memahami OOP (Object-Oriented Programming) di PHP. Laravel dibangun sepenuhnya dengan OOP — setiap Controller, Model, dan Middleware adalah sebuah class.

Kenapa OOP penting untuk Laravel?
  • Controller = class PHP biasa yang di-extend dari base Controller
  • Model = class yang merepresentasikan tabel database
  • Semua fitur Laravel (routing, middleware, dll) bekerja lewat class & method
  • Tanpa OOP, kode Laravel akan terasa seperti "mantra" yang tidak dipahami

Konsep 1: Class & Object

📄 contoh-oop.php 
<?php

// Mendefinisikan class
class Mahasiswa {

    // Properties (variabel milik class)
    public $nama;
    public $nim;

    // Constructor — dipanggil saat object dibuat
    public function __construct($nama, $nim) {
        $this->nama = $nama;
        $this->nim  = $nim;
    }

    // Method (fungsi milik class)
    public function perkenalan() {
        return "Halo, saya {$this->nama} ({$this->nim})";
    }
}

// Membuat object dari class
$mhs = new Mahasiswa('Budi', 'F1G121001');
echo $mhs->perkenalan();
// Output: Halo, saya Budi (F1G121001)

Konsep 2: Inheritance (Pewarisan)

Ini yang terjadi di setiap Controller Laravel — class kamu mewarisi semua fitur dari class induk.

📄 contoh-inheritance.php 
<?php

// Class induk (parent)
class Controller {
    public function response($data) {
        return json_encode($data);
    }
}

// Class anak (child) — mewarisi Controller
class PageController extends Controller {

    // Method baru milik PageController
    public function home() {
        // Bisa pakai method dari parent (Controller)
        return $this->response(['page' => 'home']);
    }
}

// Inilah yang terjadi di Laravel!
// class PageController extends Controller { ... }

Konsep 3: Namespace

Namespace seperti "folder" untuk class — mencegah konflik nama dan mengorganisir kode.

📄 contoh-namespace.php 
<?php

// Deklarasi namespace — seperti alamat class ini
namespace App\Http\Controllers;

// Import class dari namespace lain
use Illuminate\Http\Request;
use App\Models\Product;

// Inilah struktur standar Controller di Laravel
class ProductController extends Controller {
    public function index() {
        // ...
    }
}

Konsep 4: Access Modifier

Modifier Bisa diakses dari Contoh penggunaan di Laravel
public Mana saja Method Controller (public function index())
protected Class sendiri + class turunan Property Model (protected $fillable)
private Class sendiri saja Helper method internal

Perbandingan: PHP Native CRUD vs Laravel

Aspek PHP Native (yang sudah kamu pelajari) Laravel (yang akan dipelajari)
Koneksi DB mysqli_connect() / PDO manual Otomatis via .env + Eloquent
Query data $sql = "SELECT * FROM ..." Product::all()
Insert data mysqli_query($conn, $sql) Product::create($request->all())
Routing Satu file = satu halaman (tambah.php) Semua di routes/web.php
Tampilan PHP campur HTML di satu file Blade template terpisah
Keamanan Harus implementasi sendiri CSRF, XSS protection built-in
Intinya: Semua yang kamu lakukan di PHP native CRUD masih berlaku di Laravel — hanya caranya yang lebih terstruktur dan lebih singkat. Laravel tidak mengganti PHP, ia mempercepat PHP.

4. Prasyarat Instalasi Persiapan

Sebelum menginstall Laravel, pastikan semua software berikut sudah tersedia di komputer kamu. Masing-masing punya peran yang berbeda dan saling bergantung satu sama lain.

1. PHP 8.1+

Laravel 11 membutuhkan PHP versi 8.1 ke atas. PHP adalah bahasa yang menjalankan seluruh kode Laravel di server. Beberapa fitur Laravel modern seperti readonly properties, enums, dan fibers hanya tersedia di PHP 8.x.

Cara paling mudah mendapatkan PHP: Install Laragon (Windows) atau XAMPP. Keduanya sudah menyertakan PHP, MySQL, dan Apache dalam satu paket — tidak perlu install PHP secara terpisah.

Pastikan PHP sudah bisa diakses dari terminal:

Terminal / CMD 
$ php --version
PHP 8.2.12 (cli) (built: Oct 24 2023)
Copyright (c) The PHP Group

Jika muncul pesan "php is not recognized", berarti PHP belum ditambahkan ke PATH sistem. Di Laragon, klik kanan tray icon → PHP → Tambah PHP ke PATH.

2. Composer

Composer adalah dependency manager untuk PHP — fungsinya mirip seperti npm untuk JavaScript atau pip untuk Python. Laravel sendiri diinstall melalui Composer, begitu juga semua library/package yang digunakan di dalam project.

Cara kerja Composer: ia membaca file composer.json di project kamu, lalu mengunduh semua package yang dibutuhkan ke folder vendor/.

1 Download Composer Installer

Buka getcomposer.org/download dan download Composer-Setup.exe (Windows) atau ikuti instruksi untuk macOS/Linux.

2 Jalankan Installer

Saat instalasi, Composer akan meminta lokasi file php.exe. Arahkan ke PHP yang sudah terinstall, misalnya: C:\laragon\bin\php\php8.2\php.exe

3 Verifikasi

Terminal / CMD 
$ composer --version
Composer version 2.6.5 2023-10-06 10:11:52

3. Node.js & npm

Laravel menggunakan Vite sebagai build tool untuk memproses file CSS dan JavaScript. Vite berjalan di atas Node.js, sehingga Node.js wajib terinstall meskipun kamu tidak menulis kode JavaScript.

npm (Node Package Manager) sudah otomatis terinstall bersama Node.js dan digunakan untuk menjalankan perintah seperti npm run dev dan npm run build.

Download Node.js: Buka nodejs.org dan pilih versi LTS (Long Term Support) — lebih stabil untuk development.
Terminal / CMD 
$ node --version
v20.10.0

$ npm --version
10.2.3

4. MySQL

MySQL adalah database relasional yang paling umum digunakan bersama Laravel. Laravel berkomunikasi dengan MySQL melalui Eloquent ORM — kamu tidak perlu menulis query SQL secara langsung.

MySQL sudah tersedia di XAMPP maupun Laragon. Untuk mengelola database secara visual, gunakan phpMyAdmin (sudah include di XAMPP/Laragon) atau TablePlus.

Konfigurasi koneksi database dilakukan di file .env di root project Laravel: 
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=nama_database
DB_USERNAME=root
DB_PASSWORD=

5. Instalasi Laravel Praktek

Ada dua cara membuat project Laravel baru. Keduanya menggunakan Composer, hanya berbeda di perintahnya. Cara pertama (via composer create-project) adalah yang paling umum dan tidak memerlukan setup tambahan.

Cara 1: Composer Create-Project (Direkomendasikan)

1 Buka Terminal dan Masuk ke Folder Kerja

Buka terminal (CMD, PowerShell, atau terminal di VS Code), lalu navigasi ke folder tempat kamu menyimpan project. Untuk pengguna Laragon, folder default-nya adalah C:\laragon\www.

Terminal 
# Pengguna Laragon (Windows)
$ cd C:\laragon\www

# Pengguna XAMPP (Windows)
$ cd C:\xampp\htdocs

# Pengguna macOS
$ cd ~/Sites

2 Buat Project Laravel Baru

Jalankan perintah berikut. Ganti belajar-laravel dengan nama project yang kamu inginkan — nama ini akan menjadi nama folder sekaligus nama virtual host di Laragon.

Terminal 
$ composer create-project laravel/laravel belajar-laravel

Creating a "laravel/laravel" project at "./belajar-laravel"
Installing laravel/laravel (v11.x.x)
  - Installing laravel/framework (v11.x.x): Downloading (100%)
  - Installing ... (banyak package lain)
...
Application key set successfully.

Proses ini membutuhkan koneksi internet dan bisa memakan waktu 2–5 menit tergantung kecepatan internet. Composer akan mengunduh Laravel beserta semua dependensinya ke folder vendor/.

Pesan "Application key set successfully" menandakan Laravel berhasil di-generate dengan encryption key unik untuk project ini. Key ini tersimpan di file .env sebagai APP_KEY.

3 Masuk ke Folder Project

Setelah instalasi selesai, masuk ke folder project yang baru dibuat:

Terminal 
$ cd belajar-laravel

Semua perintah Artisan dan Composer selanjutnya harus dijalankan dari dalam folder ini.

4 Konfigurasi File .env

File .env adalah file konfigurasi environment — berisi pengaturan database, nama aplikasi, mode debug, dll. File ini sudah otomatis dibuat saat instalasi. Buka dan sesuaikan bagian database:

.env 
APP_NAME=BelajarLaravel
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=belajar_laravel   ← sesuaikan nama database
DB_USERNAME=root               ← username MySQL (default: root)
DB_PASSWORD=                   ← password MySQL (default: kosong di Laragon/XAMPP)
File .env berisi informasi sensitif (password, API key, dll). File ini sudah masuk ke .gitignore secara default — jangan pernah di-commit ke repository publik.

5 Buat Database di MySQL

Buka phpMyAdmin (http://localhost/phpmyadmin) atau TablePlus, lalu buat database baru dengan nama yang sama seperti yang kamu tulis di DB_DATABASE pada file .env.

Atau bisa juga lewat terminal MySQL:

Terminal 
$ mysql -u root -p
Enter password: (tekan Enter jika kosong)

mysql> CREATE DATABASE belajar_laravel;
Query OK, 1 row affected

mysql> exit

Buka http://127.0.0.1:8000 atau http://localhost:8000 di browser.

Jika muncul halaman welcome Laravel dengan tampilan seperti di bawah → Instalasi berhasil.

Pengguna Laragon bisa langsung mengakses project via virtual host otomatis: http://belajar-laravel.test — tanpa perlu menjalankan php artisan serve.

Cara 2: Laravel Installer

Cara alternatif menggunakan Laravel Installer — sebuah tool global yang perlu diinstall sekali, lalu bisa dipakai berulang kali untuk membuat project baru.

Terminal 
# Install Laravel Installer secara global (cukup sekali)
$ composer global require laravel/installer

# Buat project baru (lebih cepat dari create-project)

6. Struktur Folder Laravel Penting!

Saat pertama kali membuka project Laravel, jumlah folder dan file-nya bisa terasa membingungkan. Tapi tenang — dalam praktik sehari-hari, kamu hanya akan bekerja di 4–5 folder utama. Sisanya jarang disentuh.

belajar-laravel/ ├── app/ ← Inti aplikasi (paling sering diedit) │ ├── Http/ │ │ ├── Controllers/ ← Controller — logika request/response │ │ └── Middleware/ ← Filter request (auth, CSRF, dll) │ ├── Models/ ← Model — representasi tabel database │ └── Providers/ ← Service providers (jarang diedit) │ ├── bootstrap/ ← Bootstrap framework (jangan diedit) ├── config/ ← File konfigurasi (database, mail, dll) ├── database/ │ ├── migrations/ ← Skema tabel database (versi kontrol DB) │ ├── seeders/ ← Data awal / data dummy │ └── factories/ ← Generator data dummy untuk testing │ ├── public/ ← Document root — satu-satunya folder yang diakses browser │ ├── index.php ← Entry point seluruh aplikasi │ └── (css, js, img hasil build) │ ├── resources/ ← Source file tampilan dan aset │ ├── css/ ← Source CSS (sebelum dikompilasi Vite) │ ├── js/ ← Source JavaScript │ └── views/ ← Blade templates (.blade.php) │ ├── routes/ │ ├── web.php ← Semua route halaman web didefinisikan di sini │ └── api.php ← Route untuk REST API │ ├── storage/ ← File upload, cache, session, log │ └── logs/laravel.log ← Log error aplikasi ├── tests/ ← Unit test dan feature test ├── vendor/ ← Semua package Composer (JANGAN diedit) │ ├── .env ← Konfigurasi environment (DB, APP_KEY, dll) ├── .env.example ← Template .env untuk dibagikan ke tim ├── artisan ← Entry point Artisan CLI ├── composer.json ← Daftar dependensi PHP ├── package.json ← Daftar dependensi Node.js └── vite.config.js ← Konfigurasi Vite (asset bundler)

Penjelasan Folder Utama

app/ — Inti Aplikasi

Folder app/ adalah tempat kamu menghabiskan sebagian besar waktu. Di sinilah semua logika bisnis aplikasi berada.

Path Isi Dibuat dengan
app/Http/Controllers/ Semua class Controller. Setiap Controller menangani satu kelompok fitur (misal: ProductController untuk semua hal terkait produk). php artisan make:controller
app/Models/ Setiap Model merepresentasikan satu tabel database. Model Product terhubung ke tabel products. php artisan make:model
app/Http/Middleware/ Filter yang berjalan sebelum/sesudah request diproses. Contoh: cek apakah user sudah login sebelum mengakses halaman tertentu. php artisan make:middleware

routes/ — Definisi URL

Semua URL aplikasi didefinisikan di folder routes/. Tidak ada lagi konsep "satu file = satu halaman" seperti PHP native.

routes/web.php 
<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\ProductController;

// Route sederhana — langsung return view
Route::get('/', function () {
    return view('welcome');
});

// Route ke Controller
Route::get('/products', [ProductController::class, 'index']);
Route::get('/products/{id}', [ProductController::class, 'show']);

resources/views/ — Template Blade

Semua file tampilan disimpan di sini dengan ekstensi .blade.php. Blade adalah template engine Laravel yang memungkinkan kamu menulis PHP di dalam HTML dengan sintaks yang lebih bersih.

Struktur views biasanya diorganisir per fitur:

resources/views/ ├── layouts/ │ └── app.blade.php ← Layout utama (header, footer, nav) ├── products/ │ ├── index.blade.php ← Halaman daftar produk │ ├── show.blade.php ← Halaman detail produk │ ├── create.blade.php ← Form tambah produk │ └── edit.blade.php ← Form edit produk └── welcome.blade.php ← Halaman default Laravel

database/migrations/ — Versi Kontrol Database

Migration adalah cara Laravel mengelola skema database. Setiap perubahan struktur tabel (buat tabel baru, tambah kolom, dll) dibuat sebagai file migration baru. Ini memungkinkan seluruh tim memiliki struktur database yang sama.

database/migrations/2024_01_01_000000_create_products_table.php 
public function up(): void
{
    Schema::create('products', function (Blueprint $table) {
        $table->id();
        $table->string('name');
        $table->decimal('price', 10, 2);
        $table->text('description')->nullable();
        $table->timestamps(); // created_at & updated_at
    });
}

public/ — Satu-satunya Folder yang Diakses Browser

Ini adalah document root yang harus diarahkan oleh web server (Apache/Nginx). Browser hanya bisa mengakses file di dalam folder ini secara langsung. File public/index.php adalah pintu masuk seluruh aplikasi — semua request diarahkan ke sini, lalu Laravel yang menentukan response-nya.

Jangan pernah menyimpan file sensitif (seperti .env) di dalam folder public/. Folder public/ adalah satu-satunya yang boleh diakses dari internet.

.env — Konfigurasi Environment

File .env menyimpan semua konfigurasi yang berbeda antara environment development dan production — seperti kredensial database, API key, dan mode debug. File ini tidak di-commit ke Git.

File .env.example adalah versi "kosong" dari .env yang aman untuk dibagikan ke tim melalui Git. Setiap anggota tim meng-copy file ini menjadi .env dan mengisi nilainya sendiri.

vendor/ — Jangan Diedit

Folder vendor/ berisi semua package PHP yang diinstall via Composer, termasuk framework Laravel itu sendiri. Folder ini di-generate ulang setiap kali menjalankan composer install, sehingga tidak perlu (dan tidak boleh) diedit secara manual. Folder ini juga masuk ke .gitignore.

resources/views/ File Blade template (tampilan) Sangat sering

.env Konfigurasi database, app key, mode debug Sering (terutama awal setup)
database/migrations/ Membuat dan mengubah struktur tabel Sering

9. Routing Fundamental

Routing adalah proses menentukan URL mana yang ditangani oleh kode mana. Di Laravel, semua route untuk halaman web didefinisikan di satu file: routes/web.php. Tidak ada lagi konsep "satu file PHP = satu halaman" seperti di PHP native.

1. Route Dasar

Setiap route minimal membutuhkan dua hal: URL dan apa yang dikembalikan. Yang dikembalikan bisa berupa string, view, JSON, atau redirect.

routes/web.php 
<?php
use Illuminate\Support\Facades\Route;

// Return string langsung — berguna untuk testing cepat
Route::get('/hello', function () {
    return 'Hello, World!';
});

// Return view — paling umum untuk halaman statis
Route::get('/', function () {
    return view('welcome');
});

// Return view dengan data — data dikirim ke Blade template
Route::get('/about', function () {
    return view('about', [
        'title'   => 'Tentang Kami',
        'content' => 'Selamat datang di website kami.',
    ]);
});

// Return redirect
Route::get('/home', function () {
    return redirect('/');
});

2. HTTP Method

Route harus mencocokkan HTTP method dari request. Setiap method punya makna yang berbeda sesuai konvensi REST:

Method Digunakan untuk Contoh URL
Route::get() Menampilkan halaman / mengambil data GET /products
Route::post() Menerima data dari form (tambah data) POST /products
Route::put() Update seluruh data PUT /products/1
Route::patch() Update sebagian data PATCH /products/1
Route::delete() Menghapus data DELETE /products/1
routes/web.php 
// Tampilkan form tambah produk
Route::get('/products/create', [ProductController::class, 'create']);

// Proses form submit (tambah produk)
Route::post('/products', [ProductController::class, 'store']);

// Update produk
Route::put('/products/{id}', [ProductController::class, 'update']);

// Hapus produk
Route::delete('/products/{id}', [ProductController::class, 'destroy']);

3. Route ke Controller

Untuk aplikasi yang lebih dari sekedar halaman statis, route sebaiknya diarahkan ke Controller — bukan langsung menulis logika di dalam closure. Ini membuat kode lebih terorganisir dan mudah ditest.

routes/web.php 
<?php
use Illuminate\Support\Facades\Route;
use App\Http\Controllers\PageController;
use App\Http\Controllers\ProductController;

// Format: [NamaController::class, 'namaMethod']
Route::get('/',        [PageController::class, 'home']);
Route::get('/about',   [PageController::class, 'about']);
Route::get('/contact', [PageController::class, 'contact']);

Route::get('/products',     [ProductController::class, 'index']);
Route::get('/products/{id}', [ProductController::class, 'show']);

4. Route Parameter

Parameter route memungkinkan bagian dari URL menjadi variabel yang diteruskan ke Controller atau closure.

routes/web.php 
// Parameter wajib — URL: /user/budi → $name = 'budi'
Route::get('/user/{name}', function ($name) {
    return "Halo, {$name}!";
});

// Parameter opsional — URL: /user atau /user/budi
Route::get('/user/{name?}', function ($name = 'Guest') {
    return "Halo, {$name}!";
});

// Beberapa parameter — URL: /post/2024/laravel-routing
Route::get('/post/{year}/{slug}', function ($year, $slug) {
    return "Tahun: {$year}, Slug: {$slug}";
});

// Validasi parameter dengan where — hanya angka yang diterima
Route::get('/products/{id}', [ProductController::class, 'show'])
    ->where('id', '[0-9]+');

// Validasi parameter — hanya huruf
Route::get('/category/{slug}', [ProductController::class, 'byCategory'])
    ->where('slug', '[a-z\-]+');

5. Named Route

Named route memberi nama pada sebuah route sehingga kamu bisa membuat URL-nya dari nama tersebut — bukan dari string URL secara langsung. Keuntungannya: jika URL berubah, kamu hanya perlu mengubah di satu tempat (di web.php), tidak perlu mencari-cari di seluruh file Blade.

routes/web.php

Route::get('/',               [PageController::class, 'home'])   ->name('home');
Route::get('/about',         [PageController::class, 'about'])  ->name('about');
Route::get('/products',     [ProductController::class, 'index']) ->name('products.index');
Route::get('/products/{id}', [ProductController::class, 'show'])  ->name('products.show');

6. Route Group

Route group memungkinkan kamu menerapkan prefix URL atau middleware yang sama ke beberapa route sekaligus — tanpa mengulang kode.

});

Resource Controller adalah Controller dengan 7 method standar yang sesuai dengan konvensi REST. Ini yang paling sering dipakai untuk fitur CRUD.

<?php

namespace App\Http\Controllers;

use App\Models\Product;
use Illuminate\Http\Request;

class ProductController extends Controller
{
    // GET /products — tampilkan semua data
    public function index()
    {
        $products = Product::all();
        return view('products.index', compact('products'));
    }

    // GET /products/create — tampilkan form tambah
    public function create()
    {
        return view('products.create');
    }

    // POST /products — simpan data baru
    public function store(Request $request)
    {
        $request->validate([
            'name'  => 'required|string|max:255',
            'price' => 'required|numeric|min:0',
        ]);

        Product::create($request->only(['name', 'price', 'description']));

        return redirect()->route('products.index')
            ->with('success', 'Produk berhasil ditambahkan!');
    }

    // GET /products/{id} — tampilkan detail satu data
    public function show($id)
    {
        $product = Product::findOrFail($id); // otomatis 404 jika tidak ada
        return view('products.show', compact('product'));
    }

    // GET /products/{id}/edit — tampilkan form edit
    public function edit($id)
    {
        $product = Product::findOrFail($id);
        return view('products.edit', compact('product'));
    }

    // PUT /products/{id} — simpan perubahan data
    public function update(Request $request, $id)
    {
        $request->validate([
            'name'  => 'required|string|max:255',
            'price' => 'required|numeric|min:0',
        ]);

        $product = Product::findOrFail($id);
        $product->update($request->only(['name', 'price', 'description']));

        return redirect()->route('products.index')
            ->with('success', 'Produk berhasil diperbarui!');
    }

    // DELETE /products/{id} — hapus data
    public function destroy($id)
    {
        $product = Product::findOrFail($id);
        $product->delete();

        return redirect()->route('products.index')
            ->with('success', 'Produk berhasil dihapus!');
    }
}

11. View & Blade Template Fundamental

Apa itu Blade?

Blade adalah template engine bawaan Laravel. File Blade berekstensi .blade.php dan disimpan di folder resources/views/.

Blade bukan sekedar "PHP di dalam HTML" — ia punya sintaks khusus yang lebih bersih dan aman. Di balik layar, Laravel mengkompilasi file .blade.php menjadi file PHP biasa, lalu menyimpannya di storage/framework/views/ sebagai cache.

Bagaimana Blade bekerja?
  1. Kamu menulis resources/views/home.blade.php
  2. Saat pertama kali diakses, Laravel mengkompilasi file tersebut menjadi PHP murni
  3. Hasil kompilasi disimpan di storage/framework/views/ (cache)
  4. Request berikutnya langsung pakai cache — lebih cepat
  5. Cache otomatis diperbarui jika file Blade berubah

Contoh: {{ $nama }} dikompilasi menjadi <?php echo e($nama); ?> — fungsi e() itulah yang melakukan escaping untuk mencegah XSS.

Mengirim Data dari Controller ke View

Sebelum bisa menampilkan data di Blade, Controller harus mengirimkan data tersebut. Ada beberapa cara:

📄 app/Http/Controllers/PageController.php 
// Cara 1: Array langsung (paling umum)
return view('home', [
    'title'  => 'Beranda',
    'jumlah' => 42,
]);

// Cara 2: compact() — otomatis buat array dari nama variabel
$title  = 'Beranda';
$jumlah = 42;
return view('home', compact('title', 'jumlah'));

// Cara 3: with() — chaining
return view('home')
    ->with('title', 'Beranda')
    ->with('jumlah', 42);

Semua cara di atas menghasilkan hal yang sama: di dalam file Blade, kamu bisa langsung pakai $title dan $jumlah.

Sintaks Blade Dasar

📄 resources/views/home.blade.php 
<!DOCTYPE html>
<html>
<head>
    <title>{{ $title }}</title>
</head>
<body>

    <!-- 1. Menampilkan variabel (auto-escaped dari XSS) -->
    <h1>{{ $title }}</h1>

    <!-- 2. Nilai default jika variabel null/tidak ada -->
    <p>{{ $subtitle ?? 'Tidak ada subtitle' }}</p>

    <!-- 3. Tanpa escaping — hanya untuk konten HTML terpercaya -->
    {!! $htmlContent !!}

    <!-- 4. Komentar Blade — tidak muncul di source HTML browser -->
    {{-- Ini komentar Blade, tidak akan terlihat di inspect element --}}

    <!-- 5. Percabangan -->
    @if($title === 'Beranda')
        <p>Selamat datang di beranda!</p>
    @elseif($title === 'About')
        <p>Halaman tentang kami.</p>
    @else
        <p>Halaman lain.</p>
    @endif

    <!-- 6. Perulangan biasa -->
    @foreach($items as $item)
        <p>{{ $loop->iteration }}. {{ $item }}</p>
    @endforeach

    <!-- 7. @forelse — loop dengan fallback jika array kosong -->
    @forelse($items as $item)
        <p>{{ $item }}</p>
    @empty
        <!-- Blok ini dijalankan jika $items kosong [] -->
        <p class="text-muted">Tidak ada data.</p>
    @endforelse

</body>
</html>

Ini salah satu hal paling penting di Blade yang sering membingungkan pemula:

📄 contoh-escaping.blade.php 
<!-- Misalnya $input berisi: <script>alert('hacked')</script> -->

<!-- {{ }} — AMAN: karakter HTML di-escape -->
{{ $input }}
<!-- Output di browser: <script>alert('hacked')</script> (teks biasa) -->

<!-- {!! !!} — BERBAHAYA jika dari input user: HTML dirender apa adanya -->
{!! $input !!}
<!-- Output di browser: script dieksekusi! (XSS attack) -->

<!-- {!! !!} AMAN dipakai jika konten dari developer sendiri, bukan user -->
{!! $articleBody !!}  <!-- OK jika $articleBody dari editor admin yang terpercaya -->
⚠️ Aturan Praktis:
  • Selalu pakai {{ }} untuk data yang berasal dari input user (form, URL, database)
  • Pakai {!! !!} hanya untuk konten HTML yang kamu sendiri yang buat/kontrol
  • Contoh aman: konten artikel dari editor admin, HTML yang di-generate oleh kode kamu sendiri

Variabel $loop di @foreach

Di dalam @foreach, Blade menyediakan variabel spesial $loop yang berisi informasi tentang iterasi saat ini:

📄 contoh-loop.blade.php 
@foreach($mahasiswa as $mhs)
    <tr class="{{ $loop->even ? 'table-light' : '' }}">
        <td>{{ $loop->iteration }}</td>  <!-- Nomor urut: 1, 2, 3, ... -->
        <td>{{ $mhs->nama }}</td>

        @if($loop->first)
            <td><span class="badge bg-warning">Pertama</span></td>
        @elseif($loop->last)
            <td><span class="badge bg-secondary">Terakhir</span></td>
        @else
            <td>-</td>
        @endif
    </tr>
@endforeach

$loop->lastboolApakah ini iterasi terakhir?
$loop->evenboolApakah iterasi genap?
$loop->oddboolApakah iterasi ganjil?
$loop->depthintKedalaman loop (untuk nested loop)
$loop->parentobjectAkses $loop dari loop luar (nested)

15. Praktek: Halaman Statik Lengkap Hands-On

Kita akan membuat website dengan 3 halaman statik: Home, About, Contact. "Statik" di sini artinya data tidak berasal dari database — cocok untuk latihan alur MVC dan Blade sebelum masuk ke CRUD.

Alur yang akan kita ikuti:
  1. Buat Controller — tempat logika tiap halaman
  2. Daftarkan Route — URL apa yang memanggil method mana
  3. Buat Layout — template master yang dipakai semua halaman
  4. Buat View — konten spesifik tiap halaman

Struktur file yang akan terbentuk setelah praktek ini selesai:

resources/views/ ├── layouts/ │ └── app.blade.php ← template master (navbar, footer) └── pages/ ├── home.blade.php ← halaman beranda ├── about.blade.php ← halaman tentang kami └── contact.blade.php ← halaman kontak app/Http/Controllers/ └── PageController.php ← satu controller untuk semua halaman routes/ └── web.php ← daftar URL → method controller

1 Buat Controller

Controller adalah class PHP yang menangani request. Satu controller bisa punya banyak method — masing-masing untuk satu halaman atau aksi.

Terminal 
$ php artisan make:controller PageController

INFO  Controller [app/Http/Controllers/PageController.php] created successfully.

Artisan akan membuat file kosong di app/Http/Controllers/PageController.php. Kita akan isi method-nya di langkah berikutnya.

2 Isi Controller

Setiap method di controller bertanggung jawab untuk satu halaman. Method memanggil view() dengan dua argumen: nama file view dan data yang dikirim ke view.

📄 app/Http/Controllers/PageController.php 
<?php

namespace App\Http\Controllers;

class PageController extends Controller
{
    // Dipanggil saat user buka URL "/"
    // view('pages.home') → resources/views/pages/home.blade.php
    public function home()
    {
        return view('pages.home', [
            'title' => 'Beranda',  // tersedia sebagai $title di view
        ]);
    }

    // Dipanggil saat user buka URL "/about"
    // Data $team (array) dan $founded (integer) dikirim ke view
    public function about()
    {
        $team = [
            ['nama' => 'John Doe',  'posisi' => 'CEO'],
            ['nama' => 'Jane Smith', 'posisi' => 'CTO'],
            ['nama' => 'Ahmad Reza', 'posisi' => 'Developer'],
        ];

        return view('pages.about', [
            'title'   => 'Tentang Kami',
            'team'    => $team,    // tersedia sebagai $team di view
            'founded' => 2024,      // tersedia sebagai $founded di view
        ]);
    }

    // Dipanggil saat user buka URL "/contact"
    public function contact()
    {
        return view('pages.contact', [
            'title' => 'Hubungi Kami',
        ]);
    }
}
Konvensi penamaan view: Titik (.) dalam nama view berarti subfolder. Jadi 'pages.home' artinya file resources/views/pages/home.blade.php.

3 Definisikan Route

Route menghubungkan URL dengan method controller. Semua route web didefinisikan di routes/web.php.

📄 routes/web.php 
<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\PageController;

// Route::get(URL, [Controller, method])->name('nama_route')
Route::get('/',        [PageController::class, 'home'])->name('home');
Route::get('/about',   [PageController::class, 'about'])->name('about');
Route::get('/contact', [PageController::class, 'contact'])->name('contact');
Kenapa pakai ->name()?
Dengan memberi nama route, kita bisa generate URL di Blade pakai route('nama') alih-alih menulis URL hardcode. Keuntungannya: jika URL berubah (misal dari /about jadi /tentang), kita cukup ubah di web.php saja — semua link di view otomatis ikut berubah.
Cara Contoh Keterangan
route('about') http://localhost:8000/about ✅ Direkomendasikan — pakai nama route
url('/about') http://localhost:8000/about ⚠️ Hardcode URL — rawan salah jika URL berubah

Cek semua route yang terdaftar dengan perintah:

Terminal 
$ php artisan route:list

  GET|HEAD  /         home    PageController@home
  GET|HEAD  /about    about   PageController@about
  GET|HEAD  /contact  contact PageController@contact

4 Buat Layout Master

Layout adalah template induk yang berisi bagian-bagian yang sama di semua halaman — navbar, footer, link CSS/JS. Halaman anak nanti tinggal mengisi "slot" konten yang sudah disiapkan.

Buat folder resources/views/layouts/ lalu buat file:

📄 resources/views/layouts/app.blade.php 
<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- @yield('title', 'MyApp') = tampilkan section 'title', default 'MyApp' jika tidak diisi -->
    <title>@yield('title', 'MyApp') - MyApp</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>

<nav class="navbar navbar-expand-lg navbar-dark bg-dark">
    <div class="container">
        <a class="navbar-brand" href="{{ route('home') }}">MyApp</a>
        <button class="navbar-toggler" data-bs-toggle="collapse" data-bs-target="#nav">
            <span class="navbar-toggler-icon"></span>
        </button>
        <div class="collapse navbar-collapse" id="nav">
            <ul class="navbar-nav ms-auto">
                <li class="nav-item">
                    <!-- request()->is('/') cek apakah URL saat ini adalah "/" -->
                    <!-- Jika iya, tambahkan class 'active' ke link navbar -->
                    <a class="nav-link {{ request()->is('/') ? 'active' : '' }}" href="{{ route('home') }}">Home</a>
                </li>
                <li class="nav-item">
                    <a class="nav-link {{ request()->is('about') ? 'active' : '' }}" href="{{ route('about') }}">About</a>
                </li>
                <li class="nav-item">
                    <a class="nav-link {{ request()->is('contact') ? 'active' : '' }}" href="{{ route('contact') }}">Contact</a>
                </li>
            </ul>
        </div>
    </div>
</nav>

<main class="container py-4">
    <!-- @yield('content') = slot untuk konten tiap halaman -->
    <!-- Halaman anak mengisi slot ini via @section('content') -->
    @yield('content')
</main>

<footer class="bg-dark text-white text-center py-3 mt-auto">
    <p class="mb-0">&copy; {{ date('Y') }} MyApp - Laravel</p>
</footer>

<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
Poin penting di layout ini:
  • @yield('title', 'MyApp') — slot untuk judul tab browser. Argumen kedua adalah nilai default jika halaman anak tidak mengisi section ini.
  • @yield('content') — slot utama untuk konten halaman. Ini yang akan diisi oleh tiap halaman anak.
  • request()->is('about') — mengecek URL saat ini, dipakai untuk menandai link navbar yang sedang aktif.
