1. Pengenalan Laravel Dasar

Apa itu Laravel?

Laravel adalah framework PHP open-source yang mengikuti pola arsitektur MVC (Model-View-Controller). Dibuat oleh Taylor Otwell pada tahun 2011, Laravel menjadi framework PHP paling populer karena sintaksnya yang elegan dan fitur-fiturnya yang lengkap.

Mengapa Framework?
  • Tanpa framework = menulis kode dari nol (routing, security, database, dll)
  • Dengan framework = fitur sudah tersedia, tinggal pakai
  • Ibarat membangun rumah: framework = fondasi + struktur rangka sudah jadi

Keunggulan Laravel

Fitur Keterangan
Eloquent ORM Interaksi database tanpa menulis SQL mentah
Blade Template Template engine yang powerful dan mudah
Artisan CLI Command line tool untuk generate kode, migrasi, dll
Migration Version control untuk skema database
Routing Sistem routing yang ekspresif dan fleksibel
Middleware Filter HTTP request (auth, CSRF, dll)
Authentication Sistem auth built-in (login, register, dll)
Tinker REPL interaktif untuk eksperimen kode
Komunitas Dokumentasi lengkap, ekosistem besar

Perbandingan PHP Native vs Laravel

Aspek PHP Native Laravel
Routing Manual (file per halaman) Centralized di routes/web.php
Database Query SQL manual Eloquent ORM / Query Builder
Template PHP campur HTML Blade (layout, component, inheritance)
Security Implementasi sendiri CSRF, XSS, SQL Injection built-in
Struktur Bebas (bisa berantakan) Terstruktur dan terstandar (MVC)

2. Arsitektur MVC Konsep

MVC (Model-View-Controller) adalah pola desain yang memisahkan aplikasi menjadi 3 komponen utama:

Model

Data & Logika Bisnis

  • Berinteraksi dengan database
  • Mendefinisikan struktur data
  • Validasi aturan bisnis

View

Tampilan / UI

  • HTML + Blade template
  • Menampilkan data ke user
  • Tidak ada logika bisnis

Controller

Pengendali / Perantara

  • Menerima request dari user
  • Memproses data via Model
  • Mengirim data ke View

3. 🔗 Jembatan: OOP PHP Penting!

Sebelum masuk Laravel, kamu perlu memahami OOP (Object-Oriented Programming) di PHP. Laravel dibangun sepenuhnya dengan OOP — setiap Controller, Model, dan Middleware adalah sebuah class.

📌 Kenapa OOP penting untuk Laravel?
  • Controller = class PHP biasa yang di-extend dari base Controller
  • Model = class yang merepresentasikan tabel database
  • Semua fitur Laravel (routing, middleware, dll) bekerja lewat class & method
  • Tanpa OOP, kode Laravel akan terasa seperti "mantra" yang tidak dipahami

Konsep 1: Class & Object

📄 contoh-oop.php 
<?php

// Mendefinisikan class
class Mahasiswa {

    // Properties (variabel milik class)
    public $nama;
    public $nim;

    // Constructor — dipanggil saat object dibuat
    public function __construct($nama, $nim) {
        $this->nama = $nama;
        $this->nim  = $nim;
    }

    // Method (fungsi milik class)
    public function perkenalan() {
        return "Halo, saya {$this->nama} ({$this->nim})";
    }
}

// Membuat object dari class
$mhs = new Mahasiswa('Budi', 'F1G121001');
echo $mhs->perkenalan();
// Output: Halo, saya Budi (F1G121001)

Konsep 2: Inheritance (Pewarisan)

Ini yang terjadi di setiap Controller Laravel — class kamu mewarisi semua fitur dari class induk.

📄 contoh-inheritance.php 
<?php

// Class induk (parent)
class Controller {
    public function response($data) {
        return json_encode($data);
    }
}

// Class anak (child) — mewarisi Controller
class PageController extends Controller {

    // Method baru milik PageController
    public function home() {
        // Bisa pakai method dari parent (Controller)
        return $this->response(['page' => 'home']);
    }
}

// Inilah yang terjadi di Laravel!
// class PageController extends Controller { ... }

Konsep 3: Namespace

Namespace seperti "folder" untuk class — mencegah konflik nama dan mengorganisir kode.

📄 contoh-namespace.php 
<?php

// Deklarasi namespace — seperti alamat class ini
namespace App\Http\Controllers;

// Import class dari namespace lain
use Illuminate\Http\Request;
use App\Models\Product;

// Inilah struktur standar Controller di Laravel
class ProductController extends Controller {
    public function index() {
        // ...
    }
}

Konsep 4: Access Modifier

Modifier Bisa diakses dari Contoh penggunaan di Laravel
public Mana saja Method Controller (public function index())
protected Class sendiri + class turunan Property Model (protected $fillable)
private Class sendiri saja Helper method internal

Perbandingan: PHP Native CRUD vs Laravel

Aspek PHP Native (yang sudah kamu pelajari) Laravel (yang akan dipelajari)
Koneksi DB mysqli_connect() / PDO manual Otomatis via .env + Eloquent
Query data $sql = "SELECT * FROM ..." Product::all()
Insert data mysqli_query($conn, $sql) Product::create($request->all())
Routing Satu file = satu halaman (tambah.php) Semua di routes/web.php
Tampilan PHP campur HTML di satu file Blade template terpisah
Keamanan Harus implementasi sendiri CSRF, XSS protection built-in
Intinya: Semua yang kamu lakukan di PHP native CRUD masih berlaku di Laravel — hanya caranya yang lebih terstruktur dan lebih singkat. Laravel tidak mengganti PHP, ia mempercepat PHP.

7. Artisan CLI Tool

Artisan adalah command-line interface bawaan Laravel untuk mempercepat development.

Perintah Artisan yang Sering Digunakan

Terminal 
# Jalankan server development
$ php artisan serve

# Lihat semua perintah artisan
$ php artisan list

# Lihat semua route yang terdaftar
$ php artisan route:list

# Buat Controller baru
$ php artisan make:controller PageController

# Buat Model baru
$ php artisan make:model Product

# Buat Model + Migration + Controller sekaligus
$ php artisan make:model Product -mc

# Buat Migration baru
$ php artisan make:migration create_products_table

# Jalankan migrasi database
$ php artisan migrate

# Reset & jalankan ulang semua migrasi
$ php artisan migrate:fresh

# Bersihkan cache
$ php artisan cache:clear
$ php artisan config:clear
$ php artisan view:clear

# Masuk ke Tinker (REPL interaktif)
$ php artisan tinker

9. Routing Fundamental

Route mendefinisikan URL mana yang mengarah ke kode mana.

📄 routes/web.php 
<?php
use Illuminate\Support\Facades\Route;
use App\Http\Controllers\PageController;

// ============================
// 1. Route langsung ke View
// ============================
Route::get('/', function () {
    return view('welcome');
});

// ============================
// 2. Route dengan return string
// ============================
Route::get('/hello', function () {
    return 'Hello, World!';
});

// ============================
// 3. Route ke View dengan data
// ============================
Route::get('/about', function () {
    $nama = 'Universitas Mataram';
    return view('about', ['nama' => $nama]);
});

// ============================
// 4. Route dengan parameter
// ============================
Route::get('/user/{name}', function ($name) {
    return "Halo, $name!";
});

// Parameter opsional
Route::get('/user/{name?}', function ($name = 'Guest') {
    return "Halo, $name!";
});

// ============================
// 5. Route ke Controller ⭐
// ============================
Route::get('/',        [PageController::class, 'home']);
Route::get('/about',   [PageController::class, 'about']);
Route::get('/contact', [PageController::class, 'contact']);

// ============================
// 6. Route dengan nama (named route)
// ============================
Route::get('/about', [PageController::class, 'about'])->name('about');

// Di Blade: {{ route('about') }} → menghasilkan URL /about

HTTP Method Routes

Route::get('/form',    ...);  // Tampilkan form
Route::post('/form',   ...);  // Proses form submit
Route::put('/data/1', ...);  // Update data
Route::delete('/data/1', ...); // Hapus data

10. Controller Fundamental

Membuat Controller

Terminal 
$ php artisan make:controller PageController

INFO  Controller [app/Http/Controllers/PageController.php] created successfully.

Menulis Controller

📄 app/Http/Controllers/PageController.php 
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class PageController extends Controller
{
    // Halaman Home
    public function home()
    {
        $title = 'Beranda';
        return view('home', compact('title'));
    }

    // Halaman About
    public function about()
    {
        $data = [
            'title'   => 'Tentang Kami',
            'content' => 'Website ini dibuat untuk belajar Laravel.',
            'year'    => date('Y'),
        ];
        return view('about', $data);
    }

    // Halaman Contact
    public function contact()
    {
        return view('contact', ['title' => 'Kontak']);
    }

    // Halaman dengan parameter
    public function show($slug)
    {
        return view('page', ['slug' => $slug]);
    }
}
Cara mengirim data ke View: 
// Cara 1: Array
return view('home', ['title' => 'Beranda']);

// Cara 2: compact()
$title = 'Beranda';
return view('home', compact('title'));

// Cara 3: with()
return view('home')->with('title', 'Beranda');

11. View & Blade Template Fundamental

Apa itu Blade?

Blade adalah template engine bawaan Laravel. File Blade berekstensi .blade.php dan disimpan di folder resources/views/.

Sintaks Blade Dasar

📄 resources/views/home.blade.php 
<!DOCTYPE html>
<html>
<head>
    <title>{{ $title }}</title>
</head>
<body>

    <!-- Menampilkan variabel (auto-escaped dari XSS) -->
    <h1>{{ $title }}</h1>

    <!-- Tanpa escaping (hati-hati XSS!) -->
    {!! $htmlContent !!}

    <!-- Komentar Blade (tidak muncul di HTML) -->
    {{-- Ini komentar Blade --}}

    <!-- Kondisi -->
    @if($title === 'Beranda')
        <p>Selamat datang di beranda!</p>
    @elseif($title === 'About')
        <p>Halaman tentang kami.</p>
    @else
        <p>Halaman lain.</p>
    @endif

    <!-- Perulangan -->
    @foreach($items as $item)
        <p>{{ $loop->iteration }}. {{ $item }}</p>
    @endforeach

    <!-- Cek array kosong -->
    @forelse($items as $item)
        <p>{{ $item }}</p>
    @empty
        <p>Tidak ada data.</p>
    @endforelse

</body>
</html>

Ringkasan Sintaks Blade

Sintaks Fungsi
{{ $var }}Echo dengan escaping (aman dari XSS)
{!! $var !!}Echo tanpa escaping (raw HTML)
{{-- komentar --}}Komentar Blade
@if / @elseif / @else / @endifPercabangan
@foreach / @endforeachPerulangan
@forelse / @empty / @endforelseLoop dengan cek kosong
@for / @endforFor loop
@while / @endwhileWhile loop
@extends('layout')Mewarisi layout
@section('nama') / @endsectionMendefinisikan bagian konten
@yield('nama')Menampilkan section di layout
@include('partial')Menyisipkan file blade lain
@csrfToken CSRF untuk form
$loop->index / iteration / first / lastInfo loop di @foreach

14. 🗄️ Praktek: CRUD Sederhana Hands-On

Setelah bisa membuat halaman statik, sekarang kita buat aplikasi CRUD (Create, Read, Update, Delete) sederhana — yaitu manajemen data Produk. Ini adalah fondasi dari hampir semua aplikasi web nyata.

📌 Yang akan dibuat:
  • Menampilkan daftar produk (Read)
  • Form tambah produk baru (Create)
  • Form edit produk (Update)
  • Hapus produk (Delete)

Perbandingan CRUD: PHP Native vs Laravel

Operasi PHP Native Laravel
Read SELECT * FROM produk + loop manual Product::all()
Create INSERT INTO produk ... manual Product::create($data)
Update UPDATE produk SET ... WHERE id=? $product->update($data)
Delete DELETE FROM produk WHERE id=? $product->delete()

Langkah 2 — Buat Migration

Migration adalah cara Laravel membuat tabel database lewat kode PHP — pengganti membuat tabel manual di phpMyAdmin.

💻 Terminal 
$ php artisan make:migration create_products_table

INFO  Migration [database/migrations/2024_01_01_000000_create_products_table.php] created successfully.

Buka file migration yang baru dibuat, isi strukturnya:

📄 database/migrations/xxxx_create_products_table.php 
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('products', function (Blueprint $table) {
            $table->id();                                      // kolom id (auto increment)
            $table->string('name');                          // nama produk (VARCHAR)
            $table->text('description')->nullable();      // deskripsi (boleh kosong)
            $table->decimal('price', 10, 2);               // harga (DECIMAL)
            $table->integer('stock')->default(0);          // stok (default 0)
            $table->timestamps();                             // created_at & updated_at
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('products');
    }
};

📌 Padanan Migration vs SQL Manual:
Migration Laravel SQL Equivalent
$table->id() id INT AUTO_INCREMENT PRIMARY KEY
$table->string('name') name VARCHAR(255) NOT NULL
$table->text('description')->nullable() description TEXT NULL
$table->decimal('price', 10, 2) price DECIMAL(10,2) NOT NULL
$table->integer('stock')->default(0) stock INT DEFAULT 0
$table->timestamps() created_at TIMESTAMP, updated_at TIMESTAMP

Langkah 4 — Buat Model

Model adalah class PHP yang merepresentasikan tabel database. Satu model = satu tabel.

💻 Terminal 
$ php artisan make:model Product

INFO  Model [app/Models/Product.php] created successfully.
📄 app/Models/Product.php 
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Product extends Model
{
    // Kolom yang boleh diisi via create() / update()
    protected $fillable = [
        'name',
        'description',
        'price',
        'stock',
    ];
}
📌 Apa itu $fillable?
Daftar kolom yang boleh diisi secara massal (mass assignment). Ini fitur keamanan Laravel — kolom yang tidak ada di $fillable tidak bisa diisi via create() meskipun datanya dikirim dari form.

Langkah 5 — Buat Controller

💻 Terminal 
$ php artisan make:controller ProductController --resource

INFO  Controller [app/Http/Controllers/ProductController.php] created successfully.

Flag --resource otomatis membuat 7 method standar CRUD sekaligus.

📄 app/Http/Controllers/ProductController.php 
<?php

namespace App\Http\Controllers;

use App\Models\Product;
use Illuminate\Http\Request;

class ProductController extends Controller
{
    // READ — tampilkan semua produk
    public function index()
    {
        $products = Product::latest()->get();
        return view('products.index', compact('products'));
    }

    // CREATE — tampilkan form tambah
    public function create()
    {
        return view('products.create');
    }

    // CREATE — simpan data baru ke database
    public function store(Request $request)
    {
        $request->validate([
            'name'  => 'required|max:255',
            'price' => 'required|numeric|min:0',
            'stock' => 'required|integer|min:0',
        ]);

        Product::create($request->only(['name', 'description', 'price', 'stock']));

        return redirect()->route('products.index')
                     ->with('success', 'Produk berhasil ditambahkan!');
    }

    // UPDATE — tampilkan form edit
    public function edit(Product $product)
    {
        return view('products.edit', compact('product'));
    }

    // UPDATE — simpan perubahan ke database
    public function update(Request $request, Product $product)
    {
        $request->validate([
            'name'  => 'required|max:255',
            'price' => 'required|numeric|min:0',
            'stock' => 'required|integer|min:0',
        ]);

        $product->update($request->only(['name', 'description', 'price', 'stock']));

        return redirect()->route('products.index')
                     ->with('success', 'Produk berhasil diperbarui!');
    }

    // DELETE — hapus data dari database
    public function destroy(Product $product)
    {
        $product->delete();

        return redirect()->route('products.index')
                     ->with('success', 'Produk berhasil dihapus!');
    }
}
📌 Route Model Binding:
Perhatikan parameter Product $product di method edit, update, destroy. Laravel otomatis mencari data berdasarkan id dari URL — tidak perlu tulis Product::find($id) manual. Kalau data tidak ditemukan, Laravel otomatis return 404.

Langkah 6 — Daftarkan Route Resource

Satu baris Route::resource otomatis membuat 7 route CRUD sekaligus.

📄 routes/web.php

// Satu baris ini = 7 route CRUD otomatis

7 Route yang Dihasilkan Otomatis

Method URL Action Fungsi
GET /products index Tampilkan semua produk
GET /products/create create Tampilkan form tambah
POST /products store Simpan data baru
GET /products/{id} show Tampilkan detail satu produk
GET /products/{id}/edit edit Tampilkan form edit
PUT /products/{id} update Simpan perubahan
DELETE /products/{id} destroy Hapus data

$ php artisan route:list --path=products

GET|HEAD   products/create ...... products.create › ProductController@create
POST       products ............. products.store › ProductController@store

PUT|PATCH  products/{product} ... products.update › ProductController@update

@method('PUT')

15. Asset (CSS, JS, Gambar) Tips

Menyimpan File Statis

Jenis Lokasi Akses di Blade
CSS / JS / Gambar (statis) public/css/, public/js/, public/img/ {{ asset('css/style.css') }}
CSS / JS (source, Vite) resources/css/, resources/js/ @vite(['resources/css/app.css'])

Contoh Penggunaan asset()

<!-- CSS dari public/css/style.css -->
<link rel="stylesheet" href="{{ asset('css/style.css') }}">

<!-- JS dari public/js/app.js -->
<script src="{{ asset('js/app.js') }}"></script>

<!-- Gambar dari public/img/logo.png -->
<img src="{{ asset('img/logo.png') }}" alt="Logo">

Key Fungsi

🔴 PERINGATAN KEAMANAN:

  • Set APP_DEBUG=false di production!
