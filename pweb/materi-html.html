1. Pengertian HTML Dasar

Apa itu HTML?

HTML (HyperText Markup Language) adalah bahasa markup standar yang digunakan untuk membuat dan menyusun halaman web. HTML bukan bahasa pemrograman, melainkan bahasa markup yang menggunakan tag untuk menandai elemen-elemen pada halaman web.

Poin Penting:
  • HyperText - Teks yang memiliki link ke teks lain
  • Markup - Sistem penandaan untuk mendefinisikan elemen
  • Language - Bahasa yang dipahami oleh browser

Sejarah Singkat HTML

Versi Tahun Keterangan
HTML 1.0 1991 Versi pertama oleh Tim Berners-Lee
HTML 2.0 1995 Standarisasi pertama
HTML 4.01 1999 Versi stabil sebelum HTML5
HTML5 2014 Versi terbaru dengan fitur modern

Fitur Baru HTML5

  • Tag semantic baru (<header>, <footer>, <article>, dll)
  • Dukungan multimedia (<video>, <audio>)
  • Canvas untuk grafik 2D/3D
  • Geolocation API
  • Web Storage (localStorage & sessionStorage)
  • Form input baru (date, email, range, dll)

2. Struktur Dasar HTML5 Wajib

Setiap dokumen HTML5 memiliki struktur dasar yang harus diikuti agar dapat ditampilkan dengan benar oleh browser.

Struktur Minimal HTML5

<!-- Deklarasi tipe dokumen HTML5 -->
<!DOCTYPE html>
<html lang="id">
<head>
    <!-- Encoding karakter -->
    <meta charset="UTF-8">
    
    <!-- Responsive viewport -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- Judul halaman -->
    <title>Judul Halaman</title>
</head>
<body>
    <!-- Konten halaman disini -->
    <h1>Selamat Datang!</h1>
    <p>Ini adalah paragraf pertama.</p>
</body>
</html>

Penjelasan Setiap Bagian

Elemen Fungsi
<!DOCTYPE html> Deklarasi bahwa dokumen ini adalah HTML5
<html> Elemen root yang membungkus seluruh konten
<head> Berisi metadata (tidak ditampilkan di halaman)
<meta charset> Mendefinisikan encoding karakter
<meta viewport> Pengaturan tampilan responsive
<title> Judul di tab browser
<body> Berisi konten yang ditampilkan
Perhatian: Setiap tag yang dibuka harus ditutup! Contoh: <p>...</p>. Tag self-closing seperti <img> dan <br> tidak perlu ditutup di HTML5.

4. Semantic HTML5 HTML5

Tag semantic memberikan makna pada konten, membantu browser, search engine, dan screen reader memahami struktur halaman dengan lebih baik.

Perbandingan Non-Semantic vs Semantic

Non-Semantic Semantic Fungsi
<div id="header"> <header> Bagian kepala halaman
<div id="nav"> <nav> Navigasi
<div id="main"> <main> Konten utama
<div class="article"> <article> Artikel mandiri
<div class="section"> <section> Bagian tematik
<div id="sidebar"> <aside> Konten sampingan
<div id="footer"> <footer> Bagian kaki halaman

Contoh Layout Semantic HTML5

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Layout Semantic</title>
</head>
<body>
    <!-- Header halaman -->
    <header>
        <h1>Nama Website</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
                <li><a href="#">Contact</a></li>
            </ul>
        </nav>
    </header>

    <!-- Konten utama -->
    <main>
        <article>
            <header>
                <h2>Judul Artikel</h2>
                <time datetime="2024-01-15">15 Januari 2024</time>
            </header>
            
            <section>
                <h3>Bagian 1</h3>
                <p>Konten bagian pertama...</p>
            </section>
            
            <section>
                <h3>Bagian 2</h3>
                <p>Konten bagian kedua...</p>
            </section>
            
            <footer>
                <p>Penulis: John Doe</p>
            </footer>
        </article>

        <!-- Sidebar -->
        <aside>
            <h3>Artikel Terkait</h3>
            <ul>
                <li><a href="#">Artikel 1</a></li>
                <li><a href="#">Artikel 2</a></li>
            </ul>
        </aside>
    </main>

    <!-- Footer halaman -->
    <footer>
        <p>© 2024 Nama Website. All rights reserved.</p>
    </footer>
</body>
</html>
Keuntungan Semantic HTML:
  • SEO lebih baik - search engine memahami konten
  • Aksesibilitas - screen reader bisa navigasi lebih baik
  • Maintainability - kode lebih mudah dibaca dan dipelihara
  • Standardisasi - mengikuti standar web modern

Struktur Dasar Form

    <label for="nama">Nama:</label>
    <input type="text" id="nama" name="nama">

    <fieldset>
        <legend>Informasi Tambahan</legend>

7. Method GET dan POST CPMK 2.3

Dalam HTML Form, ada dua method utama untuk mengirim data ke server: GET dan POST.

Perbandingan GET vs POST

Aspek GET POST
Data terlihat di URL Ya (sebagai query string) Tidak (tersembunyi di body)
Batas ukuran data Terbatas (~2048 karakter) Tidak terbatas
Keamanan Kurang aman Lebih aman
Dapat di-bookmark Ya Tidak
Cache Dapat di-cache Tidak di-cache
Idempoten Ya (tidak mengubah data) Tidak (dapat mengubah data)
Penggunaan Search, filter, navigasi Login, register, upload

Contoh Method GET

<!-- Form dengan method GET -->
<form action="search.php" method="GET">
    <label for="keyword">Cari:</label>
    <input type="text" id="keyword" name="q" placeholder="Kata kunci...">
    
    <select name="kategori">
        <option value="semua">Semua</option>
        <option value="artikel">Artikel</option>
        <option value="produk">Produk</option>
    </select>
    
    <button type="submit">Cari</button>
</form>

<!-- Hasil URL: search.php?q=html&kategori=artikel -->
ðŸ“Œ Kapan Menggunakan GET:
  • Form pencarian
  • Filter dan sorting data
  • Navigasi halaman (pagination)
  • Ketika data tidak sensitif

Contoh Method POST

<!-- Form dengan method POST -->
<form action="login.php" method="POST">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>
    
    <label for="password">Password:</label>
    <input type="password" id="password" name="password" required>
    
    <button type="submit">Login</button>
</form>

<!-- Data dikirim di HTTP body, tidak terlihat di URL -->

Form Upload File (POST dengan enctype)

<!-- Upload file HARUS menggunakan POST dan enctype -->
<form action="upload.php" method="POST" enctype="multipart/form-data">
    <label for="file">Pilih File:</label>
    <input type="file" id="file" name="dokumen">
    
    <button type="submit">Upload</button>
</form>
âš ï¸ Penting untuk Upload File:
  • Method harus POST
  • Tambahkan atribut enctype="multipart/form-data"
  • Input type harus file

8. Jenis Input Form HTML5 HTML5

HTML5 memperkenalkan banyak tipe input baru yang memudahkan validasi dan pengalaman pengguna.

Tabel Jenis Input

Type Keterangan Contoh
text Teks satu baris
password Input tersembunyi
email Validasi format email
number Input angka
tel Nomor telepon
url Validasi URL
date Pemilih tanggal
time Pemilih waktu
datetime-local Tanggal dan waktu
month Pemilih bulan
week Pemilih minggu
color Pemilih warna
range Slider
search Input pencarian
file Upload file
hidden Input tersembunyi (tidak terlihat)

Atribut Input Penting

<!-- Atribut validasi dan UX -->
<input type="text"
       name="username"
       id="username"
       placeholder="Masukkan username"
       required
       minlength="3"
       maxlength="20"
       pattern="[a-zA-Z0-9]+"
       autocomplete="username"
       autofocus
>

<!-- Input number dengan range -->
<input type="number"
       name="umur"
       min="17"
       max="100"
       step="1"
       value="20"
>

<!-- Input dengan datalist (autocomplete options) -->
<input type="text" list="browsers" name="browser">
<datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Safari">
    <option value="Edge">
</datalist>
Atribut Fungsi
required Wajib diisi
placeholder Teks petunjuk
value Nilai default
readonly Hanya baca
disabled Nonaktifkan input
minlength/maxlength Panjang karakter
min/max Nilai minimum/maksimum
pattern Validasi regex
autofocus Fokus otomatis saat load
autocomplete Saran dari browser
