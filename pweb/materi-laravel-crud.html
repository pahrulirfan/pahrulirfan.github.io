1. Overview & Persiapan Hands-On

Setelah bisa membuat halaman statik, sekarang kita buat aplikasi CRUD (Create, Read, Update, Delete) sederhana — yaitu manajemen data Produk. Ini adalah fondasi dari hampir semua aplikasi web nyata.

Yang akan dibuat:
  • Menampilkan daftar produk (Read)
  • Form tambah produk baru (Create)
  • Form edit produk (Update)
  • Hapus produk (Delete)

Tabel Database yang Akan Dibuat

Dalam praktek ini kita hanya menggunakan satu tabel bernama products. Tabel ini dibuat otomatis oleh Laravel melalui migration — tidak perlu buka phpMyAdmin untuk membuat tabel secara manual.

Nama Tabel Dibuat Via Model Laravel Keterangan
products php artisan migrate App\Models\Product Data produk (nama, harga, stok, deskripsi)

Struktur Kolom Tabel products

Kolom Tipe Data Keterangan Wajib?
id BIGINT UNSIGNED Primary key, auto increment Otomatis
name VARCHAR(255) Nama produk Ya
description TEXT Deskripsi produk Boleh kosong
price DECIMAL(10,2) Harga produk (contoh: 15000.00) Ya
stock INT Jumlah stok, default 0 Ya
created_at TIMESTAMP Waktu data dibuat Otomatis
updated_at TIMESTAMP Waktu data terakhir diubah Otomatis

File yang Akan Dibuat

Berikut daftar semua file yang akan kita buat selama praktek ini, beserta fungsinya masing-masing:

File Dibuat Via Fungsi
database/migrations/xxxx_create_products_table.php make:migration Mendefinisikan struktur tabel products
app/Models/Product.php make:model Model Eloquent untuk tabel products
app/Http/Controllers/ProductController.php make:controller --resource Controller dengan 7 method CRUD
routes/web.php Edit manual Mendaftarkan route resource untuk produk
resources/views/products/index.blade.php Buat manual Halaman daftar semua produk
resources/views/products/create.blade.php Buat manual Form tambah produk baru
resources/views/products/edit.blade.php Buat manual Form edit produk
Urutan Pengerjaan: Migration → Model → Controller → Route → View. Urutan ini penting karena setiap langkah bergantung pada langkah sebelumnya — Controller butuh Model, View butuh Controller, dst.

2. Migration, Model & Controller Langkah 1–6

Langkah 1 — Konfigurasi Database

Buka file .env, sesuaikan konfigurasi database:

📄 .env 
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=belajar_laravel
DB_USERNAME=root
DB_PASSWORD=
Pastikan: Database belajar_laravel sudah dibuat di phpMyAdmin / MySQL sebelum lanjut.

Langkah 2 — Buat Migration

Migration adalah cara Laravel membuat tabel database lewat kode PHP — pengganti membuat tabel manual di phpMyAdmin.

💻 Terminal 
$ php artisan make:migration create_products_table

INFO  Migration [database/migrations/2024_01_01_000000_create_products_table.php] created successfully.

Buka file migration yang baru dibuat, isi strukturnya:

📄 database/migrations/xxxx_create_products_table.php 
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('products', function (Blueprint $table) {
            $table->id();                                      // kolom id (auto increment)
            $table->string('name');                          // nama produk (VARCHAR)
            $table->text('description')->nullable();      // deskripsi (boleh kosong)
            $table->decimal('price', 10, 2);               // harga (DECIMAL)
            $table->integer('stock')->default(0);          // stok (default 0)
            $table->timestamps();                             // created_at & updated_at
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('products');
    }
};

Langkah 3 — Jalankan Migration

💻 Terminal 
$ php artisan migrate

INFO  Running migrations.
  2024_01_01_000000_create_products_table ..... 10ms DONE

Cek di phpMyAdmin — tabel products sudah terbuat otomatis dengan kolom sesuai migration.

Padanan Migration vs SQL Manual:
Migration Laravel SQL Equivalent
$table->id() id INT AUTO_INCREMENT PRIMARY KEY
$table->string('name') name VARCHAR(255) NOT NULL
$table->text('description')->nullable() description TEXT NULL
$table->decimal('price', 10, 2) price DECIMAL(10,2) NOT NULL
$table->integer('stock')->default(0) stock INT DEFAULT 0
$table->timestamps() created_at TIMESTAMP, updated_at TIMESTAMP

Langkah 4 — Buat Model

Model adalah class PHP yang merepresentasikan tabel database. Satu model = satu tabel.

💻 Terminal 
$ php artisan make:model Product

INFO  Model [app/Models/Product.php] created successfully.
📄 app/Models/Product.php 
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Product extends Model
{
    // Kolom yang boleh diisi via create() / update()
    protected $fillable = [
        'name',
        'description',
        'price',
        'stock',
    ];
}
Apa itu $fillable?
Daftar kolom yang boleh diisi secara massal (mass assignment). Ini fitur keamanan Laravel — kolom yang tidak ada di $fillable tidak bisa diisi via create() meskipun datanya dikirim dari form.

Langkah 5 — Buat Controller

💻 Terminal 
$ php artisan make:controller ProductController --resource

INFO  Controller [app/Http/Controllers/ProductController.php] created successfully.

Flag --resource otomatis membuat 7 method standar CRUD sekaligus.

📄 app/Http/Controllers/ProductController.php 
<?php

namespace App\Http\Controllers;

use App\Models\Product;
use Illuminate\Http\Request;

class ProductController extends Controller
{
    // READ — tampilkan semua produk
    public function index()
    {
        $products = Product::latest()->get();
        return view('products.index', compact('products'));
    }

    // CREATE — tampilkan form tambah
    public function create()
    {
        return view('products.create');
    }

    // CREATE — simpan data baru ke database
    public function store(Request $request)
    {
        $request->validate([
            'name'  => 'required|max:255',
            'price' => 'required|numeric|min:0',
            'stock' => 'required|integer|min:0',
        ]);

        Product::create($request->only(['name', 'description', 'price', 'stock']));

        return redirect()->route('products.index')
                     ->with('success', 'Produk berhasil ditambahkan!');
    }

    // UPDATE — tampilkan form edit
    public function edit(Product $product)
    {
        return view('products.edit', compact('product'));
    }

    // UPDATE — simpan perubahan ke database
    public function update(Request $request, Product $product)
    {
        $request->validate([
            'name'  => 'required|max:255',
            'price' => 'required|numeric|min:0',
            'stock' => 'required|integer|min:0',
        ]);

        $product->update($request->only(['name', 'description', 'price', 'stock']));

        return redirect()->route('products.index')
                     ->with('success', 'Produk berhasil diperbarui!');
    }

    // DELETE — hapus data dari database
    public function destroy(Product $product)
    {
        $product->delete();

        return redirect()->route('products.index')
                     ->with('success', 'Produk berhasil dihapus!');
    }
}
Route Model Binding:
Perhatikan parameter Product $product di method edit, update, destroy. Laravel otomatis mencari data berdasarkan id dari URL — tidak perlu tulis Product::find($id) manual. Kalau data tidak ditemukan, Laravel otomatis return 404.

Langkah 6 — Daftarkan Route Resource

Satu baris Route::resource otomatis membuat 7 route CRUD sekaligus.

📄 routes/web.php 
<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\ProductController;

// Satu baris ini = 7 route CRUD otomatis
Route::resource('products', ProductController::class);

7 Route yang Dihasilkan Otomatis

Method URL Action Fungsi
GET /products index Tampilkan semua produk
GET /products/create create Tampilkan form tambah
POST /products store Simpan data baru
GET /products/{id} show Tampilkan detail satu produk
GET /products/{id}/edit edit Tampilkan form edit
PUT /products/{id} update Simpan perubahan
DELETE /products/{id} destroy Hapus data
Cek route via terminal: 
$ php artisan route:list --path=products

GET|HEAD   products ............. products.index › ProductController@index
GET|HEAD   products/create ...... products.create › ProductController@create
POST       products ............. products.store › ProductController@store
GET|HEAD   products/{product} ... products.show › ProductController@show
GET|HEAD   products/{product}/edit products.edit › ProductController@edit
PUT|PATCH  products/{product} ... products.update › ProductController@update
DELETE     products/{product} ... products.destroy › ProductController@destroy
HTML form hanya support GET dan POST. Untuk PUT dan DELETE, Laravel pakai method spoofing via field tersembunyi di dalam form: 
<!-- Untuk UPDATE -->
@method('PUT')

<!-- Untuk DELETE -->
@method('DELETE')

3. View (Blade Template) Langkah 7–9

Langkah 7 — Buat View: index (Daftar Produk)

Buat folder resources/views/products/ lalu buat file-file berikut.

📄 resources/views/products/index.blade.php 
@extends('layouts.app')
@section('title', 'Daftar Produk')

@section('content')
<div class="d-flex justify-content-between align-items-center mb-3">
    <h2>Daftar Produk</h2>
    <a href="{{ route('products.create') }}" class="btn btn-primary">+ Tambah Produk</a>
</div>

<!-- Pesan sukses -->
@if(session('success'))
    <div class="alert alert-success">{{ session('success') }}</div>
@endif

<table class="table table-bordered table-hover">
    <thead class="table-dark">
        <tr>
            <th>#</th>
            <th>Nama Produk</th>
            <th>Harga</th>
            <th>Stok</th>
            <th>Aksi</th>
        </tr>
    </thead>
    <tbody>
        @forelse($products as $product)
        <tr>
            <td>{{ $loop->iteration }}</td>
            <td>{{ $product->name }}</td>
            <td>Rp {{ number_format($product->price, 0, ',', '.') }}</td>
            <td>{{ $product->stock }}</td>
            <td>
                <a href="{{ route('products.edit', $product) }}"
                   class="btn btn-sm btn-warning">Edit</a>
                <form action="{{ route('products.destroy', $product) }}"
                      method="POST" class="d-inline"
                      onsubmit="return confirm('Yakin hapus produk ini?')">
                    @csrf
                    @method('DELETE')
                    <button class="btn btn-sm btn-danger">Hapus</button>
                </form>
            </td>
        </tr>
        @empty
        <tr>
            <td colspan="5" class="text-center text-muted">Belum ada produk.</td>
        </tr>
        @endforelse
    </tbody>
</table>
@endsection

Dua fitur penting di form ini:
  • @error('field') — menampilkan pesan error validasi per field, otomatis dari $request->validate() di Controller
  • old('field') — mengisi ulang nilai input setelah validasi gagal, sehingga user tidak perlu ketik ulang dari awal

Perbedaan create vs edit:
  • Form action mengarah ke products.update bukan products.store
  • Wajib ada @method('PUT')
  • old('name', $product->name) — argumen kedua adalah nilai default dari database, tampil saat pertama buka form edit
