1. Pengenalan DBMS CPMK 6.1

Apa itu DBMS?

DBMS (Database Management System) adalah perangkat lunak yang digunakan untuk membuat, mengelola, dan memanipulasi database. DBMS menjadi penghubung antara pengguna/aplikasi dengan data yang tersimpan.

Contoh DBMS dan Perbandingan

DBMS Tipe Kelebihan Digunakan Oleh
MySQL Relasional (SQL) Gratis, cepat, komunitas besar, mudah dipelajari WordPress, Facebook (awal), YouTube
PostgreSQL Relasional (SQL) Fitur lengkap, ACID compliance, extensible Instagram, Spotify
MariaDB Relasional (SQL) Fork MySQL, lebih cepat, open source penuh Google, Wikipedia
SQLite Relasional (SQL) Tanpa server, file-based, ringan Android, iOS, Browser
MongoDB NoSQL (Document) Fleksibel, skema dinamis, horizontal scaling Uber, eBay
SQL Server Relasional (SQL) Integrasi Microsoft, enterprise-grade Perusahaan besar
Mengapa MySQL?
  • Gratis dan open source
  • Sudah termasuk dalam paket XAMPP
  • Paling banyak digunakan dengan PHP
  • Mudah dipelajari untuk pemula
  • Dokumentasi dan komunitas sangat besar

3. SQL Dasar CPMK 6.3

Apa itu SQL?

SQL (Structured Query Language) adalah bahasa khusus yang digunakan untuk berkomunikasi dengan database. Bayangkan database seperti lemari arsip raksasa — SQL adalah cara kita memberi perintah: "ambilkan data ini", "tambahkan data baru", "ubah data yang salah", atau "hapus data yang tidak diperlukan".

Analogi Sederhana:
  • Tabel = seperti lembar spreadsheet Excel (baris & kolom)
  • Baris (row) = satu data/record (misal: satu produk)
  • Kolom (column) = atribut data (misal: id, merek, produsen)
  • Query = perintah yang kita kirim ke database

Perintah SQL CRUD

Semua operasi data pada database bisa diringkas menjadi 4 aksi utama yang disebut CRUD:

Operasi SQL Keterangan Analogi
CREATE INSERT INTO Menambah data baru Menulis baris baru di spreadsheet
READ SELECT Membaca/menampilkan data Membuka dan melihat isi spreadsheet
UPDATE UPDATE Mengubah data yang ada Mengedit isi sel di spreadsheet
DELETE DELETE Menghapus data Menghapus baris dari spreadsheet

SELECT — Mengambil Data

Perintah paling sering digunakan. Struktur dasarnya: SELECT [kolom] FROM [tabel] WHERE [kondisi]

-- Ambil SEMUA kolom dari tabel produk
SELECT * FROM produk;
-- Tanda * artinya "semua kolom"

-- Ambil kolom tertentu saja (lebih efisien)
SELECT merek, produsen FROM produk;

-- Ambil data dengan kondisi (WHERE)
-- Hanya tampilkan produk dari Indofood
SELECT * FROM produk WHERE produsen = 'Indofood';

-- LIKE digunakan untuk pencarian sebagian kata
-- % artinya "karakter apa saja" (wildcard)
SELECT * FROM produk WHERE merek LIKE '%indo%';
-- Akan menemukan: Indomie, Indofood, dll

-- ORDER BY untuk mengurutkan hasil
SELECT * FROM produk ORDER BY merek ASC;
-- ASC = A-Z (ascending), DESC = Z-A (descending)

-- COUNT untuk menghitung jumlah data
SELECT COUNT(*) AS total FROM produk;
-- AS total = memberi nama alias pada hasil COUNT

INSERT — Menambah Data Baru

Struktur: INSERT INTO [tabel] ([kolom1], [kolom2]) VALUES ([nilai1], [nilai2])

Kolom id tidak perlu diisi karena sudah AUTO_INCREMENT (otomatis bertambah).

-- Tambah satu data produk baru
INSERT INTO produk (merek, produsen)
VALUES ('Oreo', 'Mondelez');
-- id akan otomatis diisi oleh MySQL (AUTO_INCREMENT)

-- Tambah beberapa data sekaligus
INSERT INTO produk (merek, produsen) VALUES
    ('Milo', 'Nestle'),
    ('Ovaltine', 'Associated British Foods'),
    ('Energen', 'Kalbe');

UPDATE — Mengubah Data

Struktur: UPDATE [tabel] SET [kolom] = [nilai_baru] WHERE [kondisi]

-- Ubah merek dan produsen pada produk dengan id = 1
UPDATE produk
SET merek = 'Indomie Goreng', produsen = 'Indofood CBP'
WHERE id = 1;
-- WHERE id = 1 → hanya ubah baris yang id-nya 1

-- Ubah hanya satu kolom
UPDATE produk
SET produsen = 'PT Indofood'
WHERE produsen = 'Indofood';

DELETE — Menghapus Data

Struktur: DELETE FROM [tabel] WHERE [kondisi]

-- Hapus produk dengan id = 5
DELETE FROM produk WHERE id = 5;

-- Hapus semua produk dari produsen tertentu
DELETE FROM produk WHERE produsen = 'Mondelez';
⚠️ PERINGATAN PENTING: Selalu gunakan WHERE pada UPDATE dan DELETE!

Tanpa WHERE, perintah akan berlaku ke SEMUA baris di tabel:
  • UPDATE produk SET produsen = 'X' → semua produsen berubah jadi 'X'
  • DELETE FROM produk → semua data terhapus permanen!
Selalu cek query Anda sebelum dijalankan!

Tipe Data Kolom yang Umum Digunakan

Tipe Data Kegunaan Contoh
INT Angka bulat id, umur, stok
VARCHAR(n) Teks pendek (maks n karakter) nama, email, username
TEXT Teks panjang deskripsi, komentar
DECIMAL(m,d) Angka desimal harga (10,2) = 99999999.99
DATE Tanggal tanggal_lahir (YYYY-MM-DD)
TIMESTAMP Tanggal + waktu created_at, updated_at
BOOLEAN True/False (0 atau 1) is_active, is_admin

4. Koneksi PHP - MySQL CPMK 6.4

Bagaimana PHP Terhubung ke MySQL?

PHP tidak bisa langsung "membaca" database. Kita perlu membuat koneksi terlebih dahulu — seperti membuka pintu sebelum masuk ke dalam ruangan. PHP menyediakan dua cara untuk koneksi ke MySQL:

PHP
Kode kita
🔌 koneksi
MySQL
Database

Cara 1: MySQLi (Prosedural)

Cara lama yang masih banyak dipakai. Cocok untuk pemula karena sintaksnya lebih sederhana, tapi hanya bisa digunakan dengan MySQL.

koneksi_prosedural.php 
<?php
// Isi sesuai konfigurasi XAMPP Anda
$host   = "localhost";  // server database (biasanya localhost)
$user   = "root";       // username MySQL (default XAMPP: root)
$pass   = "";          // password MySQL (default XAMPP: kosong)
$dbname = "toko_db";   // nama database yang ingin digunakan

// Buat koneksi
$conn = mysqli_connect($host, $user, $pass, $dbname);

// Cek apakah koneksi berhasil
if (!$conn) {
    die("Koneksi gagal: " . mysqli_connect_error());
    // die() = hentikan script dan tampilkan pesan error
}

echo "Koneksi berhasil!";
?>

Cara 2: PDO — PHP Data Objects (Direkomendasikan)

Cara modern yang lebih aman dan fleksibel. PDO menggunakan pendekatan OOP (Object-Oriented Programming) dan mendukung banyak jenis database, bukan hanya MySQL. Kita akan menggunakan PDO di seluruh materi ini.

config/database.php 
<?php
// Konfigurasi koneksi
$host   = "localhost";
$dbname = "toko_db";
$user   = "root";
$pass   = "";

// try-catch: coba koneksi, kalau gagal tangkap errornya
try {
    $pdo = new PDO(
        // DSN (Data Source Name): format "driver:host=...;dbname=...;charset=..."
        "mysql:host=$host;dbname=$dbname;charset=utf8mb4",
        $user,
        $pass,
        [
            // Tampilkan error sebagai Exception (bukan diam-diam gagal)
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
            // Hasil fetch otomatis berupa array asosiatif ['kolom' => 'nilai']
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC
        ]
    );
} catch (PDOException $e) {
    // Jika koneksi gagal, tampilkan pesan error dan hentikan script
    die("Koneksi gagal: " . $e->getMessage());
}
// Variabel $pdo sekarang siap digunakan di file lain
?>

Apa itu Prepared Statement?

Prepared Statement adalah cara aman mengirim data ke database. Tanpa prepared statement, kode kita rentan terhadap SQL Injection — serangan di mana hacker menyisipkan perintah SQL berbahaya melalui form input.

Cara Contoh Keamanan
❌ Langsung (berbahaya) "SELECT * FROM users WHERE username = '$input'" Rentan SQL Injection
✅ Prepared Statement (aman) "SELECT * FROM users WHERE username = :username" Aman dari SQL Injection 
<?php
// ❌ CARA BERBAHAYA — jangan lakukan ini!
$username = $_POST['username']; // misal hacker isi: ' OR '1'='1
$query = "SELECT * FROM users WHERE username = '$username'";
// Query jadi: SELECT * FROM users WHERE username = '' OR '1'='1'
// Hacker bisa masuk tanpa password!

// ✅ CARA AMAN — gunakan Prepared Statement
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = :username");
// :username adalah placeholder, bukan nilai asli

$stmt->execute([':username' => $username]);
// PDO akan otomatis "membersihkan" nilai $username sebelum dikirim ke MySQL

$user = $stmt->fetch();
// fetch() = ambil satu baris hasil query sebagai array
?>

Perbandingan MySQLi vs PDO

Fitur MySQLi PDO
Database Support MySQL saja 12+ database (MySQL, PostgreSQL, SQLite, dll)
Prepared Statement ✅ Ada ✅ Ada
Named Parameters ❌ Tidak ada ✅ Ada (:nama)
OOP Support ✅ Ada (tapi opsional) ✅ Ada (wajib OOP)
Error Handling Manual try-catch (lebih rapi)
Rekomendasi Boleh untuk pemula ✅ Direkomendasikan
🔐 Keamanan: Selalu gunakan Prepared Statement untuk mencegah SQL Injection! Jangan pernah masukkan input user langsung ke dalam string query SQL.

5. Struktur Project CRUD CPMK 6.5

Mengapa Perlu Struktur yang Rapi?

Bayangkan membangun rumah tanpa denah — semua bahan ditumpuk sembarangan. Hasilnya pasti berantakan dan sulit diperbaiki. Hal yang sama berlaku untuk kode. Struktur folder yang baik membuat kode lebih mudah dibaca, dikelola, dan dikembangkan.

Prinsip Dasar:
  • Satu file, satu tugas — create.php hanya untuk tambah data, delete.php hanya untuk hapus
  • Pisahkan konfigurasi — koneksi database di folder config/ agar mudah diubah
  • Jangan duplikasi kode — koneksi database cukup ditulis sekali, lalu di-require di file lain

Struktur Folder Project

📁 crud_produk/
├── 📁 config/
│ └── database.php   ← koneksi database (dipakai semua file)
├── index.php        ← READ: tampil semua data produk
├── create.php       ← CREATE: form + proses tambah data
├── edit.php         ← UPDATE: form + proses edit data
├── delete.php       ← DELETE: proses hapus data
├── register.php     ← Auth: form + proses registrasi user
├── login.php        ← Auth: form + proses login
└── logout.php       ← Auth: hapus session & redirect

Alur Kerja Aplikasi CRUD

Setiap kali user melakukan aksi di browser, ada alur yang terjadi di balik layar:

1. Browser
User klik tombol
→ HTTP Request
2. PHP
Proses logika
↔ SQL Query
3. MySQL
Simpan/ambil data
→ Hasil
4. Browser
Tampilkan HTML

Contoh Alur: User Tambah Data Produk

Langkah Yang Terjadi File
1 User klik tombol "Tambah Produk" index.php
2 Browser buka halaman form tambah create.php
3 User isi form dan klik "Simpan" create.php (form)
4 PHP terima data dari $_POST, validasi input create.php (proses)
5 PHP jalankan query INSERT INTO via PDO create.php + config/database.php
6 MySQL simpan data, PHP redirect ke halaman utama index.php?pesan=tambah_sukses
7 Halaman utama tampil dengan notifikasi sukses index.php

Cara Menggunakan File Koneksi di File Lain

Kita tidak perlu menulis ulang kode koneksi di setiap file. Cukup gunakan require_once di baris paling atas:

config/database.php 
<?php
$host   = "localhost";
$dbname = "toko_db";
$user   = "root";
$pass   = "";

try {
    $pdo = new PDO(
        "mysql:host=$host;dbname=$dbname;charset=utf8mb4",
        $user,
        $pass,
        [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC
        ]
    );
} catch (PDOException $e) {
    die("Koneksi gagal: " . $e->getMessage());
}
?>
index.php / create.php / edit.php / dll 
<?php
// Baris pertama di setiap file PHP yang butuh database:
require_once 'config/database.php';
// Setelah ini, variabel $pdo langsung bisa digunakan!

// Contoh langsung query setelah require:
$stmt = $pdo->query("SELECT * FROM produk");
$data = $stmt->fetchAll();
// fetchAll() = ambil semua baris hasil query sebagai array
?>
Perbedaan require vs include:
  • require_once → wajib ada, script berhenti jika file tidak ditemukan. Gunakan untuk file penting seperti koneksi database.
  • include → opsional, script tetap jalan meski file tidak ada (hanya warning).
  • Akhiran _once → memastikan file hanya di-load satu kali meski dipanggil berkali-kali.

Konsep Penting di CREATE

Konsep Penjelasan
$_SERVER['REQUEST_METHOD'] Cek apakah request dari browser adalah GET (buka halaman) atau POST (submit form)
$_POST['nama_field'] Mengambil data yang dikirim dari form. Nama field harus sama dengan atribut name di HTML
trim() Menghapus spasi di awal dan akhir string. Mencegah input " " (spasi) dianggap valid
empty() Cek apakah variabel kosong. Validasi sederhana sebelum data disimpan
header("Location: ...") Redirect browser ke halaman lain setelah proses selesai
exit Hentikan eksekusi PHP setelah redirect. Wajib dipasang setelah header()!

    $merek    = trim($_POST['merek'] ?? '');

            ':merek'    => $merek,
            ':produsen' => $produsen
        ]);

        exit;
    } else {
        $pesan = "Semua field wajib diisi!";
    }
}

7. READ - Tampil Data READ

Bagaimana Cara Kerja READ?

Halaman index.php adalah halaman utama aplikasi. Tugasnya: ambil semua data dari database, lalu tampilkan dalam bentuk tabel HTML. Ini adalah operasi yang paling sering dijalankan.

Konsep Penting di READ

Konsep Penjelasan

fetchAll() Mengambil semua baris hasil query sekaligus, disimpan sebagai array. Cocok untuk menampilkan daftar data
foreach Perulangan untuk menampilkan setiap baris data satu per satu ke dalam tabel HTML

8. UPDATE - Edit Data UPDATE

Bagaimana Cara Kerja UPDATE?

UPDATE adalah operasi paling kompleks di antara CRUD karena melibatkan tiga tahap:

  1. Terima ID dari URL (edit.php?id=3)
  2. Ambil data lama dari database untuk ditampilkan di form
  3. Proses data baru saat form disubmit → jalankan UPDATE
Klik Edit
edit.php?id=3
→ SELECT data lama
Form terisi
data lama
→ User ubah & submit
UPDATE
di database
→ redirect
index.php
notif sukses

Konsep Penting di UPDATE

Konsep Penjelasan
$_GET['id'] Mengambil ID dari URL. Misal URL edit.php?id=3 → $id = 3
fetch() Mengambil satu baris hasil query. Berbeda dengan fetchAll() yang mengambil semua baris
Cek if (!$data) Jika ID tidak ditemukan di database (misal user iseng ubah URL), redirect ke index. Ini penting untuk keamanan
value="<?= $data['merek'] ?>" Mengisi form dengan data lama dari database, sehingga user tidak perlu mengetik ulang dari awal
ID di $_GET vs $_POST ID diambil dari URL ($_GET) bukan dari form, karena ID tidak boleh diubah oleh user
edit.php 
<?php
require_once 'config/database.php';
$pesan = '';

// Ambil data berdasarkan ID
$id = $_GET['id'] ?? 0;

Metode Keamanan Keterangan
md5() Tidak Aman Mudah di-crack, tidak ada salt

// Setiap kali dipanggil hasilnya BERBEDA (karena random salt)

// === VERIFY PASSWORD (saat Login) ===
$input_password = "rahasia123";
$hash_dari_db   = '$2y$10$...'; // dari database

11. Register (Bcrypt) CPMK 6.7

register.php 
<?php
require_once 'config/database.php';
$pesan = '';
$tipe  = '';

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $username = trim($_POST['username'] ?? '');
    $password = $_POST['password'] ?? '';
    $confirm  = $_POST['confirm'] ?? '';

    // Validasi
    if (empty($username) || empty($password)) {
        $pesan = "Username dan password wajib diisi!";
        $tipe  = "danger";
    } elseif (strlen($password) < 6) {
        $pesan = "Password minimal 6 karakter!";
        $tipe  = "danger";
    } elseif ($password !== $confirm) {
        $pesan = "Konfirmasi password tidak cocok!";
        $tipe  = "danger";
    } else {
        // Cek apakah username sudah ada
        $stmt = $pdo->prepare("SELECT id FROM users WHERE username = :username");
        $stmt->execute([':username' => $username]);

        if ($stmt->fetch()) {
            $pesan = "Username sudah digunakan!";
            $tipe  = "danger";
        } else {
            // Hash password dengan bcrypt
            $hash = password_hash($password, PASSWORD_BCRYPT);

            // Simpan ke database
            $stmt = $pdo->prepare("INSERT INTO users (username, password) VALUES (:username, :password)");
            $stmt->execute([
                ':username' => $username,
                ':password' => $hash
            ]);

            $pesan = "Register berhasil! Silakan login.";
            $tipe  = "success";
        }
    }
}

12. Login & Session CPMK 6.7

login.php 
<?php
session_start();
require_once 'config/database.php';

// Jika sudah login, redirect ke index
if (isset($_SESSION['user_id'])) {
    header("Location: index.php");
    exit;
}

$pesan = '';

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $username = trim($_POST['username'] ?? '');
    $password = $_POST['password'] ?? '';

    if (!empty($username) && !empty($password)) {
        // Cari user di database
        $stmt = $pdo->prepare("SELECT * FROM users WHERE username = :username");
        $stmt->execute([':username' => $username]);
        $user = $stmt->fetch();

        if ($user && password_verify($password, $user['password'])) {

14. Logout CPMK 6.7

logout.php 
<?php
session_start();

// Hapus semua data session
$_SESSION = [];

// Hapus cookie session
if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(
        session_name(), '',
        time() - 42000,
        $params["path"],
        $params["domain"],
        $params["secure"],
        $params["httponly"]
    );
}

// Hancurkan session
session_destroy();

// Redirect ke halaman login
header("Location: login.php");
exit;
?>
Sistem Auth Lengkap!
  • Register → password di-hash dengan password_hash() (bcrypt)
  • Login → password diverifikasi dengan password_verify()
  • Session → status login disimpan di $_SESSION
  • Proteksi → halaman dicek session sebelum ditampilkan
  • Logout → session dihancurkan dan redirect ke login
