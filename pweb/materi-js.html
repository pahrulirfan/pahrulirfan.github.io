1. Pengertian JavaScript CPMK 4.1

Apa itu JavaScript?

JavaScript adalah bahasa pemrograman yang digunakan untuk membuat halaman web menjadi interaktif dan dinamis. JavaScript berjalan di sisi client (browser) dan dapat memanipulasi konten HTML, merespons aksi pengguna, dan berkomunikasi dengan server.

Jika HTML diibaratkan sebagai kerangka rumah dan CSS sebagai cat serta dekorasi, maka JavaScript adalah listrik, tombol, sensor, dan sistem otomatis yang membuat rumah itu benar-benar hidup. Tanpa JavaScript, halaman web cenderung statis. Dengan JavaScript, website bisa merespons klik, menghitung data, memvalidasi form, menampilkan notifikasi, sampai mengambil data dari server tanpa reload penuh.

Fakta JavaScript:
  • Dibuat oleh Brendan Eich di Netscape tahun 1995
  • Awalnya bernama LiveScript, lalu diubah jadi JavaScript
  • JavaScript ≠ Java (bahasa berbeda!)
  • Standar resmi: ECMAScript (ES6/ES2015 adalah versi populer)
  • Dapat berjalan di browser dan server (Node.js)

Mengapa Mahasiswa Web Perlu Belajar JavaScript?

Di pengembangan web modern, JavaScript hampir selalu muncul. Saat kalian membuat form pendaftaran, kalkulator sederhana, validasi login, dashboard data, menu interaktif, atau halaman yang mengambil data dari API, JavaScript biasanya menjadi penghubung utamanya. Karena itu, memahami JavaScript bukan hanya belajar sintaks, tetapi belajar cara membuat web yang benar-benar bekerja.

Kebutuhan di Web Peran JavaScript
Form interaktif Memeriksa input kosong, format email, angka, password, dan memberi umpan balik langsung
Halaman dinamis Mengubah isi halaman tanpa harus reload terus-menerus
Integrasi API Mengambil data cuaca, berita, produk, jadwal, dan data lain dari server
UI modern Membuat modal, tab, dropdown, filter pencarian, carousel, dan notifikasi

Perbedaan HTML, CSS, dan JavaScript

Tiga teknologi ini hampir selalu digunakan bersama. Mahasiswa sering mengenalnya secara terpisah, padahal dalam praktik nyata ketiganya saling melengkapi.

Teknologi Fungsi Utama Contoh Peran
HTML Menyusun struktur dan isi halaman Membuat judul, paragraf, gambar, form, tabel, tombol
CSS Mengatur tampilan visual Warna, ukuran font, layout, jarak, animasi, responsif
JavaScript Mengatur logika dan interaksi Klik tombol, validasi, hitung nilai, ubah konten, ambil data
Contoh sederhana:
  • HTML membuat tombol Hitung Nilai
  • CSS membuat tombol itu terlihat rapi dan menarik
  • JavaScript menjalankan logika saat tombol diklik dan menampilkan hasil

Tren Pengembangan Web Modern dengan JavaScript

JavaScript saat ini tidak hanya dipakai untuk efek kecil di browser. Ekosistemnya berkembang sangat besar dan menjadi salah satu fondasi utama industri software modern.

Bidang Contoh Penggunaan JavaScript
Frontend Web Membangun antarmuka interaktif dengan JavaScript murni atau framework seperti React, Vue, dan Angular
Backend Membuat server dan API dengan Node.js
Mobile App Mengembangkan aplikasi mobile lintas platform dengan React Native atau Ionic
Desktop App Membuat aplikasi desktop dengan Electron, misalnya VS Code dan Discord
Full Stack Satu bahasa dipakai dari frontend sampai backend sehingga alur belajar lebih konsisten
Mengapa ini menarik untuk dipelajari?
  • Satu bahasa bisa dipakai di banyak platform
  • Ekosistem library dan framework sangat besar
  • Peluang kerja dan proyek freelance banyak menggunakan JavaScript
  • Cocok untuk membangun prototipe dengan cepat

Dari Halaman Statis ke Aplikasi Web

Dulu banyak website hanya menampilkan informasi. Sekarang banyak website berperan seperti aplikasi: bisa login, mengelola data, melakukan pencarian real-time, chat, menampilkan dashboard, bahkan menjalankan editor dokumen di browser. Perubahan besar ini sangat erat dengan peran JavaScript.

Karena itu, saat kalian belajar JavaScript, sebenarnya kalian sedang belajar dasar untuk masuk ke tahap berikutnya: membangun aplikasi web, bukan sekadar halaman web.

2. Cara Penulisan JavaScript CPMK 4.1

3 Cara Menyisipkan JavaScript

1️⃣ Inline JavaScript (di atribut event)

<!-- Tidak direkomendasikan -->
<button onclick="alert('Hello!')">Klik Saya</button>

<a href="#" onclick="console.log('Link diklik')">Link</a>
Tidak Direkomendasikan: Inline JavaScript mencampur HTML dengan JavaScript, sulit di-maintain.

2️⃣ Internal JavaScript (di dalam tag <script>)

<!DOCTYPE html>
<html>
<head>
    <title>Internal JS</title>
</head>
<body>
    <h1 id="judul">Hello World</h1>
    
    <!-- JavaScript di akhir body -->
    <script>
        // Kode JavaScript di sini
        let nama = "Mahasiswa";
        console.log("Halo, " + nama);
        
        // Mengubah konten HTML
        document.getElementById("judul").innerHTML = "Selamat Belajar!";
    </script>
</body>
</html>

3️⃣ External JavaScript (file terpisah) DIREKOMENDASIKAN

File: script.js

// File: script.js

// Variabel
let appName = "My Website";

// Function
function greet(name) {
    return "Halo, " + name + "!";
}

// Event saat DOM loaded
document.addEventListener("DOMContentLoaded", function() {
    console.log("Website siap!");
});

File: index.html

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>External JS</title>
</head>
<body>
    <h1>Belajar JavaScript</h1>
    
    <!-- External JavaScript (di akhir body) -->
    <script src="script.js"></script>
</body>
</html>

Posisi Tag <script>

Posisi Keterangan Kapan Digunakan
Di <head> Dieksekusi sebelum body di-render Jika pakai defer atau async
Di akhir <body> Dieksekusi setelah HTML selesai dimuat Pilihan paling aman dan umum

Atribut defer dan async

Kedua atribut ini digunakan saat script diletakkan di <head>. Tanpa atribut apapun, browser akan berhenti memproses HTML saat menemukan tag <script>.

Atribut Download Eksekusi Cocok untuk
<script> biasa HTML berhenti, download JS Langsung, HTML belum selesai Script di akhir <body>
defer Paralel (HTML tetap jalan) Setelah seluruh HTML selesai di-parse Script yang butuh akses DOM
async Paralel (HTML tetap jalan) Langsung setelah download selesai Script independen (analytics, ads)
Analogi sederhana:
  • Tanpa atribut — seperti antri di kasir, semua berhenti sampai selesai
  • defer — ambil nomor antrian, HTML terus jalan, JS dieksekusi setelah HTML beres. Urutan script tetap terjaga.
  • async — download sambil jalan, tapi langsung dieksekusi begitu selesai download. Urutan tidak dijamin.
Kapan pakai yang mana?
Gunakan defer untuk hampir semua kasus — script yang mengakses elemen HTML (getElementById, querySelector, dll) harus pakai defer atau diletakkan di akhir <body>. Gunakan async hanya untuk script yang tidak bergantung pada HTML, seperti Google Analytics. 
<!-- Tanpa atribut di <head>: HTML berhenti saat script didownload -->
<script src="app.js"></script>

<!-- defer: download paralel, eksekusi SETELAH HTML selesai di-parse -->
<script src="app.js" defer></script>

<!-- ⚡ async: download paralel, eksekusi SEGERA setelah download selesai -->
<script src="analytics.js" async></script>

Output JavaScript

Ada 5 cara menampilkan output di JavaScript. Berikut contoh lengkap masing-masing — simpan sebagai file .html dan buka di browser:

1. console.log() — untuk debugging

<!DOCTYPE html>
<html lang="id">
<head><meta charset="UTF-8"><title>console.log</title></head>
<body>
    <button onclick="coba()">Coba console.log</button>
    <script>
        function coba() {
            console.log("Pesan biasa");
            console.warn("Ini pesan warning");
            console.error("Ini pesan error");
        }
    </script>
</body>
</html>

Buka DevTools (F12 → Console) untuk melihat hasilnya.

2. alert() — popup pesan

<!DOCTYPE html>
<html lang="id">
<head><meta charset="UTF-8"><title>alert</title></head>
<body>
    <button onclick="tampilAlert()">Tampilkan Alert</button>
    <script>
        function tampilAlert() {
            alert("Halo! Ini adalah pesan alert.");
        }
    </script>
</body>
</html>

3. confirm() — popup OK / Cancel

<!DOCTYPE html>
<html lang="id">
<head><meta charset="UTF-8"><title>confirm</title></head>
<body>
    <button onclick="tanya()">Tampilkan Confirm</button>
    <p id="hasil"></p>
    <script>
        function tanya() {
            let jawaban = confirm("Apakah Anda setuju?");
            // jawaban bernilai true (OK) atau false (Cancel)
            if (jawaban) {
                document.getElementById("hasil").innerHTML = "Anda memilih OK";
            } else {
                document.getElementById("hasil").innerHTML = "Anda memilih Cancel";
            }
        }
    </script>
</body>
</html>

4. prompt() — popup input dari user

<!DOCTYPE html>
<html lang="id">
<head><meta charset="UTF-8"><title>prompt</title></head>
<body>
    <button onclick="tanyaNama()">Tanya Nama</button>
    <p id="salam"></p>
    <script>
        function tanyaNama() {
            let nama = prompt("Siapa nama Anda?");
            // nama berisi teks yang diketik user, atau null jika Cancel
            if (nama) {
                document.getElementById("salam").innerHTML = "Halo, <strong>" + nama + "</strong>!";
            }
        }
    </script>
</body>
</html>

5. innerHTML — mengubah konten elemen HTML paling sering dipakai

<!DOCTYPE html>
<html lang="id">
<head><meta charset="UTF-8"><title>innerHTML</title></head>
<body>
    <p id="teks">Teks awal</p>
    <button onclick="ubahTeks()">Ubah Teks</button>
    <button onclick="kembalikan()">Kembalikan</button>
    <script>
        function ubahTeks() {
            document.getElementById("teks").innerHTML = "<strong>Teks sudah diubah!</strong>";
        }
        function kembalikan() {
            document.getElementById("teks").innerHTML = "Teks awal";
        }
    </script>
</body>
</html>
Catatan: document.write() tidak direkomendasikan karena akan menimpa seluruh halaman jika dipanggil setelah halaman selesai dimuat. Gunakan innerHTML sebagai gantinya.

3. Variabel & Tipe Data CPMK 4.2

Contoh Awal: HTML Lengkap Variabel & Tipe Data

Contoh berikut menunjukkan bagaimana variabel dibuat di JavaScript lalu hasilnya ditampilkan ke halaman HTML. Ini adalah pola awal yang sangat umum saat belajar JavaScript dasar.

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Variabel dan Tipe Data</title>
</head>
<body>
    <h2>Belajar Variabel JavaScript</h2>
    <p id="output"></p>

    <script>
        let nama = "Andi";
        let umur = 20;
        let aktif = true;

        document.getElementById("output").innerHTML =
            `Nama: ${nama}<br>Umur: ${umur}<br>Aktif: ${aktif}`;
    </script>
</body>
</html>
Apa yang terjadi?
  1. HTML menyiapkan elemen paragraf dengan id output.
  2. JavaScript membuat beberapa variabel: string, number, dan boolean.
  3. Nilai variabel tersebut kemudian ditampilkan ke halaman dengan innerHTML.

Apa itu Variabel dan Tipe Data?

Variabel adalah tempat untuk menyimpan data. Data yang disimpan itu bisa berupa teks, angka, nilai benar atau salah, daftar data, dan lain-lain. Jenis isi yang disimpan oleh variabel disebut tipe data.

Saat menulis program, kita hampir selalu menyimpan input, hasil perhitungan, atau status tertentu ke dalam variabel. Karena itu, memahami variabel dan tipe data adalah fondasi untuk materi setelahnya seperti operator, percabangan, function, dan array.

Deklarasi Variabel

// var - cara lama (hindari)
var nama = "John";

// let - variabel yang bisa diubah 
let umur = 25;
umur = 26; // bisa diubah

// const - konstanta (tidak bisa diubah) 
const PI = 3.14159;
// PI = 3.14; // Error!

// Aturan penamaan variabel:
let namaDepan = "John";     // camelCase (direkomendasikan)
let nama_belakang = "Doe";  // snake_case
let $harga = 1000;          // boleh pakai $
let _private = true;        // boleh pakai _
// let 1nama = "salah";      // tidak boleh dimulai angka

Perbedaan var, let, dan const

Fitur var let const
Scope Function scope Block scope Block scope
Re-assign Bisa Bisa Tidak bisa
Re-declare Bisa Tidak bisa Tidak bisa
Hoisting Ya (undefined) Tidak Tidak
Rekomendasi Hindari Gunakan Gunakan

Tipe Data Primitif

// 1. String (teks)
let nama = "John Doe";
let pesan = 'Hello World';
let template = `Halo, ${nama}!`; // template literal

// 2. Number (angka - integer & float)
let umur = 25;
let harga = 99.99;
let negatif = -10;

// 3. Boolean (true/false)
let aktif = true;
let selesai = false;

// 4. Undefined (belum diberi nilai)
let x;
console.log(x); // undefined

// 5. Null (kosong/tidak ada nilai)
let data = null;

// 6. Symbol (ES6 - nilai unik)
let id = Symbol("id");

// 7. BigInt (angka sangat besar)
let bigNumber = 9007199254740991n;

Tipe Data Non-Primitif (Reference)

// 1. Object
let mahasiswa = {
    nama: "John",
    umur: 20,
    jurusan: "Informatika"
};

// 2. Array
let buah = ["Apel", "Jeruk", "Mangga"];
let campuran = [1, "dua", true, null];

// 3. Function
let sapa = function(nama) {
    return "Halo, " + nama;
};

Mengecek Tipe Data

let a = "Hello";
let b = 123;
let c = true;
let d = [1, 2, 3];
let e = {nama: "John"};

console.log(typeof a); // "string"
console.log(typeof b); // "number"
console.log(typeof c); // "boolean"
console.log(typeof d); // "object" (array adalah object)
console.log(typeof e); // "object"

// Cek apakah array
console.log(Array.isArray(d)); // true

4. Operator CPMK 4.2

Contoh Awal: HTML Lengkap Operator

Operator digunakan untuk melakukan proses pada data, misalnya menjumlahkan angka, membandingkan nilai, atau mengecek logika benar-salah. Perhatikan contoh HTML lengkap berikut.

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Contoh Operator</title>
</head>
<body>
    <h2>Belajar Operator JavaScript</h2>
    <p id="hasilOperator"></p>

    <script>
        let a = 10;
        let b = 5;
        let hasil = a + b;

        document.getElementById("hasilOperator").innerHTML =
            `Hasil ${a} + ${b} = ${hasil}`;
    </script>
</body>
</html>
Alur contoh:
  1. Dua variabel angka dibuat, yaitu a dan b.
  2. Operator + digunakan untuk menjumlahkan keduanya.
  3. Hasil operasi disimpan ke variabel hasil.
  4. Nilai akhirnya ditampilkan ke halaman HTML.

Apa itu Operator?

Operator adalah simbol yang digunakan untuk melakukan operasi terhadap data. Operasi itu bisa berupa perhitungan matematika, perbandingan, logika, atau penugasan nilai ke variabel.

Dengan kata lain, jika variabel adalah tempat menyimpan data, maka operator adalah alat untuk mengolah data tersebut.

Operator Aritmatika

let a = 10;
let b = 3;

console.log(a + b);  // 13 (penjumlahan)
console.log(a - b);  // 7  (pengurangan)
console.log(a * b);  // 30 (perkalian)
console.log(a / b);  // 3.333... (pembagian)
console.log(a % b);  // 1  (modulus/sisa bagi)
console.log(a ** b); // 1000 (pangkat: 10³)

// Increment & Decrement
let x = 5;
x++;  // x = 6 (increment)
x--;  // x = 5 (decrement)
++x;  // x = 6 (pre-increment)
--x;  // x = 5 (pre-decrement)

Operator Perbandingan

let a = 5;
let b = "5";

// == (sama nilai, tipe bisa beda)
console.log(a == b);   // true

// === (sama nilai DAN tipe) DIREKOMENDASIKAN
console.log(a === b);  // false

// != (tidak sama nilai)
console.log(a != b);   // false

// !== (tidak sama nilai ATAU tipe) 
console.log(a !== b);  // true

// Perbandingan lainnya
console.log(10 > 5);   // true (lebih besar)
console.log(10 < 5);   // false (lebih kecil)
console.log(10 >= 10); // true (lebih besar atau sama)
console.log(10 <= 5);  // false (lebih kecil atau sama)
Penting: Selalu gunakan === dan !== untuk menghindari kesalahan perbandingan tipe data!

Operator Logika

let x = true;
let y = false;

// && (AND) - kedua kondisi harus true
console.log(x && y);  // false
console.log(x && x);  // true

// || (OR) - salah satu kondisi true
console.log(x || y);  // true
console.log(y || y);  // false

// ! (NOT) - membalik nilai
console.log(!x);     // false
console.log(!y);     // true

// Contoh praktis
let umur = 20;
let punyaKTP = true;

if (umur >= 17 && punyaKTP) {
    console.log("Boleh membuat SIM");
}

Operator Assignment

let x = 10;

x += 5;   // x = x + 5  → 15
x -= 3;   // x = x - 3  → 12
x *= 2;   // x = x * 2  → 24
x /= 4;   // x = x / 4  → 6
x %= 4;   // x = x % 4  → 2
x **= 3;  // x = x ** 3 → 8

Operator String

let firstName = "John";
let lastName = "Doe";

// Concatenation (penggabungan)
let fullName = firstName + " " + lastName;
console.log(fullName); // "John Doe"

// Template Literal (ES6) 
let umur = 25;
let pesan = `Nama saya ${firstName} ${lastName}, umur ${umur} tahun`;
console.log(pesan); // "Nama saya John Doe, umur 25 tahun"

5. Percabangan (Control Flow) CPMK 4.3

Contoh Awal: HTML Lengkap Percabangan

Percabangan digunakan ketika program harus memilih aksi berdasarkan kondisi tertentu. Contoh berikut memperlihatkan alur sederhana untuk menentukan apakah seorang pengguna sudah cukup umur atau belum.

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Contoh Percabangan</title>
</head>
<body>
    <h2>Cek Status Umur</h2>
    <p id="statusUmur"></p>

    <script>
        let umur = 18;
        let status;

        if (umur >= 17) {
            status = "Sudah cukup umur";
        } else {
            status = "Belum cukup umur";
        }

        document.getElementById("statusUmur").innerHTML = status;
    </script>
</body>
</html>
Apa yang dilakukan kode di atas?
  1. Program membaca nilai variabel umur.
  2. Kondisi umur >= 17 diperiksa.
  3. Jika benar, program memilih status pertama. Jika salah, program memilih status kedua.
  4. Hasil keputusan ditampilkan ke halaman.

Apa itu Percabangan?

Percabangan adalah cara program mengambil keputusan berdasarkan kondisi. Saat kondisi berbeda, hasil atau aksi program juga bisa berbeda.

Ini sangat penting karena program nyata hampir selalu membutuhkan keputusan, misalnya: apakah login berhasil, apakah nilai lulus, apakah stok tersedia, atau apakah data input valid.

If Statement

let nilai = 85;

if (nilai >= 80) {
    console.log("Nilai A");
}

If-Else Statement

let umur = 15;

if (umur >= 17) {
    console.log("Boleh membuat SIM");
} else {
    console.log("Belum boleh membuat SIM");
}

If-Else If-Else Statement

let nilai = 75;
let grade;

if (nilai >= 90) {
    grade = "A";
} else if (nilai >= 80) {
    grade = "B";
} else if (nilai >= 70) {
    grade = "C";
} else if (nilai >= 60) {
    grade = "D";
} else {
    grade = "E";
}

console.log(`Nilai: ${nilai}, Grade: ${grade}`);

Ternary Operator (Shorthand If-Else)

let umur = 20;

// Syntax: kondisi ? nilaiJikaTrue : nilaiJikaFalse
let status = umur >= 17 ? "Dewasa" : "Anak-anak";
console.log(status); // "Dewasa"

// Nested ternary (tidak direkomendasikan jika terlalu kompleks)
let nilai = 85;
let grade = nilai >= 90 ? "A" : 
             nilai >= 80 ? "B" : 
             nilai >= 70 ? "C" : "D";

Switch Statement

let hari = 3;
let namaHari;

switch (hari) {
    case 1:
        namaHari = "Senin";
        break;
    case 2:
        namaHari = "Selasa";
        break;
    case 3:
        namaHari = "Rabu";
        break;
    case 4:
        namaHari = "Kamis";
        break;
    case 5:
        namaHari = "Jumat";
        break;
    case 6:
    case 7:
        namaHari = "Weekend";
        break;
    default:
        namaHari = "Tidak valid";
}

console.log(namaHari); // "Rabu"
Jangan lupa break! Tanpa break, eksekusi akan "jatuh" ke case berikutnya.

6. Perulangan (Loop) CPMK 4.3

Contoh Awal: HTML Lengkap Perulangan

Perulangan dipakai ketika kita ingin menjalankan perintah yang sama berkali-kali. Contoh berikut menampilkan angka 1 sampai 5 ke halaman web.

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Contoh Perulangan</title>
</head>
<body>
    <h2>Belajar Loop</h2>
    <div id="hasilLoop"></div>

    <script>
        let teks = "";

        for (let i = 1; i <= 5; i++) {
            teks += `Angka ke-${i}<br>`;
        }

        document.getElementById("hasilLoop").innerHTML = teks;

Alur kerja loop di atas:

Contoh Awal: HTML Lengkap dengan Function

Sebelum masuk ke konsep, perhatikan contoh file HTML lengkap berikut. Contoh ini menunjukkan bagaimana JavaScript function ditulis di dalam halaman HTML, lalu dijalankan saat tombol diklik.

  4. Function lalu mengubah isi paragraf dengan id hasil.

Apa itu Function?

Function adalah sekumpulan perintah yang dibungkus dalam satu nama agar bisa dipanggil kembali saat dibutuhkan. Dengan function, kita tidak perlu menulis kode yang sama berulang-ulang. Function membantu program menjadi lebih rapi, mudah dibaca, dan mudah digunakan ulang.

Bayangkan function seperti mesin otomatis. Kita menyiapkan mesinnya terlebih dahulu, lalu saat dibutuhkan kita menekan tombol untuk menjalankannya. Dalam JavaScript, proses ini biasanya terdiri dari dua tahap: membuat function lalu memanggil function.

Manfaat function:
  • Menghindari penulisan kode yang berulang
  • Membagi program menjadi bagian-bagian kecil yang jelas tugasnya
  • Mempermudah perawatan dan pengembangan program
  • Membuat logika program lebih terstruktur

Alur Function: Dibuat Lalu Dijalankan

Langkah Penjelasan
1. Deklarasi Kita menuliskan function dan memberi nama pada function tersebut
2. Parameter Jika diperlukan, function menerima data masukan
3. Proses Kode di dalam function dijalankan untuk mengolah data
4. Return Function bisa mengembalikan hasil, atau hanya menjalankan aksi saja
5. Pemanggilan Function baru benar-benar berjalan saat dipanggil dengan namanya

Bagian-Bagian Function

function tambah(a, b) {
    return a + b;
}
Keterangan:
  • function adalah kata kunci untuk membuat function
  • tambah adalah nama function
  • a dan b adalah parameter, yaitu data masukan
  • return a + b; berarti function mengembalikan hasil penjumlahan

10. DOM Manipulation CPMK 4.4

Apa itu DOM?

DOM (Document Object Model) adalah representasi struktur HTML sebagai objek yang dapat dimanipulasi dengan JavaScript.

Saat browser membaca file HTML, browser tidak hanya menampilkan teks dan tag apa adanya. Browser membangun sebuah struktur pohon dari elemen-elemen HTML. Struktur inilah yang disebut DOM. Dengan DOM, JavaScript dapat menemukan elemen tertentu, membaca isinya, mengubah tampilannya, menambah elemen baru, atau menghapus elemen yang sudah ada.

Karena itu, ketika kita menulis JavaScript seperti document.getElementById() atau querySelector(), sebenarnya kita sedang meminta JavaScript untuk mencari node tertentu di dalam pohon DOM, lalu melakukan sesuatu terhadap node tersebut.

Contoh Awal: HTML Lengkap dan DOM

Perhatikan contoh file HTML lengkap berikut. Contoh ini sederhana, tetapi cukup untuk melihat bagaimana HTML berubah menjadi struktur DOM yang bisa diakses JavaScript.

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>Contoh DOM</title>
</head>
<body>
    <div id="container">
        <h2 id="judul">Belajar DOM</h2>
        <p class="teks">Ini paragraf pertama</p>
        <button onclick="ubahIsi()">Ubah Judul</button>
    </div>

    <script>
        function ubahIsi() {
            document.getElementById("judul").innerHTML = "Judul sudah diubah oleh JavaScript";
        }
    </script>
</body>
</html>
Apa yang terjadi pada contoh di atas?
  1. HTML membuat elemen div, h2, p, dan button.
  2. Browser mengubah semua elemen itu menjadi node-node dalam DOM.
  3. Saat tombol diklik, function ubahIsi() dijalankan.
  4. JavaScript mencari elemen dengan id judul, lalu mengganti isi elemennya.

Hirarki DOM

DOM dapat dibayangkan seperti pohon keluarga. Ada elemen yang menjadi parent (induk), ada yang menjadi child (anak), dan ada juga yang menjadi sibling (saudara, yaitu elemen dengan parent yang sama).

Jika HTML di atas diubah menjadi hirarki DOM, gambarnya kurang lebih seperti ini:

document
└── html
    ├── head
    │   └── title
    └── body
        └── div#container
            ├── h2#judul
            ├── p.teks
            └── button
Istilah Contoh pada Struktur di Atas
Parent div#container adalah parent dari h2, p, dan button
Child h2 adalah child dari div#container
Sibling h2, p, dan button saling sibling karena parent-nya sama
Root document adalah akar utama dari seluruh DOM

Mengapa Hirarki DOM Penting?

Memahami hirarki DOM penting karena banyak operasi JavaScript bergantung pada posisi elemen di dalam struktur ini. Misalnya, saat kalian ingin mengambil semua paragraf di dalam satu div, menambahkan elemen baru sebagai anak dari container tertentu, atau menghentikan event agar tidak naik ke parent, kalian sebenarnya sedang bekerja berdasarkan hubungan antar node dalam DOM.

Dengan memahami konsep parent, child, dan sibling, mahasiswa akan lebih mudah memahami kenapa selector tertentu berhasil menemukan elemen, kenapa event bisa menyebar ke elemen induk, dan kenapa perubahan DOM bisa memengaruhi bagian tertentu dari halaman.

    console.log(event.clientX, event.clientY); // posisi di viewport
    console.log(event.pageX, event.pageY);     // posisi di halaman
});

// Keyboard event

});

class Mahasiswa {
    // Constructor
    constructor(nama, nim, jurusan) {
        this.nama = nama;
        this.nim = nim;
        this.jurusan = jurusan;
        this.ipk = 0;
    }
    
    // Methods
    perkenalan() {
        return `Halo, saya ${this.nama} dari ${this.jurusan}`;
    }
    
    setIpk(nilai) {
        if (nilai >= 0 && nilai <= 4) {
            this.ipk = nilai;
        } else {
            console.log("IPK harus antara 0-4");
        }
    }
    
    getInfo() {
        return `${this.nama} (${this.nim}) - IPK: ${this.ipk}`;
    }

        return this.nama.toUpperCase();
    }
    
    // Setter
    set ubahNama(namaBaru) {
        this.nama = namaBaru;
    }
    
    // Static method (dipanggil dari class, bukan instance)
    static info() {
        return "Ini adalah class Mahasiswa";
    }
}

Inheritance (Pewarisan)

// Parent class
class Orang {
    constructor(nama, umur) {
        this.nama = nama;
        this.umur = umur;
    }
    
    sapa() {
        return `Halo, saya ${this.nama}`;
    }
}

// Child class (mewarisi dari Orang)
class Mahasiswa extends Orang {
    constructor(nama, umur, nim, jurusan) {
        super(nama, umur); // Panggil constructor parent
        this.nim = nim;
        this.jurusan = jurusan;
    }
    
    // Override method parent
    sapa() {
        return `${super.sapa()}, mahasiswa ${this.jurusan}`;
    }
    
    // Method baru
    belajar() {
        return `${this.nama} sedang belajar`;
    }
}

// Child class lain
class Dosen extends Orang {
    constructor(nama, umur, nip, bidang) {
        super(nama, umur);
        this.nip = nip;
        this.bidang = bidang;
    }
    
    mengajar() {
        return `${this.nama} mengajar ${this.bidang}`;
    }
}

// Penggunaan
let mhs = new Mahasiswa("John", 20, "12345", "Informatika");
let dosen = new Dosen("Dr. Smith", 45, "1234567", "Pemrograman Web");

console.log(mhs.sapa());     // "Halo, saya John, mahasiswa Informatika"
console.log(mhs.belajar());  // "John sedang belajar"
console.log(dosen.sapa());   // "Halo, saya Dr. Smith"
console.log(dosen.mengajar()); // "Dr. Smith mengajar Pemrograman Web"

Contoh Praktis: Class Produk untuk E-Commerce

class Produk {
    constructor(id, nama, harga, stok) {
        this.id = id;
        this.nama = nama;
        this.harga = harga;
        this.stok = stok;
    }
    
    formatHarga() {
        return `Rp ${this.harga.toLocaleString('id-ID')}`;
    }
    
    kurangiStok(jumlah) {
        if (jumlah <= this.stok) {
            this.stok -= jumlah;
            return true;
        }
        return false;
    }
    
    tambahStok(jumlah) {
        this.stok += jumlah;
    }
    
    getDetail() {
        return {
            id: this.id,
            nama: this.nama,
            harga: this.formatHarga(),
            stok: this.stok,
            tersedia: this.stok > 0
        };
    }
}

class Keranjang {
    constructor() {
        this.items = [];
    }
    
    tambahItem(produk, qty) {
        if (produk.kurangiStok(qty)) {
            this.items.push({ produk, qty });
            return true;
        }
        return false;
    }
    
    hitungTotal() {
        return this.items.reduce((total, item) => {
            return total + (item.produk.harga * item.qty);
        }, 0);
    }
    
    lihatKeranjang() {
        return this.items.map(item => ({
            nama: item.produk.nama,
            qty: item.qty,
            subtotal: item.produk.harga * item.qty
        }));
    }
}

// Penggunaan

let mouse = new Produk(2, "Mouse Wireless", 250000, 50);

let keranjang = new Keranjang();

keranjang.tambahItem(mouse, 2);

13. jQuery CPMK 4.3

Apa itu jQuery?

jQuery adalah library JavaScript yang dibuat untuk mempermudah penulisan kode JavaScript, terutama untuk manipulasi DOM, event handling, animasi, dan AJAX. Pada masa ketika perbedaan browser masih besar, jQuery sangat populer karena membuat kode lebih singkat dan kompatibel di banyak browser.

Mengapa jQuery dulu sangat populer?
  • Sintaks lebih pendek dibanding JavaScript lama
  • Seleksi elemen HTML menjadi lebih mudah
  • Event dan animasi bisa dibuat dengan cepat
  • Membantu mengatasi perbedaan perilaku antar browser

Contoh Perbedaan JavaScript Biasa dan jQuery

Kegiatan JavaScript Biasa jQuery
Memilih elemen document.getElementById("judul") $("#judul")
Mengubah teks element.innerHTML = "Halo" $("#judul").html("Halo")
Event klik button.addEventListener("click", fn) $("#btn").click(fn)
Menyembunyikan elemen element.style.display = "none" $(".box").hide()

Contoh Kode jQuery

<!-- HTML -->
<h3 id="judul">Belajar jQuery</h3>
<button id="btnUbah">Ubah Teks</button>

<!-- jQuery -->
<script>
    $("#btnUbah").click(function() {
        $("#judul").html("Teks berhasil diubah dengan jQuery!");
    });
</script>

Apakah jQuery Masih Dipakai?

jQuery masih ditemui pada banyak website lama, panel admin, template HTML, plugin, dan proyek yang sudah berjalan bertahun-tahun. Namun, di proyek baru, banyak fitur jQuery kini bisa digantikan oleh JavaScript modern seperti querySelector, fetch, classList, dan addEventListener.

Kesimpulan tentang jQuery:
  • Penting untuk dikenal karena masih banyak dipakai di proyek lama
  • Membantu memahami sejarah perkembangan frontend web
  • Untuk proyek baru, JavaScript modern sering menjadi pilihan utama

14. Node.js dan JavaScript Modern CPMK 4.4

Apa itu Node.js?

Node.js adalah runtime yang memungkinkan JavaScript dijalankan di luar browser, terutama di sisi server. Dengan Node.js, JavaScript tidak hanya dipakai untuk frontend, tetapi juga untuk membuat backend, API, tool build, automation script, dan berbagai aplikasi lain.

Lingkungan Peran JavaScript
Browser Mengatur interaksi halaman web dan DOM
Node.js Mengatur server, file system, API, package manager, dan tooling

Mengapa Node.js Penting?

  • Memungkinkan satu bahasa dipakai untuk frontend dan backend
  • Menjadi dasar banyak tools modern seperti npm, Vite, Next.js, dan Express
  • Sangat umum dipakai dalam workflow pengembangan web modern

Apa yang Dimaksud JavaScript Modern?

JavaScript modern biasanya mengacu pada fitur-fitur ECMAScript baru yang membuat kode lebih singkat, aman, dan mudah dibaca dibanding gaya JavaScript lama.

Fitur Modern Kegunaan
let dan const Pengganti var dengan aturan scope yang lebih jelas
Arrow Function Menulis function lebih ringkas
Template Literal Menggabungkan string dengan lebih mudah memakai backtick
Destructuring Mengambil data dari array atau object dengan cepat
Spread / Rest Mempermudah pengolahan array, object, dan parameter
async / await Menangani proses asynchronous dengan lebih mudah dipahami
Module import / export Membagi kode ke beberapa file dengan lebih rapi

Contoh Gaya Lama vs Modern

// JavaScript lama
var nama = "Budi";
function sapa() {
    return "Halo, " + nama;
}

// JavaScript modern
const namaUser = "Budi";
const sapaUser = () => `Halo, ${namaUser}`;
Catatan penting:
  • Belajar JavaScript dasar tetap wajib sebelum memakai framework atau backend
  • Node.js memperluas penggunaan JavaScript ke luar browser
  • JavaScript modern membuat kode lebih dekat dengan praktik industri saat ini
