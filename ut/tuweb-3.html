Aktivitas Belajar 12 Vue.js

Component, Canvas, dan Animasi (1): Components

Bayangkan membangun rumah dari LEGO. Setiap balok LEGO bisa dipakai berulang kali di tempat yang berbeda, memiliki bentuk tersendiri, dan bisa dirakit bersama balok lain. Itulah konsep component di Vue. Component adalah potongan antarmuka (UI) yang berdiri sendiri, bisa dipakai ulang, dan bisa digabung untuk membentuk halaman yang lebih kompleks.

Tujuan Pembelajaran:
  • Mahasiswa mampu menjelaskan konsep dan manfaat component di Vue.
  • Mahasiswa mampu membuat dan mendaftarkan global component.
  • Mahasiswa mampu menerapkan konsep one root element pada template component.
  • Mahasiswa mampu menggunakan props untuk mengirim data dari parent ke child.
  • Mahasiswa mampu menggunakan emits untuk komunikasi dari child ke parent.
  • Mahasiswa mampu menggunakan slots untuk konten dinamis.
  • Mahasiswa mampu menggunakan computed dan watch di dalam component.
  • Mahasiswa mampu membuat custom directive menggunakan app.directive().

A. Mengapa Butuh Component?

Tanpa component, jika ingin menampilkan 10 kartu mahasiswa, kode HTML-nya harus ditulis 10 kali. Jika ada perubahan desain kartu, 10 tempat harus diubah satu per satu. Dengan component, kartu ditulis sekali lalu dipakai berulang kali — cukup ubah di satu tempat, semua kartu ikut berubah.

Tanpa Component Dengan Component
Kode HTML diulang untuk setiap item Kode ditulis sekali, dipakai berulang
Sulit dirawat jika ada perubahan tampilan Cukup ubah di satu tempat
File HTML sangat panjang Kode lebih pendek dan terstruktur

B. Cara Membuat Component (via CDN)

Di Vue 3 via CDN, component didaftarkan menggunakan app.component() setelah aplikasi dibuat namun sebelum app.mount(). Template component bisa ditulis sebagai string di dalam properti template.

const app = createApp({ /* opsi root */ });

// Daftarkan component sebelum mount
app.component('nama-component', {
    template: `<p>Halo dari component!</p>`,
    data() { return { /* data lokal */ }; },
    props: [],
    emits: [],
    methods: {}
});

app.mount('#app'); // mount setelah semua component didaftarkan
Aturan Penamaan: Nama component menggunakan format kebab-case (huruf kecil, dipisah tanda hubung), contoh: kartu-mahasiswa. Di HTML, digunakan sebagai tag: <kartu-mahasiswa></kartu-mahasiswa>.

One Root Element pada Template

Template component harus memiliki satu elemen root yang membungkus seluruh konten. Ini adalah aturan dasar penulisan template Vue.

// BENAR: satu elemen <div> sebagai root yang membungkus semua
app.component('kartu-info', {
    props: ['judul', 'isi'],
    template: `
        <div>
            <h3>{{ judul }}</h3>
            <p>{{ isi }}</p>
        </div>
    `
});

// SALAH: dua elemen root sejajar — menyebabkan error di Vue via CDN
app.component('kartu-info', {
    template: `
        <h3>{{ judul }}</h3>
        <p>{{ isi }}</p>
    `
});
Catatan Vue 3: Di Vue 3 Single File Component (.vue), multiple root elements (fragment) diperbolehkan. Namun saat menggunakan template string via CDN seperti pada mata kuliah ini, satu root element tetap wajib.

C. Props: Mengirim Data dari Parent ke Child

Props (properties) adalah cara parent mengirim data ke component anak. Analoginya seperti mengisi formulir — parent mengisi data, component menerima dan menampilkannya. Data mengalir satu arah: parent → child.

Di Parent (HTML) Di Component (JS)
:nama="mahasiswa.nama" props: ['nama']
:nilai="mahasiswa.nilai" props: ['nilai']

D. Emits: Component Kirim Sinyal ke Parent

Jika props adalah cara parent memberi data ke child, maka emits adalah cara child memberi tahu parent bahwa sesuatu telah terjadi. Contoh: tombol hapus di dalam kartu menggunakan emit untuk memberi tahu parent bahwa item tersebut ingin dihapus.

// Di dalam template component
<button @click="$emit('hapus')">Hapus</button>

// Di component, deklarasikan emit yang mungkin terjadi
emits: ['hapus'],

// Di parent, tangkap event tersebut
<kartu-mahasiswa @hapus="hapusMahasiswa(mhs.nim)"></kartu-mahasiswa>

E. Slots: Konten Dinamis di Dalam Component

Slot digunakan ketika ingin menempatkan konten HTML bebas di dalam component. Slot ibarat "lubang" yang bisa diisi oleh parent dengan konten apapun.

// Definisi component dengan slot
app.component('kartu-panel', {
    template: `
        <div class="panel">
            <h3>{{ judul }}</h3>
            <slot>Konten default jika slot tidak diisi.</slot>
        </div>
    `,
    props: ['judul']
});

// Pemakaian di parent — konten di antara tag adalah isi slot
<kartu-panel judul="Profil">
    <p>Ini konten yang dimasukkan dari parent.</p>
</kartu-panel>

F. Lifecycle Hooks

Setiap component melewati tahap kehidupan: dibuat, dipasang ke halaman, diperbarui, lalu dilepas. Vue menyediakan hook untuk menjalankan kode di setiap tahap tersebut.

Hook Kapan Dipanggil Kegunaan Umum
mounted() Setelah component terpasang ke DOM Memuat data dari API, akses elemen DOM
updated() Setelah data berubah dan DOM diperbarui Sinkronisasi library pihak ketiga
beforeUnmount() Sebelum component dihapus dari DOM Membersihkan event listener, timer
created() Setelah instance dibuat, sebelum mount Inisialisasi data awal

G. Computed dan Watch di dalam Component

Component dapat memiliki computed dan watch sendiri, terpisah dari root app. Ini membuat logika reaktif terisolasi di dalam component yang bersangkutan.

app.component('kartu-nilai', {
    props: ['nama', 'nilai'],
    computed: {
        predikat() {
            if (this.nilai >= 85) return 'A';
            if (this.nilai >= 70) return 'B';
            if (this.nilai >= 60) return 'C';
            return 'D';
        },
        statusLulus() {
            return this.nilai >= 60 ? 'Lulus' : 'Tidak Lulus';
        }
    },
    watch: {
        nilai(nilaiBaru, nilaiLama) {
            if (nilaiBaru < 60 && nilaiLama >= 60) {
                console.log(this.nama + ' berubah menjadi tidak lulus');
            }
        }
    },
    template: `
        <div class="kartu">
            <p><strong>{{ nama }}</strong></p>
            <p>Nilai: {{ nilai }} — Predikat: {{ predikat }}</p>
            <p>Status: {{ statusLulus }}</p>
        </div>
    `
});
Opsi Fungsi dalam Component
data() State lokal component, tidak dibagi ke luar
computed Nilai turunan dari props atau data lokal component
watch Memantau perubahan props atau data lokal component
methods Fungsi yang dipanggil dari template component

H. Custom Directive

Selain directive bawaan Vue (v-model, v-if, v-for), kita bisa membuat custom directive sendiri menggunakan app.directive(). Custom directive berguna untuk memanipulasi DOM secara langsung pada elemen HTML yang sudah ada.

Hook Directive Kapan Dipanggil
mounted(el) Saat elemen pertama kali terpasang ke DOM
updated(el, binding) Setiap kali nilai binding berubah
unmounted(el) Saat elemen dilepas dari DOM

Contoh 1: v-focus — Otomatis Fokus Input

// Daftarkan sebelum app.mount()
app.directive('focus', {
    mounted(el) {
        el.focus();
    }
});

// Penggunaan di HTML
<input v-focus placeholder="Kursor otomatis ada di sini">

Contoh 2: v-uppercase — Format Teks Menjadi Huruf Kapital

app.directive('uppercase', {
    mounted(el) {
        el.style.textTransform = 'uppercase';
    },
    updated(el) {
        el.style.textTransform = 'uppercase';
    }
});

// Penggunaan
<p v-uppercase>{{ nama }}</p>
<input v-model="judul" v-uppercase>

Contoh 3: Custom Directive dengan Nilai (Binding)

// v-warna="'red'" — ubah warna teks sesuai nilai yang diberikan
app.directive('warna', {
    mounted(el, binding) {
        el.style.color = binding.value;
    },
    updated(el, binding) {
        el.style.color = binding.value;
    }
});

// Penggunaan
<p v-warna="'red'">Teks berwarna merah</p>
<p v-warna="warnaAktif">Warna diambil dari data Vue</p>
Perbedaan Component vs Custom Directive:
  • Component: membuat elemen HTML baru yang bisa dipakai ulang, memiliki template, data, dan logika sendiri.
  • Custom Directive: menambahkan perilaku baru pada elemen HTML yang sudah ada, terutama untuk manipulasi DOM langsung.

Aktivitas Belajar 13 Canvas

Component, Canvas, dan Animasi (2): Canvas, Animasi

Selama ini kita menampilkan konten berbasis teks: paragraf, tabel, tombol. HTML Canvas membuka pintu ke tampilan yang lebih visual: grafik, animasi, hingga game sederhana. Canvas ibarat papan gambar kosong di halaman web — JavaScript-lah yang memegang kuasnya dan menggambar apa yang kita minta.

Tujuan Pembelajaran:
  • Mahasiswa mampu membuat elemen canvas dan mengakses konteksnya.
  • Mahasiswa mampu menggambar bentuk dasar menggunakan fungsi rect, line, dan arc.
  • Mahasiswa mampu membuat animasi menggunakan requestAnimationFrame.
  • Mahasiswa mampu mengintegrasikan canvas ke dalam Vue menggunakan ref.
  • Mahasiswa mampu mendefinisikan animasi CSS menggunakan @keyframes.
  • Mahasiswa mampu menggunakan property-property class animasi CSS.
  • Mahasiswa mampu mengontrol animasi CSS dari Vue menggunakan :class binding.

A. Dasar HTML5 Canvas

Elemen <canvas> adalah wadah gambar. Ukurannya ditentukan oleh atribut width dan height dalam piksel. Untuk menggambar, kita ambil konteks 2D-nya melalui JavaScript.

<!-- Di HTML -->
<canvas id="papan" width="400" height="200"></canvas>

<!-- Di JavaScript -->
const canvas = document.getElementById('papan');
const ctx = canvas.getContext('2d'); // ambil konteks 2D

B. Menggambar Bentuk Dasar

Setelah mendapat konteks (ctx), kita bisa menggambar berbagai bentuk menggunakan method yang tersedia.

Bentuk Method Contoh
Persegi panjang fillRect(x, y, lebar, tinggi) ctx.fillRect(10, 10, 100, 50)
Lingkaran arc(x, y, radius, mulai, akhir) ctx.arc(50, 50, 30, 0, Math.PI * 2)
Garis moveTo(x,y) + lineTo(x,y) ctx.moveTo(0,0); ctx.lineTo(100,100)
Teks fillText(teks, x, y) ctx.fillText('Halo', 50, 50)
Cara menggambar lingkaran: Canvas tidak punya method circle(). Lingkaran dibuat dengan arc():
  • ctx.beginPath() — mulai jalur baru
  • ctx.arc(x, y, radius, 0, Math.PI * 2) — gambar lingkaran penuh (0 sampai 2π radian)
  • ctx.fillStyle = 'warna' — pilih warna
  • ctx.fill() — isi dengan warna

C. Animasi dengan requestAnimationFrame

requestAnimationFrame adalah cara browser menjalankan animasi yang mulus dan hemat baterai. Cara kerjanya: fungsi animasi memanggil dirinya sendiri terus-menerus, sekitar 60 kali per detik. Setiap pemanggilan menghapus kanvas lalu menggambar ulang dengan posisi baru.

function animasi() {
    // 1. Hapus kanvas
    ctx.clearRect(0, 0, canvas.width, canvas.height);

    // 2. Perbarui posisi
    x += dx;
    y += dy;

    // 3. Gambar ulang di posisi baru
    ctx.beginPath();
    ctx.arc(x, y, radius, 0, Math.PI * 2);
    ctx.fillStyle = 'biru';
    ctx.fill();

    // 4. Panggil diri sendiri untuk frame berikutnya
    requestAnimationFrame(animasi);
}

animasi(); // mulai animasi

D. Integrasi Canvas dengan Vue menggunakan ref

Di Vue, kita tidak menggunakan document.getElementById() untuk mengakses elemen DOM. Sebagai gantinya, kita pakai ref. Tambahkan atribut ref="namaRef" di HTML, lalu akses elemen tersebut di JavaScript melalui this.$refs.namaRef di dalam mounted().

<!-- Di template -->
<canvas ref="papanku" width="400" height="200"></canvas>

<!-- Di script Vue -->
mounted() {
    // Canvas sudah ada di DOM saat mounted() dipanggil
    const canvas = this.$refs.papanku;
    const ctx = canvas.getContext('2d');
    // mulai menggambar...
}

E. Animasi CSS dengan @keyframes

Selain animasi JavaScript dengan requestAnimationFrame, web juga mendukung animasi CSS. Animasi CSS menggunakan aturan @keyframes untuk mendefinisikan tahapan animasi, lalu property animation untuk menerapkannya ke elemen. Animasi CSS lebih ringan karena dijalankan langsung oleh browser tanpa campur tangan JavaScript setiap frame-nya.

Syntax @keyframes

/* Cara 1: from → to (awal ke akhir) */
@keyframes fadeIn {
    from { opacity: 0; }
    to   { opacity: 1; }
}

/* Cara 2: persentase — kontrol lebih detail setiap tahap */
@keyframes bounce {
    0%   { transform: translateY(0); }
    50%  { transform: translateY(-30px); }
    100% { transform: translateY(0); }
}

/* Cara 3: multi-tahap dengan perubahan warna */
@keyframes warna-berubah {
    0%   { background-color: #e74c3c; }
    33%  { background-color: #f39c12; }
    66%  { background-color: #27ae60; }
    100% { background-color: #e74c3c; }
}

Property-Property Class Animasi CSS

Property Fungsi Contoh Nilai
animation-name Nama @keyframes yang digunakan fadeIn, bounce
animation-duration Durasi satu siklus animasi 1s, 500ms
animation-timing-function Percepatan / perlambatan animasi ease, linear, ease-in-out
animation-iteration-count Jumlah pengulangan 1, 3, infinite
animation-direction Arah pemutaran ulang normal, reverse, alternate
animation-delay Jeda sebelum animasi dimulai 0s, 0.5s
animation-fill-mode Keadaan elemen sebelum/sesudah animasi none, forwards, backwards
Shorthand — tulis semua dalam satu baris:
animation: nama durasi timing delay iterasi arah fill-mode;

Contoh:
animation: bounce 0.8s ease-in-out 0s infinite alternate;

    <script>
        const { createApp } = Vue;

        createApp({
            data() {
                return {
                    warna: '#3498db',
                    kecepatan: 3,
                    radius: 25,
                    berjalan: false,
                    animasiId: null,
                    bola: { x: 100, y: 100, dx: 1, dy: 1 }
                };
            },
            mounted() {
                // Canvas siap digunakan setelah mounted
                this.gambarDiam();
            },
            beforeUnmount() {
                // Selalu hentikan animasi saat component dihapus
                cancelAnimationFrame(this.animasiId);
            },
            methods: {
                getCtx() {
                    return this.$refs.kanvas.getContext('2d');
                },
                gambarDiam() {
                    if (this.berjalan) return;
                    const canvas = this.$refs.kanvas;
                    const ctx = this.getCtx();
                    ctx.clearRect(0, 0, canvas.width, canvas.height);
                    this.gambarBola(ctx);
                },
                gambarBola(ctx) {
                    ctx.beginPath();
                    ctx.arc(this.bola.x, this.bola.y, this.radius, 0, Math.PI * 2);
                    ctx.fillStyle = this.warna;
                    ctx.fill();
                    ctx.strokeStyle = 'rgba(0,0,0,0.2)';
                    ctx.lineWidth = 2;
                    ctx.stroke();
                },
                toggleAnimasi() {
                    if (this.berjalan) {
                        this.berjalan = false;
                        cancelAnimationFrame(this.animasiId);
                    } else {
                        this.berjalan = true;
                        this.jalankanAnimasi();
                    }
                },
                jalankanAnimasi() {
                    const canvas = this.$refs.kanvas;
                    const ctx = this.getCtx();
                    const b = this.bola;
                    const r = this.radius;

                    // Hapus frame sebelumnya
                    ctx.clearRect(0, 0, canvas.width, canvas.height);

                    // Perbarui posisi
                    b.x += b.dx * this.kecepatan;
                    b.y += b.dy * this.kecepatan;

                    // Pantulan di dinding
                    if (b.x + r >= canvas.width || b.x - r <= 0) b.dx *= -1;
                    if (b.y + r >= canvas.height || b.y - r <= 0) b.dy *= -1;

                    // Gambar bola di posisi baru
                    this.gambarBola(ctx);

                    // Lanjut ke frame berikutnya
                    if (this.berjalan) {
                        this.animasiId = requestAnimationFrame(this.jalankanAnimasi.bind(this));
                    }
                }
            }
        }).mount('#app');
    </script>

Poin Penting Animasi:
  • clearRect(0, 0, lebar, tinggi) — selalu hapus kanvas di awal setiap frame, kalau tidak gambar lama tetap terlihat.
  • cancelAnimationFrame(id) — simpan ID yang dikembalikan requestAnimationFrame agar bisa dihentikan.
  • this.$refs.kanvas hanya tersedia setelah mounted() — jangan akses di created().
  • Pantulan dicek dengan membandingkan posisi bola dengan batas kanvas.

Latihan

Modifikasi kode di atas: tambahkan dua bola sekaligus (simpan dalam array), masing-masing dengan warna dan kecepatan yang berbeda. Gunakan v-for untuk membuat kontrol warna untuk tiap bola, dan perbarui loop animasi agar menggambar semua bola di array setiap frame.

A. Apa itu Database Relasional?

Database relasional menyimpan data dalam bentuk tabel, mirip spreadsheet. Setiap tabel punya kolom (nama, tipe data) dan baris (isi data). Beberapa tabel bisa saling terhubung lewat foreign key.

B. SQL Dasar

SQL (Structured Query Language) adalah bahasa yang digunakan untuk berkomunikasi dengan database. SQL mencakup perintah untuk membuat struktur database hingga memanipulasi isinya.

Membuat Database dan Tabel

Langkah pertama sebelum menyimpan data adalah membuat database dan mendefinisikan struktur tabelnya menggunakan perintah CREATE.

-- Membuat database baru
CREATE DATABASE kampus;

-- Menggunakan database tersebut
USE kampus;

-- Membuat tabel mahasiswa
CREATE TABLE mahasiswa (
    id       INT          PRIMARY KEY AUTO_INCREMENT,
    nim      VARCHAR(10)  NOT NULL UNIQUE,
    nama     VARCHAR(100) NOT NULL,
    prodi    VARCHAR(50),
    semester INT          DEFAULT 1
);

-- Membuat tabel nilai (berelasi dengan tabel mahasiswa)
CREATE TABLE nilai (
    id           INT PRIMARY KEY AUTO_INCREMENT,
    mahasiswa_id INT,
    mata_kuliah  VARCHAR(100),
    skor         DECIMAL(5,2),
    FOREIGN KEY (mahasiswa_id) REFERENCES mahasiswa(id)
);
Tipe Data SQL Kegunaan Contoh Kolom
INT Bilangan bulat id, semester, stok
VARCHAR(n) Teks dengan panjang maksimum n karakter nim, nama, email
TEXT Teks panjang tanpa batas deskripsi, catatan
DECIMAL(p,s) Bilangan desimal (p digit, s di belakang koma) nilai, harga
DATE Tanggal format YYYY-MM-DD tanggal_lahir, created_at
BOOLEAN True / False aktif, sudah_bayar

Operasi CRUD

Setelah tabel dibuat, ada 4 operasi utama untuk memanipulasi data yang sering disebut CRUD.

Operasi SQL Arti
Create INSERT INTO Tambah data baru
Read SELECT Ambil / baca data
Update UPDATE Perbarui data yang ada
Delete DELETE FROM Hapus data

Contoh Perintah SQL

-- Ambil semua mahasiswa
SELECT * FROM mahasiswa;

-- Ambil mahasiswa dari prodi tertentu, diurutkan nama
SELECT nim, nama, semester
FROM mahasiswa
WHERE prodi = 'Informatika'
ORDER BY nama ASC;

-- Tambah mahasiswa baru
INSERT INTO mahasiswa (nim, nama, prodi, semester)
VALUES ('A004', 'Dian Rahayu', 'Informatika', 1);

-- Perbarui semester mahasiswa dengan id = 1
UPDATE mahasiswa
SET semester = 6
WHERE id = 1;

-- Hapus mahasiswa dengan nim tertentu
DELETE FROM mahasiswa
WHERE nim = 'A002';
Selalu gunakan WHERE saat UPDATE dan DELETE! Tanpa WHERE, perintah UPDATE akan mengubah semua baris dan DELETE akan menghapus seluruh isi tabel. Ini adalah kesalahan fatal yang sering terjadi.

D. Fetch API: Mengambil Data dari File JSON

fetch() adalah fungsi JavaScript bawaan untuk mengambil data dari URL atau file. Hasilnya berupa Promise, sehingga kita gunakan .then() atau async/await untuk menunggu datanya.

// Cara 1: Menggunakan .then() (Promise chaining)
fetch('data/mahasiswa.json')
    .then(response => response.json())   // ubah respons ke objek JS
    .then(data => {
        this.daftarMahasiswa = data;     // simpan ke data Vue
    })
    .catch(error => {
        console.error('Gagal memuat data:', error);
    });

// Cara 2: Menggunakan async/await (lebih mudah dibaca)
async mounted() {
    try {
        const response = await fetch('data/mahasiswa.json');
        const data = await response.json();
        this.daftarMahasiswa = data;
    } catch (error) {
        console.error('Gagal memuat data:', error);
    }
}

E. HTTP Request dengan Axios

Axios adalah library JavaScript populer untuk melakukan HTTP Request. Dibanding fetch() bawaan browser, Axios memiliki sintaks lebih ringkas, penanganan error lebih informatif, dan respons JSON sudah otomatis di-parse tanpa perlu memanggil .json() secara manual.

fetch() Axios
Parse JSON Manual — perlu .json() Otomatis — langsung di res.data
Error handling Error HTTP (404, 500) tidak masuk catch Semua error HTTP masuk catch otomatis
Kirim JSON Perlu JSON.stringify() + header manual Otomatis — cukup berikan objek JS
Instalasi Bawaan browser, tidak perlu instalasi Perlu CDN atau npm

Method Fungsi Contoh

// js/components/stock-table.js
window.StockTableComponent = {
    data() {
        return {
            semuaData: [],          // data asli dari JSON
            filterUpbjj: '',        // filter UT-Daerah
            filterKategori: '',     // filter kategori (dependent)
            filterMenipis: false,   // checkbox stok menipis
            filterKosong: false,    // checkbox stok kosong
            sortField: 'judul',     // field untuk sort
            sortAsc: true,          // arah sort
            modeEdit: null,         // nim yang sedang diedit (null = tidak ada)
            formBaru: { kode: '', judul: '', kategori: '', upbjj: '', lokasiRak: '', harga: 0, qty: 0, safety: 5 }
        };
    },
    computed: {
        // Daftar UPBJJ unik untuk dropdown
        daftarUpbjj() {
            return [...new Set(this.semuaData.map(b => b.upbjj))];
        },
        // Kategori yang muncul sesuai UPBJJ yang dipilih (dependent options)
        daftarKategori() {
            const sumber = this.filterUpbjj
                ? this.semuaData.filter(b => b.upbjj === this.filterUpbjj)
                : this.semuaData;
            return [...new Set(sumber.map(b => b.kategori))];
        },
        // Data setelah semua filter dan sort diterapkan
        dataTermpilkan() {
            let hasil = [...this.semuaData];

            if (this.filterUpbjj) hasil = hasil.filter(b => b.upbjj === this.filterUpbjj);
            if (this.filterKategori) hasil = hasil.filter(b => b.kategori === this.filterKategori);
            if (this.filterMenipis) hasil = hasil.filter(b => b.qty < b.safety && b.qty > 0);
            if (this.filterKosong) hasil = hasil.filter(b => b.qty === 0);

            hasil.sort((a, b) => {
                const va = a[this.sortField];
                const vb = b[this.sortField];
                if (va < vb) return this.sortAsc ? -1 : 1;
                if (va > vb) return this.sortAsc ? 1 : -1;
                return 0;
            });

            return hasil;
        }
    },
    watch: {
        // Watcher 1: Saat UPBJJ berubah, reset kategori
        filterUpbjj(nilaiLama, nilaiBaru) {
            this.filterKategori = '';
        },
        // Watcher 2: Log perubahan filter untuk debugging / audit
        filterMenipis(val) {
            console.log('Filter menipis:', val ? 'aktif' : 'nonaktif');
        }
    },
    async mounted() {
        const res = await fetch('data/dataBahanAjar.json');
        this.semuaData = await res.json();
    },
    methods: {
        formatHarga(angka) {
            return 'Rp ' + Number(angka).toLocaleString('id-ID');
        },
        aturSort(field) {
            if (this.sortField === field) {
                this.sortAsc = !this.sortAsc;  // toggle arah
            } else {
                this.sortField = field;
                this.sortAsc = true;
            }
        },
        mulaiEdit(kode) {
            this.modeEdit = kode;
        },
        simpanEdit(buku) {
            // Data sudah terikat v-model langsung ke objek di array
            this.modeEdit = null;
        },
        hapusBuku(kode) {
            if (confirm('Hapus buku ' + kode + '?')) {
                this.semuaData = this.semuaData.filter(b => b.kode !== kode);
            }
        },
        tambahBuku() {
            if (!this.formBaru.kode || !this.formBaru.judul) {
                alert('Kode dan judul wajib diisi!');
                return;
            }
            this.semuaData.push({ ...this.formBaru });
            // Reset form
            this.formBaru = { kode: '', judul: '', kategori: '', upbjj: '', lokasiRak: '', harga: 0, qty: 0, safety: 5 };
        },
        resetFilter() {
            this.filterUpbjj = '';
            this.filterKategori = '';
            this.filterMenipis = false;
            this.filterKosong = false;
        }
    },
    template: `#tpl-stock`  // atau inline template string
};
